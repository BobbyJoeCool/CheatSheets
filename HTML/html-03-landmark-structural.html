Landmark & Structural Tags

header · nav · main · footer · aside · ARIA landmarks

Description

Landmark elements define the major navigable regions of a page. Unlike generic <div>s, each one carries an implicit ARIA landmark role that allows screen reader users to jump directly between page regions (banner, navigation, main content, footer) using keyboard shortcuts. They also signal page structure to search engines. A page without landmark elements forces screen reader users to read every line from the top — a significant usability failure.

Notes

  • Only one <main> per page — it marks the primary, unique content
  • <header> and <footer> can appear inside <article> or <section> as well as at page level
  • Multiple <nav> elements are fine — label them with aria-label to distinguish (e.g. "Primary", "Breadcrumb")
  • These are CSS-neutral — they render as block elements with no default visual styling
  • <body> itself is a landmark (document) — you never need role="document"

Landmark Elements & Their ARIA Roles

ElementImplicit RolePurpose
<header>banner (page-level)Site header, logo, site-wide nav
<nav>navigationMajor navigation link groups
<main>mainDominant page content — one per page
<footer>contentinfo (page-level)Site footer, copyright, links
<aside>complementaryTangential content — sidebars, callouts
<section>region (if named)Thematic grouping with a heading
<form>form (if named)Interactive form landmark

Nesting Rules

RuleDetail
One <main>Never nest or repeat; hide others with hidden in SPAs
<main> not in <header><main> is a sibling of <header>, not a child
Label multiple navsAdd aria-label="Primary" when more than one <nav> exists
header/footer contextOnly get banner/contentinfo roles when direct child of <body>

♿ Accessibility

  • Screen reader users navigate by landmark with a single keystroke — landmarks replace the need to tab through everything
  • Add a "Skip to main content" link as the first focusable element: <a href="#main">Skip to content</a>
  • When two landmarks share the same role (e.g. two <nav>), add aria-label to both to distinguish them in the landmark list
  • Avoid landmark overuse — wrapping every <div> in a <section> clutters the landmark tree

Example

<!-- Skip link — must be the first focusable element on the page -->
<a href="#main-content" class="skip-link">Skip to content</a>

<!-- banner landmark (page-level header) -->
<header>
  <!-- site logo, branding -->

  <!-- navigation landmark — label required when multiple navs exist -->
  <nav aria-label="Primary">
    <!-- primary nav links (ul > li > a) -->
  </nav>
</header>

<!-- main landmark — one per page, the dominant content -->
<main id="main-content">

  <!-- complementary landmark — sidebar, callout, related content -->
  <aside aria-label="Filters">
    <!-- filter controls or supplementary content -->
  </aside>

  <!-- primary page content: articles, sections, media, etc. -->

</main>

<!-- second nav — different label so screen readers can distinguish -->
<nav aria-label="Breadcrumb">
  <ol>
    <!-- breadcrumb trail links -->
  </ol>
</nav>

<!-- contentinfo landmark (page-level footer) -->
<footer>
  <!-- copyright, legal links, footer nav -->
  <nav aria-label="Footer">
    <!-- footer links -->
  </nav>
</footer>

⚠ Common Mistakes

  • Using <div class="header"> instead of <header> — loses the landmark role entirely
  • Nesting <main> inside <header> or <footer> — <main> must be a direct child of <body>
  • Multiple unlabelled <nav> elements — screen readers list them as "navigation, navigation, navigation"
  • Putting the skip link after other focusable elements — it must be the very first focusable item
  • Wrapping all content in <section> — sections without headings provide no landmark benefit

✓ Tips & Best Practices

  • Build the landmark skeleton first, before any inner content — it enforces good document structure
  • Test landmark navigation with a screen reader (NVDA, VoiceOver) or the Accessibility Tree in DevTools
  • Every page should have at least: <header>, <main>, <footer>
  • Use aria-label on landmarks only when two or more share the same role

Quick Summary

  • Landmark elements expose page regions to screen readers — use them instead of <div>
  • One <main> per page; multiple <nav>/<header>/<footer> are allowed
  • Label duplicate landmarks with aria-label
  • Always include a skip link targeting <main> as the first focusable element
