HTML · Sheet 10

Inline & Misc Elements

br · hr · wbr · details · summary · dialog · template · noscript

Description

This sheet covers elements that don't belong to the major structural, text, or media families but are essential tools in a working HTML toolkit. <br> and <hr> are small but often misused. <details> and <dialog> are native browser interactive patterns that work without any JavaScript. <template> holds inert markup for JavaScript to clone at runtime. <noscript> provides graceful degradation when JavaScript is unavailable.

Notes

  • <br> is for meaningful line breaks (poetry, addresses) — not for adding vertical space between elements (use CSS margin)
  • <hr> is a thematic break — block-level despite often appearing inline in context
  • <details> works without JS — the open boolean attribute controls its initial state
  • <dialog> has two modes: modal (showModal() — traps focus, adds backdrop) and non-modal (show())
  • Content inside <template> is completely inert — scripts don't run, images don't load until it's cloned into the DOM

① br  ② hr  ③ wbr

<!-- ① br: meaningful line break 
			(address, poetry) -->
<address>
  123 Main Street<br />
  Springfield, IL 62701<br />
  USA
</address>

<!-- ② hr: thematic section break 
				(block-level) -->
<section>
  <h2>Chapter 1</h2>
  <!-- chapter content -->
</section>
<hr />
<section>
  <h2>Chapter 2</h2>
  <!-- chapter content -->
</section>

<!-- ③ wbr: optional word-break hint 
				for long strings -->
<p>
  https://www.example.com<wbr />
  	/very<wbr />/long<wbr />/path
</p>

④ details + summary  ⑤ dialog

<!-- ④ details + summary: native accordion — no JS -->
<details>
  <summary>What is your refund policy?</summary>
  <p><!-- answer content --></p>
</details>

<!-- open attribute: expanded by default -->
<details open>
  <summary>Expanded on load</summary>
  <p><!-- content visible immediately --></p>
</details>

<!-- ⑤ dialog: showModal() traps focus + adds backdrop -->
<dialog id="confirm-dialog" aria-labelledby="dialog-title">
  <h2 id="dialog-title">Confirm Delete</h2>
  <p>Are you sure? This cannot be undone.</p>
  <div>
    <button id="cancel-btn">Cancel</button>
    <button id="confirm-btn">Delete</button>
  </div>
</dialog>
<button id="open-btn">Delete Item</button>

<script defer>
  const dialog = document.getElementById('confirm-dialog');
  document.getElementById('open-btn')
    .addEventListener('click', () => dialog.showModal());
  document.getElementById('cancel-btn')
    .addEventListener('click', () => dialog.close('cancelled'));
  document.getElementById('confirm-btn')
    .addEventListener('click', () => dialog.close('confirmed'));
</script>

⑥ template  ⑦ noscript

<!-- ⑥ template: inert — 
not rendered until cloned by JS -->
<template id="card-template">
  <div class="card">
    <h3 class="card__title"></h3>
    <p  class="card__body"></p>
  </div>
</template>

<script>
  const clone = document
    .getElementById('card-template')
    .content.cloneNode(true);
  clone.querySelector('.card__title')
    .textContent = 'Hello';
  document.body.appendChild(clone);
</script>

<!-- ⑦ noscript: shown only 
		when JS is disabled -->
<noscript>
  <p>
    This app requires JavaScript.
    Please enable it to continue.
  </p>
</noscript>

Element Quick Reference

Element Type Purpose
<br> Void / inline Meaningful line break within a paragraph
<hr> Void / block Thematic break between content sections
<wbr> Void / inline Optional word-break hint for long strings
<details> Block Native disclosure / accordion — no JS needed
<summary> Block Visible label/trigger for <details>
<dialog> Block Native modal or non-modal dialog
<template> Inert Cloneable HTML fragment; not rendered until cloned
<noscript> Block Fallback content shown when JS is disabled

<dialog> JS API

Method / Event Description
showModal() Open as modal — traps focus, adds ::backdrop
show() Open as non-modal — no focus trap
close(val) Close; optional return value stored in returnValue
open attribute Boolean — present when dialog is open
close event Fires when dialog is closed
cancel event Fires on Escape key press

♿ Accessibility

  • <details>/<summary> are natively keyboard accessible — Tab to focus, Enter/Space to toggle; no ARIA needed
  • <dialog> opened with showModal() automatically traps focus and sets aria-modal — prefer it over a custom modal div
  • Give every <dialog> an accessible name via aria-label or aria-labelledby pointing to a heading inside it
  • <hr> has an implicit role of "separator" — announced by screen readers as a section boundary

⚠ Common Mistakes

  • Using <br> tags for vertical spacing between elements — use CSS margin instead
  • Building a custom modal <div> with JS focus-trapping instead of using native <dialog>
  • Opening <dialog> with show() when showModal() is needed — non-modal dialogs don't trap focus
  • Forgetting aria-labelledby on a <dialog> — screen readers announce "dialog" with no context
  • Expecting scripts or images inside <template> to load — they're completely inert until cloned

✓ Tips & Best Practices

  • Use <details>/<summary> for FAQs and simple accordions — zero JavaScript required
  • Always use showModal() over show() for confirmation dialogs — focus trapping and backdrop are critical UX
  • Style <dialog>'s ::backdrop pseudo-element to control the overlay appearance
  • Use <template> for repeated UI patterns (cards, list items) rendered by JS — cleaner than building HTML strings

Quick Summary

  • <br> = meaningful line break; <hr> = thematic section break
  • <details>/<summary> = native accordion, no JS; open attribute = expanded by default
  • <dialog> + showModal() = native modal with focus trap and backdrop
  • <template> holds inert markup; clone with content.cloneNode(true)
