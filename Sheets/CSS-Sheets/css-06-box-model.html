The Box Model

margin · padding · border · width · height · box-sizing

Description

Every HTML element is a rectangular box composed of four layers from inside out: content, padding, border, and margin. By default (box-sizing: content-box), width applies only to the content area — padding and border expand the total size. Setting box-sizing: border-box includes padding and border inside the declared width, which is almost always what you want and is the standard in modern CSS resets.

Notes

  • * { box-sizing: border-box } in your reset — makes element sizing intuitive project-wide
  • Margins can be negative; padding cannot go below zero
  • Vertical margins between adjacent block elements collapse to the larger of the two values — horizontal margins never collapse
  • width: auto fills available space minus margins; width: 100% fills the parent content box exactly

Example

/* Universal border-box reset — add to base/_reset.css */
*, *::before, *::after { box-sizing: border-box; }

.box {
  /* Sizing */
  width:  320px;
  height: 200px;

  /* Padding — space inside the border */
  padding: 1rem;                       /* all sides */
  padding: 0.5rem 1rem;               /* vertical | horizontal */
  padding: 0.5rem 1rem 1.5rem 1rem;   /* T R B L (clockwise) */

  /* Border */
  border: 2px solid #5c6bc0;

  /* Margin — space outside the border */
  margin: 0 auto;    /* center horizontally */
  margin-top: 2rem;
}

/* With border-box: declared width = total rendered width */
/* padding + border are INSIDE the 320px, not added on top */

Code Explained

box-sizing: border-box — width/height include padding and border
width / height — sets the box size (content only in content-box; total in border-box)
padding — inner space between content and border
padding: 0.5rem 1rem — shorthand: vertical | horizontal
padding: T R B L — four-value clockwise shorthand
border — drawn between padding and margin; takes space in content-box mode
margin — outer space beyond the border
margin: 0 auto — centers a block element horizontally

Shorthand Patterns

⚠ Common Mistakes

  • Not using box-sizing: border-box — adding padding makes elements larger than their set width
  • Using width: 100% on elements with padding — in content-box mode the total exceeds 100%
  • Relying on margin collapse — it does not happen inside flex or grid containers, or with overflow set
  • Setting margin: auto on inline elements — centering only works on block-level boxes

✓ Tips & Best Practices

  • Apply box-sizing: border-box universally via the * selector in your base reset
  • Use max-width instead of width for containers — allows shrinking on small screens
  • Padding for inner space, margin for outer space — they serve different purposes
  • margin: 0 auto with a defined max-width is the classic centered container pattern

Quick Summary

  • Layers inside-out: content → padding → border → margin
  • box-sizing: border-box: padding and border are inside the declared width
  • Padding is inside; margin is outside — neither can be negative (margin can, padding cannot)
  • Vertical block margins collapse; horizontal margins never do
