Selectors: Pseudo-Classes

:hover · :focus · :nth-child · :not · :checked · :disabled

Description

Pseudo-classes select elements based on their state or position in the DOM — without adding any classes to HTML. Interaction pseudo-classes like :hover and :focus respond to user actions. Structural ones like :nth-child() target elements by their position among siblings. Form pseudo-classes like :checked and :disabled reflect input state in real time.

Notes

  • Pseudo-classes have the same specificity as a class selector: 0,1,0
  • :nth-child() counts all sibling elements; :nth-of-type() counts only same-tag siblings
  • :focus-visible is preferred over :focus — shows rings for keyboard users only, not mouse clicks
  • :not() accepts a full selector list in modern CSS: :not(.a, .b)

Example

a:hover             { color: #7986cb; }
input:focus         { outline: 2px solid #5c6bc0; }
input:focus-visible { outline: 2px solid #5c6bc0; } /* keyboard only */

li:first-child      { font-weight: 700; }
li:last-child       { border-bottom: none; }
li:nth-child(2)     { color: #ccc; }
li:nth-child(odd)   { background: #1c2030; }  /* every odd row */
li:nth-child(3n)    { color: #9c8fff; }        /* every 3rd */
li:nth-child(3n+1)  { margin-top: 1rem; }      /* 1st of each group of 3 */

input:checked       { accent-color: #5c6bc0; }
input:disabled      { opacity: 0.4; cursor: not-allowed; }
.card:not(.featured){ opacity: 0.8; }
li:not(:last-child) { border-bottom: 1px solid #333; }

Code Explained

:hover — pointer device is over the element
:focus — element has received focus (keyboard or click)
:focus-visible — keyboard-driven focus only; skips mouse clicks
:first-child / :last-child — positional within parent
:nth-child(n) — n can be a number, keyword, or formula
odd / even — shorthand for formulas 2n+1 / 2n
3n+1 — matches 1, 4, 7, 10… (first of each group of 3)
:not(selector) — excludes elements matching the argument

⚠ Common Mistakes

  • Styling :focus for visible rings — use :focus-visible to skip mouse-click rings
  • Assuming :nth-child(2) is the second of that tag type — it counts all siblings regardless of tag
  • Setting outline: none without an alternative — breaks keyboard navigation accessibility
  • Using :hover for critical interactions — hover does not exist on touch devices

✓ Tips & Best Practices

  • Always style :focus-visible — never remove focus rings without a visible replacement
  • Use :nth-child(odd/even) for zebra-stripe tables instead of adding extra HTML classes
  • li:not(:last-child) { border-bottom: 1px solid; } — dividers without the final extra line
  • Chain pseudo-classes for compound states: input:focus:not(:disabled)

Quick Summary

  • Pseudo-classes select by state or position — no extra HTML markup needed
  • :hover / :focus-visible / :active — user interaction states
  • :nth-child() accepts numbers, odd/even keywords, and formulas like 2n+1
  • :not() / :checked / :disabled — logic and form input states
