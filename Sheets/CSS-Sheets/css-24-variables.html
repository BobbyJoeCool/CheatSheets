CSS Variables

--custom-properties · var() · fallback · :root · scope · override

Description

CSS custom properties — variables — store reusable values that can be referenced anywhere with var(). Declared with a -- prefix, they cascade and inherit like any CSS property. A variable on :root is globally available; one on a component is scoped to that subtree. Unlike Sass variables, CSS custom properties are live — they respond to media queries, can be overridden in component scope, and can be read and written by JavaScript.

Notes

  • Custom property names are case-sensitive: --Color and --color are different variables
  • var() accepts a fallback: var(--color, red) — used when the variable is undefined or invalid
  • Custom properties can store any CSS value including space-separated values, strings, and calc expressions
  • JS reads/writes them: el.style.setProperty('--x', val) and getComputedStyle(el).getPropertyValue('--x')

Code Explained

--name: value — declares a custom property on the selected element
var(--name) — inserts the variable's value at that point
var(--name, fallback) — fallback used when variable is undefined
:root — the <html> element; variables here are globally available
Component scope: redefining on .modal affects only .modal and its children
[data-theme='dark'] — toggle entire colour themes by flipping one attribute
@media + :root — update all spacing/sizing globally at one breakpoint
JS setProperty — live update without re-parsing any stylesheet

JS Interaction

SyntaxDescription
getComputedStyle(el).getPropertyValue('--x')Read variable
el.style.setProperty('--x', val)Write variable
el.style.removeProperty('--x')Remove override

Example

/* Declare on :root for global scope */
:root {
  --color-primary:   hsl(231 44% 56%);
  --color-danger:    hsl(0 70% 60%);
  --color-surface:   #1c2030;
  --spacing-sm:      0.5rem;
  --spacing-md:      1rem;
  --spacing-lg:      2rem;
  --radius-card:     8px;
  --font-body:       'DM Sans', sans-serif;
  --transition-base: 0.2s ease;
}

/* Use with var() */
.btn {
  background:    var(--color-primary);
  padding:       var(--spacing-sm) var(--spacing-md);
  border-radius: var(--radius-card);
  transition:    background var(--transition-base);
}

/* Fallback value — used if variable is undefined */
.box { color: var(--color-accent, #9c8fff); }

/* Component-scoped override */
.modal { --radius-card: 12px; }  /* only inside .modal */

/* Theme switching via data attribute */
:root               { --bg: #ffffff; --text: #111111; }
[data-theme="dark"] { --bg: #0f1117; --text: #e8eaf6; }
body { background: var(--bg); color: var(--text); }

/* Responsive — update tokens at a breakpoint */
:root { --spacing-lg: 1.5rem; }
@media (min-width: 768px) { :root { --spacing-lg: 2rem; } }

⚠ Common Mistakes

  • Declaring variables without :root and expecting global access — scope is limited to descendants of the selector
  • Forgetting case-sensitivity — --Color is not the same as --color
  • Assuming custom properties compute eagerly — they store literal text; the value is resolved at use time
  • Not providing fallbacks for critical UI values — a failed variable resolves to the property's initial value

✓ Tips & Best Practices

  • Keep all custom property declarations in a dedicated _tokens.css file — the single source of truth
  • Combine with @media on :root to update spacing, type size, and layout values globally at one breakpoint
  • Use [data-theme] attribute toggling for dark/light themes — swap every colour with one JS attribute change
  • Custom properties work inside calc(): calc(var(--spacing-md) * 2)

Quick Summary

  • Declare: --name: value · Use: var(--name) · Fallback: var(--name, default)
  • :root scope = global · component scope = descendants only
  • Unlike Sass, CSS custom properties are live — they update with media queries and JS in real time
  • Use a --category-variant token pattern for a maintainable, scalable design system
