Flexbox: Items

flex-grow · flex-shrink · flex-basis · flex · align-self · order

Description

Flex item properties are placed on the children of a flex container, not the container itself. flex-grow distributes extra space proportionally; flex-shrink controls compression when space is tight; flex-basis sets the initial size before growing or shrinking. The flex shorthand combines all three. align-self overrides the container's align-items for a single item. order changes visual position without altering DOM order.

Example

/* These properties go on the ITEM, not the container */
.item {
  flex-grow:   1;    /* share of extra space to absorb */
  flex-shrink: 1;    /* share to give up when tight */
  flex-basis:  0;    /* starting size before grow/shrink */
}

/* flex shorthand: grow | shrink | basis */
.item  { flex: 1; }          /* 1 1 0  — free grow/shrink */
.item  { flex: auto; }       /* 1 1 auto — from natural size */
.item  { flex: none; }       /* 0 0 auto — rigid */
.item  { flex: 0 1 200px; }  /* max 200px, can shrink, no grow */

/* Classic sidebar + main layout */
.sidebar { flex: 0 0 260px; } /* fixed 260px */
.main    { flex: 1; }         /* fills the rest */

/* align-self overrides container align-items for one item */
.special { align-self: flex-end; }
.tall    { align-self: stretch; }

/* order — default 0; lower numbers appear first */
.lead { order: -1; }
.last { order: 1; }

Notes

  • flex: 1 expands to flex: 1 1 0 — item grows and shrinks from a zero starting size
  • flex: auto = flex: 1 1 auto — grows and shrinks from the item's natural content size
  • flex: none = flex: 0 0 auto — completely rigid; ignores available or insufficient space
  • order is visual only — DOM order, tab focus, and screen readers are unaffected

Code Explained

flex-grow: 1 — absorbs 1 share of leftover space after fixed items are placed
flex-shrink: 1 — gives up 1 share when there is not enough room
flex-basis: 0 — starts from zero; all space is distributed via grow
flex: 1 — shorthand for grow=1, shrink=1, basis=0
flex: none — rigid item; never grows or shrinks
flex: 0 0 260px — fixed 260px sidebar that will not flex
align-self — overrides align-items for this one item only
order: -1 — appears before all order:0 items visually

⚠ Common Mistakes

  • Confusing flex: 1 (basis=0) with flex: auto (basis=auto) — they distribute space differently when items have varying content
  • Using order to reorder meaningful content — DOM order still controls focus and accessibility
  • Setting flex-grow without resetting flex-basis — use the shorthand flex: 1 to set all three correctly
  • Applying flex item properties to the container element instead of the child items

✓ Tips & Best Practices

  • flex: 1 on content + flex: 0 0 Npx on sidebar = robust two-column layout
  • margin-left: auto on a flex item absorbs all available space — great for pushing nav items right
  • Different flex-grow values create proportional layouts: 2 vs 1 gives a 2:1 ratio
  • Always use the flex shorthand rather than individual properties — it correctly resets all three

Quick Summary

  • flex: grow shrink basis — the item's complete sizing recipe
  • flex: 1 = freely growing · flex: none = rigid · flex: auto = from natural size
  • align-self overrides the container's align-items for a single item only
  • order changes visual order only — DOM and accessibility order are unchanged
