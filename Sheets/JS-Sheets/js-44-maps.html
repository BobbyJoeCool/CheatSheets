Map Examples

// ── Creating a Map ────────────────────────────────────────────
const map = new Map();
const map2 = new Map([["a", 1], ["b", 2], ["c", 3]]);

// ── CRUD ──────────────────────────────────────────────────────
map.set("name", "Alice");
map.set("age", 30);
map.set(42, "number key");     // non-string key — not possible with {}
map.set({ id: 1 }, "obj key"); // object as key (by reference)

map.get("name");               // "Alice"
map.get("missing");            // undefined
map.has("age");                // true
map.size;                      // 4

map.delete("age");             // true — removed
map.has("age");                // false

// ── Iteration ─────────────────────────────────────────────────
for (const [key, val] of map) {  // same as .entries()
  console.log(key, val);
}

[...map.keys()];               // ["name", 42, {id:1}]
[...map.values()];             // ["Alice", "number key", "obj key"]
[...map.entries()];            // [["name","Alice"], ...]

// ── Map ↔ Object conversion ───────────────────────────────────
const obj = { a: 1, b: 2 };
const fromObj = new Map(Object.entries(obj)); // object → Map
const toObj   = Object.fromEntries(fromObj);  // Map → object

// ── Common patterns ───────────────────────────────────────────
// Frequency counter
const words = ["apple", "banana", "apple", "cherry", "banana", "apple"];
const freq = new Map();
for (const word of words) {
  freq.set(word, (freq.get(word) ?? 0) + 1);
}
// Map { apple → 3, banana → 2, cherry → 1 }

// Cache / memoisation
const cache = new Map();
function expensive(n) {
  if (cache.has(n)) return cache.get(n);
  const result = n * n;        // expensive computation
  cache.set(n, result);
  return result;
}

// ── WeakMap — keys must be objects; not iterable ──────────────
const wm = new WeakMap();
const keyObj = {};
wm.set(keyObj, "private data");
wm.get(keyObj);                // "private data"
// When keyObj is GC'd, entry is automatically removed

// ── TypeScript typed Map ──────────────────────────────────────
const scores: Map<string, number> = new Map();
scores.set("Alice", 95);
scores.set("Bob", 87);
const s: number | undefined = scores.get("Alice"); // 95

Notes

Map keys can be any value — objects, functions, numbers, even NaN (which equals itself as a Map key)
.get() returns undefined for missing keys — always check with .has() or use ?? default
Object keys as Map keys use reference equality — two different objects with the same shape are different keys
WeakMap — keys must be objects; entries are garbage-collected when the key has no other references; not iterable; ideal for private class data
Frequency counter pattern — map.get(key) ?? 0 safely initialises to 0 for missing keys without a separate .has() check
Use Map over plain objects when: keys are non-strings, insertion order matters critically, or you frequently add/delete entries

Common Mistakes

  • Using an object key and expecting lookup by value — Map uses reference equality for objects
  • Using JSON.stringify(map) — produces {}; convert with Object.fromEntries(map) first
  • Iterating a WeakMap — it is not iterable by design; use a regular Map if you need iteration
  • Forgetting .get() returns undefined for missing keys — accessing properties on undefined throws

Tips

  • Use freq.set(key, (freq.get(key) ?? 0) + 1) as the standard frequency-counting idiom
  • Use WeakMap to associate private data with DOM elements or class instances without preventing garbage collection
  • Convert between Map and object with new Map(Object.entries(obj)) and Object.fromEntries(map)

Summary

  • Map — any key type · ordered · .set/.get/.has/.delete/.size
  • Iterates natively with for...of · .keys()/.values()/.entries()
  • Prefer Map over objects for non-string keys, frequent mutations, or large key sets
  • WeakMap — object keys only · GC-friendly · not iterable · ideal for private metadata
