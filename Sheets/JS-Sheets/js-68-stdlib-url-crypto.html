URL, Crypto & Buffer

The URL API parses and constructs URLs in a structured way — always prefer it over string manipulation for URLs. URLSearchParams manages query strings. The Web Crypto API (crypto.subtle) provides cryptographically secure operations; use crypto.randomUUID() for IDs and crypto.getRandomValues() for secure random bytes. Buffer (Node-only) handles raw binary data as sequences of bytes — encoding/decoding between binary and text (Base64, hex, UTF-8). TextEncoder/TextDecoder are the web-standard equivalent available in both browser and Node.

URL Properties

  • .href — full URL string
  • .protocol — "https:"
  • .hostname — "example.com"
  • .port — "8080" or ""
  • .pathname — "/path/to/page"
  • .search — "?key=val"
  • .searchParams — URLSearchParams object
  • .hash — "#section"

URL, Crypto & Buffer Examples

// ── URL parsing ───────────────────────────────────────────────
const url = new URL("https://example.com:8080/path?foo=1&bar=2#top");
url.protocol;      // "https:"
url.hostname;      // "example.com"
url.port;          // "8080"
url.pathname;      // "/path"
url.search;        // "?foo=1&bar=2"
url.hash;          // "#top"

// Relative URL (needs a base)
const rel = new URL("/new-path?x=5", "https://example.com");
rel.href;          // "https://example.com/new-path?x=5"

// ── URLSearchParams ───────────────────────────────────────────
const params = url.searchParams;
params.get("foo");          // "1"
params.has("bar");          // true
params.set("foo", "99");    // update
params.append("tag", "a");  // add without replacing
params.delete("bar");       // remove
params.toString();          // "foo=99&tag=a"

// Build query string from object
const qs = new URLSearchParams({ page: "2", limit: "10", sort: "name" });
qs.toString();     // "page=2&limit=10&sort=name"
`/api/users?${qs}`; // "/api/users?page=2&limit=10&sort=name"

// ── Crypto: secure random ─────────────────────────────────────
// UUID — available in browser and Node 14.17+
crypto.randomUUID();  // "9b1deb4d-3b7d-4bad-9bdd-2b0d7b3dcb6d"

// Secure random bytes (browser + Node)
const buf = new Uint8Array(16);
crypto.getRandomValues(buf); // fills with cryptographically secure bytes

// ── Crypto: hashing (SubtleCrypto) ────────────────────────────
async function sha256(message) {
  const encoded = new TextEncoder().encode(message);
  const hash    = await crypto.subtle.digest("SHA-256", encoded);
  return Array.from(new Uint8Array(hash))
    .map(b => b.toString(16).padStart(2, "0"))
    .join("");
}
await sha256("hello"); // "2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c..."

// ── Buffer (Node only) ────────────────────────────────────────
// Create
const buf2 = Buffer.from("Hello, world!", "utf8");
const hex  = Buffer.from("ff0a3d", "hex");
const b64  = Buffer.from("SGVsbG8=", "base64");

// Convert
buf2.toString("utf8");      // "Hello, world!"
buf2.toString("hex");       // "48656c6c6f..."
buf2.toString("base64");    // "SGVsbG8sIHdvcmxkIQ=="
buf2.length;                // 13 (bytes)

// ── TextEncoder / TextDecoder (browser + Node) ────────────────
const enc   = new TextEncoder();           // always UTF-8
const bytes = enc.encode("Hello");         // Uint8Array

const dec  = new TextDecoder("utf-8");
const text = dec.decode(bytes);            // "Hello"

// ── Base64 (browser built-ins) ────────────────────────────────
btoa("Hello, world!");            // "SGVsbG8sIHdvcmxkIQ=="
atob("SGVsbG8sIHdvcmxkIQ==");    // "Hello, world!"
// Note: btoa/atob only handle Latin-1; use Buffer for Unicode

Notes

new URL() throws on invalid URLs — wrap in try/catch when parsing user input; great for validating URLs
URLSearchParams automatically encodes/decodes special characters — no manual encodeURIComponent needed
crypto.randomUUID() — use for generating IDs; available globally in browsers and Node 19+; no library needed
crypto.subtle.digest is async — returns a Promise; always await it; the result is an ArrayBuffer, not a hex string
Buffer is Node-only — for cross-platform code, use TextEncoder/TextDecoder or Uint8Array
btoa/atob — browser built-ins for Base64; only handle ASCII/Latin-1; for Unicode use Buffer.from(str,"utf8").toString("base64")

Common Mistakes

  • Using Math.random() for security tokens — use crypto.getRandomValues() or crypto.randomUUID()
  • String-manipulating URLs with regex — always use new URL() to avoid encoding bugs
  • Using btoa with non-ASCII characters — throws; use Buffer or encode to UTF-8 bytes first
  • Forgetting crypto.subtle.digest is async — missing await gives a Promise, not a hash

Tips

  • Use new URLSearchParams(obj).toString() to build query strings from objects — handles encoding automatically
  • Use crypto.randomUUID() instead of libraries like uuid — it's built-in and fast
  • Prefer TextEncoder/TextDecoder over Buffer for portable code that runs in both browser and Node

Summary
