TypeScript Array & Tuple Types

TypeScript arrays are typed with T[] or Array<T> — both are equivalent. Adding readonly prevents mutation at compile time. Tuples are fixed-length arrays where each position has a specific type — they are TypeScript's way of typing multi-value returns, structured pairs, and coordinate systems. Tuple elements can be labeled for documentation, optional with ?, and use rest elements for variadic patterns. The as const assertion infers the narrowest possible tuple type from an array literal.

Key Points

  • T[] and Array<T> are identical
  • readonly T[] = ReadonlyArray<T>
  • Tuple: [string, number] — fixed length, typed positions
  • Labeled tuple: [name: string, age: number]
  • Optional element: [string, number?]
  • Rest in tuple: [string, ...number[]]
  • as const infers exact tuple from array literal

TS Array & Tuple Examples

// ── Typed arrays ──────────────────────────────────────────────
const scores: number[] = [95, 87, 72];
const names: string[] = ["Alice", "Bob"];
const flags: boolean[] = [true, false, true];
const matrix: number[][] = [[1, 2], [3, 4]];

// Generic syntax — identical to T[]
const ids: Array<number> = [1, 2, 3];
const pairs: Array<[string, number]> = [["a", 1], ["b", 2]];

// ── Readonly arrays ───────────────────────────────────────────
const fixed: readonly number[] = [1, 2, 3];
fixed.push(4);        // ✖ Error: Property 'push' does not exist
fixed[0] = 99;        // ✖ Error: Index signature is readonly

const also: ReadonlyArray<string> = ["x", "y"]; // equivalent

// ── Basic tuples ─────────────────────────────────────────────
type Point = [number, number];
const p: Point = [3, 4];
p[0];                 // number
p[2];                 // ✖ Error: Tuple type has no index 2

type Entry = [string, number];
const e: Entry = ["age", 30];
const [key, val] = e; // key: string, val: number

// ── Labeled tuples (documentation) ──────────────────────────
type Range = [start: number, end: number];
type RGB   = [red: number, green: number, blue: number];

const r: Range = [0, 100];
const color: RGB = [255, 128, 0];

// ── Optional tuple elements ───────────────────────────────────
type WithOptional = [string, number, boolean?];
const a: WithOptional = ["hi", 1];        // ✔
const b: WithOptional = ["hi", 1, true];  // ✔

// ── Rest elements in tuples ───────────────────────────────────
type StringThenNumbers = [string, ...number[]];
const c: StringThenNumbers = ["label", 1, 2, 3, 4]; // ✔

type AtLeastTwo = [number, number, ...number[]];
const d: AtLeastTwo = [1, 2];             // ✔
const e2: AtLeastTwo = [1, 2, 3, 4, 5];  // ✔
const f: AtLeastTwo = [1];               // ✖ Error

// ── as const — infer tuple from literal ──────────────────────
const pair = [1, "hello"] as const;
// type: readonly [1, "hello"]  — not number | string[]

// Derive a union from a tuple
const STATUSES = ["active", "inactive", "pending"] as const;
type Status = typeof STATUSES[number]; // "active" | "inactive" | "pending"

// ── Function returning a tuple ────────────────────────────────
function useState<T>(initial: T): [T, (value: T) => void] {
  let state = initial;
  const setState = (value: T) => { state = value; };
  return [state, setState];
}
const [count, setCount] = useState(0);
// count: number, setCount: (value: number) => void

Notes

readonly T[] — prevents push, pop, splice, and index assignment at compile time; runtime array is still mutable
Tuples enforce both length and position types — accessing an out-of-range index is a compile error
Labeled tuple elements are documentation only — they do not affect assignability or runtime behaviour
Optional elements (boolean?) must come after required elements; rest elements must be last
as const on an array — freezes the type to a readonly tuple of exact literal values instead of a mutable array of the wider type
Returning tuples from functions (like React's useState) — label the return type explicitly so destructuring gets correct types

Common Mistakes

  • Returning [value, setter] without a tuple return type — TS infers a wider union array type, losing positional typing
  • Using as const on an array then trying to push to it — readonly prevents mutation
  • Forgetting that readonly is compile-time only — the underlying JS array can still be mutated at runtime if you bypass TS

Tips

  • Always annotate the return type of functions that return tuples — prevents TS from inferring a plain array union type
  • Use labeled tuples for anything beyond a simple pair — [start: number, end: number] is self-documenting
  • Use as const on string arrays to derive union types: type Dir = typeof DIRS[number]

Summary

  • T[] = Array<T> · readonly T[] = ReadonlyArray<T>
  • Tuples — fixed-length, position-typed: [string, number]
  • Labels, optional elements, and rest elements extend tuple expressiveness
  • as const — narrows array literal to readonly tuple of exact types
