Linked Lists

Node pattern · singly linked · prepend · append · traversal · TS typed

Linked Lists in JavaScript

JavaScript has no built-in linked list — you implement one with node objects. A singly linked list consists of nodes where each node holds a value and a reference (next) to the next node. The list tracks a head (first node) and optionally a tail (last node). Linked lists shine when you need O(1) prepend/append and frequent insertion or deletion at known positions — operations where arrays are O(n) due to shifting. However, they have O(n) random access (no indexing) and higher memory overhead per element. In most JS applications, arrays are the right choice; linked lists appear in algorithm problems and low-level queue implementations.

Complexity Comparison

  • Prepend: linked list O(1) · array O(n)
  • Append: linked list O(1)* · array O(1) amortised
  • Insert middle: linked list O(1)† · array O(n)
  • Delete: linked list O(1)† · array O(n)
  • Access by index: linked list O(n) · array O(1)
  • Search: both O(n)
  • * O(1) with tail pointer · † given a reference to the node

Linked List Implementation

// ── Node and LinkedList types ─────────────────────────────────
class ListNode<T> {
  constructor(
    public value: T,
    public next: ListNode<T> | null = null
  ) {}
}

class LinkedList<T> {
  head: ListNode<T> | null = null;
  tail: ListNode<T> | null = null;
  size = 0;

  // O(1) — add to end
  append(value: T): void {
    const node = new ListNode(value);
    if (!this.tail) {
      this.head = this.tail = node;
    } else {
      this.tail.next = node;
      this.tail = node;
    }
    this.size++;
  }

  // O(1) — add to front
  prepend(value: T): void {
    const node = new ListNode(value, this.head);
    this.head = node;
    if (!this.tail) this.tail = node;
    this.size++;
  }

  // O(n) — remove first occurrence of value
  remove(value: T): boolean {
    if (!this.head) return false;
    if (this.head.value === value) {
      this.head = this.head.next;
      if (!this.head) this.tail = null;
      this.size--;
      return true;
    }
    let curr = this.head;
    while (curr.next) {
      if (curr.next.value === value) {
        if (curr.next === this.tail) this.tail = curr;
        curr.next = curr.next.next;
        this.size--;
        return true;
      }
      curr = curr.next;
    }
    return false;
  }

  // O(n) — convert to array
  toArray(): T[] {
    const result: T[] = [];
    let curr = this.head;
    while (curr) {
      result.push(curr.value);
      curr = curr.next;
    }
    return result;
  }

  // Make iterable
  [Symbol.iterator](): Iterator<T> {
    let curr = this.head;
    return {
      next(): IteratorResult<T> {
        if (curr) {
          const value = curr.value;
          curr = curr.next;
          return { value, done: false };
        }
        return { value: undefined as T, done: true };
      },
    };
  }
}

// ── Usage ─────────────────────────────────────────────────────
const list = new LinkedList<number>();
list.append(1);
list.append(2);
list.append(3);
list.prepend(0);

list.toArray();           // [0, 1, 2, 3]
list.remove(2);
list.toArray();           // [0, 1, 3]
list.size;                // 3

for (const val of list) {
  console.log(val);       // 0  1  3
}

Notes

Tracking a tail pointer is essential for O(1) append — without it, you'd traverse to the end on every append
Removal requires finding the previous node — you need to keep a curr.next lookahead, not just curr
[Symbol.iterator] makes the list usable with for...of and spread — any iterable protocol method works
A doubly linked list adds a prev pointer — enables O(1) backwards traversal and deletion given only the node reference
In practice, JavaScript arrays are backed by V8's highly optimised internal representations — use linked lists only when you have a measured performance reason
Linked lists appear heavily in coding interviews — practise reversal, cycle detection (Floyd's algorithm), and merge sorted lists

Common Mistakes

  • Forgetting to update tail on append, prepend, or remove — causes corruption of the list
  • Not handling the empty list case — head and tail are null; every method needs a null check
  • Losing the head reference while traversing — always keep a separate pointer; never advance this.head to traverse

Tips

  • Add a sentinel (dummy) head node to simplify edge cases — removes special handling for empty list and single-element list
  • Implement [Symbol.iterator] to make your list first-class in JS — works with for...of, spread, destructuring, and Array.from
  • For most real JS use cases, an array-based queue/deque is better — only reach for a linked list when you need guaranteed O(1) prepend at scale

Summary

  • Node pattern: { value, next } · list tracks head and tail
  • O(1) prepend/append (with tail) · O(n) access · O(1) delete (given node reference)
  • Implement [Symbol.iterator] for for...of compatibility
  • In JS, arrays are usually faster in practice — use linked lists for specific algorithmic needs
