Express Basics

app.use · routing · middleware · req · res · body parsing · static files

req — Request Object

Property / MethodNotes
req.paramsRoute params: /users/:id → { id: "1" }
req.queryQuery string: ?page=2 → { page: "2" }
req.bodyParsed body (requires body-parser middleware)
req.headersAll request headers as an object
req.method"GET" | "POST" | "PUT" | "PATCH" | "DELETE"
req.pathPath without query string
req.ipClient IP address
req.get(header)Get a specific request header by name

res — Response Object

MethodNotes
res.json(data)Send JSON; sets Content-Type automatically
res.send(body)Send string, Buffer, or object
res.status(code)Set status code (chainable)
res.sendStatus(code)Set status + send its text as body
res.redirect(url)302 redirect; use res.redirect(301, url) for permanent
res.set(header, val)Set a response header
res.cookie(name, val)Set a cookie
res.sendFile(path)Stream a file to the client

Express Examples

// npm install express
// npm install -D @types/express  (TypeScript)
import express from "express";
const app = express();

// ── Middleware ────────────────────────────────────────────────
app.use(express.json());                    // parse JSON bodies
app.use(express.urlencoded({ extended: true })); // parse form data
app.use(express.static("public"));          // serve static files

// Custom middleware — runs on every request
app.use((req, res, next) => {
  console.log(`${req.method} ${req.path}`);
  next();                                   // must call next() or respond
});

// ── Route handlers ────────────────────────────────────────────
// GET /
app.get("/", (req, res) => {
  res.send("Hello, Express!");
});

// GET /users/:id — route parameter
app.get("/users/:id", (req, res) => {
  const { id } = req.params;               // always a string
  res.json({ id: Number(id), name: "Alice" });
});

// GET /search?q=foo&page=2 — query string
app.get("/search", (req, res) => {
  const { q = "", page = "1" } = req.query;
  res.json({ query: q, page: Number(page) });
});

// POST /users — request body
app.post("/users", (req, res) => {
  const { name, email } = req.body;        // parsed by express.json()
  res.status(201).json({ id: 1, name, email });
});

// PUT /users/:id
app.put("/users/:id", (req, res) => {
  res.json({ id: req.params.id, ...req.body });
});

// DELETE /users/:id
app.delete("/users/:id", (req, res) => {
  res.sendStatus(204);
});

// ── Chaining route methods ────────────────────────────────────
app.route("/items")
  .get((req, res) => res.json([]))
  .post((req, res) => res.status(201).json(req.body));

// ── Error handler (4 args — must be last) ────────────────────
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(err.status ?? 500).json({ error: err.message });
});

app.listen(3000, () => console.log("Express on :3000"));

Notes

app.use() — registers middleware; runs in declaration order; always call next() to pass control or send a response to end the chain
express.json() populates req.body — must be registered before any routes that read req.body
req.params values are always strings — convert with Number() or zod when expecting integers
res.status(code).json(data) — chain .status() before .json(); .json() ends the response
Error middleware must have exactly 4 parameters (err, req, res, next) — Express identifies it by arity
express.static("public") serves files from the public/ folder at the root path; use app.use("/static", express.static(...)) to prefix

Common Mistakes

  • Registering express.json() after route definitions — req.body is undefined in those routes
  • Forgetting next() in middleware — request hangs with no response
  • Error handler with 3 params — Express won't recognise it as an error handler; the error propagates unhandled
  • Sending multiple responses — calling res.json() twice throws "Cannot set headers after they are sent"

Tips

  • Wrap async route handlers: app.get("/", asyncHandler(async (req, res) => { ... })) — catches rejected promises and forwards to error middleware
  • Use express.Router() (Sheet 90) to group related routes into separate files — keeps app.js clean
  • Validate req.body and req.params with zod at the top of each handler — reject bad input before it reaches business logic

Summary

  • app.use(fn) middleware · app.get/post/put/patch/delete(path, fn) routes
  • express.json() before routes · express.static(dir) for files
  • req.params · req.query · req.body · req.headers
  • res.status(n).json(data) · error handler needs 4 params · always end the response
