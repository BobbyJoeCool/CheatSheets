Sheet 06

Console Output

console.log · warn · error · table · dir · group · time · count

console Methods — Quick Reference

MethodLog LevelNotes
console.log()infoGeneral output — most common
console.info()infoIdentical to log; semantic label for informational messages
console.warn()warnYellow in DevTools; used for deprecation warnings
console.error()errorRed; writes to stderr in Node — separate from stdout
console.debug()debugHidden by default in DevTools; shown at verbose log level
console.table()infoRenders array of objects as formatted table
console.dir()infoDeep-inspects an object; useful for DOM nodes
console.group()infoStarts collapsible indent group in DevTools
console.groupEnd()—Closes the current indent group
console.time()—Starts a named timer
console.timeEnd()—Stops timer and prints elapsed time
console.count()—Counts how many times called with a given label
console.countReset()—Resets a count label to zero
console.assert()errorLogs error only if first argument is falsy
console.clear()—Clears the console (browser/terminal)

Examples

// ── Basic output ─────────────────────────────────────────────
console.log("Hello");                  // Hello
console.log(42, true, [1, 2]);         // 42 true [ 1, 2 ]
console.log("Val:", myVar);            // Val: <myVar>

// ── Log levels ───────────────────────────────────────────────
console.info("Server ready on :3000"); // ℹ informational
console.warn("Deprecated: use v2 API");// ⚠ yellow
console.error("Connection failed");    // ✖ red; → stderr in Node
console.debug("State:", state);        // verbose level only

// ── Structured output ────────────────────────────────────────
console.table([
  { name: "Alice", age: 30 },
  { name: "Bob",   age: 25 },
]);
// ┌─────────┬───────┬─────┐
// │ (index) │ name  │ age │
// ├─────────┼───────┼─────┤
// │    0    │ Alice │  30 │
// │    1    │  Bob  │  25 │

console.dir(document.body, { depth: 2 }); // DOM node properties

// ── Grouping ─────────────────────────────────────────────────
console.group("Request cycle");
console.log("→ GET /api/users");
console.log("← 200 OK");
console.groupEnd();

// ── Timing ───────────────────────────────────────────────────
console.time("fetch");
const data = await getData();
console.timeEnd("fetch");              // fetch: 142.3ms

// ── Counting ─────────────────────────────────────────────────
console.count("click");               // click: 1
console.count("click");               // click: 2
console.countReset("click");

// ── Assertion ────────────────────────────────────────────────
console.assert(1 === 2, "Math broke");// Assertion failed: Math broke
console.assert(1 === 1, "Never seen");// (silent — condition is true)

// ── Format specifiers ────────────────────────────────────────
console.log("Name: %s, Age: %d", "Alice", 30); // Name: Alice, Age: 30
console.log("%cStyled", "color:hotpink");        // browser only

Notes

log accepts multiple arguments — all are printed space-separated
console.error writes to stderr in Node — pipe separately with 2>
console.table is the fastest way to visualise an array of objects during debugging
console.dir shows object properties; unlike log, it doesn't render DOM nodes as HTML
%s string · %d number · %o object · %c CSS (browser only)
console.time/timeEnd — instant benchmarking; no imports required
console.assert — only fires on falsy; great for lightweight runtime checks in dev

Common Mistakes

  • Leaving console.log calls in production code — use a linter rule to catch them
  • Using console.log for errors — use console.error so they go to stderr
  • Forgetting console.groupEnd() — subsequent logs stay indented in DevTools

Tips

  • Use console.table when inspecting API responses or arrays of objects — far more readable than nested log
  • Add "no-console": "warn" to your ESLint config to flag leftover debug logs before committing
  • console.time/timeEnd is the quickest way to profile a block without any profiler setup

Summary

  • log/info/warn/error/debug — same output, different severity levels and colours
  • error writes to stderr in Node; all others write to stdout
  • table/dir/group/time/count — structured debugging helpers
  • Format specifiers: %s string · %d number · %c CSS (browser)
