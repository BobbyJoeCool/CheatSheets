Stack & Queue Examples

// ── Array-based stack ─────────────────────────────────────────
const stack: number[] = [];
stack.push(1);           // [1]
stack.push(2);           // [1, 2]
stack.push(3);           // [1, 2, 3]
stack.pop();             // 3 — [1, 2]
stack.at(-1);            // 2 — peek without removing
stack.length === 0;      // false

// Real-world: undo history
const history: string[] = [];
function doAction(action: string) { history.push(action); }
function undo() { return history.pop(); }

// Real-world: balanced brackets check
function isBalanced(str: string): boolean {
  const stack: string[] = [];
  const pairs: Record<string, string> = { ")": "(", "]": "[", "}": "{" };
  for (const ch of str) {
    if ("([{".includes(ch)) { stack.push(ch); }
    else if (")]}" .includes(ch)) {
      if (stack.pop() !== pairs[ch]) return false;
    }
  }
  return stack.length === 0;
}
isBalanced("({[]})");    // true
isBalanced("({[}])");    // false

// ── Array-based queue ─────────────────────────────────────────
const queue: string[] = [];
queue.push("first");     // ["first"]
queue.push("second");    // ["first", "second"]
queue.push("third");     // ["first", "second", "third"]
queue.shift();           // "first" — ["second", "third"]

// Real-world: task queue
async function processQueue(tasks: (() => Promise<void>)[]) {
  const q = [...tasks];
  while (q.length) {
    const task = q.shift()!;
    await task();
  }
}

// ── Class wrapper for type safety ────────────────────────────
class Stack<T> {
  private items: T[] = [];
  push(item: T): void   { this.items.push(item); }
  pop(): T | undefined  { return this.items.pop(); }
  peek(): T | undefined { return this.items.at(-1); }
  get size(): number    { return this.items.length; }
  get isEmpty(): boolean{ return this.items.length === 0; }
}

class Queue<T> {
  private items: T[] = [];
  enqueue(item: T): void  { this.items.push(item); }
  dequeue(): T | undefined{ return this.items.shift(); }
  peek(): T | undefined   { return this.items[0]; }
  get size(): number      { return this.items.length; }
  get isEmpty(): boolean  { return this.items.length === 0; }
}

const s = new Stack<number>();
s.push(1); s.push(2);
s.pop();                 // 2

const q = new Queue<string>();
q.enqueue("a"); q.enqueue("b");
q.dequeue();             // "a"

Notes

Stack (LIFO) — Last In, First Out; push + pop are both O(1)
Queue (FIFO) — First In, First Out; push is O(1) but shift is O(n) because all elements are re-indexed
For high-performance queues (millions of operations), use a linked-list based queue or a circular buffer — array shift is too slow
The class wrapper pattern adds type safety and a clear API — prevents accidentally using wrong array methods on your stack/queue
Stack use cases: undo/redo, browser history, expression parsing, depth-first search, call stack simulation
Queue use cases: task scheduling, BFS traversal, event processing, rate limiting, producer-consumer patterns

Common Mistakes

  • Using arr.shift() as the dequeue operation in performance-critical code — it is O(n)
  • Not checking isEmpty before pop()/shift() — both return undefined on empty arrays silently
  • Using a queue where a stack is needed (or vice versa) — mixing FIFO and LIFO patterns causes subtle ordering bugs

Tips

  • For small task queues, the array + shift() approach is perfectly fine — only optimise when queue length grows into the thousands
  • Use the generic class wrapper in TypeScript projects — it prevents mixing stack and queue operations accidentally
  • The balanced-brackets check is a classic coding interview problem — practise it as the archetypal stack algorithm

Summary

  • Stack (LIFO) — push + pop — both O(1); uses: undo, DFS, parsing
  • Queue (FIFO) — push + shift — shift is O(n); uses: task scheduling, BFS
  • Array-based implementations are idiomatic JS; wrap in a class for type safety
  • For high-throughput queues, use a linked-list or circular buffer instead
