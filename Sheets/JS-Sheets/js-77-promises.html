Promise Examples

// ── Creating a Promise ────────────────────────────────────────
const p = new Promise((resolve, reject) => {
  // async work here
  setTimeout(() => {
    if (Math.random() > 0.5) resolve("success");
    else reject(new Error("failed"));
  }, 1000);
});

// ── .then / .catch / .finally ─────────────────────────────────
p
  .then(value => console.log("Resolved:", value))
  .catch(err  => console.error("Rejected:", err.message))
  .finally(()  => console.log("Always runs"));

// ── Chaining ──────────────────────────────────────────────────
fetch("/api/users")
  .then(res  => res.json())          // res → data
  .then(data => process(data))       // data → result
  .then(result => save(result))      // result → saved
  .catch(err   => console.error(err));

// ── Promise.all — parallel, fail fast ────────────────────────
const [user, posts, comments] = await Promise.all([
  fetchUser(1),
  fetchPosts(1),
  fetchComments(1),
]);
// All run in parallel; throws if any rejects

// ── Promise.allSettled — parallel, never fails ────────────────
const results = await Promise.allSettled([
  fetchUser(1),
  fetchUser(999),   // 404 — rejected
  fetchUser(2),
]);
results.forEach(r => {
  if (r.status === "fulfilled") console.log(r.value);
  else                          console.error(r.reason);
});

// ── Promise.race — timeout pattern ───────────────────────────
function withTimeout(promise, ms) {
  const timeout = new Promise((_, reject) =>
    setTimeout(() => reject(new Error("Timeout")), ms)
  );
  return Promise.race([promise, timeout]);
}
const data = await withTimeout(fetchSlowData(), 5000);

// ── Promise.any — first success ───────────────────────────────
const fastest = await Promise.any([
  fetch("https://mirror1.example.com/file"),
  fetch("https://mirror2.example.com/file"),
  fetch("https://mirror3.example.com/file"),
]);
// First successful response wins

// ── Promisify a callback-based API ───────────────────────────
function readFilePromise(path) {
  return new Promise((resolve, reject) => {
    fs.readFile(path, "utf8", (err, data) => {
      if (err) reject(err);
      else     resolve(data);
    });
  });
}
// Modern Node: use fs/promises instead

Notes

Promises have three states: pending (initial) · fulfilled (resolved) · rejected — states are final; a settled promise never changes
.then(onFulfilled, onRejected) — the second arg is rarely used; prefer a separate .catch() at the end of the chain
.finally() — runs on both resolve and reject; does not receive a value; ideal for cleanup (spinner off, connection close)
Promise.all — rejects immediately when any promise rejects (fast-fail); use allSettled when partial failures are acceptable
Promise.allSettled — every result has a status of "fulfilled" (with .value) or "rejected" (with .reason)
Unhandled promise rejections crash Node 15+ — always attach a .catch() or use try/await

Common Mistakes

  • Forgetting to return the Promise in a .then() chain — creates a "floating" promise that isn't part of the chain
  • Using Promise.all when partial failure is acceptable — one rejection kills everything; use allSettled
  • Not attaching a .catch() — unhandled rejections crash Node and cause silent failures in browsers
  • Nesting .then() inside .then() — creates callback pyramid; chain them flat instead

Tips

  • Prefer async/await over chains for new code — same power, much more readable (Sheet 78)
  • Use Promise.all to parallelise independent async operations — sequential await calls are up to N× slower
  • Use Promise.race([fetch(url), withTimeout(5000)]) as a universal timeout wrapper for external calls

Summary

  • new Promise((resolve, reject) => {}) · .then/.catch/.finally
  • Promise.all — parallel, fast-fail · allSettled — parallel, never rejects
  • Promise.race — first to settle · Promise.any — first to resolve
  • Always handle rejections · prefer async/await for readability
