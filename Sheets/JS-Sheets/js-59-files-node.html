fs/promises — Common Methods

MethodNotes
fs.readFile(path, enc)Read entire file as string or Buffer
fs.writeFile(path, data)Write/overwrite file; creates if absent
fs.appendFile(path, data)Append to file; creates if absent
fs.mkdir(path, opts)Create directory; { recursive: true } for nested
fs.readdir(path, opts)List directory contents; { withFileTypes: true } for Dirents
fs.stat(path)File metadata: size, mtime, isDirectory()
fs.access(path, mode)Check if path exists and is accessible
fs.rename(src, dest)Move or rename file/directory
fs.copyFile(src, dest)Copy a file
fs.unlink(path)Delete a file
fs.rm(path, opts)Delete file or directory; { recursive: true }
fs.watch(path, cb)Watch for file/directory changes

File I/O Examples

import { readFile, writeFile, appendFile,
         mkdir, readdir, stat, unlink } from "fs/promises";
import { existsSync, readFileSync } from "fs"; // sync versions

// ── Read ──────────────────────────────────────────────────────
const text   = await readFile("./data.txt", "utf8");     // string
const buffer = await readFile("./image.png");             // Buffer

// Sync (blocks — only use at startup or in scripts)
const cfg = readFileSync("./config.json", "utf8");

// ── Write / Overwrite ─────────────────────────────────────────
await writeFile("./output.txt", "Hello, world!", "utf8");
await writeFile("./data.json", JSON.stringify(obj, null, 2));

// ── Append ────────────────────────────────────────────────────
await appendFile("./log.txt", `[${new Date().toISOString()}] Event\n`);

// ── Check existence ───────────────────────────────────────────
if (existsSync("./config.json")) {           // sync check
  const cfg = await readFile("./config.json", "utf8");
}

// Using stat (async)
try {
  await stat("./file.txt");                  // throws if not found
  console.log("exists");
} catch {
  console.log("not found");
}

// ── Directory operations ──────────────────────────────────────
await mkdir("./output/nested", { recursive: true }); // no error if exists

const entries = await readdir("./src", { withFileTypes: true });
for (const entry of entries) {
  if (entry.isDirectory()) console.log("DIR:", entry.name);
  else                     console.log("FILE:", entry.name);
}

// ── Delete ────────────────────────────────────────────────────
await unlink("./temp.txt");                          // file only
// await rm("./dist", { recursive: true, force: true }); // dir

// ── Error handling ────────────────────────────────────────────
try {
  const data = await readFile("./missing.txt", "utf8");
} catch (err) {
  if (err.code === "ENOENT") console.error("File not found");
  else throw err;                   // rethrow unexpected errors
}

// ── Streaming large files ─────────────────────────────────────
import { createReadStream, createWriteStream } from "fs";
import { pipeline } from "stream/promises";
import { createGzip } from "zlib";

await pipeline(
  createReadStream("./big.log"),
  createGzip(),
  createWriteStream("./big.log.gz")
);

Notes

fs/promises — modern async API; all methods return Promises; use with await
Sync methods (readFileSync) block the event loop — only use in startup scripts or CLI tools, never in a server handler
Common error codes: ENOENT file not found · EACCES permission denied · EEXIST already exists · EISDIR is a directory
readFile without encoding returns a Buffer — call .toString("utf8") or pass "utf8" as the second argument
Streaming with createReadStream + pipeline — processes large files chunk by chunk without loading everything into memory
{ recursive: true } on mkdir — creates all intermediate directories and does not throw if the directory already exists

Common Mistakes

  • Using sync methods inside async request handlers — blocks the entire Node event loop for all concurrent requests
  • Not catching file errors — unhandled ENOENT crashes the process in Node 15+
  • Using readFile on large files (GB+) — loads everything into memory; use streams instead
  • Forgetting "utf8" encoding — readFile returns a Buffer without it, not a string

Tips

  • Import from fs/promises not fs for clean async code — no callback pyramid
  • Use stream/promises pipeline() for file transformations — it handles backpressure and cleans up on error automatically
  • Always check err.code in catch blocks — handle ENOENT gracefully; rethrow everything else

Summary

  • fs/promises — async API; readFile · writeFile · appendFile · mkdir · readdir · stat · unlink
  • Pass "utf8" to get a string; omit for a Buffer
  • Error codes: ENOENT not found · EACCES permission · EEXIST exists
  • Large files → use streams with pipeline() instead of readFile
