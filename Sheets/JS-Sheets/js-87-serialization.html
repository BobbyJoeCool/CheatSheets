structuredClone · JSON · MessageChannel · Buffer · Base64 · deep clone patterns

Serialization Examples

// ── structuredClone — best general-purpose deep clone ─────────
const original = {
  name: "Alice",
  scores: [1, 2, 3],
  created: new Date(),
  prefs: new Map([["theme", "dark"]]),
  tags: new Set(["js", "ts"]),
};

const clone = structuredClone(original);
clone.scores.push(4);
original.scores;                   // [1, 2, 3] — unaffected
clone.created instanceof Date;     // true  — Date preserved
clone.prefs instanceof Map;        // true  — Map preserved

// Circular references work
const circular = { self: null };
circular.self = circular;
const cloned = structuredClone(circular); // ✔ no error

// ── structuredClone does NOT support ─────────────────────────
// Functions, DOM nodes, class instances (become plain objects),
// getters/setters, prototype chain, WeakMap/WeakSet

// ── JSON round-trip (fast but lossy) ─────────────────────────
const data = { name: "Alice", created: new Date(), fn: () => {} };
const str  = JSON.stringify(data);
// '{"name":"Alice","created":"2024-01-15T10:00:00.000Z"}'
// fn is omitted; Date became a string

const back = JSON.parse(str);
back.created instanceof Date;      // false — it's a string now

// ── Serialise with custom toJSON ─────────────────────────────
class Money {
  constructor(public amount: number, public currency: string) {}
  toJSON() { return { amount: this.amount, currency: this.currency }; }
  static fromJSON(obj) { return new Money(obj.amount, obj.currency); }
}
const price = new Money(9.99, "USD");
JSON.stringify(price);             // '{"amount":9.99,"currency":"USD"}'

// ── Reviver to restore types on parse ────────────────────────
const reviver = (key, value) => {
  if (typeof value === "string" && /^\d{4}-\d{2}-\d{2}T/.test(value)) {
    return new Date(value);        // ISO strings → Date
  }
  return value;
};
const restored = JSON.parse(str, reviver);
restored.created instanceof Date;  // true  — Date restored

// ── Buffer encoding (Node) ────────────────────────────────────
// Binary → Base64
const bin  = Buffer.from([0xff, 0x0a, 0x3d]);
const b64  = bin.toString("base64");          // "/wo="
const hex  = bin.toString("hex");             // "ff0a3d"

// Base64 → Buffer
const back2 = Buffer.from(b64, "base64");
back2.equals(bin);                // true

// String → Buffer → Base64
const encoded = Buffer.from("Hello, 世界", "utf8").toString("base64");
const decoded  = Buffer.from(encoded, "base64").toString("utf8");
// "Hello, 世界"  — handles Unicode correctly (unlike btoa/atob)

// ── MessageChannel — structured clone for workers ────────────
const { port1, port2 } = new MessageChannel();
const bigData = { items: new Array(100000).fill(0) };

port2.onmessage = ({ data }) => console.log("Received:", data.items.length);
port1.postMessage(bigData);        // deep clones via structured clone

// Transfer instead of copy (zero-copy for ArrayBuffer)
const buf = new ArrayBuffer(1024 * 1024);    // 1MB
port1.postMessage(buf, [buf]);               // transferred — buf is now empty

Notes

structuredClone — use for most deep copy needs; handles Maps, Sets, Dates, and circular refs; available in Node 17+ and all modern browsers
JSON round-trip — fast but lossy; undefined and functions are silently dropped; Date becomes a string; Map/Set become {}
The toJSON pattern — classes control their own serialisation; pair with a static fromJSON for symmetric serialisation/deserialisation
Reviver function — restore specific types (Dates, Money, etc.) during JSON parse; run bottom-up so nested objects are restored before parents
Buffer vs btoa/atob — btoa/atob only handle Latin-1; Buffer handles full UTF-8 correctly for Base64 encoding of Unicode strings
MessageChannel transfer — ArrayBuffer can be transferred (not copied) to a worker; the original becomes detached and unusable — zero-copy performance

Common Mistakes

  • Using JSON.parse(JSON.stringify(obj)) for deep cloning — silently corrupts Dates, drops undefined/functions, throws on circular refs
  • Expecting structuredClone to preserve class instances — prototype chain is lost; instances become plain objects
  • Using btoa on a string containing non-Latin characters — throws in browsers; use Buffer.from(str, "utf8").toString("base64")
  • Transferring an ArrayBuffer and then trying to use the original — the buffer is detached after transfer; it's zero-length

Tips

  • Use structuredClone as your default deep clone — it's built-in, fast, and handles far more types than JSON
  • Implement toJSON + fromJSON on domain objects that need to survive serialisation — keeps the logic in the class
  • Use a JSON reviver registered globally in your app startup to automatically restore ISO date strings to Date objects

Summary

  • structuredClone() — best deep clone; handles Map, Set, Date, circular; no class instances
  • JSON round-trip — fast, universal, lossy; restore types with a reviver
  • toJSON / fromJSON — classes control their own serialisation
  • Buffer (Node) — UTF-8 safe Base64 · MessageChannel — structured clone / zero-copy transfer
