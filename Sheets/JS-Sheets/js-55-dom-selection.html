DOM Selection Methods

MethodReturnsNotes
document.querySelector(css)Element | nullFirst match of any CSS selector
document.querySelectorAll(css)NodeList (static)All matches; not live; iterable
document.getElementById(id)HTMLElement | nullFastest single-element lookup by id
document.getElementsByClassName(cls)HTMLCollection (live)Live collection — updates as DOM changes
document.getElementsByTagName(tag)HTMLCollection (live)All elements of that tag; live
element.querySelector(css)Element | nullScope search to element's descendants
element.querySelectorAll(css)NodeList (static)Scoped to element's subtree

DOM Selection Examples

// ── querySelector / querySelectorAll ─────────────────────────
const btn    = document.querySelector("#submit");          // by id
const first  = document.querySelector(".card");            // first .card
const input  = document.querySelector("input[type=email]");// attribute
const nested = document.querySelector("nav > ul > li");    // complex

const all    = document.querySelectorAll(".card");         // NodeList
const inputs = document.querySelectorAll("input, select"); // multiple

// NodeList is iterable but NOT an array
[...all].forEach(el => el.classList.add("visible"));
Array.from(inputs).map(el => el.value);

// ── getElementById — fastest for id lookups ───────────────────
const header = document.getElementById("main-header");     // no "#"
const modal  = document.getElementById("modal");

// ── Null check — always required ─────────────────────────────
const el = document.querySelector(".missing");
if (el) {
  el.textContent = "found";   // safe
}
// Or non-null assertion (TS) when certain element exists:
const nav = document.querySelector("nav")!;

// ── Scoped queries ────────────────────────────────────────────
const sidebar = document.querySelector(".sidebar");
const links   = sidebar?.querySelectorAll("a");    // only links in sidebar

// ── closest — walk up the DOM tree ───────────────────────────
document.addEventListener("click", (e) => {
  const card = e.target.closest(".card");          // nearest ancestor
  if (card) {
    console.log("Card clicked:", card.dataset.id);
  }
});

// ── matches — test selector on an element ────────────────────
const items = document.querySelectorAll("li");
items.forEach(item => {
  if (item.matches(".active, .selected")) {
    item.style.fontWeight = "bold";
  }
});

// ── TypeScript: typed element selection ──────────────────────
// querySelector returns Element | null — cast to specific type
const input2 = document.querySelector("#email") as HTMLInputElement;
input2.value;                    // ✔ typed string

// Generic overload — type inferred from tag name
const canvas = document.querySelector("canvas");    // HTMLCanvasElement | null
const form   = document.querySelector("form");      // HTMLFormElement | null

// Manual cast with null check
const btn2 = document.getElementById("submit") as HTMLButtonElement | null;
if (btn2) btn2.disabled = true;

Notes

querySelectorAll returns a static NodeList — a snapshot taken at call time; changes to the DOM afterward don't update it
getElementsByClassName/getElementsByTagName return live HTMLCollections — they update automatically, which can cause infinite loops if you mutate the DOM while iterating
getElementById is the fastest DOM lookup — use it when selecting by a unique id
.closest(selector) — starts at the element itself and walks up the tree; returns the first ancestor (or self) that matches; essential for event delegation
TS: querySelector("canvas") returns HTMLCanvasElement | null — TS knows the return type from the tag name string literal
Always check for null before using a selected element — if the selector doesn't match, you get null, not an error

Common Mistakes

  • Not checking for null after querySelector — calling methods on null throws TypeError
  • Passing "#id" to getElementById — it takes the raw id without the hash; use querySelector("#id") if you want the hash syntax
  • Iterating a live HTMLCollection while mutating the DOM — the collection updates in real time, causing skipped items or infinite loops
  • Using overly broad selectors like document.querySelectorAll("div") — slow and fragile; be specific

Tips

  • Scope queries to a container element (container.querySelectorAll(".item")) — faster and more reliable than querying the whole document
  • Use .closest() in event handlers to find the nearest matching ancestor — the foundation of event delegation
  • Convert NodeList to array with [...nodeList] or Array.from(nodeList) to use map, filter, etc.

Summary

  • querySelector/querySelectorAll — any CSS selector; static NodeList
  • getElementById — fastest id lookup (no # needed)
  • closest(css) — walk ancestors · matches(css) — test self
  • Always null-check results · TS infers element type from tag name literals
