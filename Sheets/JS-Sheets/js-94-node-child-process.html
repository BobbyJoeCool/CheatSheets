Child Processes

exec · execFile · spawn · fork · stdio · shell commands · TS types

Child Process Functions Compared

FunctionShellOutputBest For
exec(cmd, cb)✔Buffered stringShort shell commands; output fits in memory
execFile(file, args, cb)✖Buffered stringRun a binary directly; safer than exec
spawn(cmd, args)✖StreamsLong-running processes; large output
fork(modulePath)✖Streams + IPCSpawn another Node.js script with IPC channel
execSync / spawnSync✔/✖Buffered (blocks)Scripts only — blocks the event loop

Child Process Examples

import { exec, execFile, spawn, fork } from "child_process";
import { promisify } from "util";

const execAsync = promisify(exec);

// ── exec — run a shell command ────────────────────────────────
const { stdout, stderr } = await execAsync("ls -la ./src");
console.log(stdout);

// With options
const { stdout: out } = await execAsync("git log --oneline -5", {
  cwd: "/path/to/repo",
  timeout: 5000,              // kill if takes longer than 5s
});

// ── execFile — run binary without shell ───────────────────────
const { stdout: version } = await promisify(execFile)("node", ["--version"]);
console.log(version.trim());  // "v20.11.0"

// ── spawn — streaming output (long-running) ───────────────────
const proc = spawn("npm", ["install"], { cwd: "./my-project" });

proc.stdout.on("data", chunk => process.stdout.write(chunk));
proc.stderr.on("data", chunk => process.stderr.write(chunk));

const exitCode = await new Promise((resolve, reject) => {
  proc.on("close", resolve);
  proc.on("error", reject);
});
console.log("Exit code:", exitCode);

// ── spawn with pipe — capture output as string ────────────────
function runCommand(cmd, args, options = {}) {
  return new Promise((resolve, reject) => {
    const child = spawn(cmd, args, { ...options, stdio: "pipe" });
    const out = [], err = [];
    child.stdout.on("data", d => out.push(d));
    child.stderr.on("data", d => err.push(d));
    child.on("close", code => {
      if (code === 0) resolve(Buffer.concat(out).toString());
      else reject(new Error(Buffer.concat(err).toString()));
    });
  });
}
const result = await runCommand("git", ["status", "--short"]);

// ── fork — spawn another Node script with IPC ─────────────────
// child.js:
process.on("message", ({ task, data }) => {
  const result = heavyWork(data);
  process.send({ task, result });
});

// main.js:
const child = fork("./child.js");
child.send({ task: "compute", data: [1, 2, 3] });
child.on("message", ({ result }) => console.log(result));
child.on("exit",    code => console.log("Done:", code));

// ── stdio options ─────────────────────────────────────────────
spawn("node", ["script.js"], {
  stdio: "inherit",           // child uses parent's stdin/stdout/stderr
});
spawn("node", ["script.js"], {
  stdio: "pipe",              // capture all stdio
});
spawn("node", ["script.js"], {
  stdio: ["inherit", "pipe", "pipe"], // stdin inherit, stdout+err piped
});

Notes

exec uses a shell — allows pipes and shell features (ls | grep); never pass unsanitised user input to exec (shell injection risk)
execFile skips the shell — faster and safer for running known binaries; arguments are passed as an array, not concatenated
spawn returns streams — handles GB of output without buffering in memory; the right choice for long-running processes
fork is spawn specialised for Node scripts — adds a built-in IPC channel; process.send() and .on("message") work automatically
stdio: "inherit" — child shares parent's terminal; useful for interactive processes or forwarding coloured output
Always handle "error" events — if the binary isn't found, spawn emits an error (not an exit code)

Common Mistakes

  • Passing user input to exec() — shell injection vulnerability; use execFile() or spawn() with args array instead
  • Using execSync in a server — blocks the entire event loop for all concurrent requests
  • Not setting a timeout on exec — a hung process keeps your Node process alive indefinitely
  • Ignoring stderr — exit code 0 doesn't mean success; check stderr for warnings and errors

Tips

  • Use promisify(exec) for clean async/await syntax with short commands; use spawn for streaming output
  • Always set { cwd } explicitly when running commands relative to a specific directory — don't rely on the inherited working directory
  • Use fork + IPC instead of worker threads when you want to run a completely separate Node script with its own module graph

Summary

  • exec — shell, buffered · execFile — no shell, safer · spawn — streaming · fork — Node + IPC
  • Never pass user input to exec — use spawn with args array
  • Sync variants block the event loop — CLI/scripts only
  • Always handle "error" + "close" events; set a timeout
