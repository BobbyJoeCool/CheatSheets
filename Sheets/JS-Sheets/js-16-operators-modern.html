ES2020+ Operators

Several operators added in ES2015–ES2021 are now ubiquitous in modern JS/TS and deserve focused attention. Nullish coalescing (??) provides defaults only for null/undefined — unlike || which also triggers on 0, "", and false. Optional chaining (?.) short-circuits property access, method calls, and array indexing when the left side is nullish. The spread (...) and rest (...) syntax share the same token but do opposite things depending on position.

At a Glance

  • ?? — default if null/undefined only
  • || — default if any falsy value
  • ?. — safe property/method/index access
  • ??= — assign only if null/undefined
  • ||= — assign only if falsy
  • &&= — assign only if truthy
  • ...arr — spread: expand into positions
  • ...rest — rest: collect remaining into array
  • a ? b : c — ternary conditional

Modern Operator Examples

// ── ?? nullish coalescing ────────────────────────────────────
const port = process.env.PORT ?? 3000;  // 3000 if PORT is null/undefined
const name = user.name ?? "Anonymous"; // "Anonymous" only if null/undefined

// vs || which also triggers on 0, "", false:
const count = userCount || 10;   // 10 even if userCount is 0 (bug!)
const count2 = userCount ?? 10;  // 10 only if userCount is null/undefined

// ── ?. optional chaining ─────────────────────────────────────
const city = user?.address?.city;     // undefined if any step is null
const len  = str?.length;             // undefined if str is null
const val  = arr?.[0];                // undefined if arr is null
const res  = obj?.method?.();         // undefined if method missing

// Combining with ?? for a default:
const city2 = user?.address?.city ?? "Unknown";

// ── ??= ||= &&= logical assignment ───────────────────────────
let cache = null;
cache ??= expensiveLoad();    // assign only if null/undefined

let label = "";
label ||= "Default";          // assign because "" is falsy → "Default"

let isAdmin = true;
isAdmin &&= checkPermissions();// assign only if currently truthy

// ── Spread operator (...) ────────────────────────────────────
// Arrays — expand into individual elements
const a = [1, 2, 3];
const b = [...a, 4, 5];       // [1, 2, 3, 4, 5]
const c = [...a, ...b];       // [1, 2, 3, 1, 2, 3, 4, 5]
Math.max(...a);               // same as Math.max(1, 2, 3)

// Objects — shallow copy / merge
const defaults = { color: "blue", size: 10 };
const custom   = { size: 20, weight: "bold" };
const merged   = { ...defaults, ...custom }; // { color:"blue", size:20, weight:"bold" }

// ── Rest (...) — collect remaining ──────────────────────────
function sum(...nums) {           // nums is an array
  return nums.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3, 4);               // 10

const [first, second, ...rest] = [1, 2, 3, 4, 5];
// first=1, second=2, rest=[3,4,5]

const { a: alpha, ...others } = { a: 1, b: 2, c: 3 };
// alpha=1, others={b:2, c:3}

// ── Ternary ──────────────────────────────────────────────────
const label2 = isActive ? "Active" : "Inactive";
const cls    = score > 90 ? "A" : score > 80 ? "B" : "C"; // chained

Notes

?? only checks for null/undefined — values like 0, "", false pass through unchanged
?. short-circuits the entire chain — if any step is nullish, the whole expression evaluates to undefined
Spread creates a shallow copy — nested objects are still shared by reference
In object spread, later keys override earlier ones — useful for applying overrides to defaults
Rest must be the last parameter/element — function(a, ...b, c) is a SyntaxError
Chained ternaries are valid but hard to read — prefer if/else or a lookup table for 3+ branches

Common Mistakes

  • Using || for defaults when 0 or "" are valid — those falsy values get replaced unexpectedly
  • Assuming spread is a deep clone — nested objects are still references
  • Putting rest param in the middle of a parameter list — must be last
  • Deeply chaining ternaries — becomes unreadable; use early returns or a map instead

Tips

  • Default to ?? over || for fallback values — it is more precise and rarely surprising
  • Use structuredClone(obj) when you need a true deep clone of an object or array
  • Use ??= to lazily initialise cache entries: map.get(key) ??= computeValue(key)

Summary

  • ?? nullish default · ?. safe access · always prefer over || and manual null checks
  • ??= · ||= · &&= — logical assignment; assign conditionally based on current value
  • ...spread — expand array/object · ...rest — collect remainder (must be last)
  • Spread is shallow — use structuredClone() for deep copies
