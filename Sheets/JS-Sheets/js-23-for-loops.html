For Loops

for · for...of · for...in · break · continue · label · loop patterns

For Loop Types — Quick Reference

LoopUse WhenNotes
for (init; cond; step)Need index · counting · fixed iterationsMost explicit; full control over iteration
for...ofIterating values of arrays, strings, Maps, SetsModern standard; works with any iterable
for...inIterating keys of a plain objectReturns string keys; avoid on arrays
Array.forEach()Side-effect per array elementCannot break early; no await inside
for await...ofAsync iterables, streams, generatorsMust be inside async function

For Loop Examples

// ── Classic for loop ─────────────────────────────────────────
for (let i = 0; i < 5; i++) {
  console.log(i);               // 0 1 2 3 4
}

// Count down
for (let i = 5; i > 0; i--) {
  console.log(i);               // 5 4 3 2 1
}

// Step by 2
for (let i = 0; i <= 10; i += 2) {
  console.log(i);               // 0 2 4 6 8 10
}

// ── for...of — iterate values ────────────────────────────────
const fruits = ["apple", "banana", "cherry"];
for (const fruit of fruits) {
  console.log(fruit);           // apple  banana  cherry
}

// With index using entries()
for (const [i, fruit] of fruits.entries()) {
  console.log(i, fruit);        // 0 apple  1 banana  2 cherry
}

// Iterating a string
for (const char of "hello") {
  console.log(char);            // h e l l o
}

// Iterating a Map
const map = new Map([["a", 1], ["b", 2]]);
for (const [key, val] of map) {
  console.log(key, val);        // a 1  b 2
}

// ── for...in — iterate object keys ───────────────────────────
const user = { name: "Alice", age: 30, city: "NYC" };
for (const key in user) {
  console.log(key, user[key]);  // name Alice  age 30  city NYC
}
// Avoid for...in on arrays — iterates index strings + prototype keys

// ── break and continue ───────────────────────────────────────
for (let i = 0; i < 10; i++) {
  if (i === 3) continue;        // skip 3
  if (i === 6) break;           // stop at 6
  console.log(i);               // 0 1 2 4 5
}

// ── Labeled break — exit outer loop ─────────────────────────
outer: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (j === 1) break outer;   // exits both loops
    console.log(i, j);          // 0 0
  }
}

// ── for await...of — async iteration ────────────────────────
async function readLines(stream) {
  for await (const line of stream) {
    console.log(line);
  }
}

Notes

for...of — works on any iterable (arrays, strings, Maps, Sets, generators); preferred over classic for for most array work
for...in — iterates enumerable string keys including inherited ones; always use hasOwnProperty or Object.keys() instead for plain objects
Array.forEach() — cannot be exited early with break; use for...of when you need to break
break exits the loop entirely; continue skips to the next iteration
Labels are rare — only use when breaking out of nested loops is genuinely the clearest solution
for await...of — requires the enclosing function to be async; used with async generators and Node streams

Common Mistakes

  • Using for...in on arrays — iterates index strings and may include prototype properties
  • Using var in a for loop — all iterations share the same variable due to hoisting; use let
  • Trying to break inside .forEach() — it doesn't work; use for...of instead
  • Off-by-one errors — i < arr.length vs i <= arr.length (the latter goes out of bounds)

Tips

  • Default to for...of for arrays — it is cleaner and avoids index management
  • Use arr.entries() with for...of when you need both the index and the value
  • Use Object.keys(), Object.values(), or Object.entries() with for...of instead of for...in for objects

Summary

  • for (init; cond; step) — use when you need an index or precise iteration control
  • for...of — preferred for arrays, strings, Maps, Sets; supports break
  • for...in — object keys only; avoid on arrays
  • break exits · continue skips · labels exit nested loops
