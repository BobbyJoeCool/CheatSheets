Regex Examples

// ── Creating regexes ──────────────────────────────────────────
const re1 = /hello/i;                 // literal syntax (preferred)
const re2 = new RegExp("hello", "i"); // constructor (for dynamic patterns)
const pattern = "world";
const re3 = new RegExp(`\\b${pattern}\\b`, "gi"); // escape user input!

// ── test — check for match ────────────────────────────────────
/^\d{5}$/.test("12345");              // true  — 5-digit zip code
/^\d{5}$/.test("1234");               // false
/[a-z]/i.test("Hello");               // true

// ── match — extract matches ───────────────────────────────────
"Hello World".match(/\b\w+/g);        // ["Hello", "World"]
"2024-01-15".match(/(\d{4})-(\d{2})-(\d{2})/);
// ["2024-01-15", "2024", "01", "15"]  index 0=full, 1,2,3=groups

// Named groups
const m = "2024-01-15".match(/(?<y>\d{4})-(?<m>\d{2})-(?<d>\d{2})/);
m?.groups;                            // { y:"2024", m:"01", d:"15" }

// ── matchAll — all matches with capture groups ────────────────
const str = "cat bat hat";
const matches = [...str.matchAll(/(\w)at/g)];
matches.map(m => m[1]);              // ["c", "b", "h"]

// ── replace ───────────────────────────────────────────────────
"foo bar baz".replace(/\b\w/g, c => c.toUpperCase()); // "Foo Bar Baz"
"2024-01-15".replace(
  /(?<y>\d{4})-(?<m>\d{2})-(?<d>\d{2})/,
  "$<d>/$<m>/$<y>"                      // reorder with named groups
);                                    // "15/01/2024"

// ── split ─────────────────────────────────────────────────────
"one, two,three ,four".split(/\s*,\s*/); // ["one","two","three","four"]

// ── Common patterns ───────────────────────────────────────────
const EMAIL    = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
const URL_RE   = /^https?:\/\/[^\s/$.?#].[^\s]*$/i;
const UUID     = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;
const HEX_COLOR= /^#([0-9a-f]{3}|[0-9a-f]{6})$/i;
const ISO_DATE = /^\d{4}-\d{2}-\d{2}$/;
const SLUG     = /^[a-z0-9]+(?:-[a-z0-9]+)*$/;

// ── Escape user input for dynamic regex ──────────────────────
function escapeRegex(str) {
  return str.replace(/[.*+?^${}()|[\]\\]/g, "\\$&");
}
const search = escapeRegex(userInput);
const re = new RegExp(search, "gi");

Notes

Regex literal /pattern/flags is compiled at parse time — use the RegExp constructor only when the pattern is dynamic
str.match() with /g returns all full matches as an array; without /g returns the first match with capture groups
matchAll requires the /g flag and returns an iterator — spread with [...] to get an array
Named groups (?<name>...) — access via match.groups.name; reference in replace with $<name>
Always escape user input before inserting into a RegExp constructor — special chars like . and * have regex meaning
Greedy vs lazy — .* matches as much as possible; .*? matches as little as possible

Common Mistakes

  • Forgetting to escape user input in new RegExp(userInput) — special chars break the pattern or create security issues
  • Using match() expecting capture groups with /g — the /g flag returns only full matches; use matchAll for groups
  • Using stateful regex (/pattern/g) shared between calls — lastIndex persists between calls on the same regex object
  • Using regex for HTML/XML parsing — regex can't handle nested structures; use a parser

Tips

  • Use named capture groups for complex patterns — (?<year>\d{4}) is self-documenting and robust to position changes
  • Test regexes at regex101.com — interactive testing with explanation and match highlighting
  • Prefer str.replaceAll() over /pattern/g for simple string replacements — clearer intent, no regex escaping needed

Summary

  • /pattern/flags literal or new RegExp(str, flags) for dynamic patterns
  • test → boolean · match → first or all · matchAll → all with groups
  • Named groups (?<name>...) → match.groups.name → $<name> in replace
  • Always escape user input before using in new RegExp()
