TS Access Modifiers

public · private · protected · readonly · abstract · parameter properties

Access Modifier Reference

ModifierAccessible FromInheritedNotes
publicAnywhere✔Default — no modifier needed; explicit for clarity
privateThis class only✖Compile-time only — use # for runtime enforcement
protectedThis class + subclasses✔Not accessible on instances from outside
readonlyAnywhere (read only)✔Can only be set in constructor or at declaration
abstractN/A — must be overridden✔Class and/or members must be implemented by subclass
staticVia class name✔On the class, not instances; can combine with private/readonly
overrideSubclass method—Explicit override marker (TS 4.3+); errors if parent lacks the method

Access Modifier Examples

// ── All modifiers on a class ──────────────────────────────────
class Person {
  public  name: string;           // accessible everywhere (default)
  private age: number;            // compile-time private
  protected email: string;        // accessible in subclasses
  readonly id: string;            // set once; cannot reassign

  constructor(name: string, age: number, email: string) {
    this.name  = name;
    this.age   = age;
    this.email = email;
    this.id    = crypto.randomUUID();
  }

  public  greet() { return `Hi, I'm ${this.name}`; }
  private validate() { return this.age >= 0; }
  protected getInfo() { return `${this.name} <${this.email}>`; }
}

const p = new Person("Alice", 30, "a@b.com");
p.name;           // ✔
p.age;            // ✖ Error: private
p.email;          // ✖ Error: protected
p.id = "new";     // ✖ Error: readonly

// ── Parameter properties — shorthand ─────────────────────────
// Instead of declaring + assigning manually:
class Point {
  constructor(
    public  x: number,            // declares AND assigns in one step
    public  y: number,
    private label: string = ""
  ) {}                            // no body needed

  toString() { return `${this.label}(${this.x},${this.y})`; }
}
const pt = new Point(3, 4, "P");
pt.x;             // 3  ✔
pt.label;         // ✖ Error: private

// ── protected — accessible in subclasses ─────────────────────
class Employee extends Person {
  constructor(name: string, age: number, email: string,
              public department: string) {
    super(name, age, email);
  }

  getEmployeeInfo() {
    return `${this.getInfo()} — ${this.department}`; // ✔ protected
    // this.age would ✖ Error — private, not accessible in subclass
  }
}

// ── abstract class ────────────────────────────────────────────
abstract class Animal {
  abstract sound(): string;       // must be implemented by subclass
  abstract name: string;          // abstract property

  describe() {                    // concrete method — inherited as-is
    return `${this.name} says ${this.sound()}`;
  }
}
// new Animal();   // ✖ Error: cannot instantiate abstract class

class Cat extends Animal {
  name = "Cat";
  sound() { return "meow"; }      // must implement
}

new Cat().describe();             // "Cat says meow"

// ── override keyword (TS 4.3+) ───────────────────────────────
class GuideDog extends Employee {
  constructor(name: string, email: string) {
    super(name, 0, email, "Assistance");
  }

  override greet() {              // ✔ parent has greet()
    return `Woof! I'm ${this.name}`;
  }
}

Notes

TS private is compile-time only — at runtime the property is accessible; use #field (JS private) for true runtime enforcement
Parameter properties — adding a modifier in the constructor parameter list declares and assigns in one step; significantly less boilerplate
protected — accessible within the class and any subclass, but not on instances from outside the hierarchy
readonly — can be set in the constructor or at declaration; any reassignment after that is a compile error
abstract class — cannot be instantiated directly; serves as a base that defines required member shapes
override keyword — signals intent and errors if the parent class doesn't have the method; prevents accidental method name typos

Common Mistakes

  • Relying on TS private for runtime security — it's compile-time only; use #field for actual enforcement
  • Accessing a protected member on an instance from outside its class hierarchy — TS errors even though it looks like a normal property
  • Forgetting to implement all abstract members in a subclass — TS errors at the class definition, not the call site
  • Overriding a method without the override keyword — works but silently breaks if the parent method is renamed

Tips

  • Use parameter properties to eliminate repetitive this.x = x assignments — especially useful for value objects and DTOs
  • Enable noImplicitOverride in tsconfig to require the override keyword on all overriding methods
  • Use abstract classes to define contracts that must be implemented — it is the idiomatic TS equivalent of abstract base classes

Summary

  • public (default) · private (compile) · protected (subclasses) · readonly (set once)
  • TS private ≠ JS #private — use # for runtime enforcement
  • Parameter properties — constructor(public x: T) declares and assigns in one step
  • abstract — cannot instantiate; subclass must implement · override — explicit intent marker
