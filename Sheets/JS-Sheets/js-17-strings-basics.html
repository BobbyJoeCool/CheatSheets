String Basics

Literals · quotes · template literals · escape sequences · indexing · immutability

String Fundamentals

Strings in JavaScript are immutable sequences of UTF-16 characters. You can create them with single quotes ('), double quotes ("), or backticks (`). Single and double quotes are functionally identical — pick one and be consistent. Backtick strings are template literals — they support multi-line content and embedded expressions with ${ } interpolation. Strings are zero-indexed, and individual characters can be read with bracket notation or .at(). Because strings are immutable, all string methods return a new string — they never modify the original.

Escape Sequences

  • \' — single quote (inside single-quoted string)
  • \" — double quote (inside double-quoted string)
  • \\ — literal backslash
  • \n — newline
  • \t — tab
  • \r — carriage return
  • \u0041 — Unicode code point → "A"
  • \u{1F600} — Unicode with braces (emoji etc.)
  • \0 — null character

String Examples

// ── Creating strings ─────────────────────────────────────────
const a = 'single quotes';
const b = "double quotes";
const c = `backtick / template literal`;

// ── Concatenation ────────────────────────────────────────────
const greeting = "Hello, " + "world!";  // "Hello, world!"
const name = "Alice";
const msg = "Hello, " + name + "!";     // "Hello, Alice!"

// ── Template literals ────────────────────────────────────────
const msg2 = `Hello, ${name}!`;         // "Hello, Alice!"
const calc = `Sum: ${2 + 2}`;          // "Sum: 4"
const expr = `${isAdmin ? "Admin" : "User"}`;

// Multi-line (no \n needed)
const multi = `Line one
Line two
Line three`;

// Nested template
const html = `
  • ${item}
    • `;

// ── Indexing & length ────────────────────────────────────────
const str = "Hello";
str[0]          // "H"
str[4]          // "o"
str[-1]         // undefined  ← use .at() instead
str.at(-1)      // "o"  (negative index from end)
str.at(0)       // "H"
str.length      // 5

// ── Strings are immutable ────────────────────────────────────
let s = "hello";
s[0] = "H";     // silently fails (or throws in strict mode)
s               // "hello" — unchanged
s = s[0].toUpperCase() + s.slice(1); // must build new string
s               // "Hello"

// ── Comparing strings ────────────────────────────────────────
"apple" === "apple"         // true
"apple" < "banana"          // true (lexicographic)
"Z" < "a"                   // true ("Z" is 90, "a" is 97)

// Case-insensitive compare:
"Hello".toLowerCase() === "hello".toLowerCase() // true

// ── TypeScript string types ───────────────────────────────────
const s1: string = "plain string type";
const s2: "north" | "south" = "north"; // literal union
function greet(name: string): string {
  return `Hello, ${name}!`;
}

    Notes

    Template literals can embed any JS expression inside ${ } — function calls, ternaries, arithmetic
    .at(index) — supports negative indices (-1 = last character); cleaner than str[str.length - 1]
    String methods never mutate — they always return a new string; reassign if you want to "change" it
    String comparison is lexicographic (character by character) — use .localeCompare() for language-aware sorting
    Strings are UTF-16 — emoji and some Unicode chars are two code units; .length counts code units, not characters
    Template literals are great for HTML generation, SQL fragments, and log messages — keep logic simple inside ${ }

    Common Mistakes

    • Using str[-1] to get the last character — returns undefined; use str.at(-1)
    • Trying to mutate a string in place — str[0] = "X" silently fails; build a new string instead
    • Comparing strings with >/< across locales — use .localeCompare() for correct alphabetical order
    • Forgetting backticks for template literals — "Hello ${name}" prints literally, not interpolated

    Tips

    • Prefer template literals over concatenation — they are easier to read and support multi-line without \n
    • Use .at(-1) instead of [str.length - 1] for the last character — cleaner and expressive
    • Use String.raw`path\to\file` for raw strings where backslashes should not be interpreted as escapes

    Summary

    • Three quote styles: ' ' · " " · ` ` — backticks enable interpolation and multi-line
    • Strings are immutable — methods return new strings; originals are unchanged
    • str[i] · str.at(i) — zero-indexed; .at() supports negative indices
    • ${expression} inside backticks — embeds any JS expression
