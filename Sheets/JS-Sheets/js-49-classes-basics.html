JavaScript Classes

Classes (introduced in ES6) are syntactic sugar over JavaScript's prototype-based inheritance. A class defines a blueprint for objects; instances are created with new. The constructor runs once per instance at creation. Instance fields (declared at class level or in constructor) belong to each object. Private fields (prefixed with #) are enforced at runtime — truly inaccessible from outside the class, unlike the convention of prefixing with _. Getters and setters let you define computed properties with validation logic while keeping a clean property-access syntax.

Class Member Types

  • Instance field — unique to each instance
  • Method — function on the prototype; shared
  • Private field (#name) — runtime enforced; truly private
  • Getter (get prop()) — computed read property
  • Setter (set prop(v)) — validated write property
  • Static — on the class, not instances (Sheet 50)
  • Classes are always in strict mode

Class Examples

// ── Basic class ───────────────────────────────────────────────
class Animal {
  // Instance field (ES2022 class fields)
  species = "unknown";

  constructor(name, sound) {
    this.name  = name;   // instance property via constructor
    this.sound = sound;
  }

  // Instance method — on prototype; shared across all instances
  speak() {
    return `${this.name} says ${this.sound}`;
  }

  // toString override
  toString() {
    return `Animal(${this.name})`;
  }
}

const dog = new Animal("Rex", "woof");
dog.speak();             // "Rex says woof"
dog instanceof Animal;   // true
dog.constructor === Animal; // true

// ── Private fields (#) — truly private, runtime enforced ─────
class BankAccount {
  #balance = 0;          // private — cannot be accessed outside
  #owner;

  constructor(owner, initialBalance = 0) {
    this.#owner   = owner;
    this.#balance = initialBalance;
  }

  deposit(amount) {
    if (amount <= 0) throw new RangeError("Amount must be positive");
    this.#balance += amount;
    return this;          // allow chaining
  }

  withdraw(amount) {
    if (amount > this.#balance) throw new Error("Insufficient funds");
    this.#balance -= amount;
    return this;
  }

  // ── Getter ────────────────────────────────────────────────
  get balance() {
    return this.#balance; // read-only public access to private field
  }

  get owner() {
    return this.#owner;
  }
}

const acct = new BankAccount("Alice", 100);
acct.deposit(50).withdraw(30); // chaining
acct.balance;                  // 120
acct.#balance;                 // ✖ SyntaxError — truly private

// ── Getter + Setter pair ──────────────────────────────────────
class Temperature {
  #celsius = 0;

  constructor(celsius) { this.#celsius = celsius; }

  get fahrenheit() { return this.#celsius * 9/5 + 32; }
  set fahrenheit(f) { this.#celsius = (f - 32) * 5/9; }

  get celsius() { return this.#celsius; }
  set celsius(c) {
    if (c < -273.15) throw new RangeError("Below absolute zero");
    this.#celsius = c;
  }
}

const temp = new Temperature(100);
temp.fahrenheit;      // 212
temp.fahrenheit = 32;
temp.celsius;         // 0

Notes

Class declarations are not hoisted like function declarations — you cannot use a class before it is defined
Methods defined in the class body are placed on the prototype — all instances share one copy; more memory-efficient than assigning functions in the constructor
Private fields (#name) are runtime-enforced — accessing them outside the class throws a SyntaxError at parse time
The _name convention (underscore prefix) is not actually private — it is just a naming convention; anyone can still access it
Returning this from methods enables method chaining: acct.deposit(50).withdraw(30)
Getters/setters look like property access to the caller — they hide the implementation while allowing validation or computed values

Common Mistakes

  • Forgetting new when instantiating — calling a class without new throws a TypeError
  • Using _privateProp expecting privacy — it is just a convention; use #privateProp for real privacy
  • Defining methods as arrow functions in the class body — they become instance fields, not prototype methods; each instance gets its own copy
  • Forgetting that class bodies are always in strict mode — implicit this is undefined in methods called without a receiver

Tips

  • Use #privateField for internal state — it is the only way to guarantee encapsulation in JavaScript
  • Use getters for derived/computed values and for controlled read-access to private fields; add a setter only when you need validation on write
  • Return this from mutating methods to enable fluent/chaining APIs

Summary

  • class · constructor · instance fields · prototype methods
  • #field — truly private (runtime enforced); _field is convention only
  • get prop() / set prop(v) — computed read/write with clean call syntax
  • Return this from methods to enable method chaining
