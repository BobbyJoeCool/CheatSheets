Lodash Examples

// npm install lodash  (+ @types/lodash for TS)
import _ from "lodash";
// Or tree-shake individual functions:
import debounce from "lodash/debounce";
import groupBy  from "lodash/groupBy";

// ── Deep access / set ─────────────────────────────────────────
const obj = { user: { profile: { name: "Alice" } } };
_.get(obj, "user.profile.name");         // "Alice"
_.get(obj, "user.profile.age", 0);       // 0  (default)
_.get(obj, "user.missing.deep");         // undefined — no throw

_.set(obj, "user.profile.age", 30);      // mutates obj
_.set(obj, "settings.theme.color", "dark"); // creates intermediate objects

// ── merge vs Object.assign ────────────────────────────────────
const defaults = { a: 1, nested: { x: 1, y: 2 } };
const overrides = { nested: { y: 99, z: 3 } };

Object.assign({}, defaults, overrides);
// { a: 1, nested: { y: 99, z: 3 } }  ← nested is replaced entirely

_.merge({}, defaults, overrides);
// { a: 1, nested: { x: 1, y: 99, z: 3 } }  ← nested is merged

// ── cloneDeep ─────────────────────────────────────────────────
const original = { user: { scores: [1, 2, 3] } };
const clone    = _.cloneDeep(original);
clone.user.scores.push(4);
original.user.scores;                   // [1, 2, 3] — unaffected

// ── debounce — delay until idle ───────────────────────────────
const handleSearch = _.debounce((query) => {
  fetchResults(query);
}, 300);
input.addEventListener("input", e => handleSearch(e.target.value));
// Only fires 300ms after the user stops typing

// ── throttle — limit frequency ────────────────────────────────
const handleScroll = _.throttle(() => {
  updateScrollPosition();
}, 100);
window.addEventListener("scroll", handleScroll);
// Fires at most once per 100ms regardless of scroll speed

// ── groupBy ───────────────────────────────────────────────────
const orders = [
  { id: 1, status: "pending", amount: 100 },
  { id: 2, status: "shipped", amount: 200 },
  { id: 3, status: "pending", amount: 150 },
];
_.groupBy(orders, "status");
// { pending: [{id:1,...},{id:3,...}], shipped: [{id:2,...}] }

// ── orderBy ───────────────────────────────────────────────────
_.orderBy(orders, ["status","amount"], ["asc","desc"]);

// ── uniqBy ────────────────────────────────────────────────────
const users = [{ id:1,name:"Alice"},{id:2,name:"Bob"},{id:1,name:"Alice2"}];
_.uniqBy(users, "id");  // [{id:1,name:"Alice"},{id:2,name:"Bob"}]

// ── pick / omit ───────────────────────────────────────────────
const user = { id:1, name:"Alice", password:"secret", email:"a@b.com" };
_.pick(user, ["id","name","email"]); // { id:1, name:"Alice", email:"a@b.com" }
_.omit(user, ["password"]);          // { id:1, name:"Alice", email:"a@b.com" }

Notes

_.get is still useful despite optional chaining (?.) — it works with string paths at runtime, allowing dynamic property access
_.merge vs Object.assign — assign shallow-replaces nested objects; merge deeply combines them
_.cloneDeep handles Dates, RegExps, and circular references better than JSON.parse(JSON.stringify()) — use it or native structuredClone
_.debounce — fires after the user stops calling for ms; use for search inputs, resize handlers
_.throttle — fires at most once per ms; use for scroll handlers, mouse move events
Modern JS covers much of lodash — Array.groupBy (ES2024), structuredClone, optional chaining; use lodash selectively where native is lacking

Common Mistakes

  • Importing the entire lodash bundle — use lodash/debounce instead of lodash for tree-shaking
  • Using _.merge when you want a shallow merge — it recursively combines nested objects which may not be the intent
  • Not cancelling debounced functions on component unmount — call debouncedFn.cancel() in cleanup to prevent stale updates

Tips

  • Import individual functions for tree-shaking: import debounce from "lodash/debounce" — keeps bundle size small
  • Use _.debounce(fn, ms, { leading: true }) to fire on the first call then wait — useful for button click handlers
  • Before adding lodash, check if native JS has the feature — Array.flat, structuredClone, and optional chaining cover many common cases

Summary

  • _.get/set/has — safe deep access · _.merge — deep merge · _.cloneDeep — deep clone
  • _.debounce — fire after idle · _.throttle — fire at most once per interval
  • _.groupBy · _.orderBy · _.uniqBy · _.pick · _.omit · _.chunk
  • Import individual functions for tree-shaking · check native JS first
