TS Function Types

Typed params · return types · optional · overloads · void · never · callbacks

Typing Functions in TypeScript

TypeScript can infer the return type of most functions, but explicitly annotating it is best practice for public APIs — it serves as documentation and prevents accidentally returning the wrong type. Parameters are typed inline; optional params use ? and are typed as T | undefined. Function types can be expressed as inline annotations, type aliases, or interface call signatures. Overloads let you declare multiple call signatures for a function that behaves differently based on argument types. The this parameter is a special first-parameter annotation that disappears at runtime but tells TS what this refers to inside the function.

Key Points

  • Always annotate params; return type is recommended for exports
  • param? — optional; type is T | undefined
  • param: T = default — default implies optional
  • void — function returns nothing meaningful
  • never — function never returns (throws or loops)
  • Overload signatures must be compatible with implementation
  • this: Type first param is erased at runtime

TS Function Type Examples

// ── Basic typed function ──────────────────────────────────────
function add(a: number, b: number): number {
  return a + b;
}

// Arrow with type annotation
const multiply = (a: number, b: number): number => a * b;

// ── Optional and default parameters ──────────────────────────
function greet(name: string, greeting?: string): string {
  return `${greeting ?? "Hello"}, ${name}!`;
}
greet("Alice");             // "Hello, Alice!"
greet("Alice", "Hi");       // "Hi, Alice!"

function padNum(n: number, width: number = 3): string {
  return String(n).padStart(width, "0");
}

// ── void and never ────────────────────────────────────────────
function logMsg(msg: string): void {
  console.log(msg);          // no return value expected
}

function fail(msg: string): never {
  throw new Error(msg);      // never returns
}

// ── Function type aliases ─────────────────────────────────────
type Transformer = (input: string) => string;
type Predicate = (value: T) => boolean;
type Callback = (err: Error | null, result?: string) => void;

const upper: Transformer = (s) => s.toUpperCase();
const isPositive: Predicate = (n) => n > 0;

// ── Interface call signature ──────────────────────────────────
interface Formatter {
  (value: string, options?: { trim: boolean }): string;
}

// ── Overloads ─────────────────────────────────────────────────
function format(value: number): string;
function format(value: string): string;
function format(value: number | string): string {  // implementation
  if (typeof value === "number") return value.toFixed(2);
  return value.trim();
}
format(3.14159);  // "3.14"  ← TS knows return is string
format("  hi  "); // "hi"

// ── this parameter ────────────────────────────────────────────
interface Counter {
  count: number;
  increment(this: Counter): void;
}
const counter: Counter = {
  count: 0,
  increment(this: Counter) {
    this.count++;              // TS knows `this` is Counter
  },
};

// ── Typing callbacks ─────────────────────────────────────────
function doWork(
  data: string[],
  onComplete: (results: number[], errors: string[]) => void
): void {
  // ...
  onComplete([1, 2, 3], []);
}

// ── ReturnType utility ────────────────────────────────────────
function getConfig() {
  return { port: 3000, host: "localhost" };
}
type Config = ReturnType<typeof getConfig>;
// Config = { port: number; host: string }

Notes

void return type — tells callers not to rely on the return value; function implicitly returns undefined
never return type — function never completes normally; TS uses this for exhaustiveness checking
Optional param? — the value is T | undefined inside the function; always check or provide a default
Overload signatures define the public API; the implementation signature is wider and not visible to callers
this parameter — first param that TS erases at runtime; ensures this is only called in the right context
ReturnType<typeof fn> — infer the return type of a function without repeating it; stays in sync automatically

Common Mistakes

  • Marking a param optional? but not handling the undefined case inside — runtime crash
  • Writing overload signatures that contradict each other — TS won't error but callers get incorrect types
  • Using void when you mean undefined — they are related but not interchangeable in all contexts
  • Forgetting that overload implementation signature is private — it must be compatible with all overloads but is not callable directly

Tips

  • Use ReturnType<typeof fn> to derive a type from a function's return — stays in sync as the function evolves
  • Prefer type Fn = (x: T) => R aliases for reusable callback types — makes signatures cleaner throughout the codebase
  • Use overloads sparingly — often a union parameter type is simpler; overloads are best when the return type also varies with input

Summary

  • Annotate all params; annotate return types on exported/public functions
  • param? optional → T | undefined · param = val default → implicitly optional
  • void — no return value · never — never returns
  • Overloads — multiple call signatures; ReturnType<typeof fn> — infer return type
