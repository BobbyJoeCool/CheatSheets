HTTP Server

http.createServer · request · response · routing · headers · serving files

Node's Built-in HTTP Module

Node's http module lets you create a web server without any framework. http.createServer(callback) returns a server instance; the callback receives an IncomingMessage (request) and a ServerResponse (response) on every connection. While real applications almost always use Express or Fastify on top, understanding the raw HTTP layer is essential for debugging and for writing lightweight utilities. The server runs in the event loop — each request fires a new callback invocation; the server itself never blocks.

Key Objects

  • req.method — "GET" | "POST" | "PUT" …
  • req.url — path + query string e.g. "/users?id=1"
  • req.headers — object of request headers
  • req is a Readable stream — body arrives in chunks
  • res.writeHead(status, headers) — send status + headers
  • res.setHeader(name, value) — set individual header
  • res.end(body) — send body and close response
  • res.write(chunk) — stream body in chunks

HTTP Server Examples

import http from "http";
import fs   from "fs";
import path from "path";
import { fileURLToPath } from "url";

const __dirname = path.dirname(fileURLToPath(import.meta.url));

// ── Minimal server ────────────────────────────────────────────
const server = http.createServer((req, res) => {
  res.writeHead(200, { "Content-Type": "text/plain" });
  res.end("Hello, Node!\n");
});
server.listen(3000, () => console.log("Listening on :3000"));

// ── Manual routing ────────────────────────────────────────────
http.createServer(async (req, res) => {
  const url = new URL(req.url, `http://localhost`);

  if (req.method === "GET" && url.pathname === "/") {
    res.writeHead(200, { "Content-Type": "text/html" });
    res.end("
Home
");

  } else if (req.method === "GET" && url.pathname === "/api/users") {
    const users = [{ id: 1, name: "Alice" }];
    res.writeHead(200, { "Content-Type": "application/json" });
    res.end(JSON.stringify(users));

  } else if (req.method === "POST" && url.pathname === "/api/users") {
    const body = await readBody(req);
    const data = JSON.parse(body);
    res.writeHead(201, { "Content-Type": "application/json" });
    res.end(JSON.stringify({ id: 2, ...data }));

  } else {
    res.writeHead(404, { "Content-Type": "application/json" });
    res.end(JSON.stringify({ error: "Not found" }));
  }
}).listen(3000);

// ── Read request body (streams in as chunks) ──────────────────
function readBody(req) {
  return new Promise((resolve, reject) => {
    const chunks = [];
    req.on("data",  chunk  => chunks.push(chunk));
    req.on("end",   ()     => resolve(Buffer.concat(chunks).toString()));
    req.on("error", reject);
  });
}

// ── Serve a static file ───────────────────────────────────────
http.createServer((req, res) => {
  const filePath = path.join(__dirname, "public", req.url);
  fs.readFile(filePath, (err, data) => {
    if (err) {
      res.writeHead(404); res.end("Not found"); return;
    }
    res.writeHead(200);
    res.end(data);
  });
}).listen(3000);

// ── HTTPS server ──────────────────────────────────────────────
import https from "https";
const options = {
  key:  fs.readFileSync("./key.pem"),
  cert: fs.readFileSync("./cert.pem"),
};
https.createServer(options, (req, res) => {
  res.end("Secure!");
}).listen(443);

Notes

req.url is a raw string — always parse with new URL(req.url, base) to safely extract pathname and query params
The request body is a stream — it arrives in chunks; collect all chunks then join with Buffer.concat(chunks).toString()
res.writeHead() must be called before any res.write() or res.end() — headers cannot be sent after the body starts
res.end() is mandatory — forgetting it leaves the connection hanging and the client waiting indefinitely
For production, use Express (Sheet 89) or Fastify — they handle routing, body parsing, and error handling automatically
For local HTTPS dev, use mkcert to generate trusted certificates without browser warnings

Common Mistakes

  • Forgetting res.end() — request hangs; client times out
  • Calling res.writeHead() after writing body — throws "Cannot set headers after they are sent"
  • Reading req.url directly without parsing — misses query strings and can't decode encoded characters
  • Not handling the "error" event on req — unhandled stream errors crash the process

Tips

  • Use the raw http module for ultra-lightweight proxies, health-check endpoints, or learning — use a framework for everything else
  • Wrap your server callback in a try/catch and call res.writeHead(500) in the catch — unhandled errors otherwise leave the connection open
  • Use server.listen(0) in tests — Node picks a random available port; read it back with server.address().port

Summary

  • http.createServer(cb) · server.listen(port)
  • req.method · req.url · req.headers — request info · body is a stream
  • res.writeHead(status, headers) · res.end(body) — always call end
  • For production: use Express (Sheet 89) or Fastify
