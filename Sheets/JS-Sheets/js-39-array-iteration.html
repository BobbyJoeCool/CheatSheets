Array Iteration & Functional

map · filter · reduce · forEach · find · some · every · flat · flatMap

Array Iteration Methods

MethodReturnsMutatesPurpose
.forEach(fn)undefined✖Side effect per element; cannot break early
.map(fn)new array✖Transform each element; same length result
.filter(fn)new array✖Keep elements where fn returns truthy
.reduce(fn, init)single value✖Accumulate all elements into one value
.reduceRight(fn, init)single value✖Like reduce but right-to-left
.find(fn)element | undefined✖First element where fn is truthy
.findIndex(fn)number✖Index of first match; -1 if none
.findLast(fn)element | undefined✖Last element where fn is truthy (ES2023)
.some(fn)boolean✖True if at least one element passes fn
.every(fn)boolean✖True if all elements pass fn
.flat(depth?)new array✖Flatten nested arrays to given depth
.flatMap(fn)new array✖map then flat(1) in one pass

Iteration Examples

const nums = [1, 2, 3, 4, 5];
const users = [
  { name: "Alice", age: 30, active: true },
  { name: "Bob",   age: 17, active: false },
  { name: "Carol", age: 25, active: true },
];

// ── forEach — side effects only ───────────────────────────────
nums.forEach((n, i) => console.log(i, n)); // 0 1  1 2  2 3 ...

// ── map — transform ───────────────────────────────────────────
nums.map(n => n * 2);            // [2, 4, 6, 8, 10]
users.map(u => u.name);          // ["Alice", "Bob", "Carol"]
users.map(u => ({ ...u, role: "user" })); // add field to each

// ── filter — keep matching ────────────────────────────────────
nums.filter(n => n % 2 === 0);   // [2, 4]
users.filter(u => u.active);     // [Alice obj, Carol obj]
users.filter(u => u.age >= 18);  // [Alice obj, Carol obj]

// ── reduce — accumulate ───────────────────────────────────────
nums.reduce((sum, n) => sum + n, 0);       // 15
nums.reduce((max, n) => n > max ? n : max, -Infinity); // 5

// Group by property
users.reduce((groups, u) => {
  const key = u.active ? "active" : "inactive";
  (groups[key] ??= []).push(u);
  return groups;
}, {});
// { active: [Alice, Carol], inactive: [Bob] }

// ── find / findIndex ──────────────────────────────────────────
users.find(u => u.name === "Bob");          // Bob object
users.findIndex(u => u.name === "Bob");     // 1
users.find(u => u.name === "Dave");         // undefined

// ── some / every ─────────────────────────────────────────────
users.some(u => u.age < 18);     // true  (Bob is 17)
users.every(u => u.age >= 18);   // false (Bob is 17)
users.every(u => u.name.length > 0); // true

// ── flat / flatMap ────────────────────────────────────────────
[[1,2],[3,4],[5]].flat();         // [1,2,3,4,5]
[1,[2,[3,[4]]]].flat(Infinity);   // [1,2,3,4]

// flatMap: map + flat(1) in one pass — great for one-to-many
const sentences = ["hello world", "foo bar"];
sentences.flatMap(s => s.split(" ")); // ["hello","world","foo","bar"]

// Chaining
const result = users
  .filter(u => u.active)
  .map(u => u.name.toUpperCase())
  .sort();
// ["ALICE", "CAROL"]

Notes

forEach — cannot break early; use for...of when you need to exit the loop
map — always returns an array of the same length; if you need to change length, use filter or flatMap
reduce — always provide an initial value (, 0); without it, the first call skips the first element as the accumulator
find returns the element (or undefined); some returns a boolean — use the right one for the job
flatMap is significantly more efficient than .map().flat() — single pass over the array
All these methods receive (element, index, array) as callback arguments — use index when you need position

Common Mistakes

  • Using forEach when you need the result — it returns undefined; use map
  • Omitting the initial value from reduce — works on non-empty arrays but throws on empty ones
  • Using map for side effects only — use forEach instead; it signals intent
  • Chaining too many methods over large arrays — each creates a new array; consider a single reduce pass for performance-critical code

Tips

  • Use method chaining (.filter().map().sort()) to build readable data pipelines
  • Use reduce to group, index, or aggregate — it is the most general of all iteration methods
  • Use flatMap whenever you find yourself writing .map(...).flat() — it is faster and cleaner

Summary

  • map — transform · filter — keep · reduce — accumulate · forEach — side effects
  • find — first match · some — any? · every — all?
  • flat(depth) — flatten nested · flatMap — map + flat(1) in one pass
  • All return new arrays (except forEach) — originals are never mutated
