Implicit coercion · explicit conversion · parseInt · truthy/falsy

Implicit vs Explicit Conversion

JavaScript will automatically convert types in many situations — this is called implicit coercion and is one of the most surprising aspects of the language. For example, adding a number to a string produces a string, and using a non-boolean in an if condition converts it to boolean. Explicit conversion uses built-in functions like Number(), String(), and Boolean() to convert values intentionally. Always prefer explicit conversion — it makes intent clear and avoids subtle bugs from unexpected coercion.

Falsy Values

  • false — the boolean false
  • 0 and -0 — zero
  • 0n — BigInt zero
  • "" — empty string
  • null — explicit empty
  • undefined — unassigned
  • NaN — not a number
  • Everything else is truthy — including [], {}, "0", -1

Conversion Examples

// ── Implicit coercion (automatic — often surprising) ─────────
"5" + 3          // "53"   string wins with +
"5" - 3          // 2      arithmetic converts string to number
"5" * "2"        // 10     both coerced to number
true + 1         // 2      true → 1
false + 1        // 1      false → 0
null + 1         // 1      null → 0
undefined + 1    // NaN    undefined → NaN
[] + []          // ""     both to string
[] + {}          // "[object Object]"
{} + []          // 0      (as statement, {} is empty block)

// ── Explicit: → Number ───────────────────────────────────────
Number("42")     // 42
Number("3.14")   // 3.14
Number("")       // 0
Number(" ")      // 0
Number("abc")    // NaN
Number(true)     // 1
Number(false)    // 0
Number(null)     // 0
Number(undefined)// NaN
Number([])       // 0
Number([1])      // 1

parseInt("42px") // 42     stops at first non-numeric char
parseInt("0xFF") // 255    parses hex
parseInt("abc")  // NaN
parseInt("9", 2) // NaN    "9" invalid in base-2
parseInt("11",2) // 3      binary "11" = 3
parseFloat("3.14abc") // 3.14

// ── Explicit: → String ───────────────────────────────────────
String(42)       // "42"
String(true)     // "true"
String(null)     // "null"
String(undefined)// "undefined"
(42).toString()  // "42"
(255).toString(16)// "ff"  hex
(8).toString(2)  // "1000" binary

// ── Explicit: → Boolean ──────────────────────────────────────
Boolean(0)       // false
Boolean("")      // false
Boolean(null)    // false
Boolean(undefined)// false
Boolean(NaN)     // false
Boolean("0")     // true   non-empty string!
Boolean([])      // true   empty array is truthy!
Boolean({})      // true   empty object is truthy!

// ── Double-bang shorthand ────────────────────────────────────
!!0              // false
!!"hello"        // true
!!null           // false

Notes

+ with a string triggers string concatenation — all other arithmetic operators convert to number
parseInt(str, radix) — always pass the radix (base); default is not always 10
Number() vs parseInt() — Number("") is 0; Number("42px") is NaN; parseInt("42px") is 42
[] and {} are both truthy — a very common source of bugs
!! (double-bang) — converts any value to its boolean equivalent; idiomatic JS shorthand
TypeScript prevents most implicit coercion bugs at compile time when strict is enabled

Common Mistakes

  • Using "5" + 3 expecting 8 — gets "53" because + concatenates when either operand is a string
  • Checking if ([]) as falsy — empty arrays are truthy in JS
  • Using parseInt() without a radix — can parse "08" as octal in some environments
  • Relying on Number(null) === 0 — explicit check for null is always clearer

Tips

  • Always use explicit conversion functions rather than relying on coercion — Number(x) over +x
  • Use parseInt(str, 10) — always pass 10 as the radix for decimal numbers
  • Use x == null to check for both null and undefined in one expression

Summary

  • Implicit coercion is automatic and often surprising — prefer explicit conversion
  • Number() · String() · Boolean() — the three explicit conversion functions
  • parseInt(str, 10) · parseFloat(str) — parse strings that start with numbers
  • Falsy: false · 0 · "" · null · undefined · NaN — everything else is truthy
