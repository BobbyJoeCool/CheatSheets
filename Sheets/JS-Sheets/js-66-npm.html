package.json Reference

{
  "name": "my-app",
  "version": "1.2.3",          // semver: major.minor.patch
  "description": "My app",
  "type": "module",             // "module" = ESM; "commonjs" = CJS (default)
  "main": "./dist/index.cjs",   // CJS entry point
  "module": "./dist/index.mjs", // ESM entry point (bundlers)
  "types": "./dist/index.d.ts", // TypeScript declarations

  // Modern: conditional exports per environment
  "exports": {
    ".": {
      "import":  "./dist/index.mjs",
      "require": "./dist/index.cjs",
      "types":   "./dist/index.d.ts"
    }
  },

  "scripts": {
    "dev":   "tsx src/index.ts",       // run dev server
    "build": "tsc --noEmit && vite build",
    "test":  "vitest",
    "lint":  "eslint src --ext .ts",
    "start": "node dist/index.js"
  },

  "dependencies": {           // shipped with the package
    "express": "^4.18.0",     // ^ = compatible with (>=4.18.0 <5)
    "zod":     "~3.22.0"      // ~ = patch updates only (>=3.22.0 <3.23)
  },

  "devDependencies": {        // development only, not shipped
    "typescript":  "^5.0.0",
    "vitest":      "^1.0.0",
    "@types/node": "^20.0.0"
  },

  "peerDependencies": {       // must be installed by the consumer
    "react": ">=18.0.0"
  },

  "engines": {
    "node": ">=18.0.0"        // minimum Node version required
  },

  "files": ["dist", "README.md"], // what to include when publishing

  "private": true             // prevent accidental npm publish
}

Notes

Semver: ^1.2.3 allows >=1.2.3 <2.0.0 · ~1.2.3 allows >=1.2.3 <1.3.0 · 1.2.3 (exact) allows only that version
Lockfiles (package-lock.json, yarn.lock, pnpm-lock.yaml) — pin exact resolved versions; always commit to version control
devDependencies — not installed in production (npm install --production skips them); keep tooling, tests, and type defs here
peerDependencies — declares compatibility requirements; the consuming package must install the peer itself; common in plugins and UI libraries
npx pkg — runs a binary from npm without installing it globally; uses a temp cache
pnpm uses a content-addressable store + symlinks — dramatically faster installs and uses far less disk space than npm/yarn for monorepos

Common Mistakes

  • Not committing the lockfile — teammates get different versions; CI breaks unexpectedly
  • Putting everything in dependencies — type defs, linters, and test frameworks should be devDependencies
  • Using npm install --no-save — package isn't added to package.json; next install won't include it
  • Publishing a package without "files" field — entire project including node_modules and source gets published

Tips

  • Use pnpm in monorepos — workspaces + shared store = dramatically faster installs
  • Run npm audit fix regularly — automatically upgrades vulnerable dependencies within semver constraints
  • Use npm run (no args) to list all available scripts — helpful in unfamiliar projects

Summary

  • dependencies — shipped · devDependencies — tooling only · peerDependencies — consumer must install
  • Semver: ^ minor+patch · ~ patch only · exact no prefix
  • Always commit lockfiles · use npx/pnpm dlx for one-off binaries
  • scripts — define project commands; run with npm run <name>
