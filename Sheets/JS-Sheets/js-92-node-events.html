EventEmitter

on · emit · once · off · removeAllListeners · typed events · extending

EventEmitter API

MethodReturnsNotes
.on(event, listener)emitterAdd listener; fires every time event is emitted
.once(event, listener)emitterAdd listener that fires exactly once then auto-removes
.emit(event, ...args)booleanFire event synchronously; returns true if listeners exist
.off(event, listener)emitterRemove specific listener (same reference required)
.removeAllListeners(event?)emitterRemove all listeners for event, or all events if omitted
.listeners(event)Function[]Array of registered listeners for an event
.listenerCount(event)numberNumber of listeners for an event
.setMaxListeners(n)emitterDefault max is 10; raise to suppress memory leak warnings
.prependListener(ev, fn)emitterAdd listener to the front of the queue
EventEmitter.once(emitter, ev)PromiseStatic — await the next emission of an event

EventEmitter Examples

import { EventEmitter } from "events";

// ── Basic usage ───────────────────────────────────────────────
const emitter = new EventEmitter();

emitter.on("data", (value) => {
  console.log("Received:", value);
});
emitter.on("data", (value) => {
  console.log("Also received:", value);
});

emitter.emit("data", 42);            // both listeners fire: 42, 42
emitter.emit("data", "hello");       // 42 → "hello"

// ── once — fire exactly once ──────────────────────────────────
emitter.once("connect", () => {
  console.log("Connected!");
});
emitter.emit("connect");             // "Connected!"
emitter.emit("connect");             // (nothing — already removed)

// ── Removing listeners ────────────────────────────────────────
function handler(msg) { console.log(msg); }
emitter.on("msg", handler);
emitter.off("msg", handler);         // must pass the SAME reference

// ── Error events — special handling ──────────────────────────
// If "error" is emitted with no listener, Node throws
emitter.on("error", (err) => {
  console.error("Caught:", err.message); // always add an error listener
});
emitter.emit("error", new Error("Something broke"));

// ── Extending EventEmitter ────────────────────────────────────
class Store extends EventEmitter {
  #data = {};

  set(key, value) {
    this.#data[key] = value;
    this.emit("change", { key, value });   // notify listeners
  }

  get(key) { return this.#data[key]; }
}

const store = new Store();
store.on("change", ({ key, value }) => {
  console.log(`${key} changed to`, value);
});
store.set("theme", "dark");            // "theme changed to dark"

// ── TypeScript: typed EventEmitter ────────────────────────────
import { TypedEmitter } from "tiny-typed-emitter"; // npm install tiny-typed-emitter

interface AppEvents {
  "user:login":  (userId: number) => void;
  "user:logout": (userId: number) => void;
  "data:update": (key: string, value: unknown) => void;
  "error":       (err: Error) => void;
}

class AppEmitter extends TypedEmitter<AppEvents> {}
const app = new AppEmitter();

app.on("user:login", (userId) => {    // userId is typed as number
  console.log("User logged in:", userId);
});
app.emit("user:login", 42);           // ✔
app.emit("user:login", "not-a-number"); // ✖ TS Error

// ── Await next event with EventEmitter.once ───────────────────
const { once } = EventEmitter;
const [value] = await once(emitter, "ready"); // waits for "ready"

Notes

emit() is synchronous — all listeners run to completion before emit() returns; use setImmediate or async listeners for non-blocking handlers
Error events are special — if "error" is emitted with no listener, Node throws an uncaught exception and crashes; always add an error listener
Max listeners warning — Node warns at 10+ listeners per event (memory leak signal); call setMaxListeners(0) to disable if you legitimately need more
.off() requires the exact same function reference — arrow functions stored inline cannot be removed; always assign to a variable
Extending EventEmitter — the standard Node pattern for observable objects (streams, servers, and process itself all extend it)
tiny-typed-emitter — lightweight TS wrapper that adds full type checking to event names and payload types

Common Mistakes

  • Emitting "error" with no listener — Node throws an unhandled exception; always add an error handler
  • Trying to remove an anonymous function with .off() — impossible without a reference; assign listeners to named variables
  • Adding listeners in a loop without removing them — creates a memory leak; use removeAllListeners() in cleanup
  • Expecting async listeners to be awaited — emit() is sync; async listeners fire but their Promises are not awaited

Tips

  • Use EventEmitter.once(emitter, "event") to await a one-time event in async code — cleaner than wrapping in a Promise manually
  • Use tiny-typed-emitter or write a typed wrapper — catches misspelled event names and wrong payload types at compile time
  • Always remove event listeners when tearing down objects (servers closing, components unmounting) to prevent memory leaks

Summary

  • .on(ev, fn) — subscribe · .once(ev, fn) — one shot · .off(ev, fn) — unsubscribe (same ref)
  • .emit(ev, ...args) — synchronous; fires all listeners immediately
  • Always handle "error" events — unhandled error events crash Node
  • Extend EventEmitter for observable domain objects; use tiny-typed-emitter for TS type safety
