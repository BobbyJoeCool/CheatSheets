CommonJS Examples

// ── Exporting (math.js) ───────────────────────────────────────
// Option 1: export an object
module.exports = {
  PI: 3.14159,
  add: (a, b) => a + b,
  sub: (a, b) => a - b,
};

// Option 2: shorthand — exports is a reference to module.exports
exports.PI  = 3.14159;
exports.add = (a, b) => a + b;
// ✖ exports = { PI, add }  — reassigning exports breaks the link

// Option 3: export a single value (class, function, etc.)
module.exports = class Logger {
  log(msg) { console.log(msg); }
};

// ── Importing ─────────────────────────────────────────────────
const math   = require("./math");           // import whole module
const { add } = require("./math");          // destructure named exports
const Logger  = require("./logger");        // import default class

// Built-in modules
const path = require("path");
const fs   = require("fs");
const http = require("http");

// ── Dynamic require — can be inside functions/conditions ──────
function loadPlugin(name) {
  try {
    return require(`./plugins/${name}`);    // runtime path
  } catch {
    return null;
  }
}

if (process.env.DEBUG) {
  const devTools = require("./dev-tools"); // conditional load
}

// ── require.resolve — find module path without loading ────────
const p = require.resolve("./config.json");
// "/absolute/path/to/config.json"

// ── CJS interop with ESM ──────────────────────────────────────
// From ESM: import a CJS module
import logger from "./logger.cjs";          // ✔ default import
import { createRequire } from "module";
const require = createRequire(import.meta.url);
const legacy  = require("./legacy-cjs.js"); // require in ESM file

// ── package.json "exports" — conditional exports ──────────────
// In package.json: serve ESM to bundlers, CJS to old Node:
// "exports": {
//   ".": {
//     "import": "./dist/index.mjs",   // for ESM consumers
//     "require": "./dist/index.cjs",  // for CJS consumers
//     "types": "./dist/index.d.ts"    // for TypeScript
//   }
// }

// ── Module caching ────────────────────────────────────────────
// CJS modules are cached after first require
const a = require("./singleton");
const b = require("./singleton");
a === b;  // true — same cached object

// Clear cache (rarely needed)
delete require.cache[require.resolve("./config")];
const fresh = require("./config");

Notes

exports is a reference to module.exports — adding properties to exports works; reassigning exports = ... breaks the link
require() is synchronous — it blocks while reading/executing the module; never use it inside a hot code path in a server
Module caching — require caches modules on first load; subsequent calls return the same object, making CJS modules effectively singletons
Dynamic require — can be used anywhere in code (inside functions, conditions, loops); ESM's import() is the async equivalent
createRequire(import.meta.url) — lets you call require() from inside an ESM file when you need to load a CJS module
Conditional exports in package.json — modern packages ship both ESM and CJS builds; Node/bundlers pick the right one automatically

Common Mistakes

  • Reassigning exports = { ... } — breaks the reference to module.exports; use module.exports = { ... } instead
  • Using require in an ESM file without createRequire — throws ReferenceError: require is not defined
  • Relying on module caching for state sharing — works but is implicit; use an explicit singleton pattern for clarity
  • Mixing CJS and ESM in the same file — you cannot use both import and require in a standard file

Tips

  • New projects should default to ESM — CJS is legacy; use it only when a dependency or toolchain requires it
  • When publishing a library, use conditional exports in package.json to ship both ESM and CJS builds
  • Use createRequire(import.meta.url) as a bridge when you need a CJS module from an ESM codebase

Summary

  • require() — sync import · module.exports / exports.x — export
  • Modules are cached after first load — same object on every require
  • Dynamic require works anywhere; ESM's import() is the async async equivalent
  • New code → use ESM · Legacy/libs → CJS · Publishing → ship both with conditional exports
