Sheet 21

If / Else

if · else if · else · ternary · short-circuit · early return · guard clauses

Conditional Execution

if/else is the fundamental branching construct. The condition is any expression — JS converts it to a boolean automatically (truthy/falsy rules apply). Curly braces are technically optional for single statements but always use them — omitting them is a common source of bugs. The ternary operator (condition ? a : b) is a concise inline expression for simple two-branch choices. Short-circuit evaluation with && and || is a common pattern for conditional execution without a full if block. Guard clauses (early returns) keep the happy path unindented and are strongly preferred over deeply nested if/else chains.

Key Points

  • Condition can be any expression — converted to boolean via truthy/falsy
  • Always use curly braces — even for single-line bodies
  • Ternary returns a value — use it in expressions, not for side effects
  • expr && doThis() — call only if expr is truthy
  • expr || fallback() — call only if expr is falsy
  • Guard clauses — return early to avoid deep nesting
  • TS narrows types inside if blocks automatically

Conditional Examples

// ── Basic if / else if / else ────────────────────────────────
const score = 75;

if (score >= 90) {
  console.log("A");
} else if (score >= 80) {
  console.log("B");
} else if (score >= 70) {
  console.log("C");
} else {
  console.log("F");
}                               // "C"

// ── Omitting braces — avoid this ────────────────────────────
if (score > 50) console.log("pass"); // works but risky

// ── Ternary operator ─────────────────────────────────────────
const label = score >= 60 ? "Pass" : "Fail";  // "Pass"
const cls   = isActive ? "active" : "inactive";

// Nested ternary — hard to read; prefer if/else for 3+ branches
const grade = score >= 90 ? "A" : score >= 80 ? "B" : "C";

// ── Short-circuit evaluation ─────────────────────────────────
isAdmin && showAdminPanel();        // call only if truthy
user ?? createUser();               // call only if null/undefined
const name = user?.name || "Guest"; // fallback

// ── Guard clauses (early return) ─────────────────────────────
// Avoid: deeply nested
function processOrder(order) {
  if (order) {
    if (order.items.length > 0) {
      if (order.isPaid) {
        ship(order);
      }
    }
  }
}

// Prefer: guard clauses — flat and readable
function processOrder2(order) {
  if (!order) return;
  if (order.items.length === 0) return;
  if (!order.isPaid) return;
  ship(order);
}

// ── TypeScript: type narrowing in if blocks ───────────────────
function greet(name: string | null) {
  if (name === null) {
    return "Hello, stranger!";
  }
  return `Hello, ${name}!`;    // TS knows name is string here
}

function processInput(val: string | number) {
  if (typeof val === "string") {
    console.log(val.toUpperCase()); // val is string
  } else {
    console.log(val.toFixed(2));    // val is number
  }
}

Notes

Only one else per if chain — it is the catch-all for all unmatched conditions
Ternary is an expression — it produces a value and can be used inline; if/else is a statement
&& short-circuit — right side only evaluates if left is truthy; returns left side if falsy, right side if truthy
|| short-circuit — returns first truthy value; beware it also triggers on 0 and ""
Guard clauses improve readability by eliminating nesting — the happy path is always at the lowest indentation level
TypeScript automatically narrows the type within an if block based on the condition — no manual cast needed

Common Mistakes

  • Omitting curly braces — adding a second line to a braceless if looks correct but doesn't behave as expected
  • Using a ternary for side effects — condition ? doA() : doB() is valid but an if/else is clearer
  • Using || for defaults when 0 or "" are valid — use ?? instead
  • Deeply nesting if/else — flatten with guard clauses and early returns

Tips

  • Invert conditions to return early — "fail fast" at the top, success logic at the bottom
  • Use ternary for value assignment only — keep them simple and single-line
  • ESLint rule no-else-return — flags unnecessary else after a return, pushing you toward guard clauses

Summary

  • if / else if / else — always use curly braces; conditions are truthy/falsy
  • Ternary a ? b : c — inline expression for two-branch value selection
  • Short-circuit && / || — conditional execution without a full block
  • Guard clauses — return early to keep code flat and readable
