Static Members

static fields · static methods · static blocks · static private · singleton pattern

Static Members

Static members belong to the class itself, not to instances. They are accessed via the class name (MyClass.method()), not via this on an instance. Static fields hold shared state across all instances — counters, caches, configuration. Static methods are utility functions that do not need instance state — factory methods, validators, parsers. Static blocks (ES2022) run once when the class is first loaded and are used for complex static initialisation that can't be expressed in a single field assignment. Static members can also be private with #.

Key Points

  • Static members are on the class, not instances
  • Access via ClassName.member, not this.member
  • Static methods cannot access instance fields via this
  • this inside a static method refers to the class
  • Static fields are shared state — mutations affect all code
  • Static private: static #field
  • Static blocks run once at class definition time

Static Member Examples

// ── Static field and method ───────────────────────────────────
class MathUtils {
  static PI = 3.14159265358979;    // shared constant

  static circleArea(r) {
    return MathUtils.PI * r * r;
  }

  static clamp(val, min, max) {
    return Math.min(Math.max(val, min), max);
  }
}

MathUtils.circleArea(5);           // 78.539...
MathUtils.clamp(15, 0, 10);        // 10
// No new MathUtils() needed — pure utility class

// ── Instance counter ──────────────────────────────────────────
class Widget {
  static #count = 0;               // private static

  static get count() { return Widget.#count; }

  constructor(name) {
    this.name = name;
    Widget.#count++;
  }

  static reset() { Widget.#count = 0; }
}

const a = new Widget("A");
const b = new Widget("B");
Widget.count;                      // 2
Widget.reset();
Widget.count;                      // 0

// ── Factory method pattern ────────────────────────────────────
class Color {
  constructor(r, g, b) {
    this.r = r; this.g = g; this.b = b;
  }

  static fromHex(hex) {             // factory — named constructor
    const n = parseInt(hex.replace("#", ""), 16);
    return new Color((n >> 16) & 255, (n >> 8) & 255, n & 255);
  }

  static fromCSS(str) {             // another factory
    const [r, g, b] = str.match(/\d+/g).map(Number);
    return new Color(r, g, b);
  }

  toString() { return `rgb(${this.r},${this.g},${this.b})`; }
}

Color.fromHex("#ff8000").toString(); // "rgb(255,128,0)"
Color.fromCSS("rgb(0, 128, 255)").toString(); // "rgb(0,128,255)"

// ── Static block (ES2022) — complex initialisation ────────────
class Config {
  static debug;
  static apiUrl;

  static {                          // runs once at class load
    Config.debug  = process.env.NODE_ENV !== "production";
    Config.apiUrl = Config.debug
      ? "http://localhost:3000"
      : "https://api.example.com";
    console.log("Config initialised");
  }
}

// ── Singleton pattern ─────────────────────────────────────────
class Database {
  static #instance = null;

  static getInstance() {
    if (!Database.#instance) {
      Database.#instance = new Database();
    }
    return Database.#instance;
  }

  private constructor() { /* connect */ }

  query(sql) { /* ... */ }
}

const db1 = Database.getInstance();
const db2 = Database.getInstance();
db1 === db2;                       // true — same instance

Notes

this inside a static method refers to the class itself — use it to call other static members: this.PI or ClassName.PI
Static private #field — inaccessible even via the class name from outside; only reachable from within the class body
Factory methods (static fromX()) are a clean pattern for multiple named constructors — more readable than overloading a single constructor with many optional params
Static blocks run exactly once when the class definition is evaluated — ideal for async-free initialisation that requires logic
Singleton pattern — store the single instance in a static private field; the constructor can be made private in TypeScript
Utility classes (all-static) are common for math, string helpers, and formatters — but a plain module exporting functions is often cleaner in modern JS

Common Mistakes

  • Accessing a static member via this inside an instance method — use the class name instead
  • Using static fields for state that should be per-instance — static state is shared globally across all uses
  • Calling a static method on an instance — new MyClass().staticMethod() works but is confusing; call on the class directly

Tips

  • Use static factory methods to provide multiple named constructors — Color.fromHex() is far more readable than new Color(null, null, null, "#ff0000")
  • Use static blocks for environment-aware initialisation that reads process.env or performs conditional setup
  • Prefer a plain module with exported functions over an all-static utility class — simpler to tree-shake and test

Summary

  • Static members belong to the class — accessed via ClassName.member
  • Static methods: utilities · factories · validators — no instance state needed
  • Static fields: shared counters, caches, config, singletons
  • Static blocks (ES2022) — one-time complex initialisation at class load
