Routers, Middleware & TypeScript

express.Router() creates a mini-app with its own middleware and routes — mount it on a path to keep your main app.ts clean. Middleware factories are functions that return middleware, allowing configuration (e.g. authenticate({ role: "admin" })). Async route handlers need a wrapper to forward rejected Promises to the error handler — Express 4 doesn't catch them automatically (Express 5 does). TypeScript requires augmenting the Request interface to add custom properties like req.user.

Project Structure

  • src/app.ts — create app, register middleware
  • src/server.ts — call app.listen()
  • src/routes/users.ts — Router for /users
  • src/middleware/auth.ts — auth middleware
  • src/middleware/validate.ts — zod validation
  • src/types/express.d.ts — Request augmentation
  • Separate app from server.listen — easier to test

Advanced Express Examples

import express, { Request, Response, NextFunction, Router } from "express";

// ── express.Router — group related routes ─────────────────────
// src/routes/users.ts
const router = Router();

router.get("/",      getUsers);
router.get("/:id",   getUserById);
router.post("/",     createUser);
router.patch("/:id", updateUser);
router.delete("/:id",deleteUser);

export default router;

// src/app.ts
import usersRouter from "./routes/users.js";
app.use("/users", usersRouter);        // routes → /users, /users/:id …

// ── Async handler wrapper (Express 4) ─────────────────────────
const asyncHandler = (fn: (req: Request, res: Response,
                           next: NextFunction) => Promise) =>
  (req: Request, res: Response, next: NextFunction) =>
    fn(req, res, next).catch(next);    // forward rejection to error mw

router.get("/:id", asyncHandler(async (req, res) => {
  const user = await db.users.findById(Number(req.params.id));
  if (!user) return res.status(404).json({ error: "Not found" });
  res.json(user);
}));

// ── Auth middleware factory ────────────────────────────────────
function authenticate(options: { role?: string } = {}) {
  return async (req: Request, res: Response, next: NextFunction) => {
    const token = req.headers.authorization?.split(" ")[1];
    if (!token) return res.status(401).json({ error: "No token" });
    try {
      const payload = verifyJWT(token);
      if (options.role && payload.role !== options.role) {
        return res.status(403).json({ error: "Forbidden" });
      }
      req.user = payload;              // attach to request
      next();
    } catch {
      res.status(401).json({ error: "Invalid token" });
    }
  };
}

router.get("/admin", authenticate({ role: "admin" }), adminHandler);

// ── Zod validation middleware ─────────────────────────────────
import { z, ZodSchema } from "zod";

function validate(schema: ZodSchema) {
  return (req: Request, res: Response, next: NextFunction) => {
    const result = schema.safeParse(req.body);
    if (!result.success) {
      return res.status(400).json({ errors: result.error.flatten().fieldErrors });
    }
    req.body = result.data;            // replace with parsed+coerced data
    next();
  };
}

const CreateUserSchema = z.object({
  name:  z.string().min(1),
  email: z.string().email(),
});
router.post("/", validate(CreateUserSchema), createUser);

// ── TypeScript: augment Request type ──────────────────────────
// src/types/express.d.ts
declare global {
  namespace Express {
    interface Request {
      user?: { id: number; role: string };
    }
  }
}
// Now req.user is typed throughout the app

Notes

Router() — scoped middleware and routes; mounting at /users prepends that prefix to all router paths
Async wrapper — in Express 4, unhandled Promise rejections in route handlers crash the process; wrapping with .catch(next) forwards them to the error handler
Middleware factories — functions that return middleware allow runtime configuration; much cleaner than checking flags inside a single middleware
Validation middleware — replacing req.body with zod's parsed output means downstream handlers get typed, coerced data
Request augmentation — the express.d.ts declaration file extends Express's Request interface globally; include it in tsconfig.json include array
Express 5 (stable 2024) handles async errors natively — no wrapper needed; check your Express version

Common Mistakes

  • Not wrapping async handlers in Express 4 — unhandled rejections crash the server in Node 15+
  • Forgetting export default router then importing the file — gets an empty object
  • Augmenting Request but not including the .d.ts in tsconfig — TS doesn't see the custom properties
  • Mounting the error handler before routes — it never receives errors from routes registered after it

Tips

  • Create a single asyncHandler utility and use it on every async route — one missed wrapper can crash your server
  • Separate app.ts (middleware + routes) from server.ts (listen) — makes the app importable in tests without starting a port
  • Use router.param("id", cb) to preload and validate a resource once for all routes that share the same param

Summary

  • express.Router() — scoped routes; mount with app.use("/prefix", router)
  • Async wrapper — fn(req,res,next).catch(next) required in Express 4
  • Middleware factories — return middleware to enable configuration
  • TS: augment Express.Request in a .d.ts file to add req.user etc.
