Math & Date Examples

// ── Math utilities ────────────────────────────────────────────
Math.max(...[3, 1, 4, 1, 5, 9]);    // 9  (spread array)
Math.min(...[3, 1, 4, 1, 5, 9]);    // 1

// Random integer in range [min, max] inclusive
function randInt(min, max) {
  return Math.floor(Math.random() * (max - min + 1)) + min;
}
randInt(1, 6);  // simulates a die roll

// Random element from array
const pick = arr => arr[Math.floor(Math.random() * arr.length)];

// Clamp a value
const clamp = (val, min, max) => Math.min(Math.max(val, min), max);
clamp(15, 0, 10);                   // 10

// Rounding to N decimal places
const round2 = n => Math.round(n * 100) / 100;
round2(3.14159);                    // 3.14

// ── Date: reading values ──────────────────────────────────────
const now = new Date();
now.getFullYear();      // 2024
now.getMonth();         // 0-11 (January = 0!)
now.getDate();          // 1-31
now.getDay();           // 0-6 (Sunday = 0)
now.getHours();         // 0-23
now.getMinutes();       // 0-59
now.getSeconds();       // 0-59
now.getTime();          // Unix timestamp in ms
now.toISOString();      // "2024-01-15T10:30:00.000Z"
now.toLocaleDateString();// "1/15/2024" (locale-dependent)
now.toLocaleString();   // "1/15/2024, 10:30:00 AM"

// ── Date arithmetic ───────────────────────────────────────────
const tomorrow = new Date(Date.now() + 86400_000); // +1 day in ms
const nextWeek = new Date();
nextWeek.setDate(nextWeek.getDate() + 7);

const diff = Date.now() - new Date("2024-01-01").getTime();
const days = Math.floor(diff / 86400_000);

// ── Intl.DateTimeFormat — locale-aware formatting ─────────────
const fmt = new Intl.DateTimeFormat("en-US", {
  year: "numeric", month: "long", day: "numeric",
  hour: "2-digit", minute: "2-digit",
});
fmt.format(now);   // "January 15, 2024 at 10:30 AM"

new Intl.DateTimeFormat("de-DE").format(now);  // "15.1.2024"
new Intl.DateTimeFormat("ja-JP").format(now);  // "2024/1/15"

// ── Intl.NumberFormat ─────────────────────────────────────────
new Intl.NumberFormat("en-US", { style: "currency", currency: "USD" })
  .format(1234.5);                  // "$1,234.50"
new Intl.NumberFormat("de-DE", { style: "currency", currency: "EUR" })
  .format(1234.5);                  // "1.234,50 €"

Notes

Math.random() is NOT cryptographically secure — use crypto.getRandomValues() for security-sensitive random numbers
getMonth() returns 0–11 (January = 0) — the most notorious Date gotcha; always add 1 when displaying
Date arithmetic works in milliseconds — store as 86400_000 (one day) or compute diff / 1000 / 60 / 60 / 24
Intl.DateTimeFormat and Intl.NumberFormat — built-in locale-aware formatting; no external library needed
For serious date work (timezones, durations, parsing), use Temporal (Stage 3 proposal) or a library like date-fns or dayjs
Date.now() — fastest way to get the current timestamp; no object allocation

Common Mistakes

  • getMonth() returns 0-indexed months — displaying it directly shows January as 0
  • Using Math.random() for security (tokens, passwords) — use crypto.getRandomValues()
  • Parsing locale-specific date strings with new Date("15/01/2024") — format is implementation-dependent; use ISO strings
  • Mutating dates with setDate() when you want a new date — always create a copy first

Tips

  • Use date.toISOString() for storage and API transport — always unambiguous regardless of timezone
  • Use Intl.DateTimeFormat / Intl.NumberFormat for UI display — zero dependencies, handles all locales
  • For complex date logic, use date-fns (tree-shakeable) or dayjs (lightweight) instead of the native Date API
