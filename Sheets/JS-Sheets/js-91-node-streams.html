Stream Examples

import { createReadStream, createWriteStream } from "fs";
import { createGzip, createGunzip }            from "zlib";
import { Transform, PassThrough }              from "stream";
import { pipeline }                            from "stream/promises";

// ── pipeline — connect streams safely ────────────────────────
// Handles errors and cleanup automatically — always prefer over .pipe()
await pipeline(
  createReadStream("./big.log"),
  createGzip(),
  createWriteStream("./big.log.gz")
);
// File is gzipped; all streams closed even on error

// ── Decompress ────────────────────────────────────────────────
await pipeline(
  createReadStream("./big.log.gz"),
  createGunzip(),
  createWriteStream("./big.log")
);

// ── Custom Transform stream ───────────────────────────────────
class UpperCaseTransform extends Transform {
  _transform(chunk, encoding, callback) {
    this.push(chunk.toString().toUpperCase()); // push transformed chunk
    callback();                               // signal chunk is done
  }
}

await pipeline(
  createReadStream("./input.txt"),
  new UpperCaseTransform(),
  createWriteStream("./output.txt")
);

// ── Collect all chunks from a Readable ────────────────────────
async function streamToString(readable) {
  const chunks = [];
  for await (const chunk of readable) {
    chunks.push(Buffer.isBuffer(chunk) ? chunk : Buffer.from(chunk));
  }
  return Buffer.concat(chunks).toString("utf8");
}

// ── Create a Readable from a string or array ──────────────────
import { Readable } from "stream";

const readable = Readable.from(["line one\n", "line two\n", "line three\n"]);
const readable2 = Readable.from(async function* () {
  yield "chunk 1";
  await sleep(100);
  yield "chunk 2";
}());

// ── Pipe HTTP response to file ────────────────────────────────
import https from "https";
await new Promise((resolve, reject) => {
  const file = createWriteStream("./download.zip");
  https.get("https://example.com/file.zip", res => {
    pipeline(res, file).then(resolve).catch(reject);
  });
});

// ── Backpressure — respect Writable's drain ───────────────────
async function writeLines(writable, lines) {
  for (const line of lines) {
    const canContinue = writable.write(line + "\n");
    if (!canContinue) {
      await new Promise(r => writable.once("drain", r)); // wait for drain
    }
  }
  writable.end();
}

// ── PassThrough — inspect bytes mid-pipeline ──────────────────
let byteCount = 0;
const counter = new PassThrough();
counter.on("data", chunk => { byteCount += chunk.length; });

await pipeline(
  createReadStream("./file.bin"),
  counter,
  createWriteStream("./copy.bin")
);
console.log("Bytes copied:", byteCount);

Notes

pipeline() from stream/promises — always prefer over manual .pipe(); handles errors in every stage and properly closes all streams
Transform._transform(chunk, enc, cb) — call this.push(data) to emit output, then callback() to signal readiness for next chunk
for await (const chunk of readable) — the async iterator protocol works on any Node Readable stream; clean alternative to manual event listeners
Backpressure — writable.write() returns false when the internal buffer is full; wait for the "drain" event before writing more to avoid memory buildup
Readable.from(iterable) — create a Readable from any sync or async iterable; excellent for testing or building data sources
Streams process data chunk-by-chunk — memory usage stays constant even for GB-scale files

Common Mistakes

  • Using .pipe() instead of pipeline() — .pipe() doesn't propagate errors or close downstream streams on failure
  • Ignoring backpressure — writing without checking the return value of write() buffers data in memory and can OOM the process
  • Forgetting to call callback() in _transform — the stream stalls and no more chunks are processed
  • Not handling "error" events on streams — unhandled stream errors crash the process

Tips

  • Always use pipeline() from stream/promises — it's the modern, safe way to connect streams
  • Use for await...of to consume a Readable — it handles backpressure automatically and integrates cleanly with async/await
  • Use PassThrough to add observability (byte counting, logging) mid-pipeline without modifying the data

Summary

  • 4 types: Readable · Writable · Duplex · Transform
  • pipeline(r, t, w) from stream/promises — always prefer over .pipe()
  • for await (const chunk of readable) — clean async consumption
  • Backpressure — check write() return; await "drain" when false
