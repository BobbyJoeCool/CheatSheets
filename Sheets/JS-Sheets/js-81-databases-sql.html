SQL Database Examples

// ── PostgreSQL with pg ────────────────────────────────────────
// npm install pg
import { Pool } from "pg";

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  max: 10,                          // max connections in pool
  idleTimeoutMillis: 30_000,
  connectionTimeoutMillis: 2_000,
});

// Query (parameterised — NEVER interpolate user input)
const { rows } = await pool.query(
  "SELECT * FROM users WHERE id = $1",
  [userId]
);
const user = rows[0];               // undefined if not found

// Insert with returning
const { rows: [created] } = await pool.query(
  "INSERT INTO users (name, email) VALUES ($1, $2) RETURNING *",
  [name, email]
);

// ── MySQL with mysql2 ─────────────────────────────────────────
// npm install mysql2
import mysql from "mysql2/promise";

const conn = await mysql.createConnection({
  host:     process.env.DB_HOST,
  user:     process.env.DB_USER,
  password: process.env.DB_PASS,
  database: process.env.DB_NAME,
});

const [rows2] = await conn.execute(
  "SELECT * FROM users WHERE email = ?",
  [email]
);

// Pool (preferred for servers)
const pool2 = mysql.createPool({
  host: process.env.DB_HOST,
  /* ... */
  waitForConnections: true,
  connectionLimit: 10,
});
const [results] = await pool2.execute("SELECT * FROM products WHERE active = ?", [1]);

// ── SQLite with better-sqlite3 ────────────────────────────────
// npm install better-sqlite3
import Database from "better-sqlite3";

const db = new Database("./app.db");  // creates if not exists

// Create table
db.exec(`
  CREATE TABLE IF NOT EXISTS users (
    id    INTEGER PRIMARY KEY AUTOINCREMENT,
    name  TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL
  )
`);

// Prepare statements (reusable, efficient)
const insert = db.prepare("INSERT INTO users (name, email) VALUES (?, ?)");
const findById = db.prepare("SELECT * FROM users WHERE id = ?");

insert.run("Alice", "alice@example.com");
const found = findById.get(1);       // returns row object or undefined
const all   = db.prepare("SELECT * FROM users").all(); // returns array

// ── Transactions ──────────────────────────────────────────────
// better-sqlite3 (sync)
const transferFn = db.transaction((from, to, amount) => {
  db.prepare("UPDATE accounts SET balance = balance - ? WHERE id = ?")
    .run(amount, from);
  db.prepare("UPDATE accounts SET balance = balance + ? WHERE id = ?")
    .run(amount, to);
});
transferFn(1, 2, 100);              // atomic — rolls back on throw

// pg (async)
const client = await pool.connect();
try {
  await client.query("BEGIN");
  await client.query("UPDATE accounts SET balance = balance - $1 WHERE id = $2", [100, fromId]);
  await client.query("UPDATE accounts SET balance = balance + $1 WHERE id = $2", [100, toId]);
  await client.query("COMMIT");
} catch (err) {
  await client.query("ROLLBACK");
  throw err;
} finally {
  client.release();                 // return to pool
}

Notes

Always use parameterised queries — never interpolate user input into SQL strings; SQL injection is the most critical web security vulnerability
Connection pools — reuse connections across requests; creating a new connection per query is expensive; set pool size based on DB limits
pg uses $1, $2 placeholders · mysql2 uses ? · better-sqlite3 uses ? or named :name
pool.connect() + client.release() — required for transactions to keep them on one connection; always release in finally
better-sqlite3 is synchronous — do not use in a high-concurrency HTTP server; ideal for CLIs, tests, and Electron apps
Transactions are atomic — if any statement throws, rolling back undoes all statements in the transaction

Common Mistakes

  • String-interpolating user input into SQL: `SELECT * FROM users WHERE id = ${req.params.id}` — SQL injection vulnerability
  • Not releasing the client in a pg transaction — pool exhausts after a few requests
  • Creating a new connection per request instead of using a pool — crushes the database with connection overhead
  • Not handling unique constraint violations — insert errors with code 23505 (pg) should return 409, not 500

Tips

  • Use prepared statements in better-sqlite3 (db.prepare()) — parse the SQL once, run many times; faster and safer
  • Use pg's pool.query() for simple queries (auto-manages connection) · pool.connect() only for transactions
  • Store the connection string in an environment variable — never hardcode credentials in source

Summary

  • Always use parameterised queries — $1/$2 (pg) or ? (mysql2/sqlite)
  • Use connection pools for servers · better-sqlite3 is sync-only (CLI/tests)
  • Transactions — BEGIN/COMMIT/ROLLBACK for atomic operations; always release client in finally
  • Connection string from env vars — never hardcode credentials
