Mocking & Spying

A mock replaces a real function or module with a controlled substitute that records calls and returns preset values. A spy wraps a real function to observe calls without replacing its behaviour. Use mocks to isolate the unit under test from external dependencies (APIs, databases, file system). In Vitest, the mock API is on the vi object; in Jest it is on jest. TypeScript adds the challenge of keeping mock return types compatible — the vi.mocked() helper and explicit type annotations solve this cleanly.

Mock API (vi / jest)

  • vi.fn() — create a mock function
  • .mockReturnValue(val) — always return val
  • .mockResolvedValue(val) — return Promise.resolve(val)
  • .mockRejectedValue(err) — return Promise.reject(err)
  • .mockReturnValueOnce(val) — return val once only
  • .mockImplementation(fn) — replace with function
  • vi.mock("module") — mock entire module
  • vi.spyOn(obj, "method") — spy on a method

Mock & Spy Examples

import { describe, it, expect, vi, beforeEach } from "vitest";

// ── Mock function ─────────────────────────────────────────────
const mockFn = vi.fn();
mockFn.mockReturnValue(42);
mockFn();                          // 42
mockFn.mock.calls;                 // [[]]  — array of call args
mockFn.mock.calls.length;          // 1
expect(mockFn).toHaveBeenCalled();
expect(mockFn).toHaveBeenCalledTimes(1);
expect(mockFn).toHaveBeenCalledWith(/* args */);

// Return different values
mockFn.mockReturnValueOnce("first").mockReturnValueOnce("second");
mockFn();  // "first"
mockFn();  // "second"
mockFn();  // 42  — back to default

// Async mock
const fetchUser = vi.fn().mockResolvedValue({ id: 1, name: "Alice" });
const user = await fetchUser(1);    // { id: 1, name: "Alice" }

// ── Mocking a module ──────────────────────────────────────────
vi.mock("./emailService.js", () => ({
  sendEmail: vi.fn().mockResolvedValue({ sent: true }),
}));

import { sendEmail } from "./emailService.js";
// sendEmail is now a vi.fn() — calls are recorded, nothing sent

// ── Spy on a method ───────────────────────────────────────────
const console_spy = vi.spyOn(console, "log").mockImplementation(() => {});
doSomethingThatLogs();
expect(console_spy).toHaveBeenCalledWith("Expected log message");
console_spy.mockRestore();          // restore original console.log

// ── TypeScript: typed mocks ───────────────────────────────────
import type { UserService } from "./userService.js";

// Create a typed mock of an interface
const mockUserService: UserService = {
  findById: vi.fn().mockResolvedValue({ id: 1, name: "Alice" }),
  create:   vi.fn().mockResolvedValue({ id: 2, name: "Bob" }),
  delete:   vi.fn().mockResolvedValue(true),
};

// vi.mocked() — assert a function is a mock for type-safe assertions
import { readFile } from "fs/promises";
vi.mock("fs/promises");
const mockedReadFile = vi.mocked(readFile);
mockedReadFile.mockResolvedValue("file contents" as never);

// ── Reset between tests ────────────────────────────────────────
beforeEach(() => {
  vi.clearAllMocks();              // clear call counts and instances
  // vi.resetAllMocks();           // also remove return values
  // vi.restoreAllMocks();         // restore all spies
});

// ── Partial module mock ───────────────────────────────────────
vi.mock("./config.js", async (importOriginal) => {
  const actual = await importOriginal();  // keep real implementation
  return { ...actual, API_URL: "http://localhost:3000" }; // override one
});

Notes

vi.mock() is hoisted to the top of the file by Vitest — it runs before imports, allowing module substitution; same behaviour as jest.mock()
mockReturnValue vs mockImplementation — use returnValue for simple values; implementation when the mock needs logic
vi.spyOn — wraps the real method; .mockRestore() is required to put it back; use in afterEach
vi.clearAllMocks() — clears call history but keeps implementation · resetAllMocks() also removes return values · restoreAllMocks() undoes spies
Typed mock interfaces — create a plain object matching the interface shape with vi.fn() for each method; TS enforces the shape
Partial module mocks with importOriginal — keep most of a module real and only override specific exports; cleaner than mocking everything

Common Mistakes

  • Not calling vi.clearAllMocks() in beforeEach — mock call counts accumulate across tests, causing false failures
  • Forgetting mockRestore() on spies — the original method stays patched for subsequent tests
  • Mocking too much — if you mock all dependencies, you're testing the mock, not the code; only mock boundaries (I/O, external services)
  • Using vi.mock() with a relative path that doesn't match the import exactly — path must match the import string precisely

Tips

  • Use vi.clearAllMocks() in a global beforeEach in your test setup file — avoids repeating it in every describe block
  • Create typed mock objects matching your interfaces rather than using as any casts — maintains type safety in tests
  • Use partial module mocks (importOriginal) to keep real utility functions while mocking side-effectful ones

Summary

  • vi.fn() — mock function · .mockReturnValue/ResolvedValue/Implementation()
  • vi.mock("module") — replace module · vi.spyOn(obj, "method") — observe real method
  • clearAllMocks → history · resetAllMocks → values too · restoreAllMocks → undo spies
  • Typed mocks — match interface shape with vi.fn() per method · vi.mocked() for type-safe assertions
