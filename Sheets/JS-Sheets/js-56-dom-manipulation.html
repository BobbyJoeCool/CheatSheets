DOM Manipulation

createElement · append · remove · innerHTML · textContent · classList · style · dataset

Common DOM Element Properties

Property / MethodNotes
.innerHTMLGet/set HTML string — parses as HTML; XSS risk with user input
.textContentGet/set plain text — safe for user data; no HTML parsing
.innerTextLike textContent but respects CSS visibility; slower
.valueInput/textarea/select current value (always string)
.classListDOMTokenList — add, remove, toggle, contains, replace
.styleInline styles as CSSStyleDeclaration — camelCase property names
.datasetRead/write data-* attributes as a DOMStringMap
.setAttribute(k, v)Set any HTML attribute
.getAttribute(k)Get attribute value as string or null
.removeAttribute(k)Remove attribute entirely
.hiddenBoolean — sets/reads the hidden attribute
.disabledBoolean — applies to form elements

DOM Manipulation Examples

// ── Creating and inserting elements ──────────────────────────
const li = document.createElement("li");
li.textContent = "New item";
li.classList.add("item", "active");

const ul = document.querySelector("ul");
ul.appendChild(li);                   // add at end (legacy)
ul.append(li);                        // modern — accepts strings too
ul.prepend(li);                       // add at beginning
ul.append("plain text node");         // strings become text nodes

// Insert relative to existing element:
const ref = ul.querySelector("li:first-child");
ref.before(li);                       // insert before ref
ref.after(li);                        // insert after ref
ref.replaceWith(li);                  // replace ref with li

// ── Removing elements ─────────────────────────────────────────
li.remove();                          // remove self (modern)
ul.removeChild(ref);                  // legacy; remove a child

// ── innerHTML vs textContent ──────────────────────────────────
const div = document.querySelector("#output");

div.innerHTML = "<strong>Hello</strong>"; // parsed as HTML
div.innerHTML = userInput;             // ✖ XSS risk! Never do this

div.textContent = "<strong>Hello</strong>"; // shown as literal text
div.textContent = userInput;           // ✔ safe — no parsing

// ── classList ─────────────────────────────────────────────────
const btn = document.querySelector(".btn");
btn.classList.add("active");
btn.classList.remove("disabled");
btn.classList.toggle("open");         // add if absent, remove if present
btn.classList.toggle("open", isOpen); // force add/remove by boolean
btn.classList.contains("active");     // true/false
btn.classList.replace("old", "new");

// ── style — inline CSS ───────────────────────────────────────
btn.style.backgroundColor = "red";   // camelCase property names
btn.style.fontSize = "16px";
btn.style.display = "none";           // hide
btn.style.display = "";               // reset to stylesheet value

// Prefer classes over inline styles for anything beyond show/hide

// ── dataset — data-* attributes ──────────────────────────────
// HTML: <div data-user-id="42" data-role="admin">
const el = document.querySelector("[data-user-id]");
el.dataset.userId;                    // "42"  (camelCase in JS)
el.dataset.role;                      // "admin"
el.dataset.newProp = "value";         // sets data-new-prop="value"
delete el.dataset.role;               // removes data-role attribute

// ── attributes ────────────────────────────────────────────────
btn.setAttribute("aria-expanded", "true");
btn.getAttribute("aria-expanded");    // "true"
btn.removeAttribute("aria-expanded");
btn.hasAttribute("disabled");         // false

Notes

innerHTML — never set from user input; always sanitise or use textContent for untrusted data (XSS vulnerability)
append() (modern) vs appendChild() (legacy) — append accepts multiple nodes and strings; prefer it
classList.toggle(cls, force) — the second argument is a boolean that forces add (true) or remove (false)
style property names are camelCase — background-color → backgroundColor, font-size → fontSize
dataset auto-converts hyphenated attribute names to camelCase — data-user-id → dataset.userId
Prefer toggling CSS classes over setting inline styles — keeps styling in CSS and logic in JS, easier to maintain

Common Mistakes

  • Setting innerHTML from user input — opens XSS vulnerabilities; use textContent or sanitise with DOMPurify
  • Using hyphenated property names in .style — JS requires camelCase: style.backgroundColor not style["background-color"]
  • Creating elements inside a loop then appending one by one — batching with a DocumentFragment or building a string and setting innerHTML once is faster
  • Forgetting dataset converts hyphenated names — accessing dataset["user-id"] won't work; use dataset.userId

Tips

  • Use DocumentFragment when adding many elements — batch them into a fragment then append once to avoid repeated reflows
  • Use classList.toggle(cls, condition) instead of if/else add/remove — one line is clearer
  • Use el.hidden = true instead of el.style.display = "none" — cleaner and works with CSS [hidden] selectors

Summary

  • createElement + append/prepend/before/after — create and insert nodes
  • textContent — safe text · innerHTML — parsed HTML (never from user input)
  • classList.add/remove/toggle/contains — manage CSS classes cleanly
  • dataset.camelCase — read/write data-* attributes
