Axios Examples

// npm install axios
import axios from "axios";

// ── Basic requests ────────────────────────────────────────────
const { data } = await axios.get("/api/users");        // auto-parsed JSON
const res      = await axios.post("/api/users", { name: "Alice" });
await axios.put(`/api/users/${id}`, updatedUser);
await axios.delete(`/api/users/${id}`);
await axios.patch(`/api/users/${id}`, { name: "Bob" });

// ── TypeScript: typed responses ───────────────────────────────
interface User { id: number; name: string; email: string; }

const { data: user } = await axios.get<User>(`/api/users/1`);
// user is typed as User

const { data: users } = await axios.get<User[]>("/api/users");
// users is User[]

// ── Creating an instance ──────────────────────────────────────
const api = axios.create({
  baseURL: "https://api.example.com/v1",
  timeout: 10_000,                       // 10 second timeout
  headers: { "Content-Type": "application/json" },
});

// All requests use the base URL and default headers
const { data: profile } = await api.get("/users/me");

// ── Request interceptor — add auth token ─────────────────────
api.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem("token");
    if (token) config.headers.Authorization = `Bearer ${token}`;
    return config;                        // must return config
  },
  (error) => Promise.reject(error)
);

// ── Response interceptor — handle errors globally ────────────
api.interceptors.response.use(
  (response) => response,                // pass through successes
  async (error) => {
    if (error.response?.status === 401) {
      await refreshToken();               // try to refresh
      return api(error.config);          // retry original request
    }
    return Promise.reject(error);        // re-throw other errors
  }
);

// ── Error handling ────────────────────────────────────────────
try {
  await api.get("/protected");
} catch (err) {
  if (axios.isAxiosError(err)) {
    console.error("Status:", err.response?.status);
    console.error("Message:", err.response?.data?.message);
    console.error("URL:", err.config?.url);
  } else {
    throw err;                           // unexpected error
  }
}

// ── Request with query params ─────────────────────────────────
const { data: results } = await api.get("/search", {
  params: { q: "typescript", page: 2, limit: 10 },
});
// GET /search?q=typescript&page=2&limit=10

// ── Cancellation ──────────────────────────────────────────────
const controller = new AbortController();
const { data: stream } = await api.get("/feed", {
  signal: controller.signal,
});
// Later:
controller.abort();

Notes

Axios throws on 4xx/5xx responses — no manual res.ok check needed; the error is in err.response
Auto JSON — axios parses JSON responses and serialises JSON request bodies automatically; no JSON.stringify/parse needed
Instances with baseURL — each axios instance has its own interceptors and defaults; create one per API endpoint domain
Request interceptors — run before the request is sent; use for auth headers, request IDs, logging
Response interceptors — run before the caller receives the response; use for 401 refresh flows, global error handling, response transforms
axios.isAxiosError(err) — type guard that narrows the error to AxiosError with typed response, config, and request fields

Common Mistakes

  • Not using axios.isAxiosError() in catch — accessing err.response on a non-Axios error throws
  • Adding auth headers in every request instead of using a request interceptor — hard to maintain and easy to forget
  • Forgetting to return config from a request interceptor — the request hangs or throws
  • Creating a new axios instance for every request — instances should be singletons, created once and reused

Tips

  • Create one axios instance per API and export it — consistent baseURL, timeout, and interceptors across the whole feature
  • Use a response interceptor for 401 token refresh — centralises auth logic so every request benefits automatically
  • For simple projects, native fetch + a small wrapper is often enough; reach for axios when you need interceptors or older Node support

Summary

  • Axios throws on 4xx/5xx · auto-parses JSON · no res.ok check needed
  • Instances — axios.create({ baseURL, timeout, headers }) for per-API config
  • Interceptors — request (add auth) · response (refresh token, global errors)
  • axios.get<User>(url) — TypeScript generic types the data field
