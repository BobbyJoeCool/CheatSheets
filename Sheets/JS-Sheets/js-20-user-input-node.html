User Input — Node

readline · createInterface · process.stdin · @inquirer/prompts · piped input

Reading Input in Node.js

Node has no built-in prompt() — terminal input is handled through the readline module from the standard library. The modern async approach uses readline/promises (Node 17+), which returns Promises and works cleanly with await. For complex CLI interactions — menus, password fields, confirmations, autocomplete — the @inquirer/prompts package (formerly Inquirer.js) is the community standard. For piped or streamed input (e.g. cat file | node script.js), read from process.stdin directly.

readline Interface Methods

  • rl.question(prompt) — ask a question, get answer string
  • rl.close() — close the interface (required or process hangs)
  • rl[Symbol.asyncIterator]() — iterate lines with for await
  • process.stdin.setEncoding("utf8") — set encoding for raw stream
  • process.stdin.on("data", cb) — raw chunks from stdin
  • process.stdin.on("end", cb) — fires when stdin closes (EOF)

Node Input Examples

// ── readline/promises (Node 17+ — recommended) ───────────────
import { createInterface } from "readline/promises";
import { stdin, stdout } from "process";

const rl = createInterface({ input: stdin, output: stdout });

const name = await rl.question("What is your name? ");
const age  = await rl.question("What is your age? ");
console.log(`Hello, ${name}! You are ${Number(age)} years old.`);
rl.close();   // IMPORTANT — process hangs without this

// ── readline (callback / Node <17) ──────────────────────────
import readline from "readline";
const rl2 = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});
rl2.question("Enter value: ", (answer) => {
  console.log(`You entered: ${answer}`);
  rl2.close();
});

// ── Reading all lines from stdin (piped input) ───────────────
// Usage: echo "line1\nline2" | node script.js
const rl3 = createInterface({ input: process.stdin });
const lines = [];
for await (const line of rl3) {
  lines.push(line);
}
console.log(lines);  // ["line1", "line2"]

// ── process.stdin raw stream ─────────────────────────────────
process.stdin.setEncoding("utf8");
let buffer = "";
process.stdin.on("data", (chunk) => { buffer += chunk; });
process.stdin.on("end", () => {
  console.log("Full input:", buffer.trim());
});

// ── @inquirer/prompts — rich CLI interactions ────────────────
// npm install @inquirer/prompts
import { input, confirm, select, password, checkbox } from "@inquirer/prompts";

const username = await input({ message: "Username:" });
const pass = await password({ message: "Password:", mask: "*" });
const ok   = await confirm({ message: "Continue?", default: true });
const lang  = await select({
  message: "Pick a language:",
  choices: [
    { value: "js",  name: "JavaScript" },
    { value: "ts",  name: "TypeScript" },
    { value: "py",  name: "Python" },
  ],
});
const features = await checkbox({
  message: "Select features:",
  choices: [{ value: "lint" }, { value: "tests" }, { value: "ci" }],
});
console.log({ username, lang, features, ok });

Notes

rl.close() is mandatory — without it the process stays alive waiting for more input
readline/promises requires Node 17+ — for older Node use the callback API or a library
for await (const line of rl) — cleanest way to read all lines from a file or piped stdin
rl.question() returns a string — always convert to Number()/parseInt() when numeric input is expected
@inquirer/prompts handles arrow keys, terminal colouring, validation, and defaults automatically
When building a real CLI tool, also look at commander or yargs for parsing command-line arguments

Common Mistakes

  • Forgetting rl.close() — the Node process hangs indefinitely waiting for more input
  • Not converting rl.question() result to a number — it always returns a string
  • Using readline without await on the question — callbacks get messy fast; use readline/promises
  • Importing readline/promises in Node 16 or older — it does not exist; use callback API or upgrade

Tips

  • For any CLI beyond a single prompt, use @inquirer/prompts — it handles edge cases (Ctrl+C, invalid input) automatically
  • Use for await (const line of rl) to process large files line-by-line without loading the whole file into memory
  • Pipe data into your script (cat data.txt | node process.js) and read from process.stdin for Unix-style composability

Summary

  • readline/promises — modern async API; use await rl.question(); always call rl.close()
  • for await (const line of rl) — read all piped stdin lines into an array
  • process.stdin — raw stream; use for piped/binary input
  • @inquirer/prompts — rich CLI menus, selects, passwords, checkboxes
