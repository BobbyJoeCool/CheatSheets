Common expect() Matchers

MatcherPasses When
.toBe(val)Strict equality (===); for primitives
.toEqual(val)Deep equality; for objects and arrays
.toStrictEqual(val)Like toEqual but also checks object types and undefined properties
.toBeTruthy()Value is truthy
.toBeFalsy()Value is falsy
.toBeNull()Value is null
.toBeUndefined()Value is undefined
.toBeDefined()Value is not undefined
.toBeGreaterThan(n)Number is > n
.toBeLessThan(n)Number is < n
.toBeCloseTo(n, dp?)Float equality within decimal places
.toContain(item)Array contains item or string contains substring
.toHaveLength(n)Array/string has given length
.toThrow(err?)Function throws; optionally match message or type
.toMatchObject(obj)Object contains at least the given subset of properties
.toMatchSnapshot()Matches a saved snapshot (auto-created on first run)

Test Examples

// ── Setup (vitest.config.ts or jest.config.ts) ────────────────
// Vitest:  npm install -D vitest
// Jest:    npm install -D jest @types/jest ts-jest

// ── Basic test structure ──────────────────────────────────────
import { describe, it, expect, beforeEach, afterEach } from "vitest";
import { add, divide } from "./math.js";

describe("Math utilities", () => {

  it("adds two numbers", () => {
    expect(add(2, 3)).toBe(5);
  });

  it("handles negative numbers", () => {
    expect(add(-1, -2)).toBe(-3);
    expect(add(0, 0)).toBe(0);
  });

  it("throws on divide by zero", () => {
    expect(() => divide(10, 0)).toThrow("Division by zero");
    expect(() => divide(10, 0)).toThrow(RangeError);
  });
});

// ── Setup and teardown ────────────────────────────────────────
describe("Database tests", () => {
  let db;

  beforeEach(async () => {    // runs before EACH test
    db = await createTestDb();
    await db.seed();
  });

  afterEach(async () => {     // runs after EACH test
    await db.close();
  });

  // beforeAll / afterAll run once for the whole describe block

  it("finds a user", async () => {
    const user = await db.users.find(1);
    expect(user).toMatchObject({ id: 1, name: expect.any(String) });
  });
});

// ── Object and array assertions ───────────────────────────────
const user = { id: 1, name: "Alice", email: "a@b.com", role: "admin" };

expect(user).toMatchObject({ name: "Alice", role: "admin" }); // subset ok
expect(user).toHaveProperty("email");
expect(user).toHaveProperty("id", 1);

const arr = [1, 2, 3, 4, 5];
expect(arr).toContain(3);
expect(arr).toHaveLength(5);
expect(arr).toEqual(expect.arrayContaining([2, 4]));

// ── Async tests ───────────────────────────────────────────────
it("fetches user data", async () => {
  const user = await fetchUser(1);
  expect(user.id).toBe(1);
  expect(user.name).toBeDefined();
});

it("rejects on 404", async () => {
  await expect(fetchUser(999)).rejects.toThrow("Not found");
});

// ── test.each — parameterised tests ──────────────────────────
it.each([
  [1, 1, 2],
  [2, 3, 5],
  [-1, 1, 0],
])("add(%i, %i) = %i", (a, b, expected) => {
  expect(add(a, b)).toBe(expected);
});

Notes

toBe uses === — use for primitives; for objects always use toEqual (deep equality)
toMatchObject — the object only needs to contain the given subset; extra properties are ignored; great for API response checks
toThrow — the function must be wrapped in an arrow: expect(() => fn()).toThrow(); calling expect(fn()).toThrow() would throw before expect sees it
beforeEach/afterEach run around every individual test; beforeAll/afterAll run once per describe block
Vitest is a drop-in replacement for Jest with native ESM, TypeScript, and Vite integration — prefer it for new projects
it.each / test.each — runs the same test with multiple input sets; avoids copy-pasted test cases

Common Mistakes

  • Using toBe to compare objects — always fails as it checks reference equality; use toEqual
  • Not wrapping throwing functions: expect(fn()).toThrow() — the error fires before expect can catch it
  • Forgetting await on async tests — test passes before the assertion runs
  • Sharing mutable state between tests — use beforeEach to reset state; tests should be independent

Tips

  • Use it.each for boundary conditions and multiple input scenarios — fewer lines, same coverage
  • Name tests as full sentences: "returns null when user is not found" — failing tests read as clear bug descriptions
  • Use expect.any(String), expect.any(Number) inside toMatchObject to assert type without knowing the exact value

Summary

  • describe groups · it/test cases · expect(val).matcher()
  • toBe strict · toEqual deep · toMatchObject subset · toThrow exceptions
  • beforeEach/afterEach — per test · beforeAll/afterAll — per describe block
  • Async tests need async/await · use it.each for parameterised cases
