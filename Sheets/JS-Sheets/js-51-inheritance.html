Inheritance & Polymorphism

extends · super · method overriding · instanceof · prototype chain · polymorphism

Inheritance in JavaScript

The extends keyword creates a subclass that inherits all methods and properties from a parent class. The super() call in the constructor invokes the parent constructor — it is mandatory before accessing this in a derived class. super.method() calls the parent version of an overridden method. Polymorphism means different subclasses can be treated as the same base type — each responding to the same method call in its own way. JavaScript supports only single inheritance (one parent class), but mixins can simulate multiple inheritance patterns.

Key Points

  • extends — subclass inherits parent's methods
  • super() — must be called before this in derived constructor
  • super.method() — call parent's overridden method
  • Method overriding — same name, different implementation
  • instanceof — checks prototype chain
  • JS is single-inheritance only (one parent class)
  • All classes implicitly extend Object

Inheritance & Polymorphism Examples

// ── Base class ────────────────────────────────────────────────
class Shape {
  constructor(color = "black") {
    this.color = color;
  }

  area() {
    throw new Error("area() must be implemented by subclass");
  }

  toString() {
    return `${this.constructor.name}(color=${this.color}, area=${this.area().toFixed(2)})`;
  }
}

// ── Subclasses ────────────────────────────────────────────────
class Circle extends Shape {
  constructor(radius, color) {
    super(color);               // must call super() before using this
    this.radius = radius;
  }

  area() {                      // override parent method
    return Math.PI * this.radius ** 2;
  }
}

class Rectangle extends Shape {
  constructor(width, height, color) {
    super(color);
    this.width  = width;
    this.height = height;
  }

  area() {
    return this.width * this.height;
  }
}

class Square extends Rectangle {
  constructor(side, color) {
    super(side, side, color);   // reuse Rectangle's constructor
  }
  // inherits area() from Rectangle automatically
}

// ── Polymorphism — treat all as Shape ────────────────────────
const shapes = [
  new Circle(5, "red"),
  new Rectangle(4, 6, "blue"),
  new Square(3, "green"),
];

shapes.forEach(s => console.log(s.toString()));
// Circle(color=red, area=78.54)
// Rectangle(color=blue, area=24.00)
// Square(color=green, area=9.00)

const totalArea = shapes.reduce((sum, s) => sum + s.area(), 0);

// ── super.method() ────────────────────────────────────────────
class Animal {
  speak() { return "..."; }
}
class Dog extends Animal {
  speak() {
    const parent = super.speak(); // call parent version
    return `Woof! (parent said: ${parent})`;
  }
}
new Dog().speak();  // "Woof! (parent said: ...)"

// ── instanceof — check prototype chain ───────────────────────
const c = new Circle(3);
c instanceof Circle;    // true
c instanceof Shape;     // true  — inherits from Shape
c instanceof Object;    // true  — all objects inherit from Object
c instanceof Rectangle; // false

// ── Mixin — simulate multiple inheritance ────────────────────
const Serializable = (Base) => class extends Base {
  serialize()   { return JSON.stringify(this); }
  toJSON()      { return { ...this }; }
};

const Validatable = (Base) => class extends Base {
  validate() { return Object.keys(this).every(k => this[k] != null); }
};

class User extends Serializable(Validatable(Shape)) {
  constructor(name) { super(); this.name = name; }
  area() { return 0; }
}
const u = new User("Alice");
u.serialize();   // '{"color":"black","name":"Alice"}'
u.validate();    // true

Notes

super() must be the first statement in a derived class constructor — accessing this before calling it throws a ReferenceError
Omitting the constructor in a subclass — JS automatically generates constructor(...args) { super(...args); }
this.constructor.name — returns the actual class name of the instance, not the parent's name; useful for logging
Throwing in the base area() — enforces that subclasses must override it; this is the JS equivalent of an abstract method
Mixin pattern — a function that takes a class and returns an extended class; enables composing multiple behaviours without multiple inheritance
instanceof traverses the full prototype chain — a Square is an instanceof Rectangle, Shape, and Object

Common Mistakes

  • Accessing this before super() in a derived constructor — throws ReferenceError
  • Forgetting to call super() entirely — constructor will throw without it in a derived class
  • Deep inheritance chains — more than 2–3 levels becomes hard to follow; favour composition over deep inheritance
  • Calling super.method() outside a class method — only valid inside a class definition

Tips

  • Favour composition over inheritance for complex behaviours — mixins or injecting dependencies is usually more flexible
  • Use the mixin pattern to share behaviour across unrelated classes without forcing them into a hierarchy
  • Throw in base class methods that must be overridden — it is the closest JS gets to abstract methods without TypeScript

Summary

  • extends — inherit parent · super() — required before this in derived constructor
  • Method overriding — redefine in subclass; call parent with super.method()
  • Polymorphism — treat subclass instances as their base type; each responds uniquely
  • Mixin pattern — functions that extend a class to simulate multiple inheritance
