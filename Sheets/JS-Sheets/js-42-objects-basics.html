Object Examples

// ── Object literal ────────────────────────────────────────────
const user = {
  name: "Alice",
  age: 30,
  greet() { return `Hi, I'm ${this.name}`; }, // method shorthand
};

// ── Property access ───────────────────────────────────────────
user.name               // "Alice"  — dot notation
user["age"]             // 30       — bracket notation
user["non-existent"]    // undefined (no error)
user.address?.city      // undefined (optional chaining)

// ── Property shorthand ────────────────────────────────────────
const name = "Bob";
const age  = 25;
const bob  = { name, age }; // same as { name: name, age: age }

// ── Computed property keys ────────────────────────────────────
const key = "color";
const theme = { [key]: "blue" };          // { color: "blue" }
const prefix = "user";
const obj = {
  [`${prefix}Name`]: "Alice",
  [`${prefix}Age`]: 30,
};                                         // { userName: "Alice", userAge: 30 }

// ── Spread — shallow copy and merge ──────────────────────────
const defaults = { theme: "light", lang: "en", fontSize: 14 };
const prefs    = { theme: "dark", fontSize: 16 };
const config   = { ...defaults, ...prefs };
// { theme: "dark", lang: "en", fontSize: 16 }  — prefs wins

const copy = { ...user };   // shallow copy

// ── Object.keys / values / entries ───────────────────────────
Object.keys(user);          // ["name", "age", "greet"]
Object.values(user);        // ["Alice", 30, fn]
Object.entries(user);       // [["name","Alice"],["age",30],["greet",fn]]

// Iterate with for...of + entries
for (const [k, v] of Object.entries(user)) {
  console.log(k, v);
}

// ── Object.fromEntries — transform objects ────────────────────
const upper = Object.fromEntries(
  Object.entries(user).map(([k, v]) =>
    [k, typeof v === "string" ? v.toUpperCase() : v]
  )
);

// Convert Map → object
const map = new Map([["a", 1], ["b", 2]]);
Object.fromEntries(map);    // { a: 1, b: 2 }

// ── Object.assign ─────────────────────────────────────────────
const target = { a: 1 };
Object.assign(target, { b: 2 }, { c: 3 });
// target = { a: 1, b: 2, c: 3 }  — mutated

// ── Object.hasOwn (modern preferred) ─────────────────────────
Object.hasOwn(user, "name");   // true
Object.hasOwn(user, "email");  // false

Notes

Bracket notation — required when key is dynamic, contains special chars, or is stored in a variable
Spread objects are shallow — nested objects are shared references; use structuredClone for deep copies
In spread merges, later keys overwrite earlier ones — put defaults first, overrides last
Object.assign mutates the target — use { ...a, ...b } spread instead when you don't want mutation
Object.hasOwn(obj, key) — preferred over obj.hasOwnProperty(key); works correctly even if hasOwnProperty is overridden
Object.fromEntries + Object.entries — the idiom for transforming object values without reaching for a library

Common Mistakes

  • Assuming object spread is a deep clone — nested objects are still shared references
  • Using Object.assign(target, src) without realising it mutates target — use spread instead if mutation is unwanted
  • Iterating with for...in and getting inherited prototype properties — use Object.keys() or Object.hasOwn()
  • Accessing a property that doesn't exist expecting an error — JS returns undefined silently

Tips

  • Use Object.fromEntries(Object.entries(obj).map(...)) to transform object values in a clean one-liner
  • Use property shorthand ({ name, age }) whenever the variable name matches the key — reduces visual noise
  • Use computed keys ({ [varName]: value }) to build dynamic objects without needing a separate assignment step

Summary

  • Dot notation for known keys · bracket notation for dynamic keys
  • Shorthand { name } · computed keys { [expr]: val } · spread { ...obj }
  • Object.keys/values/entries — iterate own properties · fromEntries — rebuild from pairs
  • Spread is shallow · Object.assign mutates target · Object.hasOwn for safe key checks
