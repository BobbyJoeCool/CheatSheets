Working with CSV

CSV (Comma-Separated Values) looks simple but has many edge cases: fields with commas must be quoted, quotes inside fields are escaped by doubling them (""), and line endings vary (\r\n vs \n). Writing a production-grade CSV parser from scratch is error-prone — use papaparse (browser + Node) for parsing, and build simple output with array methods for small cases. In Node.js, large CSV files should be streamed rather than loaded entirely into memory. The browser FileReader API reads user-uploaded CSV files as text for in-browser parsing.

CSV Edge Cases

  • Fields with commas must be wrapped in quotes
  • Quotes inside quoted fields are doubled: ""
  • Line endings: \r\n (Windows) or \n (Unix)
  • Some tools use ; or \t as delimiter
  • First row is often (but not always) a header
  • Numbers/booleans are always strings after naive parsing
  • Empty fields: a,,b → ["a", "", "b"]

CSV Examples

// ── Simple manual parse (no edge cases) ──────────────────────
function parseCSVSimple(text) {
  const lines = text.trim().split("\n");
  const headers = lines[0].split(",");
  return lines.slice(1).map(line => {
    const values = line.split(",");
    return Object.fromEntries(headers.map((h, i) => [h.trim(), values[i]?.trim()]));
  });
}
// Only safe for CSV with no quoted fields or embedded commas

// ── papaparse — production-grade CSV parser ───────────────────
// npm install papaparse
import Papa from "papaparse";

// Parse CSV string
const result = Papa.parse(csvString, {
  header: true,          // use first row as object keys
  dynamicTyping: true,   // auto-convert numbers and booleans
  skipEmptyLines: true,
});
result.data;             // array of objects
result.errors;           // any parse errors

// Parse from file (Node)
import { createReadStream } from "fs";
Papa.parse(createReadStream("./data.csv"), {
  header: true,
  dynamicTyping: true,
  step: (row) => {        // process row-by-row (streaming)
    console.log(row.data);
  },
  complete: (results) => {
    console.log("Done:", results.data.length, "rows");
  },
});

// Parse from browser file input
fileInput.addEventListener("change", (e) => {
  const file = e.target.files[0];
  Papa.parse(file, {
    header: true,
    dynamicTyping: true,
    complete: (results) => {
      console.log(results.data);
    },
  });
});

// ── Generating CSV ─────────────────────────────────────────────
const users = [
  { name: "Alice", age: 30, city: "NYC" },
  { name: "Bob, Jr.", age: 25, city: "LA" },  // comma in name
];

// Manual generation with proper quoting
function toCSV(rows) {
  if (!rows.length) return "";
  const headers = Object.keys(rows[0]);
  const escape  = (v) => {
    const s = String(v);
    return s.includes(",") || s.includes('"') || s.includes("\n")
      ? `"${s.replace(/"/g, '""')}"`
      : s;
  };
  const lines = [
    headers.join(","),
    ...rows.map(row => headers.map(h => escape(row[h])).join(","))
  ];
  return lines.join("\n");
}

// Using papaparse to generate
const csv = Papa.unparse(users);   // array of objects → CSV string

// ── Browser FileReader — read uploaded CSV ────────────────────
fileInput.addEventListener("change", (e) => {
  const reader = new FileReader();
  reader.onload = (event) => {
    const text = event.target.result;   // full CSV as string
    const parsed = Papa.parse(text, { header: true });
    console.log(parsed.data);
  };
  reader.readAsText(e.target.files[0]);
});

Notes

Manual parsing with split(",") breaks on fields containing commas or newlines — only safe for controlled simple data
papaparse dynamicTyping: true — auto-converts "42" to 42 and "true" to true; turn off if you need raw strings
papaparse step callback — processes one row at a time; essential for large files; no full array in memory
Proper CSV quoting — a field must be quoted if it contains a comma, quote, or newline; quotes inside are doubled (""), not backslash-escaped
FileReader.readAsText() — asynchronous; result is available in the onload callback
For download in browser: const blob = new Blob([csv], { type: "text/csv" }) then create an anchor with URL.createObjectURL(blob)

Common Mistakes

  • Splitting on "," naively — breaks for quoted fields containing commas
  • Not quoting fields with embedded commas in output — produces malformed CSV
  • Loading a 500MB CSV with readFile into memory — use streaming with papaparse step callback
  • Assuming all CSV uses , delimiter — some locales use ;; check or make configurable

Tips

  • Use papaparse for both reading and writing — it is the de facto standard and handles all edge cases
  • Always use dynamicTyping: true when you want numbers/booleans — avoids manual conversion of every field
  • For browser downloads, create a Blob and programmatically click a hidden anchor with download attribute

Summary

  • Manual split(",") — only for trivial cases; breaks on quoted fields
  • papaparse — production parser; handles all edge cases; streaming support
  • Generating CSV — quote fields with commas/quotes/newlines; double embedded quotes
  • Browser: FileReader.readAsText() → string → parse · Node: stream with step for large files
