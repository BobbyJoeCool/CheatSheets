this & Binding

this in functions · methods · classes · call · apply · bind · arrow this

this Binding Examples

// ── Method call — this is the object ─────────────────────────
const user = {
  name: "Alice",
  greet() {
    return `Hello, I'm ${this.name}`; // this = user
  },
};
user.greet();                 // "Hello, I'm Alice"

// Detaching a method loses this
const fn = user.greet;
fn();                         // "Hello, I'm undefined" (strict)

// ── Arrow function — inherits this from enclosing scope ───────
const timer = {
  seconds: 0,
  start() {
    setInterval(() => {
      this.seconds++;          // ✔ this = timer (arrow inherits)
      console.log(this.seconds);
    }, 1000);
  },
};

// Using regular function in same scenario
const timer2 = {
  seconds: 0,
  start() {
    setInterval(function() {
      this.seconds++;          // ✖ this = globalThis / undefined
    }, 1000);
  },
};

// ── call() — invoke with explicit this, args listed ──────────
function introduce(greeting, punctuation) {
  return `${greeting}, I'm ${this.name}${punctuation}`;
}
introduce.call({ name: "Bob" }, "Hi", "!");  // "Hi, I'm Bob!"

// ── apply() — like call but args as array ────────────────────
introduce.apply({ name: "Carol" }, ["Hey", "."]);  // "Hey, I'm Carol."

// Useful: spread array into a function expecting individual args
const nums = [3, 1, 4, 1, 5];
Math.max.apply(null, nums);   // 5  (use spread instead: Math.max(...nums))

// ── bind() — return new function with locked this ─────────────
const greetAlice = introduce.bind({ name: "Alice" });
greetAlice("Hello", "!");     // "Hello, I'm Alice!"

// Pre-fill args (partial application)
const helloAlice = introduce.bind({ name: "Alice" }, "Hello");
helloAlice("!");              // "Hello, I'm Alice!"

// ── Class methods and this ────────────────────────────────────
class Counter {
  count = 0;

  // Arrow as class field — this is permanently bound to instance
  increment = () => {
    this.count++;              // always safe to pass as callback
  };

  // Regular method — this depends on how it's called
  decrement() {
    this.count--;
  }
}
const counter = new Counter();
const inc = counter.increment; // safe — arrow preserves this
const dec = counter.decrement; // ✖ this lost when detached

Notes

this is determined at call time, not at definition time — except for arrow functions which fix it at definition
Arrow functions have no own this — they look up the scope chain; ideal for callbacks inside methods
call vs apply — identical except argument format; call lists args, apply takes an array
bind returns a new function permanently bound to the given this — calling bind again or using call/apply won't override it
Class field arrows (increment = () => {}) — automatically bound to the instance; safe to pass as callbacks without .bind()
DOM event handlers with regular functions — this is the element; with arrow functions this is the enclosing scope

Common Mistakes

  • Detaching a method and calling it standalone — this is lost; use .bind() or an arrow wrapper
  • Using a regular function as a setTimeout/setInterval callback inside a method — this won't be the object
  • Using an arrow function as an object method expecting this to be the object — arrows inherit outer this, not the object
  • Calling bind() multiple times expecting to change this — only the first bind sticks

Tips

  • Use arrow functions inside methods for callbacks (setTimeout, forEach, event listeners) to avoid this issues
  • Use class field arrow functions for event handler methods that will be passed as callbacks: handleClick = () => {}
  • When in doubt, console.log(this) at the top of the function — it shows exactly what this is at runtime

Summary

  • this is set at call time — method call sets it to the object; standalone loses it
  • Arrow functions have no own this — they inherit lexically; perfect for callbacks
  • call(ctx, args) · apply(ctx, [args]) — invoke with explicit this
  • bind(ctx) — returns new permanently-bound function
