How to Execute JS / TS

CommandWhat It Does
node file.jsRun a JS file with Node.js
node --watch file.jsRun and re-run on file change (Node 18+)
tscCompile all TS files per tsconfig.json → outputs JS
tsc --watchCompile in watch mode — recompiles on save
tsc --noEmitType-check only — no JS output (CI-friendly)
npx ts-node file.tsRun TS directly in Node (transpiles on the fly)
npx tsx file.tsFaster TS runner via esbuild — no type checking
deno run file.tsRun TS natively in Deno
bun run file.tsRun TS natively in Bun
node dist/index.jsRun compiled output after tsc

Loading JS in the Browser

Tag / AttributeBehaviour
<script src="...">Blocks HTML parsing while loading & executing (legacy)
<script defer>Downloads in parallel; executes after HTML parsed — recommended
<script async>Downloads in parallel; executes immediately when ready (order not guaranteed)
<script type="module">Enables ES module imports; always deferred; always strict mode

Common Mistakes

  • Using .js for TypeScript source files — use .ts
  • Forgetting type="module" in the script tag when using import in the browser
  • Running tsc and expecting it to execute the code — it only compiles

Tips

  • Use tsx for fast dev iteration — it skips type checking for speed
  • Use tsc --noEmit in CI to catch type errors without producing output files
  • Prefer defer over inline <script> at bottom of body — same effect, cleaner HTML

Summary

  • .js/.mjs/.cjs — JavaScript variants; .ts/.tsx — TypeScript
  • Node runs JS; Deno and Bun run TS natively
  • tsc compiles TS → JS; tsx/ts-node run TS directly
  • Use type="module" in HTML for ES module support
