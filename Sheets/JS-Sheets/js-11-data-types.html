Type Examples & typeof

// ── typeof operator ──────────────────────────────────────────
typeof 42              // "number"
typeof 3.14            // "number"
typeof "hello"         // "string"
typeof true            // "boolean"
typeof undefined       // "undefined"
typeof null            // "object"  ← famous JS quirk / bug
typeof Symbol()        // "symbol"
typeof 42n             // "bigint"
typeof {}              // "object"
typeof []              // "object"  ← arrays are objects
typeof function(){}    // "function"

// ── Special number values ────────────────────────────────────
const a = 1 / 0;        // Infinity
const b = -1 / 0;       // -Infinity
const c = 0 / 0;        // NaN  (Not a Number)
const d = parseInt("abc"); // NaN

isNaN(NaN);             // true
isNaN("hello");         // true  (coerces first — unreliable)
Number.isNaN(NaN);      // true  (strict — preferred)
Number.isNaN("hello");  // false (no coercion)
Number.isFinite(Infinity); // false
Number.isFinite(42);    // true

// NaN is the only value not equal to itself
NaN === NaN;            // false  ← use Number.isNaN() instead

// ── null vs undefined ────────────────────────────────────────
let x;                  // undefined — declared, no value
let y = null;           // null — explicitly empty

x == null;              // true  (loose — catches both null & undefined)
x === null;             // false (strict)
x === undefined;        // true  (strict)

// ── TypeScript primitive types ───────────────────────────────
const n: number    = 42;
const s: string    = "hi";
const b2: boolean  = true;
const nil: null    = null;
const u: undefined = undefined;
const big: bigint  = 100n;
const sym: symbol  = Symbol("key");

// Union types
let id: number | string = 42;
id = "abc";             // ✔ also valid

// Literal types
let dir: "north" | "south" | "east" | "west" = "north";

Notes

typeof null === "object" — a historical bug in JS; never fixed for backward compatibility
typeof [] returns "object" — use Array.isArray() to check for arrays
Primitives are immutable and compared by value; reference types are compared by reference (memory address)
NaN !== NaN — always use Number.isNaN(), never == or === to check for NaN
null must be assigned explicitly; undefined is the default unassigned state
In TS, strictNullChecks: true forces you to handle null and undefined explicitly

Common Mistakes

  • Using typeof x === "null" — always returns "object"; use x === null instead
  • Using isNaN() instead of Number.isNaN() — the global version coerces its argument first
  • Comparing objects/arrays with === expecting value equality — compares references, not contents
  • Mixing number and bigint in arithmetic — throws a TypeError

Tips

  • Use Array.isArray(x) to check for arrays — typeof returns "object" for both arrays and objects
  • Use x == null (loose equality) as a shorthand that catches both null and undefined
  • In TypeScript, turn on strictNullChecks (included in strict: true) — it makes null/undefined handling explicit and safe

Summary

  • 7 primitives: number · bigint · string · boolean · null · undefined · symbol
  • typeof — returns type as a string; typeof null === "object" is a known bug
  • NaN — not equal to itself; always use Number.isNaN() to check
  • Primitives compared by value; objects/arrays compared by reference
