@param · @returns · @throws · @example · @typedef · @deprecated · IntelliSense

Common JSDoc Tags

TagPurposeExample

JSDoc Examples

/**
 * Calculates the area of a rectangle.
 *
 * @param {number} width  - Width in pixels
 * @param {number} height - Height in pixels
 * @returns {number} The area (width × height)
 * @example
 * area(4, 5);  // 20
 * area(0, 10); // 0
 */
function area(width, height) {
  return width * height;
}

/**
 * Fetches a user by ID.
 *
 * @param {string}  id        - The user's UUID
 * @param {boolean} [cache=true] - Use cached result (optional)
 * @returns {Promise<User>} Resolves with the user object
 * @throws {Error} If the user is not found (404)
 * @throws {TypeError} If id is not a string
 */
async function getUser(id, cache = true) { /* ... */ }

// ── @typedef — define reusable types in .js files ────────────
/**
 * @typedef {{ id: number, name: string, email: string }} User
 */

/** @type {User[]} */
const users = [];

// ── @template — generics in .js files ───────────────────────
/**
 * Returns the first element of an array.
 * @template T
 * @param {T[]} arr
 * @returns {T | undefined}
 */
function first(arr) { return arr[0]; }

// ── @deprecated ──────────────────────────────────────────────
/**
 * @deprecated Use `getUser()` instead — will be removed in v3
 */
function fetchUser(id) { return getUser(id); }

// ── TypeScript: JSDoc for descriptions, not types ────────────
// In .ts files, prefer TS annotations; use JSDoc for docs only
/**
 * @param width  - Width in device pixels (not CSS pixels)
 * @param height - Height in device pixels (not CSS pixels)
 */
function areaTS(width: number, height: number): number {
  return width * height;
}

Notes

JSDoc uses /** */ (double star) — single /* */ is not parsed by tooling
VS Code shows JSDoc comments as formatted tooltips on hover — no plugin required
Optional params are wrapped in square brackets: @param {boolean} [cache=true]
@typedef + @type gives you type checking and IntelliSense in plain .js files
In TypeScript files, @param types are redundant — TS already knows them; use JSDoc for descriptions only
@deprecated causes VS Code to render the function name with a strikethrough everywhere it is called
Generate HTML docs with npx jsdoc src/ -r -d docs/

Common Mistakes

  • Using /* */ instead of /** */ — single-star blocks are not parsed as JSDoc
  • Duplicating type info in both TS annotations and @param {type} — redundant and can go out of sync
  • Forgetting to update JSDoc when changing a function signature — stale docs are worse than none

Tips

  • Type /** then Enter in VS Code — it auto-generates a JSDoc stub with @param and @returns for the function below
  • Add // @ts-check at the top of a .js file to get full TS IntelliSense powered by your JSDoc types
  • Use TypeDoc (npm) instead of plain JSDoc for richer TS-aware documentation generation

Summary

  • JSDoc uses /** */ — parsed by VS Code, TypeDoc, and JSDoc CLI
  • @param · @returns · @throws · @example — the core four tags for most functions
  • @typedef · @type · @template — bring type checking to plain .js files
  • In .ts files, use JSDoc for descriptions only — let TypeScript handle the types
