Special TypeScript Types

TypeMeaningWhen to Use
anyOpts out of type checking entirelyLast resort; disables all TS safety for that value
unknownUnknown type — must narrow before useSafer alternative to any; use for untrusted input
neverValue that never occursFunctions that always throw or loop forever; exhaustive checks
voidFunction returns nothing meaningfulReturn type of functions that return undefined
undefinedExplicitly undefinedOptional values; unset properties
nullExplicitly nullIntentionally absent value
objectAny non-primitiveRarely used directly; prefer specific types

TS Type Examples

// ── Basic annotations ────────────────────────────────────────
const age: number = 30;
const name: string = "Alice";
const active: boolean = true;
const data: null = null;
const x: undefined = undefined;

// ── any — escape hatch (avoid if possible) ───────────────────
let val: any = 42;
val = "now a string";         // ✔ no error — any accepts all
val.foo.bar.baz;              // ✔ no error — dangerous!

// ── unknown — safe alternative to any ───────────────────────
let input: unknown = getUserInput();
input.toUpperCase();          // ✖ Error: must narrow first
if (typeof input === "string") {
  input.toUpperCase();        // ✔ narrowed to string
}

// ── never — impossible values ────────────────────────────────
function fail(msg: string): never {
  throw new Error(msg);       // never returns
}
function loop(): never {
  while (true) {}             // never returns
}
// Exhaustive check pattern
type Shape = "circle" | "square";
function area(s: Shape): number {
  if (s === "circle") return 3.14;
  if (s === "square") return 1;
  const _check: never = s;    // TS errors if a case is missing
  return _check;
}

// ── void ─────────────────────────────────────────────────────
function logMsg(msg: string): void {
  console.log(msg);           // returns undefined implicitly
}

// ── Union types ──────────────────────────────────────────────
let id: number | string = 42;
id = "abc";                   // ✔ both valid

function format(val: string | number): string {
  return String(val);
}

// ── Intersection types ───────────────────────────────────────
type Named  = { name: string };
type Aged   = { age: number };
type Person = Named & Aged;   // has both name AND age

const p: Person = { name: "Alice", age: 30 }; // ✔

// ── Literal types ────────────────────────────────────────────
let dir: "north" | "south" | "east" | "west" = "north";
let status: 200 | 404 | 500 = 200;

// ── as const ─────────────────────────────────────────────────
const DIRS = ["north", "south", "east", "west"] as const;
// type: readonly ["north", "south", "east", "west"]
type Dir = typeof DIRS[number]; // "north" | "south" | "east" | "west"

Notes

any — completely disables type checking; propagates through expressions silently
unknown — like any but safe: you must narrow the type before using it
never — useful for exhaustive switch/if checks; TS will error if you reach an unreachable branch
void vs undefined — void says "caller should not use return value"; undefined is the actual value
| union — value is one of the listed types
& intersection — value must satisfy all of the listed types simultaneously
as const — narrows string/number literals to their exact values and makes arrays/objects readonly

Common Mistakes

  • Reaching for any when stuck — try unknown first; it forces you to handle the type safely
  • Annotating return type as void when the function actually returns a value — use the correct type
  • Confusing union (|) and intersection (&) — union is "or", intersection is "and"
  • Forgetting as const on config objects — without it, string literals widen to string

Tips

  • Use unknown for any value coming from outside your codebase (API responses, user input, JSON.parse)
  • Use the never exhaustive check pattern in discriminated unions to get a compile error when you add a new case and forget to handle it
  • Use as const on lookup arrays to auto-derive a union type with typeof ARR[number]

Summary

  • any — no checking; unknown — safe escape hatch; never — unreachable; void — no return
  • A | B union — one or the other; A & B intersection — both at once
  • Literal types — exact string/number values as types: "north" | "south"
  • as const — freeze literal types and make structures readonly
