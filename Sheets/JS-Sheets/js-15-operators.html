Common Mistakes

  • Using == instead of === — loose equality has unexpected coercion rules
  • Using || for defaults when 0 or "" are valid values — use ?? instead
  • Forgetting ++x vs x++ — prefix returns new value; postfix returns original

Tips

  • Always use === and !== — enable the ESLint eqeqeq rule to enforce this
  • Use ??= to lazily initialise: cache.key ??= expensiveCompute()
  • Use ?. chaining to safely access deeply nested properties without null checks: user?.address?.city

Summary

  • Always use === / !== — avoid == / !=
  • ?? — nullish coalescing; only falls back on null/undefined (not 0 or "")
  • ?. — optional chaining; short-circuits to undefined if left side is null/undefined
  • ??= · ||= · &&= — logical assignment operators; assign conditionally
