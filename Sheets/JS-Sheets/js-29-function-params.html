Sheet 29

Parameters & Arguments

Default values · rest params · spread args · arguments object · TS overloads

Parameters vs Arguments

Parameters are the names listed in the function definition. Arguments are the values passed when the function is called. JS does not enforce argument count — you can pass more or fewer arguments than declared parameters without error. Extra arguments are ignored (or accessible via arguments); missing ones become undefined. Default parameters provide a fallback when an argument is undefined. Rest parameters (...name) collect all remaining arguments into an array. The spread operator expands an array into individual arguments at the call site.

Key Points

  • Missing args → undefined; extra args → ignored
  • Default params fire on undefined, not null
  • Defaults can reference earlier params: (a, b = a * 2)
  • ...rest must be the last parameter
  • arguments object — not available in arrow functions
  • Spread at call site: fn(...arr) expands array
  • TS overloads — multiple call signatures, one implementation

Parameter Examples

// ── Default parameters ────────────────────────────────────────
function greet(name = "stranger") {
  return `Hello, ${name}!`;
}
greet("Alice");       // "Hello, Alice!"
greet();              // "Hello, stranger!"
greet(undefined);     // "Hello, stranger!"  ← undefined triggers default
greet(null);          // "Hello, null!"       ← null does NOT trigger default

// Default can reference earlier params
function rect(width = 10, height = width) {
  return width * height;
}
rect();               // 100  (10 × 10)
rect(5);              // 25   (5 × 5)
rect(4, 6);           // 24

// Default can be an expression or function call
function timestamp(date = new Date()) {
  return date.toISOString();
}

// ── Rest parameters ───────────────────────────────────────────
function sum(...nums) {               // nums is a real Array
  return nums.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3, 4);      // 10
sum();                 // 0

function log(level, ...messages) {
  console.log(`[${level}]`, ...messages);
}
log("INFO", "started", "pid:", 123);  // [INFO] started pid: 123

// ── arguments object (legacy — avoid in modern code) ─────────
function oldStyle() {
  console.log(arguments[0]);  // first arg
  console.log(arguments.length);
  // arguments is NOT a real array — no .map(), .filter() etc.
  const arr = Array.from(arguments); // convert if needed
}
// Not available in arrow functions — use rest params instead

// ── Spread at call site ───────────────────────────────────────
const nums = [3, 1, 4, 1, 5];
Math.max(...nums);    // 5  — same as Math.max(3, 1, 4, 1, 5)

function add3(a, b, c) { return a + b + c; }
add3(...nums);        // 8  (3 + 1 + 4)

// ── TypeScript: typed parameters ─────────────────────────────
function add(a: number, b: number = 0): number {
  return a + b;
}

function joinAll(sep: string, ...words: string[]): string {
  return words.join(sep);
}
joinAll(", ", "a", "b", "c");  // "a, b, c"

// TS overloads — multiple call signatures
function format(value: number): string;
function format(value: string): string;
function format(value: number | string): string {
  return typeof value === "number" ? value.toFixed(2) : value.trim();
}
format(3.14159);  // "3.14"
format("  hi  "); // "hi"

Notes

Default params only fire on undefined — passing null explicitly does not trigger the default
Rest params create a real Array — you can use .map(), .filter() etc. directly; arguments is not a real array
arguments is not available in arrow functions — always use rest params in modern code
Rest must be last — function(a, ...b, c) is a SyntaxError
Spread at call site is the modern replacement for Function.apply(null, arr)
TS overloads — the implementation signature is not part of the public API; callers only see the overload signatures

Common Mistakes

  • Passing null expecting a default — null is not undefined; use ?? inside the function body instead
  • Using arguments in an arrow function — it doesn't exist; use rest params
  • Placing rest params before other params — must be last or it's a SyntaxError
  • Mutating default parameter objects — defaults are evaluated once per call, but object/array defaults can cause shared-reference bugs if mutated

Tips

  • Use object destructuring as the last parameter for optional named args: function fn(req, { timeout = 5000, retries = 3 } = {})
  • Prefer rest params over arguments — rest gives you a real array with all modern methods
  • Use spread to pass an array to functions that expect individual args: Math.min(...values)

Summary

  • Default params — fire on undefined only; can reference earlier params
  • Rest ...params — collects remaining args into a real array; must be last
  • Spread fn(...arr) — expands array into individual arguments at call site
  • TS overloads — multiple call signatures with one implementation body
