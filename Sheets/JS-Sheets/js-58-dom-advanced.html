Advanced DOM Patterns

Event delegation attaches one listener to a parent element and uses e.target.closest() to handle clicks on dynamically created children — far more efficient than attaching individual listeners to each item. MutationObserver watches for DOM changes (added nodes, attribute changes, text changes) asynchronously without polling. IntersectionObserver efficiently detects when elements enter or leave the viewport — the modern replacement for scroll event listeners for lazy loading, infinite scroll, and animation triggers. Both observers must be disconnected when no longer needed to prevent memory leaks.

Key Points

  • Event delegation — one listener handles many children
  • Use e.target.closest(selector) to identify clicked child
  • MutationObserver — async DOM change notifications
  • IntersectionObserver — viewport visibility detection
  • Always call .disconnect() to avoid memory leaks
  • TS: narrow EventTarget before accessing properties
  • ResizeObserver — element size change detection

Advanced DOM Examples

// ── Event delegation ──────────────────────────────────────────
// One listener on parent handles all current AND future children
const list = document.querySelector("#todo-list");

list.addEventListener("click", (e) => {
  // Find the closest ancestor matching the selector
  const item   = e.target.closest(".todo-item");
  const delBtn = e.target.closest(".delete-btn");

  if (delBtn) {
    delBtn.closest(".todo-item")?.remove();
    return;
  }
  if (item) {
    item.classList.toggle("done");
  }
});
// Works for items added dynamically later — no re-binding needed

// ── MutationObserver — watch DOM changes ─────────────────────
const observer = new MutationObserver((mutations) => {
  for (const mutation of mutations) {
    if (mutation.type === "childList") {
      mutation.addedNodes.forEach(node => {
        console.log("Node added:", node);
      });
    }
    if (mutation.type === "attributes") {
      console.log(`Attr "${mutation.attributeName}" changed`);
    }
    if (mutation.type === "characterData") {
      console.log("Text changed:", mutation.target.textContent);
    }
  }
});

const target = document.querySelector("#dynamic-content");
observer.observe(target, {
  childList: true,       // watch for added/removed children
  subtree: true,         // watch entire subtree (not just direct children)
  attributes: true,      // watch attribute changes
  characterData: true,   // watch text node changes
  attributeFilter: ["class", "data-state"], // only specific attributes
});

observer.disconnect();   // stop observing when done

// ── IntersectionObserver — viewport visibility ────────────────
const io = new IntersectionObserver((entries) => {
  for (const entry of entries) {
    if (entry.isIntersecting) {
      entry.target.classList.add("visible");
      io.unobserve(entry.target); // stop watching once visible
    }
  }
}, {
  root: null,            // null = viewport
  rootMargin: "0px",     // extend/shrink root bounds
  threshold: 0.1,        // fire when 10% visible
});

document.querySelectorAll(".lazy-image").forEach(img => {
  io.observe(img);
});

// ── ResizeObserver — element size changes ────────────────────
const ro = new ResizeObserver((entries) => {
  for (const entry of entries) {
    const { width, height } = entry.contentRect;
    console.log("New size:", width, height);
  }
});
ro.observe(document.querySelector(".resizable"));
// ro.disconnect() when done

// ── TypeScript: narrowing event targets ──────────────────────
document.addEventListener("click", (e: MouseEvent) => {
  const target = e.target as HTMLElement | null;
  if (!target) return;

  if (target instanceof HTMLButtonElement) {
    target.disabled = true;         // ✔ typed as HTMLButtonElement
  }
  if (target instanceof HTMLInputElement) {
    console.log(target.value);      // ✔ typed as HTMLInputElement
  }
});

Notes

Event delegation — especially valuable for lists, tables, or any container where items are added/removed dynamically; prevents listener leaks
MutationObserver callbacks are batched and fire asynchronously (microtask queue) — you receive all mutations since the last microtask flush at once
IntersectionObserver threshold — 0 fires as soon as 1px is visible; 1 fires only when fully visible; array of values fires at each threshold
io.unobserve(entry.target) inside the callback stops watching a specific element — use for one-shot animations
ResizeObserver replaces window.addEventListener("resize") for element-level size tracking — fires per element, not per window resize
TS: EventTarget has no element properties — cast with as HTMLElement or narrow with instanceof before accessing .value, .classList, etc.

Common Mistakes

  • Not calling .disconnect() on observers — creates memory leaks, especially in single-page apps where components mount/unmount
  • Using a scroll event listener for lazy loading — fires hundreds of times per scroll; IntersectionObserver is far more efficient
  • Attaching individual listeners to each list item instead of delegating — breaks for dynamically added items and wastes memory

Tips

  • Use event delegation as the default approach for any list, table, or repeating UI — it scales to any number of items
  • Use IntersectionObserver for all scroll-based triggers (lazy images, infinite scroll, scroll-triggered animations) — replace scroll event listeners
  • Use MutationObserver to react to third-party DOM changes without polling — cleaner than setInterval(checkDom, 100)

Summary

  • Event delegation — one parent listener + e.target.closest() handles all children
  • MutationObserver — async DOM change callbacks; watch childList, attributes, text
  • IntersectionObserver — viewport visibility; replaces scroll listeners for lazy loading
  • Always .disconnect() observers; TS: narrow EventTarget with instanceof
