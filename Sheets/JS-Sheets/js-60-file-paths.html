Path Examples

import path from "path";
import { fileURLToPath } from "url";

// ── CommonJS: __dirname and __filename ────────────────────────
// Available automatically in CJS modules
console.log(__dirname);          // /home/user/project/src
console.log(__filename);         // /home/user/project/src/index.js

// ── ESM: recreate __dirname from import.meta.url ──────────────
const __filename = fileURLToPath(import.meta.url);
const __dirname  = path.dirname(__filename);

// ── path.join — OS-safe path building ────────────────────────
path.join(__dirname, "data", "users.json");
// /home/user/project/src/data/users.json

path.join("src", "..", "dist", "index.js");
// dist/index.js  (resolves ..)

// NEVER string-concat paths — breaks on Windows
const bad  = __dirname + "/data/" + file; // ✖ Windows uses backslash
const good = path.join(__dirname, "data", file); // ✔ cross-platform

// ── path.resolve — absolute path from CWD ────────────────────
path.resolve("src", "index.ts");
// /cwd/src/index.ts  (prefixes with process.cwd())

path.resolve("/absolute", "file.txt");
// /absolute/file.txt  (absolute path stops resolution)

// ── path.parse / basename / extname ──────────────────────────
const p = "/project/src/utils.service.ts";
path.parse(p);
// { root: "/", dir: "/project/src", base: "utils.service.ts",
//   ext: ".ts", name: "utils.service" }

path.basename(p);              // "utils.service.ts"
path.basename(p, ".ts");       // "utils.service"
path.extname(p);               // ".ts"
path.dirname(p);               // "/project/src"

// ── path.relative ─────────────────────────────────────────────
path.relative("/project/src", "/project/dist/index.js");
// "../dist/index.js"

// ── Common patterns ───────────────────────────────────────────
// Config file next to current file
const configPath = path.join(__dirname, "config.json");

// Project root (2 levels up from src/utils/)
const root = path.resolve(__dirname, "..", "..");

// List only .ts files
const files = await readdir("./src", { withFileTypes: true });
const tsFiles = files
  .filter(f => f.isFile() && path.extname(f.name) === ".ts")
  .map(f => f.name);

Notes

path.join — builds a path from parts using the OS separator; resolves .. and .; does not make it absolute
path.resolve — like join but always returns an absolute path; if no absolute segment is given, prepends process.cwd()
import.meta.url is a file:// URL — always wrap with fileURLToPath() before passing to path methods
__dirname does not exist in ESM — use the fileURLToPath(import.meta.url) + path.dirname() pattern
process.cwd() — where the Node process was started, not where the source file lives; they differ when running from a different directory
Never concatenate paths with string operators — always use path.join() to stay cross-platform

Common Mistakes

  • Using string concatenation for paths — always breaks on Windows where the separator is \
  • Using __dirname in an ESM file — it doesn't exist; use fileURLToPath(import.meta.url)
  • Confusing process.cwd() with __dirname — cwd is where Node was launched; __dirname is where the source file lives
  • Passing import.meta.url directly to readFile — it's a file:// URL, not a plain path; convert first

Tips

  • Create an ESM-compatible __dirname once at the top of each file: const __dirname = path.dirname(fileURLToPath(import.meta.url))
  • Use path.resolve(__dirname, "../../") to reliably find your project root regardless of where Node is invoked from
  • Use path.extname(filename) to filter files by extension — cleaner than a regex or string .endsWith()

Summary

  • path.join() — OS-safe path building · path.resolve() — always returns absolute path
  • CJS: __dirname/__filename built-in · ESM: derive from fileURLToPath(import.meta.url)
  • dirname · basename · extname · parse — decompose paths
  • Never concatenate paths with strings — use path.join()
