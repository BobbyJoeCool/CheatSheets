AbortController & Storage Examples

// ── AbortController with fetch ────────────────────────────────
const controller = new AbortController();

// Cancel after 5 seconds
const timeoutId = setTimeout(() => controller.abort("Timeout"), 5000);

try {
  const res = await fetch("/api/data", { signal: controller.signal });
  clearTimeout(timeoutId);
  return await res.json();
} catch (err) {
  if (err.name === "AbortError") console.warn("Request aborted:", err.message);
  else throw err;
}

// ── AbortSignal.timeout — cleaner timeout ─────────────────────
const res = await fetch("/api/data", {
  signal: AbortSignal.timeout(5000),  // no manual cleanup needed
});

// ── Cancel on component unmount (React pattern) ───────────────
useEffect(() => {
  const controller = new AbortController();
  fetchUser(id, { signal: controller.signal })
    .then(setUser)
    .catch(err => {
      if (err.name !== "AbortError") setError(err);
    });
  return () => controller.abort();   // cleanup cancels in-flight request
}, [id]);

// ── localStorage — persist across sessions ────────────────────
// Write
localStorage.setItem("theme", "dark");
localStorage.setItem("user", JSON.stringify({ id: 1, name: "Alice" }));

// Read (always string | null)
const theme = localStorage.getItem("theme");          // "dark"
const raw   = localStorage.getItem("user");
const user  = raw ? JSON.parse(raw) : null;           // parse back

// Delete
localStorage.removeItem("theme");
localStorage.clear();                                  // delete everything

// Iterate
for (let i = 0; i < localStorage.length; i++) {
  const key = localStorage.key(i);
  console.log(key, localStorage.getItem(key));
}

// ── sessionStorage — tab-only, clears on close ───────────────
sessionStorage.setItem("draft", JSON.stringify(formData));
const draft = JSON.parse(sessionStorage.getItem("draft") ?? "null");

// ── Storage event — sync across tabs ─────────────────────────
window.addEventListener("storage", (e) => {
  if (e.key === "theme") {
    applyTheme(e.newValue);   // other tabs changed the theme
  }
});

// ── TypeScript: typed storage helper ─────────────────────────
function getStorageItem<T>(key: string): T | null {
  const raw = localStorage.getItem(key);
  if (!raw) return null;
  try { return JSON.parse(raw) as T; }
  catch { return null; }
}
const savedUser = getStorageItem<User>("user");

Notes

AbortController.abort(reason) — the reason is available as signal.reason and as the error thrown to awaiting code
AbortSignal.timeout(ms) (ES2022) — creates a signal that automatically fires after ms; no manual clearTimeout needed
localStorage stores strings only — always JSON.stringify before writing and JSON.parse after reading non-string values
localStorage.getItem() returns null (not undefined) for missing keys — always check with ?? null or an existence check
The storage event fires in other tabs/windows of the same origin, not the one that made the change — useful for cross-tab sync
localStorage is synchronous — avoid large reads/writes on the main thread; for large data use IndexedDB (async)

Common Mistakes

  • Not handling AbortError separately — it gets logged as an unexpected error in generic catch blocks
  • Storing objects in localStorage without JSON.stringify — gets stored as "[object Object]"
  • Not wrapping JSON.parse on localStorage reads in try/catch — corrupt data throws and crashes
  • Storing sensitive data (tokens, passwords) in localStorage — accessible to any JS on the page (XSS risk)

Tips

  • Use AbortSignal.timeout(ms) instead of manual setTimeout + clearTimeout — less code, auto-cleaned up
  • Create a typed storage helper that handles JSON serialisation and parse errors — use it everywhere instead of raw localStorage calls
  • Store auth tokens in httpOnly cookies, not localStorage — cookies are inaccessible to JavaScript and safe from XSS

Summary

  • AbortController — cancel fetch requests, timeouts, React unmounts · handle AbortError separately
  • AbortSignal.timeout(ms) — auto-aborting signal; no cleanup needed
  • localStorage — persists · sessionStorage — tab-only · both store strings only
  • Always JSON.stringify/parse · never store secrets · use storage event for cross-tab sync
