JSON in JavaScript

JSON (JavaScript Object Notation) is the universal data interchange format. JSON.stringify() converts a JS value to a JSON string; JSON.parse() converts a JSON string back to a JS value. The replacer argument to stringify filters or transforms output; the reviver argument to parse transforms values during parsing — useful for deserialising Date strings back to Date objects. JSON only supports a subset of JS types — values that cannot be serialised are either omitted, converted to null, or cause an error.

JSON Type Support

  • Supported: string · number · boolean · null · object · array
  • undefined → omitted from objects, null in arrays
  • function → omitted silently
  • Symbol → omitted silently
  • Date → converted to ISO string
  • NaN / Infinity → converted to null
  • Map / Set → {} — use toJSON() or convert first
  • Circular references → throws TypeError

JSON Examples

// ── JSON.stringify ────────────────────────────────────────────
const obj = { name: "Alice", age: 30, active: true };

JSON.stringify(obj);
// '{"name":"Alice","age":30,"active":true}'

JSON.stringify(obj, null, 2);   // pretty-print with 2-space indent
// {
//   "name": "Alice",
//   "age": 30,
//   "active": true
// }

// ── Replacer — filter or transform keys ───────────────────────
// Array replacer: only include listed keys
JSON.stringify(obj, ["name", "age"]);
// '{"name":"Alice","age":30}'

// Function replacer: transform or omit values
JSON.stringify(obj, (key, value) => {
  if (key === "age") return undefined; // omit
  return value;
});
// '{"name":"Alice","active":true}'

// ── JSON.parse ────────────────────────────────────────────────
const json = '{"name":"Bob","score":42}';
const data = JSON.parse(json);
data.name;  // "Bob"

// Always wrap in try/catch — invalid JSON throws SyntaxError
try {
  const parsed = JSON.parse(userInput);
} catch (err) {
  console.error("Invalid JSON:", err.message);
}

// ── Reviver — transform values during parse ───────────────────
const withDate = '{"name":"Alice","createdAt":"2024-01-15T10:00:00.000Z"}';
const revived  = JSON.parse(withDate, (key, value) => {
  if (key === "createdAt") return new Date(value); // string → Date
  return value;
});
revived.createdAt instanceof Date; // true

// ── Deep clone with JSON (simple but lossy) ───────────────────
const clone = JSON.parse(JSON.stringify(obj)); // copies plain objects
// ✖ Loses: undefined, functions, Dates become strings, Map/Set → {}
// ✔ Use structuredClone() for proper deep cloning

// ── Reading/writing JSON files ────────────────────────────────
import { readFile, writeFile } from "fs/promises";

// Read
const raw    = await readFile("./config.json", "utf8");
const config = JSON.parse(raw);

// Write (formatted)
await writeFile("./output.json", JSON.stringify(data, null, 2));

// ── TypeScript: typing parsed JSON ───────────────────────────
interface ApiResponse {
  users: { id: number; name: string }[];
  total: number;
}

const raw2 = await readFile("./data.json", "utf8");
const response = JSON.parse(raw2) as ApiResponse; // type assertion
// Or safer with a runtime validator like zod (Sheet 75)

// Custom toJSON method
class Money {
  constructor(public amount: number, public currency: string) {}
  toJSON() { return `${this.amount} ${this.currency}`; }
}
JSON.stringify(new Money(42, "USD")); // '"42 USD"'

Notes

The third argument to stringify is the indent — a number for spaces, or a string (e.g. "\t") for tab indentation
Reviver function receives every key-value pair bottom-up — use it to convert ISO date strings back to Date objects consistently
JSON.parse(JSON.stringify(obj)) deep clones plain objects but is lossy — use structuredClone() instead for safe deep cloning
toJSON() method — if a value has a toJSON method, JSON.stringify calls it and serialises the return value instead
TS: JSON.parse() returns any — always assert a type or validate with zod/typebox for untrusted data
Circular references throw synchronously — detect them with a replacer or use a library like flatted for circular JSON

Common Mistakes

  • Not wrapping JSON.parse in try/catch — invalid JSON throws synchronously and crashes
  • Using JSON.parse(JSON.stringify(obj)) as a deep clone — loses Dates, Maps, Sets, and undefined values
  • Expecting Map/Set to serialise — they become {}; convert to array/object first
  • Treating JSON.parse return type as typed — it returns any; validate before using

Tips

  • Use JSON.stringify(data, null, 2) for human-readable output in config files and logs
  • Use a reviver to deserialise Date strings: parse once at the boundary, then treat dates as Date objects everywhere else
  • Use zod (Sheet 75) to parse and validate JSON from APIs — you get both runtime safety and TypeScript types in one step

Summary

  • JSON.stringify(val, replacer, indent) — JS → JSON string
  • JSON.parse(str, reviver) — JSON string → JS value; always wrap in try/catch
  • Unsupported: undefined · function · Symbol · Map · Set · circular refs
  • TS: parse returns any — assert a type or validate with zod
