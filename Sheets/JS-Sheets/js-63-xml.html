XML Examples

// ── Browser: parse XML string with DOMParser ──────────────────
const xmlStr = `
<catalog>
  <book id="bk101">
    <title>XML Developer's Guide</title>
    <price>44.95</price>
  </book>
  <book id="bk102">
    <title>Midnight Rain</title>
    <price>5.95</price>
  </book>
</catalog>`;

const parser = new DOMParser();
const doc    = parser.parseFromString(xmlStr, "text/xml");

// Check for parse errors
const parseError = doc.querySelector("parsererror");
if (parseError) throw new Error(parseError.textContent);

// Query like HTML DOM
const books  = doc.querySelectorAll("book");
books.forEach(book => {
  const id    = book.getAttribute("id");
  const title = book.querySelector("title").textContent;
  const price = Number(book.querySelector("price").textContent);
  console.log(id, title, price);
});
// bk101 XML Developer's Guide 44.95
// bk102 Midnight Rain 5.95

// ── Browser: serialise DOM back to XML string ─────────────────
const serializer = new XMLSerializer();
const xmlOutput  = serializer.serializeToString(doc);

// ── Building XML string manually ─────────────────────────────
function buildXML(books) {
  const items = books
    .map(b => `  <book id="${escAttr(b.id)}">
    <title>${escText(b.title)}</title>
    <price>${b.price}</price>
  </book>`)
    .join("\n");
  return `<?xml version="1.0" encoding="UTF-8"?>\n<catalog>\n${items}\n</catalog>`;
}
const escAttr = s => s.replace(/&/g,"&amp;").replace(/"/g,"&quot;");
const escText = s => s.replace(/&/g,"&amp;").replace(/</g,"&lt;");

// ── Node: fast-xml-parser ─────────────────────────────────────
// npm install fast-xml-parser
import { XMLParser, XMLBuilder } from "fast-xml-parser";

const fparser = new XMLParser({ ignoreAttributes: false,
                                 attributeNamePrefix: "@_" });
const jsObj   = fparser.parse(xmlStr);
// { catalog: { book: [{ "@_id":"bk101", title:"...", price:44.95 }, ...] } }

// Build XML from object
const builder = new XMLBuilder({ ignoreAttributes: false,
                                  attributeNamePrefix: "@_" });
const rebuilt = builder.build(jsObj);

// ── Node: xml2js ──────────────────────────────────────────────
// npm install xml2js
import { parseStringPromise } from "xml2js";
const result = await parseStringPromise(xmlStr, { explicitArray: false });
result.catalog.book[0].title;  // "XML Developer's Guide"

Notes

DOMParser.parseFromString — always check for a <parsererror> element; malformed XML does not throw, it returns an error document
CSS selectors work on XML DOM — same API as HTML; use querySelector/querySelectorAll to navigate XML structure
XML requires escaping: & → &amp;, < → &lt;, " in attributes → &quot; — always escape user data
fast-xml-parser — the fastest Node XML parser; maps attributes with a prefix (@_ by default) to distinguish from child elements
xml2js — wraps single child elements in arrays by default; use explicitArray: false to avoid this
For large XML files (logs, exports), use a SAX streaming parser (sax, saxes) — event-based, no full document in memory

Common Mistakes

  • Not checking for <parsererror> after DOMParser.parseFromString — malformed XML silently returns an error document
  • Not escaping user data when building XML — unescaped < or & produces invalid XML
  • Using regex to parse XML — notoriously unreliable; always use a proper parser
  • Assuming xml2js arrays — with default settings, a single child element is an array of one; use explicitArray: false

Tips

  • In the browser, prefer DOMParser + CSS selectors — no library needed and the DOM API is familiar
  • In Node, use fast-xml-parser for simple parse/build tasks — it is fast, has no dependencies, and handles attributes well
  • Always validate XML at the boundary of your system — fail loudly on malformed input rather than silently processing garbage

Summary

  • Browser: DOMParser → parse · XMLSerializer → serialise · CSS selectors work on XML docs
  • Always check for <parsererror> after parsing; always escape user data when building
  • Node: fast-xml-parser (fast, zero-dep) · xml2js (popular, configurable)
  • Large XML → SAX streaming parser (sax/saxes) to avoid loading into memory
