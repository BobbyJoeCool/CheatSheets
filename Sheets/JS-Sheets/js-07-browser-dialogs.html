Browser Dialogs & process

alert · confirm · prompt · process.stdout · process.exit · process.env

Browser Dialog Methods

MethodReturnsNotes
alert("msg")undefinedShows message; OK button only; blocks execution
confirm("msg")booleanOK → true · Cancel → false; blocks execution
prompt("msg")string | nullReturns entered text; null if cancelled
prompt("msg", "default")string | nullSecond arg pre-fills the input field

Node.js process Object

APINotes
process.stdout.write(str)Write to stdout; no automatic newline
process.stderr.write(str)Write to stderr; no automatic newline
process.exit(0)Exit cleanly — code 0 means success
process.exit(1)Exit with error — any non-zero signals failure
process.env.NAMERead environment variable NAME
process.argvArray of CLI arguments: [node, script, ...args]
process.cwd()Current working directory as a string
process.versionNode.js version string e.g. "v20.11.0"
process.platform"linux" | "darwin" | "win32" | ...

Examples

// ── Browser dialogs (browser only — will throw in Node) ──────
alert("Save successful!");             // shows modal; returns undefined

const ok = confirm("Delete this item?");
if (ok) deleteItem();                  // ok is true or false

const name = prompt("Your name?");     // null if cancelled
const city = prompt("City?", "London");// pre-filled with "London"

// ── process.stdout vs console.log ───────────────────────────
process.stdout.write("Loading");       // no newline
process.stdout.write("...");           // same line: "Loading..."
process.stdout.write("\n");            // now newline

console.log("Done");                   // stdout + automatic \n
console.error("Failed");              // stderr + automatic \n

// ── process.exit ─────────────────────────────────────────────
if (!configFile) {
  console.error("Missing config");
  process.exit(1);                     // non-zero = failure for CI/scripts
}
process.exit(0);                       // explicit clean exit (often not needed)

// ── process.env ──────────────────────────────────────────────
const port = process.env.PORT ?? "3000";        // read with fallback
const apiKey = process.env.API_KEY;             // undefined if not set

// TS: process.env values are always string | undefined
const port2: string = process.env.PORT ?? "3000";

// ── process.argv ─────────────────────────────────────────────
// node script.js --name Alice
// process.argv → ["node", "/path/script.js", "--name", "Alice"]
const args = process.argv.slice(2);    // ["--name", "Alice"]

// ── Detecting environment ────────────────────────────────────
if (process.platform === "win32") {
  console.log("Windows detected");
}
console.log("Node:", process.version); // Node: v20.11.0
console.log("CWD:", process.cwd());   // /home/user/project

Notes

alert/confirm/prompt are browser-only — Node has no UI thread
All three dialogs block the main thread — page freezes until dismissed
prompt returns null (not empty string) when the user cancels — always check
process.stdout.write gives fine-grained control (no auto newline); useful for progress bars
process.exit(1) is important in scripts — shells and CI check exit codes
process.env values are always string | undefined — always provide a fallback
process.argv[0] is the node binary; [1] is the script path; real args start at [2]

Common Mistakes

  • Using alert() in production web apps — it blocks the page and is disabled in iframes
  • Not checking for null from prompt() — crashes when user hits Cancel
  • Reading process.env.MY_VAR without a fallback — is undefined if not set
  • Calling process.exit() inside async code before callbacks finish

Tips

  • Replace prompt() in real apps with a proper UI modal or input element
  • Use dotenv to load .env files into process.env in development
  • Use process.stdout.write + \r to overwrite the same line for progress indicators

Summary

  • alert/confirm/prompt — browser-only blocking dialogs; avoid in production
  • process.stdout/stderr.write — raw output control; no automatic newline
  • process.exit(code) — 0 = success, non-zero = error; important for scripts & CI
  • process.env/argv/cwd/platform/version — runtime environment information
