Command-Line Tools in Node

Node is an excellent platform for CLI tools — it has great file system access, the npm ecosystem, and TypeScript support. A CLI entry point needs a shebang line and executable permission. Commander.js is the most popular argument parsing library, handling subcommands, options, help text, and validation. Chalk adds ANSI colour to terminal output. Ora provides elegant spinners for long-running operations. Inquirer (covered in Sheet 20) handles interactive prompts.

CLI Toolkit

  • commander — argument parsing, subcommands, help
  • chalk — terminal colour and styling
  • ora — loading spinners
  • @inquirer/prompts — interactive prompts (Sheet 20)
  • cli-table3 — formatted tables in terminal
  • boxen — boxed messages
  • figures — cross-platform Unicode symbols
  • tsup / esbuild — bundle TS CLI to single JS file

CLI Examples

#!/usr/bin/env node
// ^ shebang — tells the OS to run this file with node

// npm install commander chalk ora

import { Command } from "commander";
import chalk       from "chalk";
import ora         from "ora";
import { readFile, writeFile } from "fs/promises";

const program = new Command();

// ── Program metadata ──────────────────────────────────────────
program
  .name("mytool")
  .description("A sample CLI tool")
  .version("1.0.0");             // adds --version flag automatically

// ── Simple command ────────────────────────────────────────────
program
  .command("greet <name>")       // <> = required arg
  .description("Greet someone")
  .option("-l, --loud",  "shout the greeting")
  .option("-t, --times <n>", "how many times", "1")  // default "1"
  .action((name, options) => {
    const msg = `Hello, ${name}!`;
    const out = options.loud ? msg.toUpperCase() : msg;
    for (let i = 0; i < Number(options.times); i++) {
      console.log(chalk.green(out));
    }
  });

// ── Command with async action ─────────────────────────────────
program
  .command("convert <input> [output]") // [] = optional arg
  .description("Convert a JSON file")
  .option("-p, --pretty", "pretty print output")
  .action(async (input, output = "output.json", options) => {
    const spinner = ora(`Reading ${input}`).start();
    try {
      const raw  = await readFile(input, "utf8");
      const data = JSON.parse(raw);
      spinner.text = "Writing output";
      const str  = options.pretty
        ? JSON.stringify(data, null, 2)
        : JSON.stringify(data);
      await writeFile(output, str);
      spinner.succeed(chalk.green(`Done → ${output}`));
    } catch (err) {
      spinner.fail(chalk.red(`Error: ${err.message}`));
      process.exit(1);
    }
  });

// ── Global options ────────────────────────────────────────────
program
  .option("-v, --verbose", "enable verbose output")
  .hook("preAction", (cmd) => {
    if (cmd.opts().verbose) console.log(chalk.dim("Verbose mode on"));
  });

// ── Subcommands from separate files ──────────────────────────
program.addCommand((await import("./commands/deploy.js")).default);

// ── Parse argv ───────────────────────────────────────────────
program.parse(process.argv);      // must be last

// ── Chalk colour examples ─────────────────────────────────────
console.log(chalk.blue("Info message"));
console.log(chalk.yellow.bold("Warning!"));
console.log(chalk.red("Error: something failed"));
console.log(chalk.green("✔ Success"));
console.log(chalk.dim("# comment-style text"));
console.log(chalk.bgBlue.white(" HEADER "));

// ── Ora spinner ───────────────────────────────────────────────
const spinner2 = ora("Fetching data...").start();
await fetchData();
spinner2.succeed("Data loaded");   // green ✔
// or: spinner.fail("Failed")      // red ✖
// or: spinner.warn("Slow")        // yellow ⚠

// ── package.json: expose as a binary ─────────────────────────
// "bin": { "mytool": "./dist/cli.js" }
// npm link  → makes 'mytool' available globally in dev

Notes

Shebang #!/usr/bin/env node — first line of the entry file; makes the script executable directly without node prefix; also run chmod +x cli.js
Commander <arg> — required; [arg] — optional; ...args — variadic (rest); arguments come before options in the action callback
Option values — -t, --times <n> accepts a value; the third argument is the default; option values are always strings — convert with Number()
Spinner pattern — start before async work, call .succeed() or .fail() after; keeps the terminal responsive while work happens
process.exit(1) in error paths — signals failure to shell scripts and CI; exit 0 is success, non-zero is failure
The "bin" field in package.json — maps command names to entry files; npm link installs globally in dev; npm publish makes it available via npx

Common Mistakes

  • Forgetting the shebang line — the script runs with node file.js but not as ./file.js directly
  • Not calling process.exit(1) on error — the script exits 0 even on failure; CI considers it a success
  • Assuming chalk colours are always enabled — they're disabled when stdout is piped; use chalk.level check or --no-color support
  • Leaving ora spinner running on error — always call spinner.fail() or spinner.stop() in catch blocks

Tips

  • Use tsup to bundle your TypeScript CLI — produces a single dist/cli.js with no runtime dependencies needed for distribution
  • Use npm link during development — makes your CLI available globally without publishing; npm unlink to remove
  • Add a --json flag for machine-readable output — makes your CLI composable with other tools and scriptable in CI

Summary

  • Shebang + "bin" in package.json + chmod +x — makes a Node script into a CLI command
  • Commander — argument parsing, options, subcommands, auto-help
  • Chalk — terminal colour · Ora — spinners · Inquirer — interactive prompts
  • process.exit(1) on error · bundle with tsup · test with npm link
