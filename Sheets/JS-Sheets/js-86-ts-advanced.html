Advanced TypeScript Types

TypeScript's type system is Turing-complete — you can perform sophisticated transformations purely at the type level. Template literal types combine string literal types like template strings. Conditional types (T extends U ? A : B) branch based on type relationships. infer extracts a type from within a conditional. Mapped types transform every property of an existing type. These features power utility types like Partial, ReturnType, and Awaited — and let you write your own.

Key Concepts

  • Template literal types: \`prefix_${T}\`
  • Conditional: T extends U ? A : B
  • infer X — capture a type within conditional
  • Distributive conditionals — applied to each union member
  • Mapped type: { [K in keyof T]: ... }
  • +readonly / -readonly — add/remove modifier
  • const type params — preserve literal types in generics

Advanced Type Examples

// ── Template literal types ────────────────────────────────────
type Greeting = `Hello, ${string}!`;
const g: Greeting = "Hello, World!";    // ✔
const bad: Greeting = "Hi there";       // ✖

type EventName = "click" | "focus" | "blur";
type Handler   = `on${Capitalize<EventName>}`;
// "onClick" | "onFocus" | "onBlur"

// Build typed CSS property names
type CSSProperty = "margin" | "padding" | "border";
type CSSDir = "Top" | "Right" | "Bottom" | "Left";
type CSSProp = `${CSSProperty}${CSSDir}`;
// "marginTop" | "marginRight" | "paddingTop" | ...

// ── Conditional types ─────────────────────────────────────────
type IsString<T> = T extends string ? "yes" : "no";
type A = IsString<string>;  // "yes"
type B = IsString<number>;  // "no"

// Distributive — applies to each union member
type ToArray<T> = T extends unknown ? T[] : never;
type C = ToArray<string | number>;  // string[] | number[]

// Non-distributive (wrap in tuple to prevent distribution)
type ToArrayND<T> = [T] extends [unknown] ? T[] : never;
type D = ToArrayND<string | number>; // (string | number)[]

// ── infer — extract a type from within a conditional ─────────
type UnpackPromise<T> = T extends Promise<infer U> ? U : T;
type E = UnpackPromise<Promise<string>>;  // string
type F = UnpackPromise<number>;           // number

type FirstArg<T> = T extends (first: infer A, ...rest: any[]) => any ? A : never;
type G = FirstArg<(x: number, y: string) => void>; // number

// ── Mapped types ──────────────────────────────────────────────
// Make all properties optional (like Partial)
type MyPartial<T> = { [K in keyof T]?: T[K] };

// Make all properties readonly
type MyReadonly<T> = { +readonly [K in keyof T]: T[K] };

// Remove readonly from all properties
type Mutable<T> = { -readonly [K in keyof T]: T[K] };

// Remap keys
type Prefixed<T> = { [K in keyof T as `get_${string & K}`]: T[K] };
interface User { id: number; name: string }
type PrefixedUser = Prefixed<User>;
// { get_id: number; get_name: string }

// ── const type parameters (TS 5.0) ───────────────────────────
function makeArray<const T>(items: T[]) { return items; }
const arr = makeArray(["a", "b", "c"]);
// arr is ["a","b","c"] not string[] — preserves literal types

// ── Template literal inference ────────────────────────────────
type GetRoute<T extends string> =
  T extends `${string}:${infer Param}/${infer Rest}`
    ? Param | GetRoute<Rest>
    : T extends `${string}:${infer Param}`
      ? Param
      : never;
type Params = GetRoute<"/users/:id/posts/:postId">;
// "id" | "postId"

Notes

Template literal types use the same syntax as template strings but operate on type-level string unions — they multiply out over unions automatically
Conditional types are distributive over naked type parameters — T extends U where T is a union applies the condition to each member separately
infer — declares a type variable that TS infers from the matched position; only valid inside extends clauses of conditional types
Mapped types with as clause — remaps the key names; combined with template literals enables powerful key transformation
-readonly — removes the readonly modifier from mapped properties; used to create a mutable version of a frozen type
const type params (TS 5.0) — equivalent to as const at the call site; preserves literal types without the caller needing to know

Common Mistakes

  • Forgetting that conditional types are distributive — wrap in [T] to prevent distribution when you want the union treated as one type
  • Deeply recursive conditional types — can hit TS's instantiation depth limit; restructure or use tail recursion
  • Over-engineering with advanced types — if a simpler approach works, use it; type-level programming adds maintenance cost

Tips

  • Use template literal types to derive event handler union types from event name unions — keeps them in sync automatically
  • Use infer to extract types from function signatures rather than repeating them — Parameters<T> and ReturnType<T> are built on this pattern
  • Enable strictNullChecks + noUncheckedIndexedAccess — most advanced type patterns assume strict mode

Summary

  • Template literal types — string-level type algebra; distributes over unions
  • Conditional T extends U ? A : B — distributive over unions; wrap in [T] to prevent
  • infer X — capture sub-types inside conditional extends clauses
  • Mapped types — transform every key; as remaps keys; +/-readonly modifies modifiers
