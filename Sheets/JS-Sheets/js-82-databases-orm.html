Prisma & Drizzle Examples

// ═══════════════════════════════════════════════
// PRISMA
// ═══════════════════════════════════════════════
// schema.prisma:
// model User {
//   id    Int    @id @default(autoincrement())
//   name  String
//   email String @unique
//   posts Post[]
// }

// npm install @prisma/client
// npx prisma migrate dev --name init
import { PrismaClient } from "@prisma/client";
const prisma = new PrismaClient();

// CRUD
const user  = await prisma.user.findUnique({ where: { id: 1 } });
const users = await prisma.user.findMany({
  where:   { email: { contains: "@example.com" } },
  orderBy: { name: "asc" },
  take:    10, skip: 0,
});
const created = await prisma.user.create({
  data: { name: "Alice", email: "alice@example.com" },
});
const updated = await prisma.user.update({
  where: { id: 1 },
  data:  { name: "Alice Updated" },
});
await prisma.user.delete({ where: { id: 1 } });

// Relations
const withPosts = await prisma.user.findUnique({
  where:   { id: 1 },
  include: { posts: true },
});

// Transaction
await prisma.$transaction([
  prisma.user.update({ where: { id: 1 }, data: { name: "A" } }),
  prisma.post.create({ data: { title: "Hi", authorId: 1 } }),
]);

// ═══════════════════════════════════════════════
// DRIZZLE
// ═══════════════════════════════════════════════
// schema.ts:
// export const users = pgTable("users", {
//   id:    serial("id").primaryKey(),
//   name:  text("name").notNull(),
//   email: text("email").unique().notNull(),
// });

import { drizzle } from "drizzle-orm/node-postgres";
import { Pool } from "pg";
import { users } from "./schema.js";
import { eq, like } from "drizzle-orm";

const pool2 = new Pool({ connectionString: process.env.DATABASE_URL });
const db2   = drizzle(pool2);

// CRUD
const all    = await db2.select().from(users);
const found  = await db2.select().from(users).where(eq(users.id, 1));
const ins    = await db2.insert(users)
  .values({ name: "Bob", email: "bob@example.com" })
  .returning();
await db2.update(users)
  .set({ name: "Bob Updated" })
  .where(eq(users.id, ins[0].id));
await db2.delete(users).where(eq(users.id, 1));

Notes

Prisma generates a type-safe client from schema.prisma — run prisma generate after every schema change
Drizzle schema is TypeScript — no code generation step needed; types are inferred directly
Prisma findUnique returns null (not throws) when not found; always null-check
Drizzle is closer to raw SQL — eq, like, and, or are composable query builder functions
Both support transactions — Prisma's $transaction handles rollback automatically
Prisma adds a query engine binary (~50MB); Drizzle is a thin wrapper with a much smaller footprint — critical for edge deployments

Common Mistakes

  • Forgetting prisma generate after schema changes — client is stale and queries fail or have wrong types
  • Creating a new PrismaClient per request — creates too many DB connections; instantiate once and export
  • Not running migrations before deploying — production DB schema out of sync with code

Tips

  • Instantiate Prisma as a singleton: export const prisma = global.prisma ??= new PrismaClient() — prevents connection exhaustion in dev with hot reload
  • Use Drizzle for edge runtimes (Cloudflare Workers, Vercel Edge) — Prisma's query engine binary doesn't run in edge environments
  • Use prisma.$transaction(async (tx) => { ... }) (interactive transactions) for complex multi-step operations that need conditional logic

Summary

  • Prisma — DSL schema · generated client · great DX · heavier binary
  • Drizzle — TS schema · thin wrapper · SQL-close · edge-friendly
  • Both provide full type safety for CRUD and relations
  • Singleton clients · always run migrations before deploy
