Function Basics

Declaration · expression · arrow · hoisting · return · TS typed functions

Function Syntax Comparison

StyleSyntaxHoistedOwn thisNotes
Declarationfunction name() {}✔✔Callable before definition; preferred for named top-level functions
Expressionconst fn = function() {}✖✔Assigned to variable; not callable before declaration
Named expr.const fn = function name() {}✖✔Name visible only inside function (useful for recursion)

Function Examples

// ── Function declaration ─────────────────────────────────────
function greet(name) {
  return `Hello, ${name}!`;
}
greet("Alice");               // "Hello, Alice!"

// Declarations are hoisted — callable before they appear
sayHi();                      // ✔ "Hi!" — works due to hoisting
function sayHi() { console.log("Hi!"); }

// ── Function expression ───────────────────────────────────────
const double = function(n) {
  return n * 2;
};
double(5);                    // 10

// Named function expression — name only visible inside
const factorial = function fact(n) {
  return n <= 1 ? 1 : n * fact(n - 1); // fact usable here
};

// ── Arrow functions ───────────────────────────────────────────
const add = (a, b) => a + b;          // implicit return
const square = n => n * n;             // single param: no parens needed
const noop = () => {};                 // no params
const getUser = () => ({ name: "Alice" }); // return object: wrap in ()

const multiply = (a, b) => {
  const result = a * b;
  return result;                       // explicit return with block body
};

// ── Return values ─────────────────────────────────────────────
function noReturn() {}                 // returns undefined
function early(n) {
  if (n < 0) return 0;                // early return
  return n * 2;
}

// ── TypeScript: typed functions ───────────────────────────────
function add2(a: number, b: number): number {
  return a + b;
}

const greetTS = (name: string): string => `Hello, ${name}!`;

// Optional parameter
function log(msg: string, level?: string): void {
  console.log(`[${level ?? "INFO"}] ${msg}`);
}

// Function type annotation
const transform: (input: string) => string = (s) => s.toUpperCase();

// Async function return type
async function fetchData(url: string): Promise<Response> {
  return fetch(url);
}

Notes

Hoisting — function declarations are moved to the top of their scope; expressions and arrows are not
Arrow functions have no own this — they inherit this from the enclosing scope; critical in callbacks and class methods
Arrow implicit return — () => expression returns the expression directly; no return keyword needed
To return an object literal with implicit return, wrap in parentheses: () => ({ key: value }) — otherwise {} is parsed as a block
All functions return undefined by default if no return statement is reached
TS optional param param? — value is T | undefined; must either provide a default or check for undefined

Common Mistakes

  • Returning an object literal from an arrow without parens — () => { key: val } is a block, not an object
  • Using arrow functions as methods expecting their own this — they inherit this from lexical scope instead
  • Forgetting that function expressions are not hoisted — calling before declaration throws a ReferenceError
  • Omitting the return type in TS for complex functions — inferred types can be unexpectedly wide

Tips

  • Use arrow functions for callbacks and short utilities; use declarations for named top-level functions
  • Always annotate return types in TypeScript for exported or public functions — it serves as documentation and catches accidental undefined returns
  • Use early returns to reduce nesting — check preconditions at the top and return immediately on failure

Summary

  • Declarations are hoisted · expressions and arrows are not
  • Arrow functions have no own this — they inherit it lexically
  • Arrow implicit return: (x) => x * 2 · object return needs parens: () => ({ })
  • TS: annotate param types and return type on all public/exported functions
