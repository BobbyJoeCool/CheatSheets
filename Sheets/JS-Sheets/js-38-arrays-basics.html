Array Examples

// ── Creating arrays ───────────────────────────────────────────
const a = [1, 2, 3];
const b = new Array(3);           // [empty × 3]  — avoid
const c = new Array(3).fill(0);   // [0, 0, 0]
const d = Array.from({ length: 5 }, (_, i) => i); // [0,1,2,3,4]
const e = Array.from("hello");    // ["h","e","l","l","o"]
const f = Array.of(1, 2, 3);      // [1, 2, 3]

// ── Accessing elements ────────────────────────────────────────
const arr = [10, 20, 30, 40, 50];
arr[0]            // 10
arr[arr.length-1] // 50
arr.at(-1)        // 50  ← preferred
arr.at(-2)        // 40
arr.length        // 5

// ── Checking ─────────────────────────────────────────────────
Array.isArray(arr)   // true
Array.isArray("hi")  // false

// ── push / pop (end) ─────────────────────────────────────────
arr.push(60);         // arr = [10,20,30,40,50,60], returns 6
arr.pop();            // arr = [10,20,30,40,50],  returns 60

// ── unshift / shift (beginning) ──────────────────────────────
arr.unshift(0);       // arr = [0,10,20,30,40,50], returns 6
arr.shift();          // arr = [10,20,30,40,50],   returns 0

// ── splice — remove/insert at index ──────────────────────────
const x = [1, 2, 3, 4, 5];
x.splice(1, 2);       // removes 2 items at index 1 → x=[1,4,5], returns [2,3]
x.splice(1, 0, 99);   // insert 99 at index 1, remove 0 → x=[1,99,4,5]

// ── Spread — copy and merge ───────────────────────────────────
const copy = [...arr];                    // shallow copy
const merged = [...arr, ...[6, 7, 8]];   // concatenate

// ── slice — non-mutating extract ────────────────────────────
const nums = [1, 2, 3, 4, 5];
nums.slice(1, 3)   // [2, 3]  — end exclusive
nums.slice(-2)     // [4, 5]  — last two
nums.slice()       // [1,2,3,4,5]  — full shallow copy

// ── TypeScript typed arrays ──────────────────────────────────
const scores: number[] = [95, 87, 72];
const names: string[] = ["Alice", "Bob"];
const mixed: (string | number)[] = [1, "two", 3];
const matrix: number[][] = [[1,2],[3,4]];

// Generic syntax (equivalent)
const ids: Array<number> = [1, 2, 3];

// readonly array
const fixed: readonly number[] = [1, 2, 3];
fixed.push(4);   // ✖ Error: readonly array

Notes

new Array(3) creates sparse empty slots — not the same as [undefined, undefined, undefined]; use .fill() or Array.from()
.at(-1) — negative index from end; cleaner than arr[arr.length - 1]
.splice() mutates the original array in place and returns the removed elements
.slice() never mutates — it returns a new array; often used to make a shallow copy
Spread [...arr] is the most idiomatic way to shallow-copy or merge arrays
TS readonly number[] — prevents any mutation methods at compile time; ReadonlyArray<T> is equivalent

Common Mistakes

  • Using new Array(3) expecting [undefined, undefined, undefined] — creates sparse slots that behave differently
  • Mutating an array while iterating it — use a copy or collect changes and apply after
  • Forgetting .slice() is non-mutating and .splice() is mutating — easy to mix up
  • Assuming spread is a deep clone — nested objects are still shared references

Tips

  • Use Array.from({ length: n }, (_, i) => i) to create a filled sequence array in one line
  • Prefer non-mutating operations (slice, spread, map, filter) to keep data flow predictable
  • Use readonly number[] in TypeScript for arrays that shouldn't be modified after creation — caught at compile time

Summary

  • push/pop (end) · unshift/shift (start) · splice (middle) — all mutate in place
  • slice() — non-mutating copy/extract · spread [...arr] — shallow copy/merge
  • Array.from() — create from iterables or length+map · .at(-1) — last element
  • TS: T[] or Array<T> · readonly T[] prevents mutation
