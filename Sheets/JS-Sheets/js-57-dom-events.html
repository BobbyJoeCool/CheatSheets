Event Examples

// ── addEventListener / removeEventListener ────────────────────
const btn = document.querySelector("#btn");

function handleClick(e) {
  console.log("Clicked!", e.target);
}
btn.addEventListener("click", handleClick);
btn.removeEventListener("click", handleClick); // same function reference

// One-time listener
btn.addEventListener("click", handleClick, { once: true });

// ── Event object properties ───────────────────────────────────
btn.addEventListener("click", (e) => {
  e.target;           // element that triggered the event
  e.currentTarget;    // element the listener is attached to
  e.type;             // "click"
  e.timeStamp;        // milliseconds since page load
  e.preventDefault(); // stop default browser behaviour (e.g. form submit)
  e.stopPropagation();// stop event bubbling up the DOM
});

document.querySelector("form").addEventListener("submit", (e) => {
  e.preventDefault(); // prevent page reload
  const data = new FormData(e.currentTarget);
});

// ── Keyboard events ───────────────────────────────────────────
document.addEventListener("keydown", (e) => {
  e.key;              // "Enter", "ArrowUp", "a", " " etc.
  e.code;             // "KeyA", "Space" (physical key, layout-independent)
  e.ctrlKey;          // true if Ctrl held
  e.shiftKey;         // true if Shift held
  e.metaKey;          // true if Cmd (Mac) / Win key held
  if (e.key === "Escape") closeModal();
  if (e.ctrlKey && e.key === "s") { e.preventDefault(); save(); }
});

// ── Event bubbling and capturing ──────────────────────────────
// Events bubble UP the DOM by default (from target to document)
// { capture: true } fires handler on the way DOWN instead
document.querySelector(".parent").addEventListener("click", (e) => {
  console.log("parent (bubble)");
});
document.querySelector(".child").addEventListener("click", (e) => {
  console.log("child");
  e.stopPropagation();  // prevents parent handler from firing
});

// Capture phase (fires before bubble phase):
document.addEventListener("click", handler, { capture: true });

// ── Custom events ─────────────────────────────────────────────
const event = new CustomEvent("userLogin", {
  detail: { userId: 42, name: "Alice" },
  bubbles: true,
  cancelable: true,
});
document.dispatchEvent(event);

document.addEventListener("userLogin", (e) => {
  console.log("User logged in:", e.detail.name);
});

// ── DOMContentLoaded vs load ──────────────────────────────────
document.addEventListener("DOMContentLoaded", () => {
  // DOM parsed — safe to query elements
  // Images/stylesheets may still be loading
});
window.addEventListener("load", () => {
  // Everything (images, styles) fully loaded
});

Notes

e.target — the element the user interacted with · e.currentTarget — the element the listener is attached to (useful in delegated handlers)
To remove a listener you must pass the same function reference — anonymous arrow functions cannot be removed
Bubbling — events travel up: child → parent → document; most events bubble; focus/blur do not (use focusin/focusout)
stopPropagation() stops bubbling; stopImmediatePropagation() also stops other listeners on the same element
CustomEvent.detail — pass any data payload; bubbles/cancelable control propagation and default-prevention behaviour
DOMContentLoaded fires when HTML is parsed; load fires when all resources (images, CSS, iframes) have loaded

Common Mistakes

  • Using an anonymous function and then trying to call removeEventListener — can't remove anonymous functions; store the reference
  • Calling e.preventDefault() on events that don't have a default — harmless but indicates misunderstanding
  • Using keypress — deprecated; use keydown instead
  • Forgetting e.stopPropagation() in nested listeners — parent handlers fire unexpectedly

Tips

  • Use { once: true } for one-shot listeners — cleaner than manually removing inside the handler
  • Use pointerdown/pointermove/pointerup instead of separate mouse and touch events — one set of handlers covers all input devices
  • Use CustomEvent to decouple components — components dispatch events, other parts listen; no direct coupling required

Summary

  • addEventListener(type, fn, opts) — attach · removeEventListener — requires same reference
  • e.target where it happened · e.currentTarget where listener lives · e.preventDefault() stop default
  • Events bubble up by default · stopPropagation() halts the bubble
  • CustomEvent({ detail }) — dispatch typed custom events with data payloads
