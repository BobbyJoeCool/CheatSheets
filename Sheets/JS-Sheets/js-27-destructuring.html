Sheet 27

Destructuring

Array destructuring · object destructuring · defaults · renaming · rest · nested

Destructuring Assignment

Destructuring is a concise syntax for unpacking values from arrays or properties from objects into distinct variables. Array destructuring uses position — the first variable gets the first element, the second gets the second, and so on. Object destructuring uses property names — variables are matched by key. Both support default values (used when the value is undefined), renaming (binding to a different variable name), rest (collecting remaining items), and nesting (destructuring inside destructuring). Destructuring is used everywhere: function parameters, for...of loops, imports, and swap patterns.

Key Points

  • Array: position-based — order matters
  • Object: name-based — order doesn't matter
  • = default — used when value is undefined
  • { key: alias } — rename while destructuring
  • ...rest — collect remaining; must be last
  • Works in function parameters directly
  • Nested destructuring mirrors the source structure

Destructuring Examples

// ── Array destructuring ───────────────────────────────────────
const [a, b, c] = [1, 2, 3];
console.log(a, b, c);         // 1 2 3

// Skip elements with commas
const [first, , third] = [10, 20, 30];
console.log(first, third);    // 10 30

// Default values (used if value is undefined)
const [x = 0, y = 0] = [5];
console.log(x, y);            // 5 0

// Rest — collect remaining elements
const [head, ...tail] = [1, 2, 3, 4];
console.log(head, tail);      // 1  [2, 3, 4]

// Swap variables without a temp variable
let p = 1, q = 2;
[p, q] = [q, p];
console.log(p, q);            // 2 1

// ── Object destructuring ──────────────────────────────────────
const user = { name: "Alice", age: 30, city: "NYC" };
const { name, age } = user;
console.log(name, age);       // Alice 30

// Rename while destructuring
const { name: username, age: years } = user;
console.log(username, years); // Alice 30

// Default values
const { role = "guest", name: uname } = user;
console.log(role, uname);     // guest Alice

// Rest — collect remaining properties
const { name: n, ...rest } = user;
console.log(n, rest);         // Alice  { age: 30, city: "NYC" }

// ── Nested destructuring ──────────────────────────────────────
const data = { user: { name: "Bob", address: { city: "LA" } } };
const { user: { name: bname, address: { city } } } = data;
console.log(bname, city);     // Bob  LA

// ── In function parameters ────────────────────────────────────
function greet({ name, age = 0 }) {
  return `${name} is ${age}`;
}
greet({ name: "Alice", age: 30 }); // "Alice is 30"
greet({ name: "Bob" });            // "Bob is 0"

// Array params
function sum([a, b, c = 0]) {
  return a + b + c;
}
sum([1, 2, 3]);               // 6
sum([1, 2]);                  // 3

// ── for...of with destructuring ───────────────────────────────
const entries = [["a", 1], ["b", 2], ["c", 3]];
for (const [key, val] of entries) {
  console.log(key, val);      // a 1  b 2  c 3
}

// ── TypeScript: typed destructuring ──────────────────────────
const { name: tname, age: tage }: { name: string; age: number } = user;
function display({ id, label = "n/a" }: { id: number; label?: string }) {
  console.log(id, label);
}

Notes

Defaults only apply when the value is strictly undefined — null does not trigger the default
Renaming syntax { key: alias } — the left side (key) is the source property; the right (alias) is the new variable name
Rest in object destructuring collects remaining own enumerable properties into a new object
Nested destructuring can get hard to read — consider a two-step approach for deeply nested structures
Swap pattern [a, b] = [b, a] — the right side creates a temporary array; very idiomatic JS
Destructuring function parameters is one of the most common uses — it self-documents which fields are used

Common Mistakes

  • Expecting defaults to fire on null — they only fire on undefined
  • Confusing rename syntax — { old: newName } not { newName: old }
  • Deeply nesting destructuring — becomes unreadable; break into steps
  • Forgetting that array destructuring is position-based — missing a comma to skip an element pulls the wrong value

Tips

  • Destructure function parameters directly — function fn({ id, name }) is clearer than function fn(opts)
  • Use the swap pattern [a, b] = [b, a] — no temp variable needed
  • Combine with Object.entries() and for...of for clean object iteration: for (const [k, v] of Object.entries(obj))

Summary

  • Array: position-based · Object: name-based · both support defaults, rest, and nesting
  • = default — fires on undefined only · { key: alias } — rename on extraction
  • ...rest — collects remaining items/properties; must be last
  • Works in function params, for...of, imports, and variable declarations
