Zod Examples

// npm install zod
import { z } from "zod";

// ── Define a schema ───────────────────────────────────────────
const UserSchema = z.object({
  id:       z.number().int().positive(),
  name:     z.string().min(1).max(100),
  email:    z.string().email(),
  age:      z.number().min(0).max(150).optional(),
  role:     z.enum(["admin", "user", "guest"]).default("user"),
  tags:     z.array(z.string()).default([]),
  address:  z.object({
    city:    z.string(),
    country: z.string().length(2), // ISO country code
  }).optional(),
});

// ── Infer TypeScript type from schema ─────────────────────────
type User = z.infer<typeof UserSchema>;
// User = { id: number; name: string; email: string; age?: number;
//          role: "admin"|"user"|"guest"; tags: string[]; address?: ... }

// ── parse — throws ZodError on failure ───────────────────────
const user: User = UserSchema.parse({
  id: 1, name: "Alice", email: "alice@example.com"
});
// age defaults to undefined, role defaults to "user", tags to []

// ── safeParse — returns { success, data } or { success, error } ──
const result = UserSchema.safeParse(untrustedInput);
if (!result.success) {
  console.error(result.error.flatten().fieldErrors);
  // { name: ["String must contain at least 1 character(s)"], ... }
} else {
  const user = result.data;  // fully typed
}

// ── Transform — validate then convert ─────────────────────────
const DateSchema = z.string()
  .datetime()
  .transform(s => new Date(s));    // string → Date after validation

const ISODate = DateSchema.parse("2024-01-15T10:00:00Z");
// ISODate is a Date object

// ── Coercion for env vars / query params ─────────────────────
const QuerySchema = z.object({
  page:  z.coerce.number().int().min(1).default(1),
  limit: z.coerce.number().int().min(1).max(100).default(20),
});
QuerySchema.parse({ page: "2", limit: "10" });
// { page: 2, limit: 10 } — strings coerced to numbers

// ── Parse API response ────────────────────────────────────────
const res = await fetch("/api/users");
const raw = await res.json();
const users = z.array(UserSchema).parse(raw); // typed User[]

// ── Reuse schemas ─────────────────────────────────────────────
const CreateUserSchema = UserSchema.omit({ id: true });
const UpdateUserSchema = UserSchema.partial().required({ id: true });
type CreateUserDto = z.infer<typeof CreateUserSchema>;

Notes

z.infer<typeof Schema> — derives the TypeScript type from the schema; single source of truth for both runtime validation and compile-time types
parse throws a ZodError with detailed field errors; safeParse returns a result object — use safeParse in route handlers to send proper HTTP error responses
.flatten().fieldErrors — converts the ZodError into a plain object mapping field names to arrays of error message strings; easy to return as API error bodies
z.coerce — tries to convert the input before validating; essential for query params and env vars which are always strings
.transform(fn) — runs after validation; the output type changes (string → Date); combine with .pipe() for multi-step transformations
Derived schemas (.omit/.partial/.pick) — compose from a base schema rather than redefining; keeps create/update/read DTOs in sync

Common Mistakes

  • Using parse in a request handler — it throws; use safeParse so you can return a 400 response instead of crashing
  • Defining a TypeScript type separately from the zod schema — they drift out of sync; always derive the type with z.infer
  • Not using z.coerce for numeric query params — z.number().parse("42") fails; use z.coerce.number()
  • Forgetting .optional() vs .nullable() — optional allows undefined; nullable allows null; they are different

Tips

  • Use zod for every external data boundary: API responses, form inputs, env vars, database query results
  • Derive create/update schemas from the base entity schema with .omit/.partial — one schema to update, all DTOs stay consistent
  • Use result.error.flatten().fieldErrors to format validation errors for API responses — gives field-by-field error messages

Summary

  • z.object/string/number/array/enum — build schemas with chainable validators
  • z.infer<typeof Schema> — derive TS type; single source of truth
  • parse (throws) · safeParse (returns result object) — use safeParse in request handlers
  • z.coerce for string→number/boolean · .transform() to reshape validated data
