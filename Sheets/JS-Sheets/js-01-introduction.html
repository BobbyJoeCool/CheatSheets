What is JavaScript?

JavaScript (JS) was created in 1995 by Brendan Eich at Netscape — designed in just 10 days as a scripting language for web browsers. It is now the only language that runs natively in every browser, and via Node.js it runs on servers, desktops, mobile devices, and embedded systems. JS follows the ECMAScript standard, with major yearly releases since ES6/2015 adding classes, modules, async/await, and more. It is dynamically typed, interpreted/JIT-compiled, and multi-paradigm — supporting procedural, object-oriented, and functional styles.

Key Facts

  • Created 1995 — Brendan Eich, Netscape
  • Standardized as ECMAScript (ECMA-262)
  • Yearly releases since ES2015 (ES6)
  • Dynamically typed · interpreted / JIT-compiled
  • Single-threaded with an event loop
  • Multi-paradigm: OOP · functional · procedural
  • Most-used language on GitHub since 2014

What is TypeScript?

TypeScript (TS) is a superset of JavaScript developed by Microsoft, released in 2012. Every valid JS file is valid TS. TypeScript adds a static type system — you annotate variables, parameters, and return values with types, and the TypeScript compiler (tsc) checks them before the code ever runs. TS compiles down to plain JavaScript, so it runs anywhere JS runs. Type annotations are erased at compile time and have zero runtime cost. TypeScript is a development-time tool, not a separate runtime.

JS vs TS

  • JS — dynamic types, errors caught at runtime
  • TS — static types, errors caught at compile time
  • TS adds interfaces · generics · enums · decorators
  • TS requires a build step (tsc or bundler)
  • All JS libraries work in TS (with @types/)
  • TS is opt-in strict — you control how tight it is
  • Deno and Bun run TS natively — no build step needed

Common Mistakes

  • Assuming a JS version — always check what your runtime supports
  • Thinking TypeScript prevents all bugs — types are erased at runtime
  • Mixing CommonJS (require) and ESM (import) without proper config

Tips

  • Start new projects with TypeScript from day one — retrofitting types is painful
  • Use strict: true in tsconfig.json for maximum safety
  • Check MDN or node.green for feature compatibility

Summary

  • JS is the universal language of the web — runs everywhere
  • TypeScript = JavaScript + static types; compiles to JS
  • ECMAScript yearly releases keep adding modern features
  • Vast ecosystem: frameworks, bundlers, linters, test runners
