What Is JSON?

JSON (JavaScript Object Notation) is a lightweight, text-based data-interchange format defined by RFC 8259 and ECMA-404. Despite its name, JSON is completely language-independent and is the dominant format for REST APIs, config files, and data storage. A JSON document is always encoded in UTF-8 (UTF-16 and UTF-32 are also permitted by the spec but rarely used in practice). Every valid JSON document is either a single value — an object, array, string, number, boolean, or null.

Key Rules

  • Keys must be double-quoted strings — single quotes are invalid
  • Values must be one of the six types: object, array, string, number, boolean, or null
  • Trailing commas are not allowed after the last item in an object or array
  • Comments are not part of the spec — strip them before parsing
  • Whitespace (spaces, tabs, newlines) outside strings is insignificant
  • The root value may be any JSON type, not just an object

Annotated JSON Document

{
  "name": "Alice",
  "age": 30,
  "active": true,
  "score": 9.5,
  "nickname": null,
  "tags": ["admin", "editor"],
  "address": {
    "city": "Austin",
    "zip": "78701"
  }
}

Line Notes

"name": "Alice" — string value; key and value both double-quoted
"age": 30 — integer number; no quotes around the value
"active": true — boolean literal; must be lowercase
"score": 9.5 — floating-point number
"nickname": null — null literal; must be lowercase
"tags": [...] — array of strings as a value
"address": {...} — nested object as a value

Common Mistakes

  • Using single quotes for keys or strings — only double quotes are valid
  • Leaving a trailing comma after the last object property or array element
  • Writing True, False, or None (Python style) instead of lowercase JSON literals
  • Embedding comments with // or /* */ — parsers will reject them

Tips

  • Use a linter or validator (jsonlint.com, VS Code) to catch syntax errors before runtime
  • Prefer 2-space indentation for readability; minify for production transport
  • If you need comments in config files, consider JSON5 or JSONC (see Sheet 08)

Summary

  • JSON is defined by RFC 8259 — language-independent, UTF-8 text
  • Six value types: object · array · string · number · boolean · null
  • Keys must be double-quoted; no trailing commas; no comments
  • Booleans and null are lowercase literals: true, false, null
