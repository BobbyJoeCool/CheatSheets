Sheet 05

Parsing & Serialization

parse · stringify · pretty-print · error handling · language comparison

Parse vs Serialize

Parsing (deserializing) converts a JSON string into a native data structure — a dictionary, map, object, or hash depending on the language. Serialization (stringifying / marshalling) is the reverse: converting native data into a JSON string. Every major language ships a JSON library in its standard library. The table below shows the canonical function names; all do the same thing under different names.

Universal Concepts

  • Parse — JSON string → native object/dict/map
  • Stringify — native value → JSON string
  • Pretty-print — serialize with indentation for readability
  • Always wrap parse calls in error handling — malformed JSON throws
  • Circular references in native objects cannot be serialized to JSON
  • undefined (JS), functions, and binary data are stripped on serialize

Parse & Serialize — Four Languages

// ── JavaScript ────────────────────────────────────────────────
const obj  = JSON.parse('{"name":"Alice","age":30}');
const str  = JSON.stringify(obj);              // compact
const pretty = JSON.stringify(obj, null, 2);   // indented

// ── Python ────────────────────────────────────────────────────
# import json
obj   = json.loads('{"name":"Alice","age":30}')
str   = json.dumps(obj)                        # compact
pretty = json.dumps(obj, indent=2)             # indented

// ── PHP ───────────────────────────────────────────────────────
$obj  = json_decode('{"name":"Alice","age":30}', true);
$str  = json_encode($obj);                     // compact
$pretty = json_encode($obj, JSON_PRETTY_PRINT); // indented

// ── Ruby ──────────────────────────────────────────────────────
# require 'json'
obj   = JSON.parse('{"name":"Alice","age":30}')
str   = obj.to_json                            # compact
pretty = JSON.pretty_generate(obj)             # indented

Notes

JS JSON.parse → throws SyntaxError on bad input; wrap in try/catch
JS JSON.stringify(obj, null, 2) — second arg is a replacer, third is indent spaces
Python json.loads → raises json.JSONDecodeError on bad input
Python json.dumps(obj, indent=2) also accepts sort_keys=True for stable output
PHP json_decode(..., true) — second arg true returns assoc array, not stdClass
Ruby JSON.parse → raises JSON::ParserError; rescue it explicitly

Common Mistakes

  • Not catching parse errors — malformed input from an API will crash unguarded code
  • PHP: forgetting the true second arg to json_decode and getting a stdClass instead of array
  • Serializing objects with circular references — all languages will error or loop infinitely
  • Using pretty-print in production API responses unnecessarily — adds bytes and latency

Tips

  • Always validate + parse in one step — never manipulate raw JSON strings with regex
  • Use sort_keys=True (Python) or replacer sort in JS for deterministic, diffable output
  • For streaming large files, prefer streaming parsers (Python ijson, Node.js stream-json)

Summary

  • Parse = JSON string → native; Stringify = native → JSON string
  • Every major language has a built-in JSON library — no third-party package needed for basics
  • Always handle parse errors — invalid JSON throws or returns null depending on language
  • Pretty-print with indent option for debugging; compact for production transport
