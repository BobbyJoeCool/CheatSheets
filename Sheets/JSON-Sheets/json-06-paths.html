Sheet 06

JSONPath & Querying

JSONPath syntax · dot notation · wildcards · filters · recursive descent · jq

JSONPath

JSONPath (RFC 9535, 2024) is a query language for extracting values from JSON documents, analogous to XPath for XML. An expression always starts with $ (the root node) and chains selectors with either dot notation ($.store.book) or bracket notation ($["store"]["book"]). Both forms are equivalent. JSONPath is supported natively in PostgreSQL, MySQL, AWS Step Functions, many API gateways, and libraries for every major language.

Syntax Quick Reference

  • $ — root of the document
  • .key — child property by name
  • [n] — array element at index n (zero-based)
  • [*] or .* — all children (wildcard)
  • .. — recursive descent (all descendants)
  • [?(@.price < 10)] — filter expression
  • [1:3] — array slice (elements 1 and 2)

jq — Command-Line JSON Processor

# ── Basic access ─────────────────────────────────────────────
echo '{"name":"Alice","age":30}' | jq '.name'       # "Alice"
echo '{"name":"Alice","age":30}' | jq '.age'        # 30

# ── Arrays ───────────────────────────────────────────────────
echo '[1,2,3]' | jq '.[0]'                          # 1
echo '[1,2,3]' | jq '.[]'                           # 1  2  3  (each on own line)
echo '[1,2,3]' | jq '.[1:3]'                        # [2,3]

# ── Filters & pipes ──────────────────────────────────────────
cat data.json | jq '.users[] | .name'               # each user's name
cat data.json | jq '.users[] | select(.age > 25)'   # filter by condition
cat data.json | jq '[.users[] | .name]'             # collect into array

# ── Transform ────────────────────────────────────────────────
cat data.json | jq '{id: .id, label: .name}'        # reshape object
cat data.json | jq '.users | length'                # count items
cat data.json | jq '.users | sort_by(.age)'         # sort array
cat data.json | jq '.users | map(.age) | add'       # sum all ages

# ── Output options ───────────────────────────────────────────
jq -r '.name'       # raw output (no quotes around strings)
jq -c '.'           # compact (minified) output
jq '.' file.json    # pretty-print a file

Notes

. — identity filter; outputs the input unchanged
.key — accesses object property; .a.b chains
.[] — iterates every element of an array or object
| — pipe: output of left becomes input of right
select(cond) — keeps input only if condition is true
[expr] — collect filter output into an array
-r flag — strips surrounding quotes from string output

JSONPath Expression Reference

ExpressionSelects
$Root node
$.storeChild store of root
$.store.book[0]First book
$.store.book[*]All books
$..authorAll author values anywhere
$.store.book[-1]Last book (RFC 9535)
$.store.book[0,1]First and second book
$.store.book[?(@.price<10)]Books cheaper than 10
$.store.book[?(@.isbn)]Books that have an isbn key
$..priceAll price values in document

Common Mistakes

  • Forgetting $ at the start of a JSONPath expression — it's mandatory
  • jq: omitting -r when piping string output to shell — the extra quotes break commands
  • Mixing JSONPath implementations — filter syntax varies between tools (Goessner vs RFC 9535)
  • Using .. (recursive descent) on large documents — it can be slow on deep structures

Tips

  • Test JSONPath expressions at jsonpath.com before embedding in code
  • Use jq -c to produce compact JSON for piping into other tools or curl
  • In jq, wrap multi-step transforms in a function with def name: expr; to keep scripts readable

Summary

  • JSONPath: $ is root · . is child · [*] is wildcard · .. is recursive
  • Filter expressions [?(@.field op value)] select matching array elements
  • jq is the standard CLI tool — pipe-based, filter-oriented, and very fast
  • jq's map, select, sort_by, and group_by cover most data-shaping tasks
