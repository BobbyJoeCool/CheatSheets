Character Classes

A character class (written inside […]) matches exactly one character from the set you define. Ranges like [a-z] expand to all characters between those Unicode code points. Negated classes ([^…]) match any character not in the set. Shorthand classes like \d, \w, and \s are aliases for common sets and work everywhere inside or outside brackets.

Key Rules

  • Inside […] most metacharacters lose their special meaning — [.*+] matches a literal dot, asterisk, or plus.
  • - is a range operator only when between two characters; place it first or last to match a literal hyphen: [-az].
  • ^ is a negation operator only when it is the first character inside the brackets.
  • \d matches [0-9]; with Unicode flag it can match non-ASCII digits.
  • The . metacharacter outside a class matches any char except \n (unless s flag is set).

Character Class Examples

// ── Basic classes ─────────────────────────────────────────────
/[aeiou]/.test("hello")         // true  — any vowel
/[^aeiou]/i.test("aeiou")       // false — no non-vowels
/[a-zA-Z]/.test("Hello")        // true  — any letter
/[0-9a-f]/i.test("1F")          // true  — hex char

// ── Ranges ────────────────────────────────────────────────────
"abc123".match(/[a-z]+/)        // ["abc"]
"abc123".match(/[0-9]+/)        // ["123"]
"abc123".match(/[a-z0-9]+/i)    // ["abc123"]

// ── Shorthands ────────────────────────────────────────────────
"$49.99".match(/\d+/g)          // ["49", "99"]
"foo bar".match(/\S+/g)         // ["foo", "bar"]
"  \t\n".match(/^\s+$/)         // matches — all whitespace

// ── Negation ──────────────────────────────────────────────────
"hello world".replace(/[^a-z ]/g, "") // "hello world" (strip non-alpha)
"2024-01-15".replace(/[^\d]/g, "")    // "20240115"

// ── Combining shorthands inside [ ] ───────────────────────────
/[\w\-]/.test("my-var")         // true  — word char or hyphen
/[^\s,;]+/g                     // split on whitespace/commas/semicolons

Notes

[aeiou] — explicit set; matches exactly one of those characters
[^aeiou] — negated set; matches any char not in the set
[a-z0-9] — two ranges combined; letters or digits
\d+ — one or more digits; equivalent to [0-9]+
\S+ — one or more non-whitespace chars; useful for tokenizing
[^\d] and \D are identical — both forms are correct
[\w\-] — combining a shorthand with a literal hyphen inside a class

Common Mistakes

  • Writing [a-Z] — the range goes from a (97) to Z (90) backwards, causing an error in most engines.
  • Putting - in the middle of a class without escaping: [a-z0-9-] is fine only when - is last.
  • Expecting [abc]+ to match the string "abc" as a whole — it matches each individual character, not the sequence.
  • Using . inside [.] — the dot loses its special meaning inside brackets and matches only a literal dot.

Tips

  • Prefer \d, \w, \s shorthands over spelled-out ranges — they're shorter and intention-revealing.
  • Use [^\n]+ to match a full line without consuming the newline.
  • For Unicode-aware matching in JS, add the u flag and use \p{L} instead of [a-zA-Z].

Summary

  • […] matches one character from the defined set.
  • [^…] matches any character not in the set.
  • Shorthands: \d digits · \w word chars · \s whitespace (and their uppercase negations).
  • Ranges use Unicode code points — keep them in ascending order.
