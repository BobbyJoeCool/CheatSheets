Sheet 03

Anchors & Boundaries

^ · $ · \b · \B · \A · \Z · multiline · word boundary

What Are Anchors?

Anchors are zero-width assertions — they match a position in the string rather than a character. ^ and $ assert the start and end of the string (or line when multiline mode is on). Word boundaries (\b) match the transition between a word character and a non-word character, making it possible to match whole words without consuming surrounding characters. Because anchors consume no input, they never appear in the matched text.

Key Rules

  • ^ and $ match the start/end of the entire string by default.
  • With the m (multiline) flag, ^/$ match after/before each \n.
  • \b matches between \w and \W (or the start/end of the string).
  • \B is the inverse — it matches where \b does not.
  • Python adds \A (absolute start) and \Z (absolute end) that ignore the multiline flag.
  • Anchors make validation patterns reliable: ^\d{5}$ ensures the whole string is 5 digits.

Anchor Usage Examples

// ── String-level anchors ──────────────────────────────────────
/^\d{5}$/.test("90210")         // true  — exactly 5 digits
/^\d{5}$/.test("902100")        // false — 6 digits
/^Hello/.test("Hello World")    // true  — starts with Hello
/World$/.test("Hello World")    // true  — ends with World

// ── Multiline mode ────────────────────────────────────────────
const text = "line1\nline2\nline3";
text.match(/^\w+/gm)            // ["line1", "line2", "line3"]
text.match(/^\w+/)              // ["line1"] — no m flag

// ── Word boundaries ───────────────────────────────────────────
"the cat sat".match(/\bcat\b/)  // ["cat"]
"concatenate".match(/\bcat\b/)  // null — no word boundary
"price: $100".replace(/\b\d+\b/g, "##")  // "price: $##"

// ── Python: \A and \Z ─────────────────────────────────────────
import re
re.match(r'\A\d+\Z', '12345')   # Match
re.match(r'\A\d+\Z', '123\n')   # None — \Z requires true end
re.match(r'^\d+$', '123\n', re.M)  # Match — $ allows \n before end

Notes

^\d{5}$ — anchoring both ends turns a search into a full-string validation
gm flags together — global finds all matches, multiline lets ^ anchor to each line
\bcat\b — word boundaries prevent matching "cat" inside "catfish" or "concatenate"
\b\d+\b — matches standalone numbers; won't match digits inside identifiers like "abc123"
\A / \Z — Python-preferred for validation; immune to multiline flag
Python $ allows an optional trailing \n — use \Z for strict end-of-string

Common Mistakes

  • Using ^/$ for per-line matching without the m flag — they match the entire string by default.
  • Expecting \b to work with non-ASCII letters — \b is defined by \w, which is ASCII-only in most engines.
  • Using Python $ for strict validation — it permits a trailing newline; use \Z instead.

Tips

  • Always anchor validation patterns with ^…$ (or \A…\Z in Python) to prevent partial matches.
  • Use \b when replacing words to avoid corrupting longer words containing the target.
  • Combine gm flags in JS when processing multi-line text line by line.

Summary

  • Anchors match positions, not characters — they consume zero input.
  • ^ / $ — start/end of string; per-line with m flag.
  • \A / \Z — absolute string boundaries regardless of flags (Python/Java).
  • \b / \B — word boundary and its inverse for whole-word matching.
