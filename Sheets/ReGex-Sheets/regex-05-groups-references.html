Groups & Backreferences

Capturing groups ((…)) serve two purposes: they group tokens so quantifiers apply to the whole unit, and they capture the matched text for later use in backreferences or replacements. Named groups ((?<name>…)) improve readability by labelling captures. Non-capturing groups ((?:…)) provide grouping without the overhead of capturing. Backreferences (\1, \k<name>) let you assert that a later part of the string matches something already captured.

Key Rules

  • Groups are numbered left-to-right by opening parenthesis: ((a)(b)) → groups 1, 2, 3.
  • Use (?:…) when you need grouping but not the capture overhead.
  • Named groups: (?<year>\d{4}) — access via match.groups.year in JS or m.group('year') in Python.
  • Backreference \1 matches the same text that group 1 captured — not the pattern again.
  • Alternation | has the lowest precedence — use a group to contain it: (?:cat|dog)s.

Groups in Action

// ── Capturing groups ──────────────────────────────────────────
const m = "2024-01-15".match(/(\d{4})-(\d{2})-(\d{2})/);
m[0]  // "2024-01-15"  (full match)
m[1]  // "2024"        (group 1)
m[2]  // "01"          (group 2)
m[3]  // "15"          (group 3)

// ── Named groups (JS) ─────────────────────────────────────────
const n = "2024-01-15".match(/(?\d{4})-(?\d{2})-(?\d{2})/);
n.groups.year   // "2024"
n.groups.month  // "01"

// ── Non-capturing group ───────────────────────────────────────
"cats and dogs".match(/(?:cat|dog)s?/g)  // ["cats", "dogs"]

// ── Backreferences in pattern ─────────────────────────────────
/<(\w+)>.*?<\/\1>/.test("bold")   // true  — tag must match
/<(\w+)>.*?<\/\1>/.test("bold")   // false — tags differ

// ── Backreferences in replacement ────────────────────────────
"John Smith".replace(/(\w+) (\w+)/, "$2, $1")  // "Smith, John"
"2024-01-15".replace(
  /(?\d{4})-(?\d{2})-(?\d{2})/,
  "$/$/$"
)  // "15/01/2024"

Notes

m[1], m[2] — array indices map to capture group numbers (0 = full match)
(?<year>…) — named capture; access via .groups.year property on the match
(?:…) — groups the alternation without wasting a capture slot
\1 in pattern — backreference; ensures closing tag matches opening tag exactly
$1, $2 in replacement — inserts captured group by number
$<name> in JS replacement — inserts named capture group by name

Common Mistakes

  • Counting groups incorrectly — nested groups add their own number; count by opening parentheses left-to-right.
  • Using | without a group: cat|dogs means "cat" OR "dogs", not "cat" OR "dog" + s.
  • Expecting \1 to re-run the group's pattern — it matches the same captured text, not a new match of the pattern.

Tips

  • Default to (?:…) unless you actually need the capture — it's faster and keeps group numbering clean.
  • Named groups make complex patterns self-documenting and immune to renumbering when you add groups.
  • Put longer alternation branches first: January|Jan ensures the full word is tried before the abbreviation.

Summary

  • (…) — capturing group; numbered by left paren position.
  • (?:…) — non-capturing group; for grouping only.
  • (?<name>…) — named capture; access by name in match result.
  • \1 / $1 — backreference in pattern / replacement.
