What is a Regular Expression?

A regular expression (regex) is a sequence of characters that defines a search pattern. Engines scan a string left-to-right, attempting to match the pattern at each position. Regex is built into virtually every modern language and is used for searching, validating, extracting, and replacing text. The same core syntax works across Python, JavaScript, Java, Go, and PHP — only the API wrapper differs.

Key Facts

  • Regex engines are greedy by default — they match as much as possible.
  • Matching is case-sensitive by default; use the i flag to ignore case.
  • A pattern with no anchors can match anywhere in the string.
  • /pattern/flags is JS literal syntax; Python uses r"pattern" raw strings.
  • Always test patterns on real data — edge cases are common.

Basic Usage — JavaScript & Python

// ── JavaScript ────────────────────────────────────────────────
const str = "Hello World 2024";

/hello/i.test(str);              // true  (case-insensitive)
str.match(/\d+/);                // ["2024"]
str.match(/\w+/g);               // ["Hello", "World", "2024"]
str.replace(/World/, "Regex");   // "Hello Regex 2024"
str.search(/\d/);                // 12  (index of first digit)
"a,b,,c".split(/,+/);            // ["a", "b", "c"]

// ── Python ────────────────────────────────────────────────────
import re
s = "Hello World 2024"

re.search(r'\d+', s).group()     # '2024'
re.findall(r'\w+', s)            # ['Hello', 'World', '2024']
re.sub(r'World', 'Regex', s)     # 'Hello Regex 2024'
re.split(r',+', 'a,b,,c')        # ['a', 'b', 'c']
bool(re.fullmatch(r'\d{4}','2024')) # True

Notes

/pattern/flags — JS regex literal; no quotes needed
r"..." — Python raw string; backslashes are not processed by Python before regex sees them
test() — returns boolean; faster than match() when you only need yes/no
match() vs search() — Python match() anchors to start; search() scans entire string
findall() — returns all non-overlapping matches as a list
fullmatch() — entire string must satisfy the pattern

Common Mistakes

  • Forgetting to escape . — it matches any character, not just a literal dot.
  • Using Python re.match() when re.search() is needed — match() only checks the start of the string.
  • Omitting the g flag in JS replace() — only the first occurrence is replaced without it.
  • Writing "\d" (non-raw) in Python — the backslash is consumed by Python before the regex engine sees it.

Tips

  • Use regex101.com or regexr.com to build and test patterns interactively with real-time explanations.
  • Always use raw strings r"..." in Python — it prevents double-escaping headaches.
  • Compile frequently-used patterns with re.compile() (Python) or store regex literals in const (JS) for performance.

Summary

  • Regex defines a pattern matched against a string left-to-right.
  • Metacharacters (. ^ $ * + ? | \ ( ) [ ] { }) have special meaning — escape them with \ to match literally.
  • Flags (i, g, m, s) modify how the engine interprets the pattern.
  • API differences are shallow — the same pattern works across languages.
