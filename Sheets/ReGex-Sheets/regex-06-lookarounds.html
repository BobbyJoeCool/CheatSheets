Lookarounds

lookahead · lookbehind · negative · zero-width · assertions

Zero-Width Lookaround Assertions

Lookarounds are zero-width assertions that check what comes before or after the current position without consuming any characters. A lookahead ((?=…)) succeeds if the pattern can match ahead; a negative lookahead ((?!…)) succeeds if it cannot. Lookbehinds work the same way but check what precedes the current position. Because they are zero-width, what they check is never included in the match result.

Key Rules

  • Lookarounds match a position — the checked text is not consumed or captured.
  • Lookbehind patterns must be fixed-width in most engines (no *, +, or ?). Python 3.6+ and .NET allow variable-width lookbehind.
  • JavaScript ES2018+ supports lookbehind ((?<=…) and (?<!…)).
  • Multiple lookaheads can be chained at the same position: (?=.*\d)(?=.*[A-Z]).
  • Lookarounds can be combined with capturing groups — but the lookaround itself does not capture.

Lookaround Types

Lookaround Examples

// ── Positive lookahead ────────────────────────────────────────
"100px 200em".match(/\d+(?=px)/g)     // ["100"]  — numbers before "px"
"file.txt".match(/\w+(?=\.txt)/)      // ["file"] — name without extension

// ── Negative lookahead ────────────────────────────────────────
"foo123 bar456".match(/[a-z]+(?!\d)/g) // ["ba"]  — letters NOT before digit
"color colour".match(/colou?r(?!s)/g)  // ["color", "colour"]

// ── Positive lookbehind ───────────────────────────────────────
"$100 €200 £50".match(/(?<=\$)\d+/g)  // ["100"]  — number after $
"width:200px".match(/(?<=:)\d+/)       // ["200"]  — value after colon

Notes

\d+(?=px) — matches digits only when "px" follows; "px" is not in the result
(?<=\$)\d+ — matches digits only when preceded by "$"; "$" is not in the result
Chained (?=.*[a-z])(?=.*[A-Z]) — both must be true at the same position; order doesn't matter
(?=(\d{3})+$) — lookahead used as a comma/thousands-separator insertion point
Lookbehind in JS requires ES2018+ (Chrome 62+, Node 8+, Firefox 78+)
Variable-length lookbehind not supported in JS — use fixed-width patterns inside (?<=…)

Common Mistakes

  • Trying to use + or * inside a lookbehind in JS or most PCRE engines — fixed-width only.
  • Expecting the lookaround text to appear in match results — it is never consumed or captured.
  • Confusing (?=abc) (lookahead) with (?<=abc) (lookbehind) — the < arrow points backward.

Tips

  • Use lookbehind instead of a capturing group when you only need to skip a prefix (e.g. strip the $ but keep the number).
  • Chain multiple lookaheads at position 0 to enforce all-must-pass rules (e.g. password strength).
  • Remember: the < in (?<=…) / (?<!…) is a mnemonic — it looks left (behind).

Summary

  • (?=…) lookahead · (?!…) negative lookahead — check what follows.
  • (?<=…) lookbehind · (?<!…) negative lookbehind — check what precedes.
  • All lookarounds are zero-width — checked text is never part of the match.
  • Chain lookaheads at the same position for multi-condition validation.
