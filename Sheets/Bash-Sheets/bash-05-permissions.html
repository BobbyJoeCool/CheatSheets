Sheet 05

Permissions & Ownership

chmod · chown · rwx · octal · umask · sudo · ACLs · executable

Unix Permissions Model

Every file has an owner (user), a group, and permission bits for three audiences: owner (u), group (g), and others (o). Each audience gets three bits: read (4), write (2), execute (1). Run ls -l to see them: -rwxr-xr-- means owner=rwx, group=r-x, others=r--. macOS also supports extended ACLs (Access Control Lists) visible as + on ls -l and inspected with ls -le.

Reading ls -l Output

  • -rwxr-xr-- — file type + 9 permission bits
  • Position 1: -=file, d=dir, l=symlink
  • Bits 2–4: owner (u), bits 5–7: group (g), bits 8–10: others (o)
  • r=4, w=2, x=1 — add for octal value
  • 755 = rwxr-xr-x (typical dir), 644 = rw-r--r-- (typical file)
  • @ after perms = extended attributes; + = ACL

chmod, chown & sudo

# ── chmod — change permissions ───────────────────────────────
chmod 755 script.sh          # rwxr-xr-x  (octal)
chmod 644 file.txt           # rw-r--r--  (octal)
chmod 600 secret.key         # rw-------  (owner only)
chmod 777 file               # rwxrwxrwx  (avoid — grants all)
chmod +x script.sh           # add execute for all
chmod -x script.sh           # remove execute for all
chmod u+x script.sh          # add execute for owner only
chmod g-w file.txt           # remove write from group
chmod o= file.txt            # clear all permissions for others
chmod -R 755 mydir/          # recursive chmod on directory

# ── chown — change owner / group ─────────────────────────────
chown alice file.txt         # change owner to alice
chown alice:staff file.txt   # change owner and group
chown :staff file.txt        # change group only
chown -R alice:staff mydir/  # recursive ownership change
whoami                       # print current username
id                           # uid, gid, and all groups

# ── sudo & su ────────────────────────────────────────────────
sudo command                 # run command as root
sudo -i                      # start interactive root shell
sudo -u alice command        # run as different user
su - alice                   # switch to alice (requires password)

# ── umask — default permissions for new files ────────────────
umask                        # show current umask (e.g. 022)
umask 022                    # dirs=755, files=644 (common default)
umask 077                    # dirs=700, files=600 (private)

# ── macOS ACLs ───────────────────────────────────────────────
ls -le file.txt              # show ACL entries
chmod +a "alice allow read,write" file.txt   # add ACL entry
chmod -a "alice allow read,write" file.txt   # remove ACL entry

Notes

chmod +x — the most common use; makes a script executable without touching other bits
chmod -R — recursive; apply carefully — setting wrong perms on / breaks the system
600 — for private keys and secrets; readable only by owner
chown -R — often needed after copying files or running commands as root
sudo !! — re-run the last command with sudo (very common workflow)
umask — the value is subtracted from 666 (files) or 777 (dirs) to set defaults
ls -le — macOS-specific; shows extended ACLs below each file entry

Common Mistakes

  • Setting 777 to "fix" permission errors — always find the actual minimal permission needed instead
  • Running scripts as sudo when the issue is just a missing chmod +x
  • Forgetting that directories need x permission to be entered — r alone only lets you list, not cd
  • chmod -R 777 / or similar recursive mistakes on system directories — catastrophic

Tips

  • Use sudo !! immediately after a "permission denied" error to retry with root
  • Always verify new scripts with ls -l script.sh before running — confirm the x bit is set
  • For SSH: ~/.ssh must be 700, key files 600, authorized_keys 644 — SSH rejects wrong perms
  • Use stat -f "%Sp %Su %Sg" file on macOS for a scriptable permission/owner summary

Summary

  • Three audiences: user · group · others; three bits each: r=4 w=2 x=1
  • chmod 755 (dirs) · 644 (files) · 600 (secrets) — the three defaults
  • chmod +x — makes scripts executable; required before running a script
  • sudo !! — the fastest way to retry with elevated privileges
