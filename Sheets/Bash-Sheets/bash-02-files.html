Sheet 02

Files & Directories

touch · cp · mv · rm · mkdir · find · stat · paths · symlinks

File System Basics

The macOS file system is hierarchical, rooted at /. Every file and directory has an absolute path (starts with /, e.g. /Users/alice/Documents) or a relative path (relative to your current directory, e.g. ../config). Special shorthand: ~ is your home directory, . is the current directory, and .. is the parent. macOS is case-insensitive by default but case-preserving — File.txt and file.txt are the same file on most macOS volumes.

Path Quick Reference

  • / — root of the entire file system
  • ~ — expands to /Users/username
  • . — current directory
  • .. — parent directory
  • - — previous directory (with cd)
  • Spaces in paths: use quotes "My Dir" or escape My\ Dir

File & Directory Commands

# ── Create ───────────────────────────────────────────────────
touch file.txt               # create empty file (or update timestamp)
touch file1.txt file2.txt    # create multiple files
mkdir mydir                  # create directory
mkdir -p a/b/c               # create nested dirs (no error if exists)

# ── Copy ─────────────────────────────────────────────────────
cp file.txt backup.txt       # copy file
cp -r src/ dest/             # copy directory recursively
cp -i file.txt dest/         # prompt before overwrite
cp -p file.txt dest/         # preserve permissions and timestamps

# ── Move / Rename ────────────────────────────────────────────
mv file.txt newname.txt      # rename file
mv file.txt ~/Documents/     # move to directory
mv -i file.txt dest/         # prompt before overwrite

# ── Delete ───────────────────────────────────────────────────
rm file.txt                  # delete file (no Trash — permanent!)
rm -i file.txt               # prompt before delete
rm -r mydir/                 # delete directory recursively
rm -rf mydir/                # force-delete, no prompts (be careful!)

# ── Inspect ──────────────────────────────────────────────────
stat file.txt                # full file metadata
file file.txt                # detect file type by content
du -sh mydir/                # disk usage, human-readable
du -sh *                     # size of every item in current dir

# ── Symlinks ─────────────────────────────────────────────────
ln -s /path/to/original link_name   # create symbolic link
ls -la                               # symlinks show as  link -> target
readlink link_name                   # show link target

Notes

touch — creates file if missing; also updates access/modify time on existing files
mkdir -p — the -p flag is essential; without it, parent dirs must already exist
cp -r — required for directories; plain cp fails on dirs
rm — bypasses the Trash entirely; there is no undo. Use -i to be safe
rm -rf — dangerous but common; double-check the path before running
stat — shows inode, permissions, size, timestamps in one view
ln -s — symlinks point to a path; they break if the target moves

Common Mistakes

  • rm -rf / or rm -rf ~ — catastrophic; always double-check paths before running
  • Using cp without -r on a directory silently fails or errors
  • Forgetting mkdir -p and getting "No such file or directory" for missing parent dirs
  • Trailing slash matters: cp src dest/ vs cp src dest behave differently

Tips

  • Alias rm to trash (brew install trash) to send files to macOS Trash instead of permanent deletion
  • Use find . -name "*.log" -delete instead of rm for bulk deletes — safer path matching
  • cp -rp preserves ownership/timestamps — essential when duplicating config dirs
  • Use mkdir -p dist/{css,js,img} with brace expansion to build entire folder trees at once

Summary

  • touch · mkdir -p — create files and nested dirs
  • cp -r · mv — copy and move; always -r for dirs
  • rm is permanent — use -i flag or trash CLI for safety
  • find — powerful recursive search with filters for name, type, size, date
  • stat · du · df — inspect file metadata and disk usage
