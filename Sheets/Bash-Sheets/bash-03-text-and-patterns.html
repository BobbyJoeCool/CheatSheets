Sheet 03

Viewing, Wildcards & Patterns

cat · less · head · tail · grep · wildcards · globbing · regex

Viewing File Contents

# ── Display entire file ───────────────────────────────────────
cat file.txt                 # print file to terminal
cat -n file.txt              # with line numbers
cat file1.txt file2.txt      # concatenate multiple files

# ── Paged viewing (less) ──────────────────────────────────────
less file.txt                # open pager (q=quit, /=search, n=next)
less +G file.txt             # open at end of file
less +F file.txt             # follow mode (like tail -f)

# ── Head and tail ─────────────────────────────────────────────
head file.txt                # first 10 lines (default)
head -n 20 file.txt          # first 20 lines
tail file.txt                # last 10 lines
tail -n 20 file.txt          # last 20 lines
tail -f logfile.log          # follow file (live updates — great for logs)
tail -f -n 50 app.log        # follow last 50 lines

# ── Word/line counts ─────────────────────────────────────────
wc file.txt                  # lines  words  bytes
wc -l file.txt               # line count only
wc -w file.txt               # word count only
wc -c file.txt               # byte count only

# ── grep — search content ────────────────────────────────────
grep "error" file.txt        # lines containing "error"
grep -i "error" file.txt     # case-insensitive
grep -n "error" file.txt     # show line numbers
grep -r "TODO" ./src         # search recursively
grep -l "error" *.log        # filenames only (not lines)
grep -v "debug" file.txt     # invert — lines NOT matching
grep -c "error" file.txt     # count matching lines
grep -E "err|warn" file.txt  # extended regex (alternation)
grep -A 2 -B 2 "error" file  # 2 lines after and before match

Notes

cat — best for short files; for long files use less
less — navigate with arrow keys, Space=page down, b=page up, q=quit
tail -f — indispensable for monitoring log files in real time
grep -r — searches all files recursively; combine with -l for file list
grep -E — enables extended regex; same as using egrep
grep -A/-B/-C — show context lines around matches

Common Mistakes

  • Quoting glob patterns accidentally: grep "*.txt" searches for literal *.txt; the shell expands globs, not grep
  • ** globbing requires shopt -s globstar in bash or setopt globstar in zsh — not on by default in bash
  • Forgetting -E for extended regex — +, |, and () need it in basic grep

Tips

  • Use grep --color=auto (often aliased by default) to highlight matches visually
  • tail -f combined with grep --line-buffered filters a live log stream
  • The brace trick file.txt{,.bak} expands to two args — perfect for quick backups
  • Use less +F instead of tail -f — it lets you toggle follow mode with Ctrl+C then scroll, then F to resume

Summary

  • cat · less · head · tail — four levels of file viewing
  • grep — search file content; -r, -i, -n, -E are the key flags
  • * ? [ ] — shell globs; {a,b} {1..n} — brace expansion
  • Regex in grep: basic by default; use -E for extended, -P for Perl
