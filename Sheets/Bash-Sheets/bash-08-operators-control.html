Operators & Control Flow

# ── Arithmetic ───────────────────────────────────────────────
echo $((5 + 3))              # 8
echo $((10 % 3))             # 1  (modulo)
echo $((2 ** 8))             # 256
(( count++ ))                # increment
(( total = a + b ))          # assign result
x=$(( width * height ))      # store result

# ── Conditionals: if / elif / else ───────────────────────────
if [[ $name == "Alice" ]]; then
  echo "Hello Alice"
elif [[ $age -gt 18 ]]; then
  echo "Adult"
else
  echo "Other"
fi

# ── One-liners (short-circuit) ───────────────────────────────
[[ -f file.txt ]] && echo "exists"     # run if true
[[ -z $var ]] && echo "empty"         # var is empty
mkdir -p dir || echo "failed"          # run if false

# ── case statement ───────────────────────────────────────────
case $fruit in
  apple)  echo "Apple" ;;
  banana|mango) echo "Tropical" ;;
  *)      echo "Other" ;;
esac

# ── for loops ────────────────────────────────────────────────
for i in 1 2 3 4 5; do echo $i; done
for i in {1..10}; do echo $i; done       # sequence
for i in {0..20..2}; do echo $i; done    # step by 2
for f in *.txt; do echo "$f"; done       # glob
for i in $(seq 1 5); do echo $i; done    # seq command
for (( i=0; i<5; i++ )); do echo $i; done  # C-style

# ── while / until ────────────────────────────────────────────
count=0
while [[ $count -lt 5 ]]; do
  echo $count
  (( count++ ))
done

until [[ $count -ge 5 ]]; do   # until = while NOT
  echo $count; (( count++ ))
done

# ── break / continue ─────────────────────────────────────────
for i in {1..10}; do
  [[ $i -eq 5 ]] && break       # exit loop
  [[ $i -eq 3 ]] && continue    # skip iteration
  echo $i
done

Notes

(( )) — arithmetic context; no $ needed on variable names inside
[[ ]] — preferred over [ ]; supports &&, ||, regex, and no word splitting
&& / || — short-circuit: right side runs only if left side succeeds/fails
case — cleaner than long if/elif chains; | separates alternatives per pattern
for f in *.txt — glob expands before loop runs; always quote "$f" in the body
until — runs while condition is false; inverse of while
break 2 — exits two levels of nested loops

Common Mistakes

  • Using [ ] instead of [[ ]] — single brackets don't support &&, ||, or regex; use double always in bash
  • Mixing numeric and string operators: [[ "10" > "9" ]] is false (string sort) — use -gt for numbers
  • Missing ; before then/do on the same line: if [[ ... ]] then fails; needs ]] ; then
  • Not quoting loop variables: for f in *.txt is fine but use "$f" inside the loop body

Tips

  • Use && / || for simple one-liners instead of full if blocks — [[ -d dist ]] || mkdir dist
  • Regex with =~ in [[ ]]: don't quote the pattern — [[ $str =~ ^[0-9]+$ ]]
  • Use case for 3+ branches on the same variable — far more readable than elif chains
  • for f in *.txt safely handles no matches with nullglob option (shopt -s nullglob)

Summary

  • [[ ]] for conditions · (( )) for arithmetic · never mix them
  • Numeric: -eq -lt -gt · String: == != -z -n · File: -f -d -e -x
  • if/elif/else/fi · case/esac · for/while/until loops
  • break exits loop · continue skips to next iteration
