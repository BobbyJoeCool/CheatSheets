Writing Shell Scripts

A shell script is a plain text file containing shell commands. The first line — the shebang (#!/usr/bin/env bash) — tells the OS which interpreter to use. Make it executable with chmod +x script.sh, then run it with ./script.sh. Variables are untyped by default; everything is a string unless used in an arithmetic context. Variable names are case-sensitive; by convention, local variables use lowercase and environment variables uppercase.

Variable Rules

  • No spaces around =: name="alice" ✓ — name = "alice" ✗
  • Reference with $name or ${name} — braces required before text
  • Unset variable expands to empty string (no error by default)
  • readonly VAR=value — makes variable immutable
  • unset VAR — delete a variable
  • Single quotes: literal, no expansion. Double quotes: expands $ and `

Script Syntax

#!/usr/bin/env bash
# ── Script header ─────────────────────────────────────────────
# Good practice: set -euo pipefail (see Sheet 12)

# ── Variables ────────────────────────────────────────────────
name="Alice"                 # string variable
count=42                     # integer (still stored as string)
greeting="Hello, $name!"     # interpolation in double quotes
literal='Hello, $name!'      # no interpolation in single quotes
echo $greeting               # Hello, Alice!
echo $literal                # Hello, $name!

# ── Braces & default values ──────────────────────────────────
echo "${name}s"              # Alices — brace required before s
echo "${unset_var:-default}" # use default if var is unset or empty
echo "${count:=10}"          # assign 10 if count is unset
echo "${name:+exists}"       # "exists" only if name is set
echo "${#name}"              # 5  (string length)

# ── echo & printf ────────────────────────────────────────────
echo "Hello World"           # print with newline
echo -n "no newline"         # suppress trailing newline
echo -e "tab:\there"         # interpret escape sequences
printf "%-10s %d\n" "item" 5 # formatted output (like C printf)
printf "%05d\n" 42           # 00042 — zero-padded

# ── read — get user input ────────────────────────────────────
read name                    # read from stdin into $name
read -p "Enter name: " name  # with prompt
read -s -p "Password: " pass # silent (no echo)
read -t 5 name               # timeout after 5 seconds
read -r line                 # raw mode — don't interpret backslashes
read -a arr                  # read words into array

# ── Script arguments ─────────────────────────────────────────
# ./myscript.sh Alice 42
echo $0                      # script name: ./myscript.sh
echo $1                      # first arg: Alice
echo $2                      # second arg: 42
echo $@                      # all args: Alice 42
echo $*                      # all args as single string
echo $#                      # number of args: 2
echo $?                      # exit code of last command
shift                        # shift args left ($2 becomes $1)

Notes

#!/usr/bin/env bash — preferred over #!/bin/bash; finds bash in PATH, more portable
${var:-default} — the safe expansion family; avoids "unbound variable" errors
${#name} — string length operator; works for any variable
printf — more reliable than echo for formatted or multi-line output in scripts
read -r — always use -r in scripts; prevents backslash interpretation surprises
$@ vs $* — prefer "$@" (quoted); preserves args with spaces as separate items
$? — check immediately after the command; the next command will overwrite it

Parameter Expansion

SyntaxMeaning
${var:-default}Use default if unset or empty
${var:=default}Assign default if unset
${var:+value}Use value only if var is set
${var:?message}Error with message if unset
${#var}Length of string
${var:2:5}Substring: offset 2, length 5
${var#prefix}Remove shortest prefix match
${var##prefix}Remove longest prefix match
${var%suffix}Remove shortest suffix match
${var%%suffix}Remove longest suffix match
${var/old/new}Replace first occurrence
${var//old/new}Replace all occurrences
${var^^}Uppercase all (bash 4+)
${var,,}Lowercase all (bash 4+)

Common Mistakes

  • Spaces around =: x = 5 fails — shell interprets it as a command named x
  • Using $* instead of "$@" — arguments with spaces get split incorrectly
  • Forgetting to quote variables: rm $file breaks on filenames with spaces; use rm "$file"
  • Checking $? too late — each command overwrites it; capture immediately after the command

Tips

  • Always quote variable references in scripts: "$var" — prevents word-splitting and glob expansion
  • Use ${var:?Error message} at the top of scripts to fail fast on missing required variables
  • printf is more portable than echo for scripts that need to run on multiple systems
  • Use local var=value inside functions to avoid polluting the global namespace

Summary

  • Shebang #!/usr/bin/env bash + chmod +x = runnable script
  • No spaces around =; always quote "$variables" in scripts
  • $1…$@ — script arguments; $# = count; $? = last exit code
  • ${var:-default} family — safe expansions for robust scripts
