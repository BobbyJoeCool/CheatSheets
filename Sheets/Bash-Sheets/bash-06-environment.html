Environment & Configuration

$PATH · export · alias · .zshrc · source · env · printenv · which

The Shell Environment

The shell maintains a set of environment variables — key-value pairs inherited by every child process. The most important is $PATH, a colon-separated list of directories the shell searches when you type a command name. Configuration files (~/.zshrc for Zsh, ~/.bash_profile for Bash) are sourced at startup, making aliases, variables, and functions available in every session. Use source (or .) to reload them without restarting the terminal.

Config File Load Order

  • Zsh login shell: /etc/zprofile → ~/.zprofile → ~/.zshrc
  • Bash login shell: /etc/profile → ~/.bash_profile
  • Bash interactive: ~/.bashrc (non-login)
  • macOS Terminal opens a login shell by default
  • Changes take effect next session or after source ~/.zshrc
  • ~/.zshenv — sourced for every zsh, including scripts

Environment Commands

# ── View environment ─────────────────────────────────────────
env                          # all environment variables
printenv                     # same as env
printenv PATH                # value of one variable
echo $HOME                   # /Users/alice
echo $SHELL                  # /bin/zsh
echo $USER                   # alice

# ── Set & export variables ───────────────────────────────────
MY_VAR="hello"               # set shell variable (not exported)
export MY_VAR="hello"        # export to child processes
export PATH="$PATH:/usr/local/bin"  # append to PATH
export EDITOR="vim"          # set default editor

# ── $PATH management ─────────────────────────────────────────
echo $PATH                   # show current PATH (colon-separated)
which python3                # /usr/bin/python3 — where it resolves
which -a python3             # all matches in PATH order
type python3                 # python3 is /usr/bin/python3

# ── Aliases ──────────────────────────────────────────────────
alias ll="ls -la"            # define alias
alias gs="git status"
alias reload="source ~/.zshrc"
alias ..="cd .."
alias ...="cd ../.."
unalias ll                   # remove alias
alias                        # list all aliases

# ── source — reload config files ─────────────────────────────
source ~/.zshrc              # reload zsh config
source ~/.bash_profile       # reload bash config
. ~/.zshrc                   # dot is shorthand for source

# ── Common useful variables ──────────────────────────────────
echo $HOME                   # /Users/alice
echo $PWD                    # current directory (same as pwd)
echo $OLDPWD                 # previous directory
echo $?                      # exit code of last command
echo $$                      # current shell PID
echo $RANDOM                 # random integer 0–32767

Notes

export — without it, a variable is only visible in the current shell, not scripts or subshells
$PATH — order matters; earlier dirs win; always append with "$PATH:/new" not replace
which -a — reveals if a command is shadowed by another earlier in PATH
alias — only lives for the session; add to ~/.zshrc to persist
source vs running a script — source runs in current shell; ./script.sh runs in a subshell
$? — check after any command; 0=success, non-zero=error

Common Mistakes

  • Setting PATH="/new/path" instead of PATH="/new/path:$PATH" — replaces PATH entirely, breaking all commands
  • Forgetting export — variable set without it won't be visible to scripts or child processes
  • Editing ~/.zshrc but forgetting to source it — changes won't apply until you do
  • Defining aliases with spaces around =: alias ll = "ls" fails; must be alias ll="ls"

Tips

  • Keep a ~/bin/ directory for personal scripts and add it first in $PATH — it'll always take precedence
  • Use which -a command to debug "wrong version" issues — reveals every match in PATH order
  • Add alias reload="source ~/.zshrc" to your config — you'll use it constantly
  • Use command -v tool in scripts instead of which — it's POSIX portable and returns an exit code

Summary

  • $PATH — the most important env var; always append, never replace
  • export VAR=value — makes variables available to child processes
  • alias — shortcuts; add to ~/.zshrc to persist across sessions
  • source ~/.zshrc — reload config without restarting terminal
