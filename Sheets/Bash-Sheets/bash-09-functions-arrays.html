Functions & Arrays

# ── Define and call functions ────────────────────────────────
greet() {
  echo "Hello, $1!"          # $1 = first arg to function
}
greet Alice                  # Hello, Alice!

# Both syntaxes work (prefer no 'function' keyword for portability)
function say_hi { echo "Hi!"; }

# ── local variables ──────────────────────────────────────────
counter() {
  local count=0              # local: not visible outside function
  (( count++ ))
  echo $count
}

# ── return values ────────────────────────────────────────────
is_even() {
  (( $1 % 2 == 0 ))         # return 0 (true) if even, 1 (false) if not
}
is_even 4 && echo "even"    # even

# Capture output as return value:
get_date() { date +%Y-%m-%d; }
today=$(get_date)            # 2025-01-15

# ── Indexed arrays ───────────────────────────────────────────
fruits=("apple" "banana" "cherry")
echo ${fruits[0]}            # apple
echo ${fruits[1]}            # banana
echo ${fruits[-1]}           # cherry  (last element, bash 4.2+)
echo ${fruits[@]}            # all elements
echo ${#fruits[@]}           # 3  (length)
fruits+=(mango)              # append element
fruits[1]="blueberry"        # replace element
unset fruits[2]              # delete element (leaves gap)

# ── Loop over array ──────────────────────────────────────────
for fruit in "${fruits[@]}"; do
  echo "$fruit"
done

# ── Array slices ─────────────────────────────────────────────
echo "${fruits[@]:1:2}"      # elements 1 and 2 (offset:length)

# ── Associative arrays (bash 4+) ─────────────────────────────
declare -A person
person[name]="Alice"
person[age]=30
person[city]="Chicago"
echo ${person[name]}         # Alice
echo ${!person[@]}           # all keys: name age city
echo ${person[@]}            # all values

for key in "${!person[@]}"; do
  echo "$key = ${person[$key]}"
done

# ── IFS — field separator ────────────────────────────────────
IFS="," read -ra parts <<< "a,b,c"
echo ${parts[0]}             # a
echo ${parts[@]}             # a b c

# Split a string into array:
str="one:two:three"
IFS=":" read -ra arr <<< "$str"

Notes

local — essential in functions; without it, variables leak to global scope
return — returns an exit code (0–255) only, not a value; use echo + $() for values
"${arr[@]}" — always quote with @; each element stays as a separate word
${#arr[@]} — array length; ${#arr[0]} is length of first element string
declare -A — required for associative arrays; must be declared before use
${!arr[@]} — the ! gives you keys/indices, not values
IFS — save/restore if changing it: old_IFS=$IFS; IFS=","; ...; IFS=$old_IFS

Function Patterns

PatternDescription
fn() { ...; }Define function (POSIX style)
local var=valScoped variable
return 0 / 1Success / failure exit code
result=$(fn)Capture function output
fn "$@"Pass all script args to function
declare -f fnPrint function definition
unset -f fnRemove function
source lib.shImport functions from file

Common Mistakes

  • Using ${arr[*]} instead of "${arr[@]}" — the * form collapses elements with spaces into one word
  • Forgetting declare -A before using associative arrays — silently creates a broken indexed array
  • Trying to return "a string" from a function — return only accepts integers 0–255; use echo instead
  • Modifying IFS globally without restoring it — breaks word-splitting in the rest of the script

Tips

  • Put reusable functions in a separate lib.sh file and source it — keeps scripts clean and testable
  • Use mapfile -t arr < file.txt to read a file into an array (one line per element) — cleaner than IFS tricks
  • Check if a function exists: declare -f fn_name > /dev/null && echo "exists"
  • Always use local for every variable inside functions — prevents subtle bugs from global state pollution

Summary

  • fn() { ...; } — define; call like any command; uses $1…$@ for args
  • local — always use inside functions to scope variables
  • arr=(a b c) · ${arr[@]} · ${#arr[@]} — indexed array essentials
  • declare -A map — associative arrays; ${!map[@]} for keys
