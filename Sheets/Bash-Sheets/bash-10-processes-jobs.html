Processes & Job Control

ps · kill · & · jobs · fg · bg · trap · nohup · timeout · signals

Processes & Jobs

Every running program is a process with a unique PID (Process ID). The shell can run processes in the foreground (blocking the terminal) or background (appending &). The shell's own list of background processes are called jobs, managed with jobs, fg, and bg. Processes communicate via signals — numbered messages like SIGTERM (graceful stop) and SIGKILL (force kill). Scripts can intercept signals using trap.

Key PID Variables

  • $$ — PID of the current shell / script
  • $! — PID of the last background process
  • $? — exit code of the last foreground command
  • $PPID — PID of the parent process
  • $0 — name of the current script
  • Exit 0 = success; any non-zero = error

Process Commands

# ── Viewing processes ────────────────────────────────────────
ps                           # your processes in this terminal
ps aux                       # all processes, all users (BSD style)
ps aux | grep python         # find python processes
pgrep python                 # PIDs of processes named python
top                          # interactive process viewer
htop                         # better top (brew install htop)

# ── Background & foreground ──────────────────────────────────
long_command &               # run in background; shell prompt returns
echo "Started PID: $!"       # PID of background process
jobs                         # list background jobs: [1] running...
fg                           # bring last job to foreground
fg %1                        # bring job 1 to foreground
bg %1                        # resume stopped job in background
Ctrl+Z                       # suspend foreground process (stop it)
Ctrl+C                       # interrupt / kill foreground process

# ── wait ─────────────────────────────────────────────────────
cmd1 &
cmd2 &
wait                         # wait for ALL background jobs to finish
wait $!                      # wait for last background PID

# ── Killing processes ────────────────────────────────────────
kill 1234                    # send SIGTERM (graceful stop) to PID 1234
kill -9 1234                 # send SIGKILL (force kill — no cleanup)
kill -TERM 1234              # same as default kill
killall python               # kill all processes named python
pkill -f "script.sh"         # kill by pattern match on full command

# ── nohup & timeout ──────────────────────────────────────────
nohup ./script.sh &          # keep running after logout; output → nohup.out
timeout 30 ./script.sh       # kill script if it runs > 30 seconds
timeout 10s curl example.com # with explicit unit

# ── trap — handle signals in scripts ─────────────────────────
trap "echo 'Caught SIGINT!'" SIGINT       # intercept Ctrl+C
trap "rm -f /tmp/lockfile" EXIT           # cleanup on exit (always runs)
trap "echo interrupted; exit 1" INT TERM  # handle multiple signals
trap - SIGINT                             # reset to default handler

Notes

ps aux — shows all users; a=all, u=user-oriented format, x=no tty required
& — appended to any command; immediately returns shell prompt
Ctrl+Z — suspends (pauses) the foreground job; use bg or fg to resume
kill -9 — SIGKILL cannot be caught or ignored; process is forcibly terminated by the OS
wait — essential in scripts to synchronize parallel background jobs
trap ... EXIT — runs cleanup code whether the script exits normally or via error
nohup — redirects stdout to nohup.out; process ignores SIGHUP (terminal close)

Common Mistakes

  • Using kill -9 immediately instead of kill — always try SIGTERM first; SIGKILL prevents proper cleanup
  • Forgetting wait in scripts with background jobs — script exits before jobs finish, leaving orphan processes
  • Not using trap ... EXIT for temp file cleanup — temp files accumulate if a script is interrupted
  • Running long scripts without nohup over SSH — the script dies when the connection drops

Tips

  • Use trap 'rm -f "$tmpfile"' EXIT at the top of any script that creates temp files — automatic cleanup
  • pgrep -l python shows name + PID — easier than parsing ps aux | grep
  • For long-running SSH jobs, prefer screen or tmux over nohup — they let you reconnect and observe
  • Use timeout 30 cmd || echo "timed out" to add time limits to any command in scripts

Summary

  • cmd & runs in background · jobs lists · fg/bg move between foreground/background
  • kill PID = SIGTERM (graceful) · kill -9 = SIGKILL (force)
  • trap "cleanup" EXIT — always runs on exit; use for temp file cleanup
  • wait · nohup · timeout — essential for robust background scripts
