Sheet 04

Redirection & Piping

stdin · stdout · stderr · > · | · tee · xargs · here-docs

Standard Streams

Every process has three standard streams: stdin (0 — input), stdout (1 — normal output), and stderr (2 — error output). By default all three connect to the terminal. Redirection detaches a stream and points it at a file. Piping (|) connects the stdout of one command directly to the stdin of the next, creating powerful data-processing chains without temporary files.

Stream Numbers

  • 0 — stdin (keyboard by default)
  • 1 — stdout (terminal by default)
  • 2 — stderr (terminal by default)
  • 2>&1 — redirect stderr into stdout stream
  • /dev/null — discard anything sent to it
  • Order matters: cmd > f 2>&1 not cmd 2>&1 > f

Redirection & Pipe Examples

# ── Output redirection ───────────────────────────────────────
echo "hello" > file.txt      # write stdout to file (overwrite)
echo "world" >> file.txt     # append stdout to file
ls > filelist.txt 2>&1       # stdout + stderr → file
ls 2> errors.txt             # stderr only → file
ls > out.txt 2> err.txt      # stdout and stderr to separate files
ls > /dev/null               # discard stdout entirely
ls 2>/dev/null               # discard errors only

# ── Input redirection ────────────────────────────────────────
sort < names.txt             # feed file as stdin to sort
wc -l < file.txt             # count lines from file

# ── Pipes ────────────────────────────────────────────────────
ls -la | less                # page long output
cat file.txt | sort | uniq   # chain: sort then deduplicate
ps aux | grep "python"       # filter process list
history | grep "git"         # search command history
ls *.log | wc -l             # count log files

# ── tee — split output ───────────────────────────────────────
ls | tee filelist.txt        # print AND save to file
ls | tee -a filelist.txt     # print AND append to file
cmd | tee file.txt | grep x  # branch: save all, filter to terminal

# ── xargs — pipe to arguments ────────────────────────────────
find . -name "*.tmp" | xargs rm          # delete found files
find . -name "*.txt" | xargs wc -l      # count lines in all txt files
echo "a b c" | xargs mkdir             # create dirs a, b, c
cat urls.txt | xargs -n1 curl -O        # download each URL

# ── Here-doc (multi-line stdin) ──────────────────────────────
cat << EOF
Line one
Line two
EOF

# ── Here-string (single string as stdin) ─────────────────────
wc -w <<< "count these words"   # 3
base64 <<< "hello"               # encodes string directly

# ── Command substitution ─────────────────────────────────────
now=$(date +%Y-%m-%d)        # capture command output as variable
files=$(ls *.txt)            # store file list
echo "Today: $now"

# ── Process substitution ─────────────────────────────────────
diff <(sort file1.txt) <(sort file2.txt)   # diff sorted versions

Notes

> truncates first, then writes — existing content is gone immediately
>> — safe append; never overwrites existing content
2>&1 — must come after > file; order determines what stream 2 joins
tee — invaluable in pipelines where you want to log and continue processing
xargs — converts stdin lines into command arguments; use -n1 for one arg at a time
<< EOF — here-doc; the delimiter (EOF) can be any word; indent with <<-
$(cmd) — preferred over backticks; nestable and readable
<(cmd) — process substitution; treats command output as a temporary file

Common Mistakes

  • cmd 2>&1 > file — wrong order; stderr redirect happens before stdout changes; use cmd > file 2>&1
  • Using > when you meant >> — silently destroys file contents before writing
  • Piping to xargs rm without testing with echo first — run | xargs echo rm to preview

Tips

  • Preview any destructive xargs command by replacing the action with echo first
  • Use | tee /dev/stderr | next-cmd to debug mid-pipeline without breaking the chain
  • cmd &> /dev/null is bash shorthand for > /dev/null 2>&1 — cleaner for silencing noisy commands
  • Here-docs with <<-EOF (dash) strip leading tabs — useful for indented scripts

Summary

  • > overwrites · >> appends · < reads from file
  • 2>&1 — merge stderr into stdout; order after > matters
  • | chains commands; tee splits; xargs converts lines to args
  • $(cmd) captures output · <<EOF feeds multi-line input
