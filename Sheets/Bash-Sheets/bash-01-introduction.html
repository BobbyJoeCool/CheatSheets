What is the Shell?

The shell is a command-line interpreter that lets you interact with macOS directly. macOS ships with Zsh as the default shell (since macOS Catalina 10.15), but Bash is also available at /bin/bash. Scripts written for bash run with #!/bin/bash or the portable #!/usr/bin/env bash shebang. Both shells share the vast majority of everyday commands — differences are noted where they matter. The Terminal app (Terminal.app or iTerm2) is the GUI wrapper around whichever shell is active.

Bash vs Zsh

  • echo $SHELL — shows your current shell
  • bash --version — macOS ships Bash 3.2 (GPL2); install Bash 5 via Homebrew
  • Zsh is default on macOS 10.15+; config lives in ~/.zshrc
  • Bash config lives in ~/.bash_profile (login) or ~/.bashrc (interactive)
  • chsh -s /bin/zsh — change default shell to zsh
  • chsh -s /opt/homebrew/bin/bash — change to Homebrew bash

Navigation Commands

# ── Where am I? ──────────────────────────────────────────────
pwd                          # /Users/alice  (print working directory)
echo $HOME                   # /Users/alice

# ── Change directory ─────────────────────────────────────────
cd /usr/local/bin            # absolute path
cd Documents                 # relative path
cd ..                        # up one level
cd ../..                     # up two levels
cd ~                         # home directory
cd -                         # previous directory (toggle)
cd                           # bare cd also goes home

# ── List directory contents ──────────────────────────────────
ls                           # basic list
ls -l                        # long format (permissions, size, date)
ls -la                       # include hidden files (dot-files)
ls -lh                       # human-readable sizes (KB, MB)
ls -lt                       # sort by modification time
ls -R                        # recursive listing
ls -G                        # colorized output (macOS default alias)

# ── Useful navigation shortcuts ──────────────────────────────
open .                       # open current dir in Finder
open -a "Visual Studio Code" .   # open in app by name

Notes

pwd — always shows the full absolute path; never leaves you guessing
cd - — toggles between last two dirs; great for back-and-forth work
ls -la — the most common combo; shows all files with full detail
ls -lh — pairs with -l; makes file sizes human-readable
ls -lt — sort newest-first; useful for finding recent files
open . — macOS-specific; bridges terminal and Finder

Common Mistakes

  • cd to a path with spaces requires quotes: cd "My Folder" or escape: cd My\ Folder
  • Forgetting ls -a and missing hidden dot-files like .gitignore or .env
  • man pages use q to quit and /term to search — new users often get stuck
  • Assuming bash and zsh behave identically — array syntax and some expansions differ

Tips

  • Press Tab aggressively — the shell completes paths, commands, and flags
  • Use Ctrl+R to search history instead of pressing ↑ dozens of times
  • Install tldr via Homebrew for quick, practical command examples alongside man
  • type command tells you exactly what the shell will execute — alias, function, or binary

Summary

  • pwd / cd / ls — the three core navigation commands
  • macOS default is Zsh; Bash 5 available via Homebrew
  • Tab, Ctrl+R, !! — essential productivity shortcuts
  • man and tldr — two levels of help, built-in and practical
