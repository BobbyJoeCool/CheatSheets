HTML · Sheet 11

Media

img · srcset · picture · video · audio · iframe · canvas · SVG

Description

HTML embeds images, video, audio, and other content through a family of dedicated elements. <img> is the most used — and the most often done wrong. Responsive images (srcset, sizes, <picture>) let the browser pick the best asset for the device. <video> and <audio> provide native playback controls. <iframe> embeds external content. <canvas> is a pixel-drawing surface for JavaScript. Each has specific performance, security, and accessibility requirements.

Notes

  • alt on <img> is mandatory — omitting it is an accessibility violation; use alt="" for decorative images
  • Always set width and height on <img> — the browser reserves space before the image loads, preventing layout shift
  • autoplay on <video> requires muted — browsers block autoplay with audio by default
  • <iframe> — always add sandbox for untrusted content; it restricts scripts, forms, and top-level navigation
  • Prefer modern formats: WebP or AVIF over JPEG/PNG for photos; SVG for logos and icons

① img  ② decorative  ③ srcset 

<!-- ① Basic image — 
		alt, width, height always required -->
<img src="hero.jpg"
     alt="A sunrise over the Rocky Mountains"
     width="1200" height="600"
     loading="lazy"
     decoding="async" />

<!-- ② Decorative image — alt="" 
		tells screen readers to skip -->
<img src="divider.svg" alt="" 
		width="200" height="20" />

<!-- ③ srcset: resolution switching — 
		same image, different sizes -->
<img src="photo-800.jpg"
     srcset="photo-400.jpg  400w,
             photo-800.jpg  800w,
             photo-1600.jpg 1600w"
     sizes="(max-width: 600px)  100vw,
            (max-width: 1200px)  50vw,
            800px"
     alt="Mountain lake at dusk"
     width="800" height="533" />

④ picture  ⑤ video

<!-- ④ picture: art direction — 
		different crops per breakpoint -->
<picture>
  <!-- mobile crop, modern format first -->
  <source media="(max-width: 480px)"
          srcset="hero-mobile.avif" type="image/avif" />
  <source media="(max-width: 480px)"
          srcset="hero-mobile.webp" type="image/webp" />
  <!-- desktop crop -->
  <source srcset="hero-desktop.avif" type="image/avif" />
  <source srcset="hero-desktop.webp" type="image/webp" />
  <!-- img is the required fallback — always last -->
  <img src="hero-desktop.jpg"
       alt="Aerial city at night"
       width="1440" height="810" />
</picture>

<!-- ⑤ video with controls, poster, captions -->
<video controls width="854" height="480"
       poster="poster.jpg" preload="metadata">
  <source src="intro.mp4"  type="video/mp4" />
  <source src="intro.webm" type="video/webm" />
  <!-- captions required for accessibility -->
  <track kind="captions" src="intro-en.vtt"
         srclang="en" label="English" default />
  <p>Your browser doesn't support video.
    <a href="intro.mp4">Download</a>.</p>
</video>

⑥ audio  ⑦ iframe  ⑧ canvas 

<!-- ⑥ audio -->
<audio controls preload="none">
  <source src="podcast.mp3" type="audio/mpeg" />
  <source src="podcast.ogg" type="audio/ogg" />
</audio>

<!-- ⑦ iframe — title required; 
		sandbox for untrusted content -->
<iframe
  src="https://maps.example.com/embed?q=NYC"
  title="Map of New York City"
  width="600" height="450"
  loading="lazy"
  sandbox="allow-scripts allow-same-origin">
</iframe>

<!-- ⑧ canvas — set size via attributes, 
								not CSS -->
<canvas id="chart" width="600" height="300">
  <!-- fallback for no-canvas environments -->
  <p>Bar chart: monthly sales. Text data below.</p>
</canvas>

Image Format Guide

Format Best for Notes
AVIF Photos, complex images Best compression; growing support
WebP Photos, illustrations Great compression; excellent support
SVG Icons, logos, diagrams Scales infinitely; styleable with CSS
PNG Transparency needed Lossless; large files
JPEG Photos (legacy) Lossy; no transparency; widely supported
GIF Simple animation (legacy) Prefer <video> or CSS for animation

Key Attributes by Element

Element Key attributes
<img> src, alt*, width, height, loading, decoding, srcset, sizes
<video> src, controls, autoplay, muted, loop, poster, preload, width, height
<audio> src, controls, autoplay, muted, loop, preload
<source> src, type, media, srcset (inside picture)
<iframe> src, title*, width, height, sandbox, allow, loading
<canvas> width, height (set via attr not CSS); fallback content inside

♿ Accessibility

  • Informative images: alt describes what the image conveys. Decorative images: alt="" (empty string, not missing)
  • <video> requires captions via <track kind="captions"> for deaf users and transcripts for deafblind users
  • <iframe> must have a descriptive title attribute — screen readers announce it as the frame's label
  • <canvas> has no built-in accessibility — put fallback content inside it and consider an ARIA description or offscreen text alternative

⚠ Common Mistakes

  • Omitting alt on <img> — not the same as alt=""; missing entirely is an a11y violation
  • Not setting width/height on images — causes cumulative layout shift (CLS) as images load
  • Using autoplay without muted — browsers block it and the video silently fails to play
  • Setting <canvas> size via CSS instead of width/height attributes — distorts the drawing surface
  • Using <iframe> without title — screen reader users hear "frame" with no context

✓ Tips & Best Practices

  • Use loading="lazy" on all below-the-fold images — free performance win, widely supported
  • Use <picture> with AVIF + WebP + JPEG sources for maximum compression with fallback support
  • Always include a poster on <video> — shows a meaningful thumbnail before playback
  • Add sandbox to all third-party <iframe>s — meaningful security boundary with minimal cost

Quick Summary

  • <img> always needs alt, width, height — use alt="" for decorative images
  • srcset + sizes = resolution switching; <picture> = art direction
  • autoplay requires muted; always provide <source> fallbacks for video/audio
  • <iframe> needs title + sandbox; <canvas> size set via attributes not CSS
