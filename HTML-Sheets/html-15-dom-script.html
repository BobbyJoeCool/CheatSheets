HTML · Sheet 15

DOM & Script Basics

DOM · script loading · querySelector · innerHTML · dataset · events

Description

When a browser parses HTML it builds the DOM (Document Object Model) — a live tree of objects representing every element, attribute, and text node. JavaScript interacts with the page by reading and manipulating this tree. This sheet covers the HTML side of that relationship: how <script> tags load and execute, how HTML attributes serve as JS hooks (id, class, data-*), and the core DOM methods for selecting and modifying elements. Full JavaScript syntax is covered in the JS reference set — this is the handoff point.

Notes

  • The DOM is not your HTML file — it's a live in-memory tree; JS changes to the DOM don't change the source file
  • innerHTML parses HTML strings — never use it with unescaped user input; it's an XSS vector
  • textContent sets plain text — safe from XSS; use it when inserting dynamic text
  • querySelectorAll returns a static NodeList; getElementsByClassName returns a live HTMLCollection
  • Inline event handlers (onclick="...") in HTML are valid but considered bad practice — use addEventListener in JS instead

Core DOM Selection Methods

Method Returns Matches
getElementById('id') Element | null Single element by id
querySelector('css') Element | null First matching CSS selector
querySelectorAll('css') NodeList (static) All matching CSS selectors
getElementsByClassName HTMLCollection (live) Elements by class name
getElementsByTagName HTMLCollection (live) Elements by tag name
closest('css') Element | null Nearest ancestor matching selector

Reading & Writing Elements

Property / Method Purpose
el.textContent Get/set plain text — XSS safe
el.innerHTML Get/set HTML — dangerous with user input
el.setAttribute(n, v) Set any attribute by name
el.getAttribute(n) Read any attribute by name
el.dataset.key Read/write data-* attribute (camelCase)
el.classList.add() Add a class without replacing others
el.classList.toggle() Add if absent, remove if present
el.style.property Set inline style — use sparingly
el.remove() Remove element from the DOM
el.addEventListener() Attach event handler

♿ Accessibility

  • When JS adds or removes content, update ARIA states too — toggling hidden on a panel must be paired with updating aria-expanded
  • Manage focus after dynamic changes — opening a modal should move focus inside it; closing should return focus to the trigger
  • Use textContent not innerHTML when inserting user-generated text — prevents XSS and keeps the DOM clean
  • Progressive enhancement: build HTML and CSS so the page is usable without JS, then layer JS on top — <noscript> is your fallback

Example - HTML

<!-- ① Script loading strategies in <head> -->
<!-- Blocks parsing — only for critical inline scripts -->
<script>
  document.documentElement.classList.add('js');
</script>

<!-- defer: downloads in parallel, executes after DOM is ready -->
<script src="/js/app.js" defer></script>

<!-- async: downloads in parallel, executes immediately on load -->
<script src="/js/analytics.js" async></script>

<!-- ES module: deferred by default, scoped to module -->
<script type="module" src="/js/main.mjs"></script>

<!-- ② HTML hooks that JS targets -->
<button
  id="save-btn"
  class="btn btn--primary"
  data-action="save"
  data-endpoint="/api/save">
  Save
</button>

<div id="status-region" aria-live="polite"></div>

Example - HTML

// ③ Selecting elements
const btn    = document.getElementById('save-btn');
const status = document.querySelector('#status-region');
const cards  = document.querySelectorAll('.card');   // NodeList
// ④ Reading data-* attributes (HTML: data-endpoint="/api/save")
const endpoint = btn.dataset.endpoint;  // "/api/save"
const action   = btn.dataset.action;    // "save"
// ⑤ Modifying content — textContent is XSS-safe
status.textContent = 'Saving…';           // plain text — safe
status.innerHTML   = '<strong>Done!</strong>'; // HTML — risky
// ⑥ Toggling classes and attributes
btn.classList.add('loading');
btn.classList.toggle('active');
btn.setAttribute('aria-busy', 'true');
btn.setAttribute('disabled', '');
// ⑦ Event listeners — preferred over inline onclick
btn.addEventListener('click', async () => {
  btn.classList.add('loading');
  btn.setAttribute('aria-busy', 'true');

  const res = await fetch(btn.dataset.endpoint,
    { method: 'POST' });

  btn.classList.remove('loading');
  btn.removeAttribute('aria-busy');
  status.textContent = res.ok ? 'Saved!' : 'Error saving.';
});
// ⑧ Creating and inserting elements
const card = document.createElement('div');
card.className   = 'card';
card.textContent = 'New card content';
document.querySelector('.grid').appendChild(card);
// ⑨ Removing an element
document.querySelector('#old-banner').remove();

⚠ Common Mistakes

  • Using innerHTML to insert user-provided text — always sanitize or use textContent
  • Running DOM queries before the document is parsed — use defer on the script or listen for DOMContentLoaded
  • Using inline onclick="..." handlers in HTML — mixes concerns and can't be removed without editing HTML
  • Selecting elements by tag name when a class or id is more specific — querySelectorAll('button') catches far more than you intend
  • Forgetting to update ARIA state when toggling visibility — hidden on the panel without updating aria-expanded on the trigger

✓ Tips & Best Practices

  • Use data-* attributes as JS configuration hooks — keeps logic out of class names and is refactor-safe
  • Use defer on all external scripts — parallel download, correct execution order, no parser blocking
  • Use event delegation: attach one listener to a parent instead of many listeners to children — more efficient and handles dynamic content
  • Build with progressive enhancement — the page should work without JS for the widest possible audience

Quick Summary

  • DOM = live tree of HTML in memory; JS reads and writes it at runtime
  • querySelector + CSS selectors; textContent for safe text; innerHTML with caution
  • defer for almost all scripts; data-* for HTML→JS configuration
  • Full JavaScript syntax continues in the JS reference set — this is the HTML/JS boundary
