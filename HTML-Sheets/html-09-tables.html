HTML · Sheet 9

Tables

table · thead · tbody · tfoot · th · td · caption · scope · colspan

Description

HTML tables are for tabular data — information that has a meaningful relationship between rows and columns, like a spreadsheet or schedule. They are not for page layout (that's CSS Grid and Flexbox). A well-structured table uses <thead>, <tbody>, and <caption> to communicate its structure to both humans and assistive technology. The scope attribute on <th> tells screen readers which cells a header applies to — essential for accessible tables.

Notes

  • <caption> must be the first child of <table> — it is the table's accessible name
  • <th> is a header cell; <td> is a data cell — never use <td> for header content
  • colspan spans columns; rowspan spans rows — remove the cells that the spanning cell covers
  • <tfoot> can appear before <tbody> in source — the browser always renders it at the bottom
  • <colgroup> + <col> let you apply styles to entire columns without touching each cell

Table Element Reference

ElementPurpose
<table>Root table element
<caption>Table title — must be first child, becomes accessible name
<colgroup>Groups columns for styling
<col>Single column inside colgroup; use span to cover multiple
<thead>Header row group — repeated on print page breaks
<tbody>Body row group — main data rows
<tfoot>Footer row group — totals, summaries
<tr>Table row
<th>Header cell — bold, centered by default; add scope
<td>Data cell

scope Attribute Values

ValueHeader applies to…
colAll cells in the same column below
rowAll cells in the same row to the right
colgroupAll cells in the colgroup
rowgroupAll cells in the row group (thead/tbody/tfoot)

♿ Accessibility

  • Always include <caption> — it is the accessible name of the table and the first thing screen readers announce
  • Use scope="col" on column headers and scope="row" on row headers — allows screen readers to associate cells with their headers
  • For complex tables with irregular headers, use id on each <th> and headers="id1 id2" on the corresponding <td>
  • Never use a table for layout — screen readers announce "table, N columns" which confuses users when there's no data relationship

① Full table structure

<table>
  <!-- caption: accessible name — 
  			must be first child -->
  <caption>Q1 2025 Sales by Region</caption>

  <!-- colgroup: style entire columns via CSS -->
  <colgroup>
    <col />                    <!-- Region -->
    <col class="highlight" />  <!-- Jan -->
    <col span="2" />           <!-- Feb + Mar -->
    <col class="total-col" />  <!-- Total -->
  </colgroup>

  <thead>
    <tr>
      <!-- scope="col": header applies
	  				to cells below -->
      <th scope="col">Region</th>
      <th scope="col">Jan</th>
      <th scope="col">Feb</th>
      <th scope="col">Mar</th>
      <th scope="col">Total</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <!-- scope="row": header applies 
	  			to cells to its right -->
      <th scope="row">North</th>
      <td>$42,000</td>
      <td>$38,500</td>
      <td>$51,200</td>
      <td>$131,700</td>
    </tr>
    <tr>
      <th scope="row">South</th>
      <td><!-- data cells --></td>
      <td></td><td></td><td></td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <th scope="row">Total</th>
      <td><!-- column totals --></td>
      <td></td><td></td>
      <td><strong><!-- grand total --></strong></td>
    </tr>
  </tfoot>
</table>

② colspan + rowspan

<table>
  <caption>Conference Schedule</caption>
  <thead>
    <tr>
      <th scope="col">Time</th>
      <th scope="col">Track A</th>
      <th scope="col">Track B</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">9:00 AM</th>
      <!-- colspan="2": spans both track columns -->
      <td colspan="2">Opening Keynote</td>
    </tr>
    <tr>
      <th scope="row">10:00 AM</th>
      <td>HTML Deep Dive</td>
      <!-- rowspan="2": covers the 11 AM row too -->
      <td rowspan="2">CSS Workshop</td>
    </tr>
    <tr>
      <th scope="row">11:00 AM</th>
      <td>JS Performance</td>
      <!-- NO td here — rowspan above covers it -->
    </tr>
  </tbody>
</table>

⚠ Common Mistakes

  • Using tables for page layout — they make code unmaintainable and are a major accessibility failure
  • Omitting <caption> — screen readers announce "table" with no label, leaving users without context
  • Forgetting to remove the cells covered by colspan/rowspan — causes extra phantom cells
  • Using <td> for header cells — loses bold styling and, more importantly, the screen reader announcement
  • Missing scope on <th> elements — screen readers can't associate headers with data cells

✓ Tips & Best Practices

  • Always include <thead>, <tbody>, and <caption> — minimum structure for an accessible table
  • Add scope="col" to all column headers and scope="row" to all row headers — takes seconds, big a11y win
  • Use <colgroup> to style columns — far cleaner than adding classes to every <td>
  • Test with a screen reader — navigate cell by cell to hear if headers are correctly associated

Quick Summary

  • Tables = tabular data only, never layout
  • Always include <caption> (table name), <thead>, <tbody>
  • Use scope="col|row" on every <th> for accessible header association
  • colspan/rowspan merge cells — remove the cells they replace
