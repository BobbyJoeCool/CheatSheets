Semantic Containers

section · article · figure · time · address · blockquote

Description

Semantic containers go beyond landmarks — they describe the nature of their content rather than just its position on the page. An <article> is self-contained and syndicatable (a blog post, a tweet, a product card). A <section> is a thematic grouping that needs a heading. <figure> wraps any self-contained illustration. These elements tell browsers, search engines, and assistive technology what the content is, not just where it sits — enabling richer indexing, reading modes, and accessible navigation.

Notes

  • <section> should always contain a heading (<h2>–<h6>) — a section without one is likely a <div>
  • <article> can be nested — a comment inside a blog post is an article inside an article
  • <time datetime> value must be a valid ISO 8601 format: 2025-01-15, 14:30, 2025-01-15T14:30:00
  • <address> is for contact info of the nearest <article> or <body> ancestor — not for postal addresses in general
  • <figure> content can be an image, code block, chart, table, or video — anything self-contained with a caption

Element Decision Guide

Element Use when…
<article> Content makes sense on its own — post, card, comment, widget
<section> Thematic grouping within a page or article; needs a heading
<figure> Self-contained media or illustration referenced from main content
<figcaption> Label or description for a <figure>; first or last child only
<time> Any date, time, or duration; add datetime for machine readability
<address> Contact info for the document or nearest article author
<blockquote> Extended quotation from another source; add cite attr for URL
<cite> Title of a referenced work (book, film, article) — not the author name

♿ Accessibility

  • <article> has an implicit role of article — screen readers announce "article" when entering, helping users understand content boundaries
  • <section> only becomes a named landmark region when it has an aria-label or aria-labelledby — otherwise it's invisible to landmark navigation
  • <figure> with <figcaption> is automatically associated — no extra ARIA needed; screen readers announce the caption as the figure's label
  • <time datetime> lets assistive tech present dates in user-preferred formats — always include the datetime attribute

Example

<!-- article: self-contained, could be syndicated independently -->
<article>
  <header>
    <h2>How to Grow Tomatoes</h2>

    <!-- time: human-readable text + machine-readable datetime -->
    <p>Published <time datetime="2025-06-01">June 1, 2025</time></p>

    <!-- address: contact info for this article's author -->
    <address>
      By <a href="/authors/jane">Jane Smith</a>
    </address>
  </header>

  <!-- section: thematic grouping — always needs its own heading -->
  <section>
    <h3>Choosing a Location</h3>
    <p><!-- section body content --></p>

    <!-- figure: self-contained illustration referenced from prose -->
    <figure>
      <img src="sun-diagram.jpg" alt="Diagram showing 8 hours of sun arc" />
      <!-- figcaption must be first or last child of figure -->
      <figcaption>Fig. 1 — Ideal sun exposure for a south-facing bed.</figcaption>
    </figure>
  </section>

  <section>
    <h3>Expert Advice</h3>

    <!-- blockquote: extended quote; cite attr holds the source URL -->
    <blockquote cite="https://garden.org/tomatoes">
      <p><!-- quoted text --></p>
      <!-- cite element: title of the referenced work (not the author) -->
      <footer>— <cite>National Gardening Association Guide</cite></footer>
    </blockquote>
  </section>

  <!-- figure can wrap code blocks, tables, or any self-contained media -->
  <figure>
    <pre><code><!-- code snippet --></code></pre>
    <figcaption>A twice-weekly watering schedule.</figcaption>
  </figure>

  <!-- nested article: a comment is its own article inside the post -->
  <section aria-label="Comments">
    <h3>Comments</h3>
    <article>
      <p><strong>GardenFan99</strong> — Great tips, thanks!</p>
      <time datetime="2025-06-03T09:15">June 3 at 9:15 AM</time>
    </article>
  </section>

</article>

⚠ Common Mistakes

  • Using <section> as a styled wrapper with no heading — that's a job for <div>
  • Using <address> for a physical mailing address unrelated to the author or document
  • Omitting the datetime attribute from <time> — the human-readable text alone isn't machine-parseable
  • Putting <figcaption> in the middle of a <figure> — it must be the first or last child
  • Using <cite> for author names — it's for titles of works, not people

✓ Tips & Best Practices

  • Ask "could this content be copy-pasted into an RSS feed and still make sense?" — if yes, it's an <article>
  • Ask "does this group need its own heading?" — if yes, it's a <section>; if no, it's a <div>
  • Always include datetime on <time> even when the visible text is already a date
  • Use <figure> for any illustration the prose references by number — "see Fig. 1"

Quick Summary

  • <article> = self-contained, syndicatable; <section> = thematic group with heading
  • <figure> + <figcaption> — any self-contained media with a label
  • <time datetime="ISO"> — always include the machine-readable attribute
  • If you're unsure between <section> and <div>, ask: does it need a heading? If not, use <div>
