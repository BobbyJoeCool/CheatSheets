Classes & IDs

class · id · CSS selectors · naming · specificity

Description

class and id are the primary hooks that connect HTML elements to CSS styling and JavaScript behavior. A class can be used on any number of elements and multiple classes can live on one element — making it the right tool for reusable styles. An id must be unique per page — making it the right tool for one-of-a-kind targets like a skip link anchor, a JS hook for a specific widget, or a scroll destination. Getting these right is the foundation of maintainable CSS.

Notes

  • Multiple classes are space-separated on the element: class="card card--featured active"
  • id must be unique on the page — duplicate IDs break CSS, JS (getElementById), and anchor links
  • id has higher specificity than class in CSS — an #id rule overrides a .class rule
  • Names are case-sensitive in CSS selectors: .Card ≠ .card
  • Names cannot start with a digit: .2col is invalid; use .col-2

class vs id

class id
Reusable? Yes — many elements No — unique per page
Multiple per element? Yes — space-separated No — one id only
CSS selector .class-name #id-name
JS selector querySelectorAll('.x') getElementById('x')
CSS specificity 0-1-0 (low) 1-0-0 (high)
Anchor target? No Yes — href="#id"
Best for Styles, repeated patterns Unique widgets, anchors, JS

Naming Conventions

Convention Example Use
kebab-case nav-link Standard for HTML classes
BEM Block card Component root
BEM Element card__title Part of a component
BEM Modifier card--featured State or variant
JS hook js-modal-trigger For JS only, never styled

♿ Accessibility

  • id is used to associate labels with form controls (<label for="id">) and to link ARIA descriptions (aria-describedby="id")
  • Duplicate IDs silently break aria-labelledby and aria-describedby associations — always validate for uniqueness
  • The class="sr-only" pattern (visually hidden but accessible) is a critical a11y tool — keep a utility class for it in your CSS
  • Never use a class name to convey meaning to users — that information must also be in the HTML semantics or ARIA

Example

<!-- 1) class: reusable — multiple elements, 
	multiple classes per element -->
<div class="card">
  <h2 class="card__title"><!-- title --></h2>
  <p  class="card__body"><!-- body --></p>
</div>

<!-- BEM modifier: adds a variant without duplicating base styles -->
<div class="card card--featured">
  <h2 class="card__title"><!-- title --></h2>
  <p  class="card__body"><!-- body --></p>
</div>
<!-- CSS: .card { … }  .card--featured { border-color: gold; } -->

<!-- 2) id: unique anchor target + JS hook -->
<section id="pricing">
  <!-- section content -->
</section>
<!-- Scroll link from anywhere on the page: -->
<a href="#pricing">See Pricing</a>

<!-- 3) id linking a label to its input (critical for a11y) -->
<label for="email-input">Email address</label>
<input type="email" id="email-input" name="email" />

<!-- 4) aria-describedby links an id to a hint or error message -->
<input type="password" id="pwd" aria-describedby="pwd-hint" />
<p id="pwd-hint">Must be at least 8 characters.</p>

<!-- 5) js- prefix: JS-only hook, never styled in CSS -->
<button class="btn js-open-modal" data-target="contact-modal">
  Contact Us
</button>

<!-- 6) sr-only: visually hidden but readable by screen readers -->
<span class="sr-only">Current page:</span>
<!-- CSS: .sr-only { position:absolute; width:1px; height:1px;
                     overflow:hidden; clip:rect(0,0,0,0);
                     white-space:nowrap; } -->

<!-- 7) Multiple classes for active state -->
<li class="nav-item nav-item--active">
  <a href="/dashboard" class="nav-link">Dashboard</a>
</li>

⚠ Common Mistakes

  • Reusing the same id on multiple elements — only the first is found by getElementById and CSS
  • Using id for styling everything — high specificity makes it painful to override later
  • Starting class names with a number — .2up-grid is invalid; use .grid-2up
  • Using the same class for both JS behavior and CSS styling — a refactor of one breaks the other
  • Forgetting that class names are case-sensitive — .NavLink and .navlink are different selectors

✓ Tips & Best Practices

  • Use class for styles, id for unique targets, data-* for JS configuration — keep them separate
  • Prefix JS-only classes with js- so they're never accidentally removed during CSS refactors
  • Keep a .sr-only utility class in every project — you'll need it for accessible icon buttons and skip links
  • Use BEM or another consistent naming convention — it prevents specificity conflicts as projects grow

Quick Summary

  • class = reusable, many elements; id = unique, one element per page
  • CSS: .class selector; #id selector — id has higher specificity
  • id enables anchor links, label associations, and ARIA references
  • Separate styling classes, JS hooks (js-), and ARIA IDs — don't let them cross-contaminate
