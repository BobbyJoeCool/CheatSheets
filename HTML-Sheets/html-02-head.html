HTML · Sheet 2

Head

metadata · SEO · open graph · favicons · CSS & JS loading

Description

The <head> element contains metadata about the document — information the browser and search engines need but that users never see directly. It controls character encoding, viewport scaling, the browser tab title, SEO data, social sharing previews, icons, and how external CSS and JavaScript files are loaded. Getting the <head> right is foundational: errors here affect rendering, performance, SEO, and accessibility before a single visible element appears.

Notes

  • charset must appear within the first 1024 bytes of the file — put it first
  • viewport meta is required for responsive design — without it, mobile browsers zoom out
  • <title> is the single most important SEO element on the page
  • Open Graph tags are read by social platforms (Facebook, Slack, LinkedIn) when generating link previews
  • defer and async only work on external scripts — inline <script> blocks are always render-blocking

Script Loading Strategies

Attribute Downloads Executes
none Blocks parsing Immediately, blocks render
defer Parallel After HTML parsed, in order
async Parallel As soon as downloaded
type="module" Parallel Deferred, ES module scope

Resource Hints

rel value What it does
preload Fetch this resource now, high priority (fonts, hero images)
prefetch Fetch for likely next navigation, low priority
preconnect Open TCP/TLS connection to origin early
dns-prefetch Resolve DNS for origin early, lighter than preconnect

♿ Accessibility

  • <title> is announced by screen readers when the page loads — make it unique and descriptive per page
  • lang on <html> (set in Architecture) enables correct pronunciation by screen readers — the head does not override it
  • Use <meta name="theme-color"> carefully — ensure sufficient contrast in browser UI chrome for users with low vision
  • Avoid <meta http-equiv="refresh"> auto-redirects — they disorient screen reader and keyboard users

Example

<head>

  <!-- ① Encoding: must be first, within 1024 bytes -->
  <meta charset="UTF-8" />

  <!-- ② Viewport: required for responsive layouts -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />

  <!-- ③ Title: shown in tab, bookmarks, search results -->
  <title>About Us — Acme Corp</title>

  <!-- ④ SEO meta -->
  <meta name="description" content="Acme Corp makes everything." />
  <meta name="robots" content="index, follow" />
  <link rel="canonical" href="https://acme.com/about" />

  <!-- ⑤ Open Graph — social share previews -->
  <meta property="og:title"       content="About Us — Acme Corp" />
  <meta property="og:description" content="Acme Corp makes everything." />
  <meta property="og:image"       content="https://acme.com/og.jpg" />
  <meta property="og:url"         content="https://acme.com/about" />
  <meta property="og:type"        content="website" />

  <!-- ⑥ Twitter / X card -->
  <meta name="twitter:card"  content="summary_large_image" />
  <meta name="twitter:title" content="About Us — Acme Corp" />

  <!-- ⑦ Favicon -->
  <link rel="icon" href="/favicon.svg" type="image/svg+xml" />
  <link rel="icon" href="/favicon.ico" sizes="any" />       <!-- fallback -->
  <link rel="apple-touch-icon" href="/apple-touch-icon.png" />
  <link rel="manifest" href="/site.webmanifest" />

  <!-- ⑧ CSS: link in head so it loads before body renders -->
  <link rel="stylesheet" href="/css/main.css" />

  <!-- ⑨ Preconnect to a third-party origin (e.g. Google Fonts) -->
  <link rel="preconnect" href="https://fonts.googleapis.com" />
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />

  <!-- ⑩ Preload a critical asset (hero font) -->
  <link rel="preload" href="/fonts/inter.woff2"
        as="font" type="font/woff2" crossorigin />

  <!-- ⑪ JavaScript: defer keeps it non-blocking -->
  <script src="/js/app.js" defer></script>

  <!-- ⑫ Theme color: browser chrome accent on mobile -->
  <meta name="theme-color" content="#1a1a2e" />

</head>

⚠ Common Mistakes

  • Placing <script> without defer or async in <head> — blocks HTML parsing entirely
  • Missing or generic <title> — "Home" or "Untitled" hurts SEO and confuses users with multiple tabs
  • Omitting og:image — social previews show a blank card instead of your image
  • Using async for scripts that depend on each other — execution order is not guaranteed
  • Linking CSS at the bottom of <body> — causes a flash of unstyled content (FOUC)

✓ Tips & Best Practices

  • Order: charset → viewport → title → meta → CSS → preloads → scripts
  • Keep <title> under 60 characters — search engines truncate longer titles
  • Use defer for almost all scripts — it's parallel download + correct execution order
  • Use an SVG favicon — it scales to any size and supports dark mode via prefers-color-scheme
  • Validate Open Graph with the Facebook Sharing Debugger or opengraph.xyz before launch

Quick Summary

  • charset + viewport + title are the three non-negotiable head elements
  • CSS goes in <head>; scripts get defer or go before </body>
  • Open Graph tags control how links look when shared on social platforms
  • Resource hints (preload, preconnect) are performance tools, not required
