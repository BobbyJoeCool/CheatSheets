HTML · Sheet 12

Links

href · absolute · relative · target · rel · schemes · download · states

Description

The anchor element <a> is how HTML connects documents, sections, email addresses, phone numbers, and downloadable files. The href attribute determines the destination — it can be an absolute URL, a relative path, a same-page fragment, or a special scheme like mailto: or tel:. Link text must describe the destination, not just say "click here." When opening links in a new tab, always pair target="_blank" with rel="noopener noreferrer" to close a security vulnerability.

Notes

  • <a> without href is a placeholder link — it has no role and is not focusable
  • target="_blank" without rel="noopener" gives the opened page access to your page via window.opener — a security hole
  • The download attribute works only for same-origin URLs or data URIs — cross-origin links are navigated instead
  • Fragment links (href="#id") move focus to the target element — it must exist and the target should be focusable or have tabindex="-1"
  • Avoid href="javascript:void(0)" — use a <button> for actions that don't navigate

href Value Types

Type Example Navigates to
Absolute URL https://example.com/page Full URL, any origin
Root-relative /about Same origin, from root
Relative path ../contact Relative to current file
Fragment #section-id Same-page anchor by id
mailto: mailto:hi@example.com Opens email client
tel: tel:+15555550100 Opens phone dialer
sms: sms:+15555550100 Opens SMS app

rel Attribute Values

Value Purpose
noopener Blocks opener access — always use with target="_blank"
noreferrer Hides referrer header + implies noopener
nofollow Tells search engines not to follow this link
external Indicates the link goes to an external site
prev / next Pagination relationships for SEO
author Links to the author of the document

♿ Accessibility

  • Link text must describe the destination — "click here" and "read more" are meaningless to screen readers navigating by link list
  • If link text is an icon or image, add aria-label or descriptive alt text — e.g. <a href="..."><img alt="GitHub profile"></a>
  • Warn users when a link opens in a new tab — add visible text or aria-label="... (opens in new tab)"
  • Fragment links must move keyboard focus to the target — add tabindex="-1" to non-interactive targets like <h2>

Example

<!-- ① Absolute URL -->
<a href="https://developer.mozilla.org">MDN Web Docs</a>

<!-- ② Root-relative (same origin, from site root) -->
<a href="/about">About Us</a>

<!-- ③ Relative path (relative to current file) -->
<a href="../pricing">Pricing</a>

<!-- ④ Fragment — scrolls to element with matching id -->
<a href="#contact">Jump to Contact</a>
<!-- ...target element on the same page... -->
<section id="contact">
  <h2>Contact</h2>
  <!-- contact content -->
</section>

<!-- ⑤ New tab — always add rel for security -->
<a href="https://example.com"
   target="_blank"
   rel="noopener noreferrer">
  Visit Example.com (opens in new tab)
</a>

<!-- ⑥ mailto: with subject and body prefilled -->
<a href="mailto:hello@example.com
         ?subject=Project%20Inquiry
         &body=Hi%2C%20I'd%20like%20to%20discuss...">
  Email us
</a>

<!-- ⑦ tel: phone link (mobile-friendly) -->
<a href="tel:+15555550100">(555) 555-0100</a>

<!-- ⑧ download: force file download, optionally rename it -->
<a href="/reports/q1-2025.pdf" download="Q1-2025-Report.pdf">
  Download Q1 Report (PDF)
</a>

<!-- ⑨ Icon-only link — aria-label provides the accessible name -->
<a href="https://github.com/username"
   aria-label="GitHub profile (opens in new tab)"
   target="_blank" rel="noopener noreferrer">
  <svg aria-hidden="true" focusable="false">
    <!-- icon paths -->
  </svg>
</a>

<!-- ⑩ WRONG — anchor used as a JS action button -->
<!-- <a href="javascript:void(0)" onclick="doThing()">Do Thing</a> -->
<!-- CORRECT — use a button for actions that don't navigate -->
<button type="button" onclick="doThing()">Do Thing</button>

⚠ Common Mistakes

  • Using target="_blank" without rel="noopener noreferrer" — opens a security vulnerability via window.opener
  • Non-descriptive link text — "click here", "read more", "here" tell screen reader users nothing about the destination
  • Using <a> with href="javascript:void(0)" for buttons — use <button> instead
  • Expecting download to work on cross-origin URLs — browsers navigate instead of downloading
  • Using tel: with spaces or dashes in the number — the href value should be digits only: tel:+15555550100

✓ Tips & Best Practices

  • Every external link should have rel="noopener noreferrer" — make it a default in your templates
  • Write link text that makes sense out of context — imagine a screen reader reading only the links on the page
  • Use root-relative paths (/about) for internal links — they work regardless of the current page's depth
  • Add a visible "(opens in new tab)" hint or icon alongside target="_blank" — sighted users appreciate the warning too

Quick Summary

  • <a href> navigates; <button> acts — don't use <a> for JS-only actions
  • target="_blank" always needs rel="noopener noreferrer"
  • Link text must describe the destination, not just say "click here"
  • mailto:, tel:, #fragment, download — four href patterns beyond URLs
