HTML · Sheet 8

Lists

ul · ol · li · dl · dt · dd · nested lists · nav pattern

Description

HTML has three list types: <ul> (unordered — bullets, order doesn't matter), <ol> (ordered — numbered, sequence matters), and <dl> (description list — term/definition pairs). Lists are one of the most semantically important structures in HTML: navigation menus, breadcrumbs, feature lists, and step-by-step instructions should all use list markup, not bare <div>s. Screen readers announce the list type and item count, which helps users understand content structure before reading it.

Notes

  • The only valid direct children of <ul> and <ol> are <li> elements
  • <ol> attributes: type (1, A, a, I, i), start (starting number), reversed (count down)
  • <li> inside an <ol> can have a value attribute to override its number
  • <dl> allows multiple <dt> per <dd> and vice versa — useful for glossaries
  • CSS list-style: none on a <ul> removes the bullet and the list semantics in Safari — add role="list" to restore them

Which List Type?

Use Element Example
Order doesn't matter <ul> Features, ingredients, nav links
Order matters <ol> Steps, rankings, instructions
Term + definition pairs <dl> Glossary, metadata, FAQ

<ol> Attributes

Attribute Values Effect
type 1 A a I i Marker style: numbers, letters, Roman
start any integer Starting counter value
reversed boolean Count down instead of up
value (on li) integer Override this item's counter

♿ Accessibility

  • Screen readers announce "list, N items" when entering a list — this orientation is lost with <div>s
  • Safari removes list semantics when list-style: none is applied — add role="list" on the <ul> to restore the announcement
  • Navigation menus should be <nav><ul> — screen readers announce "navigation" then "list" giving users immediate context
  • <dl> pairs are announced as definition term / definition — ideal for key-value metadata (author, date, category)

Quick Summary

  • <ul> = unordered; <ol> = ordered; <dl> = term/definition pairs
  • Only <li> is a valid direct child of <ul> and <ol>
  • Nav menus = <nav><ul><li><a> — always use list markup
  • Add role="list" when list-style: none is applied — preserves Safari screen reader semantics

Example

<!-- ① ul: unordered — order doesn't matter -->
<ul>
  <li><!-- item --></li>
  <li><!-- item --></li>
  <li><!-- item --></li>
</ul>

<!-- ② ol: ordered — sequence matters -->
<ol>
  <li>Install Node.js</li>
  <li>Run <code>npm install</code></li>
  <li>Start the server with <code>npm run dev</code></li>
</ol>

<!-- ③ ol attributes: type, start, reversed -->
<ol type="A" start="3" reversed>
  <li><!-- renders as "E" counting down --></li>
  <li><!-- renders as "D" --></li>
</ol>

<!-- ④ Nested list — child ul lives INSIDE the parent li -->
<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </li>
  <li>Backend</li>
</ul>

<!-- ⑤ dl: description / definition list -->
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language — the structure of web pages.</dd>

  <!-- multiple dt elements can share one dd -->
  <dt>JS</dt>
  <dt>JavaScript</dt>
  <dd>The programming language of the web.</dd>
</dl>

<!-- ⑥ Navigation — ul is the standard pattern for nav links -->
<nav aria-label="Primary">
  <!-- role="list" restores Safari semantics lost by list-style:none -->
  <ul role="list">
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>

<!-- ⑦ dl as a metadata block -->
<dl>
  <dt>Author</dt>   <dd>Jane Smith</dd>
  <dt>Published</dt><dd><time datetime="2025-06-01">June 1, 2025</time></dd>
  <dt>Category</dt> <dd>Web Development</dd>
</dl>

⚠ Common Mistakes

  • Putting elements other than <li> as direct children of <ul>/<ol> — invalid HTML
  • Using <ol> for items where order doesn't matter — misleads screen readers who announce "list of steps"
  • Building navigation with <div>s instead of <nav><ul><li> — loses landmark and list semantics
  • Forgetting role="list" when using list-style: none in CSS — Safari drops list semantics silently
  • Using a <dl> for generic two-column layouts — it's specifically for term/definition relationships

✓ Tips & Best Practices

  • Always wrap navigation links in <nav><ul><li> — even if CSS makes them look horizontal
  • Use <dl> for any key-value metadata — author, date, tags, file size — it's more semantic than a table
  • Add role="list" when applying list-style: none to a <ul> — takes two seconds, preserves a11y
  • Nest lists inside the <li> (not after it) — the sub-list is part of that item
