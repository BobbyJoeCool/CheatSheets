Responsive: Fluid Units & Functions

rem · em · vh · vw · % · clamp() · min() · max() · calc()
Description
Fluid units and CSS math functions let layouts and typography adapt to the viewport without breakpoints. rem scales with the root font size; em scales with the local font size. Viewport units (vw, vh) are relative to the screen. clamp(min, preferred, max) creates self-constraining fluid values — a single declaration that replaces a minimum, maximum, and in-between media query. calc() mixes units at render time.
Example
/* rem — relative to :root font-size */
:root { font-size: 100%; }          /* respects browser setting */
h1    { font-size: 2.5rem; }        /* 40px if root = 16px */
p     { font-size: 1rem; }          /* 16px */

/* em — relative to element's own font-size */
.parent { font-size: 20px; }
.child  { font-size: 0.8em; }       /* 16px — 80% of parent */
.child  { padding: 1em; }           /* 16px — relative to child's own size */

/* Viewport units */
.hero   { height: 100vh; }          /* full viewport height */
.hero   { height: 100svh; }         /* safe: excludes mobile chrome */
.banner { width: 50vw; }

/* calc() — mix any units */
.sidebar { width: calc(100% - 260px); }
.pad     { padding: calc(1rem + 2vw); }

/* clamp(minimum, preferred, maximum) */
h1          { font-size: clamp(1.5rem, 5vw, 3rem); }
.container  { width: clamp(320px, 90%, 1200px); }
.gap        { gap: clamp(1rem, 3vw, 2rem); }

/* min() and max() */
.box { width: min(100%, 600px); }   /* never wider than 600px */
.box { width: max(300px, 50%); }    /* never narrower than 300px */
Notes
1rem equals the root font-size (typically 16px) — it respects the user's browser font-size setting
em is relative to the element's own font-size for spacing; nesting compounds the value
100svh (small viewport height) avoids the mobile browser chrome problem with 100vh
Spaces around + and - in calc() are required — calc(100%-2rem) is invalid
Code Explained
rem — root-relative; predictable and respects browser font-size settings
em — parent-relative for font-size; self-relative for padding/margin
100vh — full viewport height; can include mobile browser UI
100svh — small viewport: excludes mobile address bar and tab strip
calc(100% - 260px) — mixes percentage and px; resolved at paint time
clamp(min, preferred, max) — the preferred scales fluidly, clamped by min/max
5vw as the preferred in clamp — scales with viewport width
min(100%, 600px) — always picks the smaller value; equivalent to max-width

⚠ Common Mistakes
Using em for font-size in nested components — values compound and become unpredictable
Using 100vh for full-screen mobile layouts — the address bar is included and content gets cut off
Omitting spaces around + and - in calc() — the expression fails silently
Using bare vw for font-size without clamp() — text is too small on phones, too large on wide screens
✓ Tips & Best Practices
Use clamp() for fluid typography and containers — replaces 2–3 media query breakpoints in one line
Set font-size: 100% on :root to preserve the user's browser font-size as the rem base
min(100%, 1200px) is equivalent to max-width: 1200px; width: 100% — shorter and cleaner
Combine: font-size: clamp(1rem, 2.5vw, 1.5rem) for fluid scaling with hard safety bounds
Quick Summary
rem = root-relative (predictable) · em = local-relative (compounds)
vh/vw = viewport · use svh for mobile safe full-height
clamp(min, preferred, max) — one line replaces multiple breakpoints
calc() mixes units; spaces required around + and - operators
