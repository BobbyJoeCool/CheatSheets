Selectors: Pseudo-Elements & Attribute

::before · ::after · ::placeholder · [attr] · [attr=] · [attr^=]
Description
Pseudo-elements create or target sub-parts of an element that don't exist as separate HTML nodes. ::before and ::after insert generated content and require a content property — even an empty string — to render. Attribute selectors match elements by the presence or value of HTML attributes, enabling powerful targeting without extra classes — perfect for links, inputs, and icon systems.
Notes
Double colon (::) is the modern syntax — avoid single-colon for pseudo-elements in new code
::before and ::after are inline by default — set display: block or position: absolute as needed
content: '' is required even for purely decorative pseudo-elements — without it nothing renders
Attribute selectors have the same specificity as a class: 0,1,0
Code Explained
::before — generated content inserted before the element's content
::after — generated content inserted after the element's content
content: '' — required; can hold text, url(), or counter()
::placeholder — styles input placeholder text
::selection — styles user-highlighted text
[attr] — element has the attribute, any value
[attr='val'] — exact match; [attr^=] starts; [attr$=] ends; [attr*=] contains
[lang|='en'] — exact en or starts with en-
Generated Content
/* Generated content */
.card::before {
  content: '';          /* required - even empty */
  display: block;
  height: 4px;
  background: #5c6bc0;  /* top accent */
}

.required::after {
  content: ' *';        /* literal text appended */
  color: #e57373;
}
Text Sub-parts & Attribute Selectors
/* Targeting text sub-parts */
input::placeholder { color: #5a6285; font-style: italic; }
p::first-line      { font-weight: 700; }
p::selection       { background: #5c6bc0; color: white; }

/* Attribute selectors */
[disabled]          { opacity: 0.5; }           /* attribute exists */
[type="email"]      { border: 1px solid #ccc; } /* exact value */
[class^="icon-"]    { display: inline-block; }  /* starts with */
[href$=".pdf"]      { color: #e57373; }         /* ends with */
[class*="col-"]     { float: left; }            /* contains */
[lang|="en"]        { quotes: '"' '"'; }        /* exact or en- prefix */

⚠ Common Mistakes
Forgetting content: '' on ::before/::after — the element won't render at all
Using :before (single colon) in new code — use double colon for all pseudo-elements
Putting meaningful text in ::before/::after — generated content is invisible to screen readers
Using [attr*='val'] when you need exact match — substring match can hit unintended elements
✓ Tips & Best Practices
::before/::after with position: absolute are perfect for decorative overlays without extra HTML
Use [href$='.pdf'] to auto-badge downloadable file links without adding classes
Style ::placeholder to match your design system — browsers show a faded fallback by default
::selection adds brand character — match it to your primary accent color
Quick Summary
::before/::after insert generated content — always require a content: declaration
Pseudo-elements target sub-parts of elements, not separate DOM nodes
[attr] exists · [attr='v'] exact · [attr^=] starts · [attr$=] ends · [attr*=] contains
Both pseudo-elements and attribute selectors have class-level specificity (0,1,0)
