CSS · Sheet 19

Transitions

transition-property · duration · timing-function · delay · shorthand

Example

/* Shorthand: property | duration | timing-function | delay */
.btn { transition: background-color 0.2s ease; }
.btn { transition: all 0.3s ease-in-out; }

/* Multiple properties — comma separated */
.card {
  transition:
    transform  0.3s ease,
    box-shadow 0.3s ease,
    opacity    0.2s ease 0.05s;   /* delay on opacity */
}

/* Timing functions */
.a { transition-timing-function: ease; }          /* default */
.b { transition-timing-function: linear; }
.c { transition-timing-function: ease-in; }       /* slow start */
.d { transition-timing-function: ease-out; }      /* slow end */
.e { transition-timing-function: ease-in-out; }   /* slow both */
.f { transition-timing-function: cubic-bezier(0.34, 1.56, 0.64, 1); }

/* Correct pattern — transition on BASE state */
.btn {
  background: #5c6bc0;
  transition: background 0.2s ease, transform 0.2s ease;
}
.btn:hover {
  background: #7986cb;
  transform: translateY(-2px);
}

Description

CSS transitions animate a property change smoothly from one value to another when a state change occurs — such as :hover, :focus, or a class toggle via JavaScript. You define which property to animate, how long it takes, the easing curve, and an optional delay. Transitions are reactive — they only fire when a triggering change happens, unlike animations which run independently.

Notes

  • Only animatable properties can transition — most numeric values qualify; display cannot
  • transition: all is convenient but animates every changing property — can cause unexpected results and hurt performance
  • Place transition on the base state, not the :hover — so it plays in both directions (on and off)
  • GPU-accelerated: opacity and transform — always prefer these for smooth 60 fps animations

Code Explained

transition: background 0.2s ease — property, duration, timing
Comma-separated list — each property gets its own duration and timing
transition: all — transitions every animatable property (use carefully)
ease — slow start, fast middle, slow end — natural feel (default)
ease-in — starts slowly, accelerates into the end
ease-out — fast start, decelerates — best for elements entering screen
cubic-bezier() — fully custom easing with 4 control points
Transition on the base state — it then runs in both directions automatically

⚠ Common Mistakes

  • Putting transition on the :hover state only — the reverse transition back won't animate
  • Using transition: all in production — animates properties you didn't intend to animate
  • Transitioning width or height — triggers layout recalc; use transform: scaleX() instead
  • Expecting display: none to transition — it is not an animatable property

✓ Tips & Best Practices

  • Stick to opacity and transform for performance-critical transitions — they run on the GPU
  • Keep UI transition durations short: 150–250ms feels instant; 300–400ms feels polished; longer feels sluggish
  • Use incremented transition-delay on list items for a staggered reveal effect
  • cubic-bezier(0.34, 1.56, 0.64, 1) creates a satisfying spring/overshoot bounce on hover

Quick Summary

  • Shorthand: property duration timing-function delay
  • Place transition on the base state — it animates in both directions automatically
  • GPU-accelerated: opacity and transform — prefer over layout-triggering properties
  • Multiple transitions comma-separated — each can have independent timing
