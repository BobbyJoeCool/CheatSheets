Transform

translate · rotate · scale · skew · matrix · transform-origin · 3D
Description
The transform property applies geometric operations to elements — moving, rotating, scaling, or skewing — without affecting document layout flow. Transformed elements don't push neighbouring elements. Multiple functions are chained in one transform declaration and applied right-to-left. Because transforms don't trigger layout recalculation, they are the preferred technique for animating motion at 60 fps.
Example
/* Translate — move without affecting layout */
.a { transform: translateX(50px); }
.b { transform: translateY(-20px); }
.c { transform: translate(50px, -20px); }
.d { transform: translate(50%); }       /* % of element's own size */

/* Rotate */
.e { transform: rotate(45deg); }
.f { transform: rotate(-0.25turn); }    /* turns unit */

/* Scale */
.g { transform: scale(1.2); }          /* uniform 120% */
.h { transform: scaleX(2); }           /* horizontal only */
.i { transform: scale(1.2, 0.8); }     /* x and y independently */

/* Skew */
.j { transform: skewX(15deg); }
.k { transform: skew(15deg, 5deg); }

/* Chaining — applied right to left */
.l { transform: rotate(45deg) scale(1.2) translateX(20px); }

/* Transform origin */
.m { transform-origin: top left; }
.n { transform-origin: 50% 100%; }  /* bottom centre */

/* 3D */
.wrap { perspective: 600px; }       /* set on parent */
.box  { transform: rotateY(45deg); }
.box  { transform: translateZ(60px); }
Notes
Transforms are applied right-to-left: rotate(45deg) translateX(100px) rotates first, then translates in the rotated direction
transform-origin defaults to the element centre (50% 50%) — change it to adjust the pivot point
Applying any transform creates a new stacking context — the element paints above non-transformed siblings
Modern CSS has standalone translate, rotate, and scale properties — they can be animated independently
Code Explained
translate(X, Y) — moves element; doesn't affect surrounding layout
translate(50%) — % is relative to the element's own dimensions, not the parent
rotate(45deg) — clockwise; also accepts turn, rad, grad
scale(1.2) — 120% of original size; values below 1 shrink
skew(Xdeg, Ydeg) — slants along X and/or Y axes
Chain order matters — functions apply right to left
transform-origin — pivot point for rotate and scale (default: 50% 50%)
perspective on the parent — enables visible depth for 3D child transforms

⚠ Common Mistakes
translate(50%) is 50% of the element itself, not the parent — easy to confuse
Ignoring chain order — rotate then translate is completely different from translate then rotate
Using top/left for animations instead of transform: translate() — top/left trigger layout recalc
Forgetting perspective on the parent for 3D — without it 3D transforms show no depth
✓ Tips & Best Practices
Always animate with transform and opacity — the only properties that don't trigger layout or paint
transform: translate(-50%, -50%) with absolute positioning is the classic true-centring trick
Set transform-origin: top center for dropdown menus that should grow downward from their trigger
Use standalone translate, rotate, scale properties in modern CSS for independent animation
Quick Summary
transform: translate · rotate · scale · skew — no layout impact
Functions apply right-to-left — order significantly changes the visual result
transform-origin sets the pivot; default is element centre (50% 50%)
Always prefer transform over top/left for animation — GPU-accelerated
