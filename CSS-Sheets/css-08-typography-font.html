Typography: Font Properties

font-family · font-size · font-weight · font-style · @font-face

Description

Font properties control the typeface, size, weight, and style of text. font-family accepts a comma-separated fallback stack ending with a generic family. font-size is the foundation for relative units — rem scales with the root, em with the parent. The font shorthand combines family, size, weight, style, and line-height in one declaration. @font-face loads custom fonts from files and should always include font-display: swap to avoid invisible text during load.

Example

/* Family stack — first available font wins */
body { font-family: 'DM Sans', Helvetica, Arial, sans-serif; }
code { font-family: 'Source Code Pro', Consolas, monospace; }

/* Size and weight */
h1    { font-size: 2.5rem;  font-weight: 700; }
h2    { font-size: 1.75rem; font-weight: 600; }
p     { font-size: 1rem;    font-weight: 400; }
small { font-size: 0.875rem; }

/* Style and variant */
em     { font-style: italic; }
.caps  { font-variant: small-caps; }

/* Font shorthand: style weight size/line-height family */
p { font: 400 1rem/1.6 'DM Sans', sans-serif; }

/* @font-face — self-hosted custom font */
@font-face {
  font-family: 'MyFont';
  src: url('../fonts/myfont.woff2') format('woff2'),
       url('../fonts/myfont.woff')  format('woff');
  font-weight: 400 700;   /* weight range this file covers */
  font-style:  normal;
  font-display: swap;     /* show fallback instantly */
}

Notes

  • Always end a font-family stack with a generic: serif, sans-serif, monospace, cursive, or fantasy
  • font-weight accepts keywords (bold) or numeric 100–900 in increments of 100
  • Prefer rem over px for font-size — respects the user's browser font-size preference
  • Variable fonts support fractional weights and custom axes via font-variation-settings

Code Explained

font-family — comma list; first available wins; end with a generic
font-size: 2.5rem — 2.5× the root font-size (typically 40px)
font-weight: 700 — bold; 400 = normal, 900 = black
font-style: italic — uses the italic variant of the font if available
font: 400 1rem/1.6 ... — shorthand: weight size/line-height family
@font-face — declares a custom font and its source file(s)
font-weight: 400 700 — tells the browser this file covers that weight range
font-display: swap — render the fallback immediately; swap when font loads

⚠ Common Mistakes

  • Omitting the generic fallback in font-family — no safety net if the named font fails to load
  • Using px for body font-size — prevents users from scaling text in browser settings
  • Setting font-weight: bold when the font only has 400/700 — browser fake-bolds look poor
  • Not adding font-display: swap in @font-face — causes FOIT (invisible text during font load)

✓ Tips & Best Practices

  • Set font-size: 62.5% on :root so 1rem = 10px for easier mental math throughout the project
  • Use Google Fonts or Bunny Fonts <link> for quick projects instead of self-hosted @font-face
  • Pair fonts in 2s: one distinctive display font for headings, one readable sans-serif for body
  • Use clamp(1rem, 2.5vw, 2rem) for fluid type that scales with the viewport without breakpoints

Quick Summary

  • font-family — always end the stack with a generic fallback
  • Prefer rem for font-size — scales with the user's browser preference
  • font-weight: 100 thin → 400 normal → 700 bold → 900 black
  • @font-face loads custom fonts; always use font-display: swap
