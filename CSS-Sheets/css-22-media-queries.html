Responsive: Media Queries

@media · min-width · max-width · orientation · breakpoints · print

Description

Media queries apply CSS conditionally based on device or environment characteristics. Modern responsive design uses a mobile-first approach: base styles target small screens, and min-width queries progressively enhance for larger ones. This requires fewer overrides than desktop-first and aligns with how browsers on slow connections render pages. Beyond width, media queries can target orientation, colour scheme preference, motion preference, and print.

Notes

  • Mobile-first uses min-width — styles apply from that breakpoint upward
  • Desktop-first uses max-width — styles apply from that breakpoint downward — requires more overrides
  • Choose breakpoints based on your content, not specific devices — devices change; content doesn't
  • Modern range syntax (640px <= width < 1024px) replaces chained min/max queries

Code Explained

Base styles — no @media wrapper means they apply to all sizes
min-width: 640px — activates for viewports 640px wide and above
Stack queries smallest → largest for mobile-first; no need to undo previous rules
prefers-color-scheme: dark — system dark mode is active
prefers-reduced-motion: reduce — user requests minimal animation
hover: none — device has no true hover capability (touch)
@media print — applied only in print preview and when printing
Range syntax: 640px <= width < 1024px — cleaner than chained min/max

Example

/* Mobile-first: base styles apply to all screen sizes */
.container { padding: 1rem; }

@media (min-width: 640px) {   /* sm: tablet portrait and up */
  .container { padding: 1.5rem; }
}

@media (min-width: 768px) {   /* md: tablet landscape and up */
  .container { padding: 2rem; }
  .grid { display: grid; grid-template-columns: repeat(2, 1fr); }
}

@media (min-width: 1024px) {  /* lg: desktop and up */
  .grid { grid-template-columns: repeat(3, 1fr); }
}

@media (min-width: 1280px) {  /* xl: wide desktop */
  .container { max-width: 1200px; margin: 0 auto; }
}

/* Preference queries */
@media (prefers-color-scheme: dark)   { body { background: #111; color: #eee; } }
@media (prefers-reduced-motion: reduce) { * { animation: none !important; } }
@media (hover: none)                  { .tooltip { display: none; } }

/* Print */
@media print {
  .nav, .footer, .sidebar { display: none; }
  body { font-size: 12pt; color: black; background: white; }
  a::after { content: ' (' attr(href) ')'; }
}

/* Range syntax (modern) */
@media (640px <= width < 1024px) { /* between sm and lg */ }

⚠ Common Mistakes

  • Using desktop-first max-width — leads to more override rules and greater complexity as screens shrink
  • Picking breakpoints to match specific devices — designs break as device sizes shift over time
  • Having no print stylesheet — pages can print with dark backgrounds, nav bars, and illegible colours
  • Animating without prefers-reduced-motion — causes nausea and dizziness for users with vestibular disorders

✓ Tips & Best Practices

  • Write mobile-first: add complexity for larger screens, never strip it away for smaller ones
  • Use @media (hover: hover) to restrict hover effects to devices that genuinely support them
  • Wrap all non-essential animation declarations in prefers-reduced-motion: no-preference
  • Always include a print stylesheet: hide navigation, use black text, reveal link URLs via attr(href)

Quick Summary

  • Mobile-first: base styles for small screens, min-width queries enhance upward
  • Common breakpoints: 640 / 768 / 1024 / 1280px — base on content, not device names
  • prefers-color-scheme and prefers-reduced-motion respect OS-level user settings
  • @media print — hide nav, use black text, remove backgrounds for clean printing
