Display & Overflow

display · overflow · visibility · float · clear · clip

Description

display determines how an element participates in layout — as a block, inline, flex container, grid container, or removed entirely. overflow controls what happens when content exceeds its container. visibility: hidden hides an element while keeping its space in the layout; display: none removes it from flow entirely. The legacy float is still useful for wrapping text around images but should not be used for page layout.

Notes

  • display: none removes the element from layout — space collapses. visibility: hidden hides it but keeps the space
  • display: contents makes the element's box disappear while keeping its children in the flow — useful for flex/grid semantics
  • overflow: auto is preferred over scroll — scrollbars appear only when actually needed
  • Floated elements are pulled out of flow; overflow: hidden on the parent or a clearfix restores height

Code Explained

display: block — fills available width, starts on a new line
display: inline — flows with text; width/height ignored
display: inline-block — inline flow with block sizing
display: none — element and its space are removed from layout
display: contents — the element's box is removed; children stay in flow
overflow: hidden — clips content and clears internal floats
overflow: auto — scrollbar appears only when content overflows
visibility: hidden — invisible but still occupies space

Example

/* Block vs inline */
.block   { display: block; }          /* full width, new line */
.inline  { display: inline; }         /* flows with text */
.iblock  { display: inline-block; }   /* inline flow + block sizing */
.none    { display: none; }           /* removed from layout entirely */
.content { display: contents; }       /* box gone, children remain */

/* Flex and grid */
.flex        { display: flex; }
.inline-flex { display: inline-flex; }
.grid        { display: grid; }
.inline-grid { display: inline-grid; }

/* Overflow */
.clip   { overflow: hidden; }         /* clips — also clears floats */
.scroll { overflow: auto; }           /* scrollbar when needed */
.always { overflow: scroll; }         /* always shows scrollbar */
.axes   { overflow-x: auto; overflow-y: hidden; }

/* Visibility vs display */
.hide-space { visibility: hidden; }   /* invisible, space kept */
.hide-flow  { display: none; }        /* invisible, space removed */
.see-thru   { opacity: 0; }          /* invisible, space kept, clickable */

/* Float — text wrapping only */
.figure { float: left; margin: 0 1rem 1rem 0; }
.clear  { clear: both; }

/* Modern clearfix */
.clearfix::after { content: ''; display: block; clear: both; }

display Values

Syntax Description
block Full width, new line
inline Flows with text
inline-block Inline + block sizing
flex Block-level flex container
inline-flex Inline-level flex container
Syntax Description
grid Block-level grid container
inline-grid Inline-level grid container
none Removed from layout
contents Box removed, children remain

overflow Values

Syntax Description
visible Default — spills outside
hidden Clips overflowing content
scroll Always shows scrollbars
auto Scrollbar only when needed
clip Like hidden, no scroll API

Visibility Comparison

Syntax Description
visibility: visible Shown (default)
visibility: hidden Hidden, space kept
display: none Hidden, space gone
opacity: 0 Invisible, space kept, clickable
pointer-events: none Visible but not interactive

⚠ Common Mistakes

  • Using visibility: hidden when you need display: none — the element still occupies space
  • Using overflow: hidden on a sticky element's parent — it breaks position: sticky
  • Floating elements without clearing — the parent collapses to zero height
  • Using display: none to hide content meant for screen readers — use the .sr-only pattern instead

✓ Tips & Best Practices

  • Use flex or grid for all new layouts — floats are only for text wrapping around images
  • The .sr-only pattern (position:absolute; width:1px; height:1px; overflow:hidden; clip:rect(0,0,0,0)) hides visually but keeps content for screen readers
  • overflow: hidden on a parent is a quick float-clearing trick — prefer flex/grid to avoid needing it
  • display: contents is great for making a semantically required wrapper invisible to flex/grid layout

Quick Summary

  • block = full width · inline = text flow · inline-block = both
  • display: none removes from flow · visibility: hidden hides but keeps space
  • overflow: auto for scrollable containers · hidden to clip or clear floats
  • Use flex or grid for layouts — floats are for text wrapping only
