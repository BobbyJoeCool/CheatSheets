Backgrounds

background-color · background-image · background-size · background-position · repeat

Description

The background shorthand paints a visual layer behind an element's content and padding. Multiple background layers can be stacked with comma-separated values — the first listed is on top. background-size: cover fills the container and may crop the image; contain fits the whole image inside without cropping but may leave gaps. background-color is always the bottom-most layer and cannot be layered in the image stack.

Example

/* Solid color */
.box  { background-color: #1c2030; }
.box  { background: #1c2030; }        /* shorthand equivalent */

/* Image */
.hero {
  background-image: url('../img/hero.jpg');
  background-size:     cover;
  background-position: center;
  background-repeat:   no-repeat;
}

/* Full shorthand: color image position/size repeat attachment */
.hero { background: #1c2030 url('../img/hero.jpg') center/cover no-repeat; }

/* Multiple layers — gradient overlay on top of image */
.hero {
  background:
    linear-gradient(rgba(0,0,0,0.55), rgba(0,0,0,0.55)),
    url('../img/hero.jpg') center/cover no-repeat;
}

/* Gradient text */
.grad-text {
  background: linear-gradient(135deg, #9c8fff, #64b5f6);
  -webkit-background-clip: text;
  background-clip: text;
  color: transparent;
}

Notes

  • Multiple backgrounds: comma-separated — first value is the topmost layer
  • background-color sits below all image layers; it cannot appear in the comma-separated list
  • background-attachment: fixed creates parallax but causes performance issues on mobile
  • background-clip: text + color: transparent creates gradient text effects

Code Explained

background-color — solid fill; always the bottom-most layer
background-image: url() — path to an image file or a gradient function
background-size: cover — fills the element; may crop image edges
background-size: contain — shows full image; may leave gaps
background-position: center — places the focal point of the image
background-repeat: no-repeat — prevents default tiling behaviour
Multiple layers comma-separated — first listed renders on top
background-clip: text + color: transparent = gradient text

background-size Values

Syntax Description
auto Natural image size (default)
cover Fill — may crop
contain Fit entirely — may gap
100% 100% Stretch to fill exactly
50% Half the element width
200px 100px Explicit W and H

background-position Values

Syntax Description
center Centre on both axes
top / bottom Top or bottom edge
left / right Left or right edge
50% 25% Percentage x y
20px 40px Offset from top-left
right 20px bottom 10px Edge-relative offset

background-repeat Values

Syntax Description
repeat Default — tile both axes
no-repeat No tiling
repeat-x Tile horizontal only
repeat-y Tile vertical only
space Tile with equal spacing
round Scale to fit whole tiles

⚠ Common Mistakes

  • Forgetting no-repeat on a hero image — it tiles by default
  • Using the background shorthand without specifying all needed values — omitted sub-properties reset to defaults
  • Using background-attachment: fixed on mobile — often disabled by mobile browsers for performance
  • Using background images for meaningful content — use <img alt='...'> for accessibility

✓ Tips & Best Practices

  • Use the multi-layer technique to add a dark overlay on hero images without an extra HTML element
  • background: center/cover no-repeat is the go-to pattern for any full-bleed image
  • Always provide a background-color fallback when using image backgrounds
  • Gradient text: pair background-clip: text with color: transparent

Quick Summary

  • cover fills and may crop · contain fits without cropping
  • Shorthand: color image position/size repeat — unspecified values reset to defaults
  • Multiple layers comma-separated — first listed is on top
  • Always pair image backgrounds with no-repeat and a colour fallback
