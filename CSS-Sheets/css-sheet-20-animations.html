Animations & Keyframes

@keyframes · animation-name · duration · iteration-count · fill-mode
Description
CSS animations use @keyframes rules to define intermediate steps and run independently without a trigger. Unlike transitions, they can loop, play automatically on page load, and define multiple intermediate steps. The animation shorthand sets name, duration, timing, delay, iteration count, direction, fill-mode, and play-state in one declaration.
Notes
Keyframe percentages: 0% = start, 100% = end — the keywords from and to are equivalent
animation-fill-mode: forwards — element keeps the final keyframe styles after the animation ends
animation-direction: alternate — reverses on each even iteration creating a ping-pong effect
Respect user preferences: wrap non-essential animations in @media (prefers-reduced-motion: no-preference)
Code Explained
@keyframes fadeIn — defines named animation steps referenced by animation-name
from / to — shorthand for 0% / 100%
0% 50% 100% — define as many intermediate steps as needed
animation: fadeIn 0.4s ease-out forwards — name duration timing fill-mode
infinite — loops the animation forever
forwards fill-mode — element retains the final keyframe state after finishing
both fill-mode — applies first frame before delay, retains last frame after end
animation-play-state: paused — freezes the animation in place

Example
/* Define keyframes */
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(10px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes pulse {
  0%   { transform: scale(1); }
  50%  { transform: scale(1.05); }
  100% { transform: scale(1); }
}

@keyframes spin {
  to { transform: rotate(360deg); }
}

/* Apply: name | duration | timing | delay | count | direction | fill */
.card    { animation: fadeIn 0.4s ease-out forwards; }
.spinner { animation: spin   1s   linear   infinite; }
.badge   { animation: pulse  2s   ease-in-out infinite; }

/* fill-mode: both — apply from: before delay, keep to: after end */
.item { animation: fadeIn 0.4s ease-out 0.1s both; }

/* Staggered entrance */
.item:nth-child(1) { animation: fadeIn 0.4s ease-out 0.0s both; }
.item:nth-child(2) { animation: fadeIn 0.4s ease-out 0.1s both; }
.item:nth-child(3) { animation: fadeIn 0.4s ease-out 0.2s both; }

/* Pause on hover */
.animated:hover { animation-play-state: paused; }

/* Reduced motion */
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}

⚠ Common Mistakes
Forgetting animation-fill-mode: forwards — element snaps back to original state after ending
Animating layout-triggering properties (width, margin) — use transform for 60 fps
Defining @keyframes inside a selector — keyframes must be at the top level of the stylesheet
Not wrapping decorative animations in prefers-reduced-motion — causes issues for users with vestibular disorders
✓ Tips & Best Practices
Use animation-fill-mode: both to apply the first frame before a delay and keep the last after
Stagger animation-delay on list children for a polished cascade entrance effect
Animate only opacity and transform for GPU-accelerated 60 fps performance
Name keyframes descriptively: slideInFromLeft, fadeUp, pulseShadow
Quick Summary
@keyframes name { from {} to {} } — defines the animation steps
Shorthand: name | duration | timing | delay | count | direction | fill-mode
forwards = keeps final state · both = forwards + backwards · infinite = loops
