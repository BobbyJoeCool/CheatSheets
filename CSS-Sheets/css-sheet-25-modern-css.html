Modern CSS

:is() · :where() · :has() · @layer · @container · subgrid
Description
Modern CSS features dramatically reduce the need for workarounds. :is() and :where() group selectors without repetition. :has() — the long-awaited parent selector — styles an element based on what it contains. @layer gives explicit, predictable control over cascade order. @container queries style elements based on their parent's size rather than the viewport. subgrid lets nested grids align to the parent grid's tracks.
Notes
:is() inherits the specificity of its most specific argument; :where() always has zero specificity
:has() is supported in all major browsers since late 2023 — it is no longer experimental
@layer order is set by the first declaration — subsequent rules of the same layer name append to it
@container requires container-type: inline-size (or size) on the parent element
Code Explained
:is(h1, h2, h3) — matches all listed selectors; reduces repetitive rule blocks
:where() — same matching as :is() but contributes zero specificity
.card:has(img) — selects .card elements that contain an img
:has(:invalid) — form that contains an invalid input field
@layer base, components, utilities — later layers beat earlier ones at equal specificity
@layer utilities { } — rules here always beat components
container-type: inline-size — makes element a named container query context
grid-template-columns: subgrid — child inherits parent grid's column definitions
Selector Functions
SyntaxDescription
:is(a, b, c)Match any; specificity = highest arg
:where(a, b, c)Match any; always 0 specificity
:has(selector)Parent contains matching child
:not(selector)Excludes matching elements
:is() :where()Can be nested and combined
Example
/* :is() — groups selectors; specificity = highest arg */
:is(h1, h2, h3) { line-height: 1.2; }
:is(article, section, aside) p { margin-bottom: 1rem; }

/* :where() — same as :is() but always zero specificity */
:where(h1, h2, h3, h4, h5, h6) { font-weight: 700; }

/* :has() — parent/relational selector */
.card:has(img)          { padding: 0; }       /* card containing an image */
.form:has(:invalid)     { border-color: red; }/* form with invalid input */
li:has(+ li)            { border-bottom: 1px solid #333; }
nav:has(.active)        { background: #1c2030; }

/* @layer — explicit cascade ordering */
@layer base, components, utilities;

@layer base       { body  { margin: 0; font-family: sans-serif; } }
@layer components { .btn  { padding: 0.5rem 1rem; border-radius: 4px; } }
@layer utilities  { .mt-4 { margin-top: 1rem; } }

/* @container — style based on parent container size */
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}
@container card (min-width: 400px) {
  .card { display: flex; gap: 1rem; }
}

/* subgrid — inherit parent grid tracks */
.parent { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem; }
.child  {
  grid-column: span 3;
  display: grid;
  grid-template-columns: subgrid; /* aligns columns to parent tracks */
}

⚠ Common Mistakes
Confusing :is() and :where() — use :where() for resets (zero specificity is always overridable)
Using @container without container-type on the parent — queries never fire
Assuming @layer overrides !important — !important in a lower layer still beats a higher one
Using complex :has() selectors on very large DOMs without performance testing
✓ Tips & Best Practices
Replace long repeated selector groups with :is(): :is(h1,h2,h3,h4,h5,h6) { }
Use :where() in design systems and resets — zero specificity means any consumer selector can override
:has() enables true component-level state styling: .card:has(.badge), nav:has(.active)
Combine @layer + @container in new projects for a fully modern, maintainable architecture
Quick Summary
:is() groups selectors (inherits specificity) · :where() groups with zero specificity
:has() — select a parent by what it contains; broadly supported since 2023
@layer — explicit cascade ordering; unlayered rules always beat layered ones
@container — style components based on their parent size, not the viewport
