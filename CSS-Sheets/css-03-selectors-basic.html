Selectors: Basic & Combinators

element · .class · #id · * · A B · A > B · A + B · A ~ B

Description

CSS selectors target the HTML elements you want to style. Basic selectors match by element type, class, ID, or all elements universally. Combinator selectors express structural relationships between elements — descendant, direct child, adjacent sibling, and general sibling. Every rule begins with a selector, and its specificity determines which rule wins when declarations conflict.

Notes

  • Class selectors (.class) are the most reusable — prefer them over element or ID selectors in components
  • #id specificity is 1,0,0 — avoid in stylesheets, reserve for JS hooks or skip-nav links
  • The universal selector * matches every element — powerful in resets but avoid deep nesting for performance
  • Combinators add no specificity of their own beyond their individual selector parts

    • Code Explained

    p — matches every <p> element on the page
    .card — matches every element with class card
    #hero — matches the single element with id hero
    * — universal: matches every element
    nav a — descendant: any <a> anywhere inside <nav>
    ul > li — child: only direct <li> children of <ul>
    h2 + p — adjacent: the <p> immediately after <h2>
    h2 ~ p — general sibling: all <p> after <h2> in the same parent

    Basic Selectors

    SyntaxDescription
    elementMatches by tag: p, h1, div
    .classMatches elements with that class
    #idMatches the one element with that ID
    *Universal — matches every element
    el.classElement AND class — e.g. a.active
    .a.bBoth classes present on one element

    Combinators

    SyntaxDescription
    A BDescendant — B anywhere inside A
    A > BChild — B is direct child of A
    A + BAdjacent — B immediately after A
    A ~ BSibling — all B after A in same parent

    Specificity Values

    SyntaxDescription
    *0,0,0 — no specificity
    element0,0,1 — one type
    .class0,1,0 — one class
    #id1,0,0 — one ID
    el.class0,1,1 — combined
    #id .class1,1,0 — combined

    ⚠ Common Mistakes

    • Using #id in component CSS — creates specificity debt requiring !important to override
    • Over-qualifying: div.card instead of .card — reduces reusability without benefit
    • Confusing A B (any depth) with A > B (direct only) — descendants match at any depth
    • Long chains like nav ul li a span — brittle, breaks whenever HTML structure changes

    ✓ Tips & Best Practices

    • Prefer class selectors for styling; element selectors only for base/reset rules
    • Use > child combinator to scope styles to direct children only — avoids deep inheritance leaks
    • Keep selector chains to 2–3 parts maximum for performance and maintainability
    • Use :is() to group selectors without repeating long combinator chains (see Sheet 25)

    Quick Summary

    • element · .class · #id · * — the four basic selector types
    • A B descendant · A > B child · A + B adjacent · A ~ B sibling
    • Specificity: ID (1,0,0) > class (0,1,0) > element (0,0,1) > universal (0,0,0)
    • Keep selectors short and class-based for low-specificity, reusable styles
