Grid: Container

display:grid · grid-template-columns · fr · repeat · minmax · auto-fit
Example
.grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;       /* 3 equal columns */
  grid-template-columns: repeat(3, 1fr);    /* same thing */
  grid-template-rows: auto;                 /* rows size to content */
  gap: 1rem;                                /* row and column gap */
  gap: 1rem 2rem;                           /* row-gap | column-gap */
}

/* Mixed track sizes */
.layout {
  grid-template-columns: 260px 1fr;         /* fixed sidebar + fluid main */
  grid-template-columns: 1fr 2fr 1fr;       /* 1:2:1 ratio */
}

/* Responsive — no media queries needed */
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
}

/* Named template areas */
.page {
  display: grid;
  grid-template-columns: 260px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    'header  header'
    'sidebar main'
    'footer  footer';
}

/* Implicit rows — controls height of auto-created rows */
.feed { grid-auto-rows: 200px; }
Description
CSS Grid is a two-dimensional layout system — it controls rows and columns simultaneously. display: grid on a container makes its direct children grid items. Tracks are defined with grid-template-columns and grid-template-rows. The fr unit distributes free space proportionally. repeat() reduces repetition; minmax() sets a size range. Combining auto-fit with minmax() creates fully responsive grids with no media queries.
Notes
The fr unit distributes free space — space remaining after fixed tracks are placed
auto-fit collapses empty tracks; auto-fill keeps them — both work with minmax()
gap (formerly grid-gap) sets row and column spacing; use row-gap / column-gap individually
Implicit tracks — created when items overflow — are sized by grid-auto-rows and grid-auto-columns
Code Explained
display: grid — container is grid; direct children become grid items
repeat(3, 1fr) — creates 3 equal columns using fractional units
fr — fractional unit: 1 share of remaining free space
260px 1fr — fixed sidebar + fluid main that fills the rest
auto-fit — as many tracks as fit; empty ones collapse to zero
minmax(220px, 1fr) — each track min 220px, max 1fr
gap: 1rem — spacing between all tracks, no edge space
grid-template-areas — named regions for readable item placement

⚠ Common Mistakes
Using fr alongside fixed tracks and expecting equal distribution — fr only shares the space remaining after fixed tracks
Using deprecated grid-gap — it still works but the standard is gap
Confusing auto-fit and auto-fill — use auto-fit for responsive card grids so empty tracks collapse
Forgetting grid-auto-rows — without it, implicitly created rows are sized to content and can be inconsistent
✓ Tips & Best Practices
repeat(auto-fit, minmax(250px, 1fr)) — the one-liner responsive card grid requiring zero media queries
Use grid-template-areas for page-level layout — it reads like a diagram of your design
Set grid-auto-rows to a fixed height or minmax() for consistent implicit row sizes
Combine fixed and fr tracks: 260px 1fr for a layout that adapts without math
