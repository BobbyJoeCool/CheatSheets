Grid: Items & Placement

grid-column · grid-row · grid-area · span · grid-template-areas · named lines
Description
Grid item properties control where a child is placed within the grid. By default items auto-place into the next available cell. grid-column and grid-row use 1-based line numbers to place items explicitly. The span keyword stretches an item across multiple tracks. Negative line numbers count from the end — -1 is always the last grid line. grid-area can reference a named region from grid-template-areas on the container.
Notes
Grid lines are numbered from 1; negative numbers count from the end — -1 is the last line
grid-column: 1 / -1 spans the full width regardless of the column count
grid-area shorthand order: row-start / column-start / row-end / column-end
Auto-placed items follow the grid auto-placement algorithm; explicitly placed items can coexist
Code Explained
grid-column: 2 / 4 — from column line 2 to line 4 (two columns wide)
grid-row: 1 / 3 — from row line 1 to line 3 (two rows tall)
span 2 — stretch across 2 tracks from the item's start position
1 / -1 — from first to last line = full width of grid
grid-area: header — places item into the named region on the container
grid-area: 1 / 1 / 3 / 4 — shorthand: row-start / col-start / row-end / col-end
justify-self — horizontal alignment within the item's cell
place-self: center — centres item both axes within its cell
Example
/* Line-number placement */
.item {
  grid-column: 2 / 4;     /* col line 2 → line 4  (2 tracks wide) */
  grid-row:    1 / 3;     /* row line 1 → line 3  (2 rows tall)  */
}

/* span keyword */
.wide   { grid-column: 1 / span 2; }  /* start line 1, span 2 cols */
.auto   { grid-column: span 3; }      /* span 3 from auto position */

/* Full-width item regardless of column count */
.full   { grid-column: 1 / -1; }

/* grid-area shorthand: row-start / col-start / row-end / col-end */
.item   { grid-area: 1 / 1 / 3 / 4; }

/* Named area placement — container must define grid-template-areas */
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

/* Alignment within the cell */
.item { justify-self: center; }  /* horizontal */
.item { align-self: center; }    /* vertical */
.item { place-self: center; }    /* both at once */

⚠ Common Mistakes
Using 0-based line numbers — CSS grid lines start at 1, not 0
Thinking line numbers equal column count — a 3-column grid has 4 column lines (1 through 4)
Mixing auto and explicit placement heavily without grid-auto-flow: dense — can create visible gaps
Setting a grid-area name when the container has no matching grid-template-areas
✓ Tips & Best Practices
grid-column: 1 / -1 is the safest full-width item — works regardless of how many columns there are
Use grid-template-areas for the page skeleton; use line numbers for fine-grained item tweaks
Set grid-auto-flow: dense on the container to fill gaps when items are different sizes
place-self: center is the quickest way to centre a grid item within its own cell
Quick Summary
grid-column: start / end — placement using 1-based grid line numbers
span N — stretch across N tracks from the item's start position
1 / -1 spans full width; negative numbers count from the end
grid-area: name — place into a named region from grid-template-areas
