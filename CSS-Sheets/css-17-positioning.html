CSS · Sheet 17

Positioning

position · top · right · bottom · left · z-index · stacking context

Example

/* static — default; offset properties have no effect */
.a { position: static; }

/* relative — stays in flow; offset from natural position */
.b { position: relative; top: 10px; left: 20px; }

/* absolute — out of flow; relative to nearest positioned ancestor */
.parent { position: relative; }        /* establishes context */
.badge  {
  position: absolute;
  top:   -8px;
  right: -8px;
}

/* fixed — relative to viewport; stays on scroll */
.nav { position: fixed; top: 0; left: 0; width: 100%; z-index: 100; }

/* sticky — relative until scroll threshold, then fixed */
.thead {
  position: sticky;
  top: 0;                              /* sticks when it hits top: 0 */
  z-index: 10;
  background: #1c2030;
}

/* inset shorthand — replaces top/right/bottom/left */
.overlay { position: absolute; inset: 0; }           /* all sides 0 */
.offset  { position: absolute; inset: 1rem 2rem; }   /* V | H */

/* z-index — stacking order; higher = in front */
.modal   { position: fixed; z-index: 100; }
.overlay { position: fixed; z-index:  99; }

Description

The position property controls how an element is placed in or removed from the normal document flow. static is the default — elements flow normally. relative offsets from its natural position while staying in flow. absolute removes the element from flow and anchors it to the nearest positioned ancestor. fixed anchors to the viewport; sticky behaves as relative until a scroll threshold is reached, then sticks.

Notes

  • An element is 'positioned' when its position is anything other than static
  • absolute positions relative to the nearest positioned ancestor — if none, the initial containing block
  • z-index only works on positioned elements (non-static) and flex/grid items
  • A new stacking context is created by: non-static position + z-index, opacity < 1, transform, filter, and others

Code Explained

position: static — default; top/right/bottom/left ignored
position: relative — stays in flow; offsets from its own natural position
position: absolute — removed from flow; anchored to nearest positioned parent
position: relative on parent — creates the positioning context for absolute children
position: fixed — anchored to the viewport; unaffected by page scrolling
position: sticky — flows normally until the scroll threshold, then sticks
inset: 0 — shorthand for top:0; right:0; bottom:0; left:0
z-index — higher number = closer to the viewer within the same stacking context

⚠ Common Mistakes

  • Forgetting position: relative on the parent of an absolute child — it anchors to the wrong ancestor
  • Using z-index on a static element — it has no effect without a non-static position
  • Setting position: sticky without a threshold (top, bottom) — element never sticks
  • Using overflow: hidden on a sticky item's parent — it breaks sticky behaviour entirely

✓ Tips & Best Practices

  • Use inset: 0 as a shorthand for full-coverage overlays — cleaner than four separate offset properties
  • Maintain a z-index scale in your design tokens: content=1, dropdowns=10, modals=100, toasts=200
  • Use isolation: isolate on components to contain their z-index within their own stacking context
  • For sticky table headers, ensure no ancestor has overflow: hidden or overflow: auto

Quick Summary

  • static (default) · relative (in flow, offset) · absolute (out of flow)
  • fixed = viewport-anchored · sticky = relative until scroll threshold
  • Absolute children need a position: relative ancestor as their positioning context
  • z-index only works on positioned (non-static) elements and flex/grid items
